Writing Tag Handlers (XbRC)

The next step needed to extend XbRC is writing the tag handlers for the new data type as members of CXBParser.

For each tag that CXBParser understands, CXBParser must have at least two member functions defined for it, generally of the form Begin_tag and End_tag. So, for the cube map example, you need a Begin_CubeTexture member function and an End_CubeTexture member function. The Begin_tag function instantiates a new container class based on the type of this resource. In this case, it will call new XBRCCubeMapData. Begin_tag function passes the attributes passed in from the XML parser to the constructor so that the constructor can dig the information out of the XML attributes that this resource needs. The Begin_tag function then pushes this resource container onto the main list and the stack list.

The End_tag function does whatever cleanup is necessary for the resource. It is called when the end tag is reached in the XML file. As such, it represents the end of the data for that tag. In the case of the cube map, you simply verify that the current context on the stack is a cube map container and then pop that container off of the stack. This moves you back out to the previous context so that the next tag you encounter can be parsed correctly.

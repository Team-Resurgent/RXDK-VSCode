Adding Autoupdate Support to Your Title

For a title to perform autoupdates, the files Update.xbe and DashUpdate.xbe must be located in the root directory of the original game disk. DashUpdate.xbe updates the Xbox Dashboard to the latest version. Update.xbe updates your title to a newer version, as necessary.

In addition, your game code must check for a newer version of itself when the player logs on to Xbox Live. When your title calls XOnlineLogon, the XOnlineLogon function uses its pHandle parameter to pass a task handle back to your title. Your title then calls the XOnlineTaskContinue function to tell the Xbox Live service that it is still active and waiting for a response. If XOnlineTaskContinue returns the value XONLINE_E_LOGON_UPDATE_REQUIRED, your title must perform an autoupdate.

Before your title actually begins the autoupdate, it must prompt the player. The recommended message for this is the following: 

A required update is available for the Xbox Live service. 
Press A to update or B to cancel. You cannot connect to Xbox Live 
until the update is installed.

Until they install the update, players should not be allowed to play online. They can play offline, but online play always requires the latest update. If the player indicates a willingness to install the update, your title calls the XOnlineTitleUpdate function. This launches Update.xbe, which in turn downloads and installs the updated build of your title. When the update completes, Update.xbe reboots the Xbox and launches the upated version of the title.

If you want to pass specific configuration information to the updated version of your title, pass it through the the XOnlineTitleUpdate function's dwContext parameter. Update.xbe passes this information to your updated title, which retrieves it by calling XGetLaunchInfo.

Testing Your Title's Autoupdate Support

To simplify the process of testing of your title's autoupdate support, the Xbox team created a default autoupdate package. This default test will "update" your title by downloading a default title that created solely for testing autoupdate. To use these tools, follow the steps below.

  1. Use the XBECert tool to change the version number of your title to 0x199.

    xbecert /testversion:0x199 default.xbe

  2. Verify that your title has the correct version number using ImageBld.

    imagebld /dump default.xbe | more

    In the resulting output, look for the following section to verify that the title ID is your title's ID and that the version number is 199:

    
CERTIFICATE
        1EC size of certificate
   3F8DA9BF time date stamp Wed Oct 15 13:10:39 2003
   FFFF010C title id
        207 allowed media types
   80000007 game region
   FFFFFFFF game ratings
          0 disk number
        199 version (XBE: 1, Disc: 153)
   54524150 online service id
  3. Run the Clear Xbox Live Configuration tool from within the Xbox Live submenu of the XDK Launcher on your Xbox Development or Debug Kit.
  4. Logon to PartnerNet with this version of your game. You should be prompted to autoupdate.

  5. The title then launches Update.xbe, which displays the following message:

    Xbox Live is updating your
system. Please don't turn off 
your Xbox console.

    If this happens, your title correctly supports autoupdate. If your title does not detect the need for an autoupdate when using this version, either it does not correctly support autoupdate or the default test has not yet been enabled for your title. You can verify that the default autoupdate test is enabled for your title by contacting xboxds@xbox.com.

Autoupdate Internals

Autoupdates are downloaded to the Title Persistent region of the Xbox hard disk. Specifically, they are placed in the T:\$u folder. If your title is not running, you can view this folder on your development PC using the Xbox Neighborhood by navigating to S:\[title_id]\$u. Any files that are updated in subfolders (media\font.ttf) will be placed in subfolders under T:\$u (T:\$u\media\font.ttf).

The autoupdate package on the Xbox Live servers contains some metadata about the autoupdate as well as the bitwise difference between the original files and the new files. By storing and downloading only the difference, you can decrease the size of most autoupdates so that downloading the update is quicker. The update itself is broken apart into smaller chunks (about 10 KB) to allow for resumption of a failed download.

When a title needs to perform an update, it launches Update.xbe, which does the updating. During the update, it wipes out the cache partition of your title and uses this area to store the new files created by adding the difference (the downloaded bits) to the original files (from the DVD). These updated files are placed in the Title Persistent region of the hard disk and Update.xbe then reboots.

Upon reboot, the Xbox kernel launches the default executable, which is D:\default.xbe. All Xbox Live titles are compiled with special startup code that first checks for an updated version in T:\$u. If it detects one, the startup code reboots to the updated version instead of the one from the disk.

Defines the various types of surface formats.

Constants

D3DFMT_UNKNOWN
Surface format is unknown.
D3DFMT_A8R8G8B8
32-bit ARGB pixel format with alpha. Format is stored as below: 
(alpha << 24)| (red << 16) | (green << 8) | blue
D3DFMT_X8R8G8B8
32-bit RGB pixel format where 8 bits are reserved for each color. Alpha is set to 1.0f. Format is stored as below: 
(red << 16) | (green << 8) | blue
D3DFMT_R5G6B5
16-bit RGB pixel format. Alpha is set to 1.0f. Format is stored as below: 
(red << 11) | (green << 5) | blue
D3DFMT_R6G5B5

(red << 10 ) | (green << 5) | blue

(red << 10) | (green << 5) | blue

(alpha << 12) | (red << 8) | (green << 4) | blue

32-bit pixel format where 8 bits are reserved for each channel. Format is stored as below:

(red << 8) | blue

16-bit pixel format where 8 bits are reserved for green and blue. When using this format, alpha is the same as the green channel, and red is the same as the blue channel. Format is stored as below: 
(green << 8) | blue
D3DFMT_P8
Surface is 8-bit color indexed. The index indicates which palettized color to use.
D3DFMT_L8
8-bit luminance only. This is for grayscale. When using this format, alpha is 1.0f, and red, green, and blue all equal the luminance value.

16-bit alpha luminance. When using this format, red, green, and blue all equal the luminance value. The format is stored as below: 
(alpha << 8) | luminance
D3DFMT_AL8
8-bit alpha luminance. When using this format, alpha, red, green, and blue all equal the luminance value.
D3DFMT_L16
16-bit pixel format where 8 bits are reserved for green and blue. When using this format, alpha and red are 1.0f. Format is stored as below: 
(green << 8) | blue

D3DFMT_D16_LOCKABLE
16-bit (integer) z-buffer bit depth. This is an application-lockable surface format.

16-bit (integer) z-buffer bit depth. Because all surfaces for Xbox are lockable, this format is the same as D3DFMT_D16.
D3DFMT_D24S8
32-bit (integer) z-buffer bit depth where 24 bits are reserved for the depth channel and 8 bits are reserved for the stencil channel.
D3DFMT_F16
16-bit (floating-point) z-buffer bit depth.
D3DFMT_F24S8
32-bit (floating-point) z-buffer bit depth where 24 bits are reserved for the depth channel and 8 bits are reserved for the stencil channel.
D3DFMT_UYVY
UYVY format (PC98 compliance). Two pixels are stored in each UYVY word. Each data value (U, Y0, V, Y1) is 8 bits. The U and V specify the chrominance and are shared by both pixels. The Y0 and Y1 values specify the luminance of the respective pixels. The chrominance and luminance can be computed from RGB values using the following formulas: 
Y = 0.299 * R + 0.587 * G + 0.114 * B
U = -0.168736 * R + -0.331264 * G + 0.5 * B
V = 0.5 * R + -0.418688 * G + -0.081312 * B
Before this format can be used, it must be enabled using the render state D3DRS_YUVENABLE.
D3DFMT_YUY2
YUY2 format (PC98 compliance). Two pixels are stored in each YUY2 word. Each data value (Y0, U, Y1, V) is 8 bits. This format is identical to the UYVY format described except that the ordering of the bits is changed. Before this format can be used, it must be enabled using the render state D3DRS_YUVENABLE.
D3DFMT_DXT1
DXT1 compression texture format.
D3DFMT_DXT2

D3DFMT_DXT3

Remarks

Many of the formats exposed through this enumeration have been provided only because the hardware supports them. They do not necessarily have a performance benefit.

Note that render target formats are restricted to the formats below:

  • D3DFMT_A8R8G8B8
  • D3DFMT_LIN_A8R8G8B8
  • D3DFMT_X8R8G8B8
  • D3DFMT_LIN_X8R8G8B8
  • D3DFMT_R5G6B5
  • D3DFMT_LIN_R5G6B5
  • D3DFMT_X1R5G5B5
  • D3DFMT_LIN_X1R5G5B5

D3DFMT_LIN_L8 and D3DFMT_LIN_G8B8 are also supported as render targets, but z-buffering, stenciling, and visibility testing do not work in combination with these formats. D3DFMT_LIN_L8 outputs the blue channel, and D3DFMT_LIN_G8B8 outputs the green and blue channels.

The order of the bits are from the most significant bit (MSB) first, so D3DFMT_A8L8 indicates that the high byte of this two-byte format is alpha. D3DFMT_D16 indicates a 16-bit integer value and an application-lockable surface.

All depth-stencil formats except D3DFMT_D16_LOCKABLE indicate no particular bit ordering per pixel, and are not application-lockable, and the driver is allowed to consume more than the indicated number of bits per depth channel (but not stencil channel).

Pixel formats are denoted by opaque DWORD identifiers. The format of these DWORDs has been chosen to enable the expression of IHV-defined extension formats, and also to include the well-established FOURCC method. The set of formats understood by the Microsoft® Direct3D® run-time is defined by D3DFORMAT.

Note that IHV-supplied formats and many FOURCC codes are not listed in the D3DFORMAT enumeration. The formats in this enumeration are unique in that they are sanctioned by the run-time, meaning that the reference rasterizer will operate on all these types. The IHV-supplied formats will be supported by the individual IHVs on a card-by-card basis.

Xbox Optimizations

The numerical values for these formats have changed from DirectX 8.0 for the PC to make the mapping to the NVIDIA® internal format used on Xbox easier. Most notable is that D3DFMT_UNKNOWN is no longer zero.

Most of these names have the following convention:

A = Alpha
R = Red
G = Green
B = Blue
X = Unused Bits
P = Palette
L = Luminance
U = dU coordinate for BumpMap
V = dV coordinate for BumpMap
S = Stencil
D = Depth (for example, Z or W buffer)

Further, the pieces are ordered starting from MSB; hence, D3DFMT_A8L8 indicates that the high byte of this two-byte format is alpha.

D16 indicates:

  • An integer 16-bit value.
  • An application-lockable surface.

All Depth/Stencil formats indicate:

  • No particular bit ordering per pixel, and
  • The driver is allowed to consume more than the indicated number of bits per Depth channel (but not Stencil channel).

    The constants for this enumeration have been redefined from the DirectX 8.0 for PC values.

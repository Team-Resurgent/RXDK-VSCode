XMountUtilityDrive

The XMountUtilityDrive function is used to mount the title's utility storage region. A hard disk partition is selected and assigned to the title for use as temporary storage and caching of game data.

BOOL XMountUtilityDrive(
  BOOL fFormatClean               
);

Parameters

fFormatClean

[in] Specifies whether the partition should be quick-formatted or not. If a partition that was last used by the calling title is available, that partition will be selected and assigned to the title. In that case, fFormatClean determines whether or not the partition will be formatted to remove any previous data it contained. If no such partition is available, the assigned partition will be returned in a quick-formatted state regardless of the value specified for fFormatClean.

A TRUE value for fFormatClean also indicates that the title wishes to use the utility partition for temporary data only. The partition is marked as discardable, and data written to the partition may be erased when a reboot of the console occurs.

Return Values

If the function succeeds, the return value is TRUE.

If the function fails, the return value is FALSE. To get extended error information, call GetLastError.

Remarks

This function is used to mount an extra hard disk partition that can be used for temporary scratch space, caching data from the game DVD, or any other transitory functionality as desired by the title. The partition assigned will be available to the title for exclusive use while the title is running. However, the data on the partition is not guaranteed to be available the next time the title is launched; thus, it cannot be relied on for permanent storage. The data on the partition may be available if other game titles have not used the partition in the interim.

This function need not be called if the game image file has the XINIT_MOUNT_UTILITY_DRIVE initialization flag set (which is the default), in which case the utility region is automatically mounted for the title at initialization time. The XINIT_FORMAT_UTILITY_DRIVE flag, in combination with XINIT_MOUNT_UTILITY_DRIVE, specifies that the partition should be formatted at initialization time. See the /INITFLAGS switch in the ImageBld documentation for more information.

After successfully mapping the utility region, the hard disk partition is available to the title through the Z:\ drive.

It is expected that a given title will always specify the same value for fFormatClean when mounting its utility partition. Specifying TRUE in some cases and FALSE in others can produce indeterminate behavior as to whether data on the partition is cached across a reboot or not. A title that wishes to retain cached data on its utility partition across reboots should always specify FALSE; a title that uses the utility partition for temporary storage only during a single boot of the game should always specify TRUE.

IDirect3D8::CheckDeviceType

Verifies whether or not a certain device type can be used on this adapter and expect hardware acceleration using the given formats.

HRESULT CheckDeviceType(
  UINT Adapter,
  D3DDEVTYPE CheckType,
  D3DFORMAT DisplayFormat,
  D3DFORMAT BackBufferFormat,
  BOOL Windowed
);

Parameters

Adapter
[in] Must be zero.
CheckType
[in] Must be D3DDEVTYPE_HAL.
DisplayFormat
[in] Member of the D3DFORMAT enumerated type, indicating the format of the adapter display mode for which the device type is to be checked. This must be a valid render target format.
BackBufferFormat
[in] Member of the D3DFORMAT enumerated type, indicating the format of the back buffer to be tested. This must be a valid render target format.
Windowed
[in] Must be FALSE.

Return Values

If the device can be used on this adapter, D3D_OK is returned.

D3DERR_INVALIDCALL is returned if Adapter equals or exceeds the number of display adapters in the system. This method returns D3DERR_INVALIDDEVICE if CheckType specified a device that does not exist. D3DERR_NOTAVAILABLE is returned if either surface format is not supported, or if hardware acceleration is not available for the specified formats.

Remarks

This function has been provided to ease in the porting of code from DirectX 8.0 to DirectX for the Xbox. On the Xbox, since the hardware is fixed and known, there is no use for this function.

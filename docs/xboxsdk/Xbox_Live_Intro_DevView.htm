Developer's View of Xbox Live

While subscribers primarily see Xbox Live as a means to more game content and increased interaction with other players, developers can view Xbox Live as a set of libraries and a toolset that provides a low-cost path to online functionality.

  • High-Functionality Toolset

Low-Cost Path to Online Functionality

The obstacles to adding online functionality to a game are numerous and formidable. For this reason, the Xbox Live libraries provides a low-cost, standardized, and mature solution for online games. The Xbox Live libraries enable you to bypass a lengthy and expensive development cycle by adding tested, powerful, and flexible code to your game. With the Xbox Live libraries, you can rapidly enhance your game with online features that are already compliant with the Xbox TCRs.

High-Performance Libraries

The Xbox Live libraries are specifically designed for maximum flexibility. For this reason, they use a partitioned and encapsulated feature set. The Xbox Live libraries are also designed to give your game the best performance possible in an online environment. To accomplish this, the Xbox Live libraries use an asynchronous task architecture that preserves the responsiveness of your title.

Encapsulated Functionality

To provide maximum flexibility, the Xbox Live libraries are designed so that you can select what functionality you want in your game. They do not require you to shoehorn features into your title that do not fit into your game's premise. For example, it is possible that the only online functionality you want to implement is downloading enhanced content. The Xbox Live libraries and toolset enable you to add this functionality while writing a minimum of code and without requiring your customers to subscribe to Xbox Live.

Likewise, the encapsulated functionality of the Xbox Live libraries lets you update your customers' copies of your game automatically. If you so desire, you can require that your customers subscribe to Xbox Live for more online functionality. This lets you add messaging and voice chats to your title. For even fuller online functionality, your game can also offer players the chance to participate in competitions with established systems in place to prevent cheating. Further, your game can support structured competitions involving teams or clans. The service can keep statistics for your players that give them the "bragging rights" they prize. You can even enable players to exchange game data such as maps they create with your title's map editor.

Implementing basic online features, such as content download, does not require you to implement advanced features, such as competitions and statistics. The flexibility of the Xbox Live libraries let you decide what features are appropriate for your title.

Asynchronous Task Architecture

In the past, the biggest drawback of online gaming has been network lag time. Lengthy network operations can induce a serious lag in the responsiveness of games.

To remedy the problem of network lag times, the Xbox Live libraries employ an asynchronous task architecture. For most tasks, titles call a library function to begin the task. The library function gives the title a task handle for the pending work that Xbox Live needs to perform. Titles are then expected to "pump" each of the task handles regularly by calling the XOnlineTaskContinue function and passing it the handle. This gives a time slice of work from the main thread to the Xbox Live libraries to process the next piece of the task. It also tells the Xbox Live service that the game is still functioning and waiting for results. In between calls to XOnlineTaskContinue, games can continue to process user input, render to the screen, or handle whatever other tasks need attending to.

When called, XOnlineTaskContinue returns a variety of values to indicate the progress of the task that corresponds to the handle your code passes to it. Generally, XONLINETASK_S_SUCCESS indicates that the task has completed whatever work was originally requested. When a title is done with the task, it calls the XOnlineTaskClose function to clean up resources associated with the task.

Many of the online tasks must be pumped regularly (once per frame) for the lifetime of the user's connection to the service. For example, the logon task provided by XOnlineLogon ia pumped once per frame as long as any user needs access to the online service. Closing this task or failing to pump it is equivalent to signing all current users out from Xbox Live. The FriendsStartup task from XOnlineFriendsStartup must be pumped for the lifetime of the signed in users. Similarly, the match session task, provided by XOnlineMatchSessionCreate, must be pumped by the host for the length of that session.

High-Functionality Toolset

In addition to the Xbox Live libraries, the Xbox SDK contains many tools to simplify the process of building an online title. For example, you can use the Content Download Manager to enable your title to download new content. The Content Download Manager is called by your title, and requires the addition of almost no specialized code.

Other tools in the toolset include Matchmaking Simulator (MatchSim), which helps you to rapidly create matchmaking queries, Network Simulator 2 (NetSim 2), which simulates real-world network conditions, and Xbox Live Authoring and Submission Tool (XLAST), which simplifies the process of building and submitting games to Xbox Live.

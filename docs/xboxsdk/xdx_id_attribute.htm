XDX id Attribute

The following XDX elements all contain an id attribute that is used as an identifier for the resource.

  • <cubeMap>
  • <indexBuffer>
  • <pixelShader>
  • <texture>
  • <vertexBuffer>
  • <vertexShader>
  • <volumeTexture>

For these elements, the id attribute is limited to 256 characters. If the identifier is longer than 256 characters, during compilation time the identifier will be truncated to 256 characters and XbRC will will issue a warning. Different resource types can share the same identifier. For example, a texture and a cube map may share the same id. However, different resources of the same type, such as two textures, should have different identifiers. If two or more resources of the same type share all of the same attributes (including id), those resources will be merged into a single resource at link time. If resources of the same type have the same id but otherwise different attributes, the linker will throw an error.

XOnlineCompetitionCreate

Creates a competition or tournament.

HRESULT XOnlineCompetitionCreate(
    DWORD dwUserIndex,
    DWORD dwTemplate,
    ULONGLONG qwTeamID,
    DWORD dwNumCompetitionAttributes,
    const XONLINE_ATTRIBUTE *pCompetitionAttributes,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwUserIndex
[in] Controller index (zero-based) of the player. The player must have been successfully logged on with a previous call to XOnlineLogon.
dwTemplate
[in] ID of a template describing the competition. This ID is provided by XLAST when you create your competition template.
qwTeamID
[in] Team ID. This ID is provided only when the competition being created is for teams.
dwNumCompetitionAttributes
[in] Number of competition attributes in the array specified by the pCompetitionAttributes parameter.
pCompetitionAttributes

[in] Array of XONLINE_ATTRIBUTE structures that contain the competition attributes. The dwAttributeID member of each structure must be set to one from the specified table, or an attribute ID defined for the competition dataset in XLAST. All of the following attributes must be listed, in the order they appear.
Attribute Value Description
XONLINE_COMP_ATTR_REG_OPEN 0x80010001 Date and time that players may begin registering for the competition, in FILETIME format.
XONLINE_COMP_ATTR_REG_CLOSE 0x80010002 Date that registration for the competition is scheduled to end, in FILETIME format. This date must be greater than or equal to the date specified in the XONLINE_COMP_ATTR_REG_OPEN attribute.
XONLINE_COMP_ATTR_COMP_START 0x80010003 Date that the competition is scheduled to begin, in FILETIME format. This date must be greater than or equal to the date specified in the XONLINE_COMP_ATTR_REG_CLOSE attribute.
XONLINE_COMP_ATTR_COMP_CLEANUP 0x80010004 Number of days after the last round that the competition should be deleted from the system.
XONLINE_COMP_ATTR_ROUND_FREQUENCY 0x80010005 Used in combination with XONLINE_COMP_ATTR_ROUND_INTERVAL and XONLINE_COMP_ATTR_ROUND_DAY_MASK to specify how often the competition's rounds are held. For example, you can hold rounds twice a week, once a month, three times a month, and so forth.
XONLINE_COMP_ATTR_ROUND_INTERVAL 0x80010006 Time interval between rounds. The unit of time used for the interval is specified by XONLINE_COMP_INTERVAL_UNIT.
XONLINE_COMP_ATTR_ROUND_DAY_MASK 0x80010007 Selects the day or days on which the rounds are held. Used only when XONLINE_COMP_ATTR_ROUND_INTERVAL is set to XONLINE_COMP_INTERVAL_DAILY.
XONLINE_COMP_ATTR_ROUND_ONE_START 0x8001000C Date and time that round one is scheduled to start, in FILETIME format. The date must be greater than, or equal to the date specified in the XONLINE_COMP_ATTR_COMP_START attribute plus any value that is currently specified for the XONLINE_COMP_ATTR_REMINDER_MIN attribute.
XONLINE_COMP_ATTR_ROUND_ONE_END 0x8001000D Date and time that round one is scheduled to end, in FILETIME format.
XONLINE_COMP_ATTR_IS_TEAM_COMP 0x80010010 Indicates whether the competition is played by teams rather than just two individuals.
XONLINE_COMP_ATTR_TEAM_SIZE 0x80010011

Number of players on each team in a team competition.

Note    It is left to the title to enforce this value.
XONLINE_COMP_ATTR_MAX_PUBLIC_SLOTS 0x80010012 Maximum number of publicly available slots in the competition.
XONLINE_COMP_ATTR_MAX_PRIVATE_SLOTS 0x80010013 Maximum number of privately available slots in the competition.
XONLINE_COMP_ATTR_MIN_SLOTS 0x80010014 Minimum number of slots required for this competition. If the number of private and public entrants that join is less than this value as of the date specified by the XONLINE_COMP_ATTR_REG_CLOSE attribute, the competition is canceled. The value of this attribute must be at least 2.
XONLINE_COMP_ATTR_BYES_GRANTED 0x80010017 Specifies whether one or more byes are granted.
XONLINE_COMP_ATTR_REMINDER_MIN 0x8001001A Number of minutes before the beginning of a round that a reminder should be sent to the round participants.

hWorkEvent
[in, optional] Handle of an event you want to associate with this task. If you do not want to assign an event, set this parameter to NULL.
phTask
[out] Pointer to an XONLINETASK_HANDLE type. The handle is associated with the task started by XOnlineCompetitionCreate. The handle can then be used in calls to XOnlineTaskContinue to perform work associated with this task. When the task is complete, call XOnlineCompetitionCreateGetResults to get the ID of the newly created competition. Call XOnlineTaskClose to close the task.

Return Values

Returns S_OK if successful, or an error code otherwise. These error codes can include the following.

Error Code Description
E_INVALIDARG A required attribute is missing, or one or more of the parameters or attributes specified is invalid.

Remarks

For most single-elimination competitions, XOnlineCompetitionCreateSingleElimination should be invoked instead of this function. XOnlineCompetitionCreate should be used only when the parameters provided by XOnlineCompetitionCreateSingleElimination are insufficient to configure the competition.

After XOnlineTaskContinue completes, call XOnlineCompetitionCreateGetResults to obtain the competition ID for the newly created competition.

Note    The user specified in dwUserIndex (or the team specified in qwTeamId) is not automatically enrolled in the competition. After the call to XOnlineCompetitionCreate completes successfully, the application should call XOnlineCompetitionManageEntrant to enroll the player or team.

The Competition Name attribute must always be set as an additional attribute (it is not a member of the XLAST DefaultAttribute structure). It is used by the Alerts system to deliver the competition reminders. The template for Competition Reminder alert text requires that the competition name be specified. For example, where TitleName is the title name and CompetitionName is the competition name, your title might display this alert: "Your match in the TitleName competition, CompetitionName, starts soon."

The Competition Name attribute ID is 0x8011000E, which corresponds to the XLAST attribute att_competition_name. The corresponding variable name for this attribute in the competition code generated by XLAST is pwszDisplayName.

Requirements

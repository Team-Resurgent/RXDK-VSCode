Optimizing a Game Disc

XbGameDisc includes an optimization feature that can help you make significant improvements in the performance of your game. It gaines this performance improvement by analyzing how users typically play your game. To reduce seek times, XbGameDisc watches for files that your game typically accesses sequentially. It stores these files sequentially on the game disc. In general, XbGameDisc tries to keep folders and their files in the same layer.

To produce data for the XbGameDisc tools optimization feature to analyze, you must perform an emulation run with logging enabled. Each emulation run is treated as unique. This gives you the ability to decide whether to use optimizations from previous runs. If you optimize your disc layout based on an emulation run, you can then perform another emulation run using the optimzed layout. With the data from the second run, you can further optimize your disc. Alternatively, you could do two runs without changing the layout and see which gave the best results. You might decide that the results of the second run are more valuable or accurate and optimize based strictly on the results of the second run.

As you perform your emulation run, be aware that you do not have to play your game in such a way as to cause it to load every file. XbGameDisc can provide you with significant performance improvements without having to optimize the position of every file on the disc. Optimizing the game primarily involves optimizing the layout of the most frequently used files.

During emulation runs, you should play the game the way you would expect most users to play it most of the time. Although you should collect as much emulation data as you reasonably can for optimization, you do not have to play your game completely through to get significant performance improvements. For example, if your game has 20 levels, playing through 10 of them might be enough to get the optimizations you're seeking. In this example, you would typically play a level, optimize, then repeat the process for each successive level. In a situation such as this, you'll generally see the largest performance improvements with the first few emulation runs. Later runs generally produce smaller performance gains. You end the optimization process when you determine that the performance gains are small enough that further emulation passes would not be worthwhile.

Performing an Optimization

To optimize the layout of your game disc, use the following procedure.

  1. Perform one or more emulation passes with logging enabled. For information on doing emulation passes and enabling logging, see Emulating a Game Disc.
  2. From the XbGameDisc tool's main menu, click Tools, and then click Optimize. XbGameDisc displays the following dialog box.

  3. Click Next. XbGameDisc displays the following dialog box.

  4. Select a log file by clicking the button with the ellipsis (three dots). Use the dialog box that appears to navigate to a log file produced from one or more emulation runs. After you select a log file, XbGameDisc enables the drop-down list labeled Emulation pass data. Each emulation pass is listed as a separate item in this list, as shown in the following figure.

    5. Note    Emulation logs created with versions of XbGameDisc released prior to June 20003 do not work with versions of XbGameDisc released in June 2003 or later.

  5. Use the Emulation pass data list to select the emulation pass you want to use as the basis for optimization. Click Next.
  6. In the dialog box that appears, click Finish. The optimization wizard updates your game disc's layout.

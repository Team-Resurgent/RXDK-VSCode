CreateThread

The CreateThread function creates a new thread to execute within the title. 

HANDLE CreateThread(
  LPSECURITY_ATTRIBUTES lpThreadAttributes, // SD
  DWORD dwStackSize,                        // initial stack size
  LPTHREAD_START_ROUTINE lpStartAddress,    // thread function
  LPVOID lpParameter,                       // thread argument
  DWORD dwCreationFlags,                    // creation option
  LPDWORD lpThreadId                        // thread identifier
);

Parameters

lpThreadAttributes
[in] Ignored; use NULL.
dwStackSize
[in] Specifies the initial commit size of the stack, in bytes. The system rounds this value to the nearest page. If this value is zero, the default stack size of the title is used.

Note   Currently a minimum of 12 KB will be allocated and committed for the thread's stack, even if a smaller size is specified.

lpStartAddress
[in] Pointer to the application-defined function of type LPTHREAD_START_ROUTINE to be executed by the thread; represents the starting address of the thread.
lpParameter
[in] Specifies a single parameter value passed to the thread.
dwCreationFlags
[in] Specifies additional flags that control the creation of the thread. If the CREATE_SUSPENDED flag is specified, the thread is created in a suspended state and will not run until the ResumeThread function is called. If this value is zero, the thread runs immediately after creation. At this time, no other values are supported.
lpThreadId
[out] Pointer to a variable that receives the thread identifier.

If this parameter is NULL, the thread identifier is not returned.

Return Values

If the function succeeds, the return value is a handle to the new thread.

If the function fails, the return value is NULL. To get extended error information, call GetLastError.

Note that CreateThread may succeed even if lpStartAddress points to data, code, or is not accessible. If the start address is invalid when the thread runs, an exception occurs and the thread terminates. Thread termination due to a invalid start address is handled as an exception with the title.

Remarks

The default stack size for the title can be set with the linker's STACK switch.

The stack size specified (or the default if zero is used) represents the reserved and committed size of the stack. The stack will not grow beyond this size. The end of the stack is protected by an uncommitted page entry. Overflowing the stack results in a stack overflow exception. This stack overflow exception cannot be caught by the title using an exception handler, so the exception will cause the console to freeze.

The stack is used for thread local storage as well as for program stack, so a title that uses many variables declared as '__declspec(thread)' may need to specify a larger stack size.

The number of threads a title can create is limited by available memory. If you reduce the default stack size, you can create more threads.

The thread execution begins at the function specified by the lpStartAddress parameter. If this function returns, the DWORD return value is used to terminate the thread in an implicit call to the ExitThread function. Use the GetExitCodeThread function to get the thread's return value.

The thread is created with a thread priority of THREAD_PRIORITY_NORMAL. Use the GetThreadPriority and SetThreadPriority functions to get and set the priority value of a thread.

When a thread terminates, the thread object attains a signaled state, satisfying any threads that were waiting on the object.

The thread object remains in the system until the thread has terminated and all handles to it have been closed through a call to CloseHandle.

Requirements

Header: Declared in Winbase.h, for include use Xtl.h.
Import Library: Use Xapilib.lib

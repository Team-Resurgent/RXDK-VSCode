(Remote) Target System Tools

The target system tools provide a means of using a network connection to (remotely) manage your target system from your development system.

The Tools

The tools in this section include:

  • Remote Attribute Tool (XbAttrib)
  • File Transfer Tool (XbCp)
  • Remote Delete Tool (XbDel)
  • Remote Directory Tool (XbDir)
  • Remote Make Directory Tool (XbMkDir)
  • Remote Reboot Tool (XbReboot)
  • Remote Rename Tool (XbRen)
  • Remote Color Adjust Tool (XbColorAdjust)
  • Remote Config Tool (XbSetCfg)
  • Remote Screen Capture Tool (XbCapture)
  • Security Tool (XbManage)
  • Shell Extension
  • Xbox Movie Capture Tool (XbMovie)

The "-x" Switch

Each of the target system tools accepts an -x switch. Use the -x switch to tell the tool which Xbox target system you wish to access. This value is stored in the registry the first time you specify a machine name. As a result, if you connect to only one target system, you need not specify the machine name each time you use this tool.

If you specify a target system using the Xbox name that you chose when you first configured the Xbox target system, the development system and the target system must be on the same subnet. If you specify a numeric IP, this restriction does not apply.

Remote Drives

The paths of files on the target system are specified in the form xE:\path\file.ext, where E is the remote drive. Be sure to use the preceding "x" when specifying the remote drive letter.

DriveMapping
D: Mapped to the directory from which the title was launched.
E: The drive where titles are stored during development.
F:–R: Dynamically allocated and/or reserved drives. Some drives in this region are used to map memory units.
S: Root of TDATA.
T: The drive where title data (TDATA) is stored.
U: The drive where user data (UDATA) is stored.
V: Root of UDATA.
W: Used to mount TDATA of another title.
X: Used to mount UDATA of another title.
Z: The drive containing the title's cache data.

Note   TDATA is where persistent title data, such as configuration settings and downloaded levels, is stored. UDATA is where persistent user data, such as savegames, is stored. See Accessing Storage Regions for more information.

Performance

If you are running a CPU-intensive title on the target system, the performance of the target system tools may be degraded. If performance is a problem, reboot the target system into the XDK Launcher before using one of these tools.

Numeric Xbox Names

If you set the Xbox name to purely numeric characters (0–9), the remote tools fail with the following error message:

error: Cannot connect to Xbox

The remote tools assume that a purely numeric name is a 32-bit representation of an IP address. Be sure to include at least one non-numeric character in your Xbox name.

File System Considerations

Consider the following when using the Xbox file system and command-line tools:

  • The Xbox file system does not support long file names.
  • The Xbox file system does not support Unicode characters in file names.

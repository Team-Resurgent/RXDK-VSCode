XGSetIndexBufferHeader

Manually creates an index buffer resource.

VOID WINAPI XGSetIndexBufferHeader(
  UINT Length,
  DWORD Usage,
  D3DFORMAT Format,
  D3DPOOL Pool,
  IDirect3DIndexBuffer8* pBuffer,
  UINT Data
);

Parameters

[in] Length
Ignored.
Usage
[in] Ignored.
Format
[in] Ignored.
Pool
[in] Ignored.
pBuffer
[in, out] Address of the IDirect3DIndexBuffer8 interface, representing the created index buffer resource that is to be filled.
Data
[in] Offset to the data held by this resource.

Return Values

None.

Remarks

Index buffers are memory resources used to hold indices and are similar to both surfaces and vertex buffers. They are supported as a way to index into a vertex buffer, but otherwise provide no significant performance improvements.

To use index buffers, create an index buffer, lock it, fill it with indices, unlock it, pass it to IDirect3DDevice8::SetIndices, set up the vertices, set up the vertex shader, and call IDirect3DDevice8::DrawIndexedPrimitive for rendering.

Only 16-bit indices are supported.

All resources, including index buffers, include a header that contains all of the information needed by the driver to process the resource and a data allocation that contains information used by the GPU. The resource headers are used only by the driver and so should be allocated from normal cached memory. The header contains all of the information needed by the driver to set up the hardware so that it can use the information stored in the data allocation.

Unless the title is manually creating a resource, it is recommended that the title use IDirect3DDevice8::CreateIndexBuffer.

Unlike other Direct3D resources, index buffers are read by the CPU and not by the GPU. As a result, titles should not call IDirect3DResource8::Register in order to register the memory address of the resource.

XSetFileCacheSize

The XSetFileCacheSize function changes the number of bytes reserved for the file system cache.

BOOL XSetFileCacheSize(
    SIZE_T dwCacheSize
);

Parameters

dwCacheSize
[in] Number of bytes that should be reserved for the file system cache. The number of bytes is rounded up to a multiple of the page size (4 KB on the Xbox).

Return Values

If the function succeeds, the return value is TRUE.

If the function fails, the return value is FALSE. To get extended error information, call GetLastError.

Remarks

The file system cache is used to hold file system metadata, such as directory entries and the location of file data. A title might change the amount of memory reserved for the cache in order to tune performance based on the title's file system usage characteristics.

It is possible to use XSetFileCacheSize to change the size of the file system cache to zero. If the file system cache is set to zero bytes, calls to other file I/O libraries may fail because they can't access file system metadata. The file system cache should only be changed to zero when the title no longer needs to access the file system. With the system cache set at zero, many file system operations will fail with ERROR_NOT_ENOUGH_MEMORY (STATUS_NO_MEMORY).

The default size for the file system cache is 64 KB.

The maximum size for the file system cache is 8 MB. Attempting to specify a size greater than 8 MB will cause the function to fail with ERROR_INVALID_PARAMETER.

The XSetFileCacheSize function can be safely called while other file I/O is pending.

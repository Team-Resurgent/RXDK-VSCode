XCalculateSignatureBegin

Begins the process of calculating a signature over a save game or other data set. Use XCalculateSignatureBegin to calculate a signature for data saved to the hard disk or memory units.

HANDLE XCalculateSignatureBegin(
  DWORD dwFlags
);

Parameters

dwFlags
[in] Value indicating how the signature should be calculated. Must be one of the following:
Value Meaning
XCALCSIG_FLAG_SAVE_GAMEUse the title-specific key to calculate a signature over a save game or other data set.
XCALCSIG_FLAG_NON_ROAMABLE Use the title-specific key to calculate a signature over a save game or other data set, in such a way that the data is non-roamable—that is, the data/signature pair is valid only on the current Xbox console.
XCALCSIG_FLAG_CONTENTCalculate a content signature for verification of downloaded content. Matches signatures created by the XBldSigs and BuildOffer tools, and by the XCalculateContentSignature function.
XCALCSIG_FLAG_DIGESTCalculate a digest over a data set. The digest, along with the Xbox Live signature of the digest, can be submitted to the Xbox Live signature service for verification.
XCALCSIG_FLAG_ONLINEUse the Xbox Live key to calculate a signature over a data set (usually a digest). This signature is written to storage along with the data, and can be submitted to the Xbox Live signature service for verification.

Return Values

If the function succeeds, the return value is a signature handle that can be used in subsequent calls to XCalculateSignatureUpdate and XCalculateSignatureEnd.

If the function fails, the return value is INVALID_HANDLE_VALUE. To get extended error information, call GetLastError.

Remarks

This function opens a signature calculation handle in preparation for calculating a signature over a data set. Data is added to the running hash with calls to XCalculateSignatureUpdate. Multiple calls to add data can be made if desired, and then the hash is completed (and in some cases signed) with a call to XCalculateSignatureEnd.

After calling XCalculateSignatureBegin, use the returned handle in calls to XCalculateSignatureUpdate to add the data for which the signature will be calculated, and then finally call XCalculateSignatureEnd to complete signature calculation and to free the signature handle and other resources associated with the signature.

The size of the calculated signature varies depending on the signature type. Call XCalculateSignatureGetSize to determine the signature size, allocate an appropriately sized buffer, and pass that buffer to XCalculateSignatureEnd to retrieve the calculated signature.

XCalculateSignatureBegin calculates signatures over data using various keys and digest methods. The dwFlags parameter specifies the type of signature or calculated digest. The three signature types are save game signatures, content signatures, and Xbox Live signatures.

Save Game Signatures

Use save game signatures to sign and verify data (a save game, for example) that is written to Xbox storage and later used in offline or system-link play (in other words, while not connected to the Xbox Live service).

When calculating a save game signature, a hash is computed over the data and the hash is signed using a per-title key. Only data that has not been tampered with and that is signed by the current title will compute to the same signature. A title can save a game along with the calculated signature for that game, and then, while later loading the game, recalculate a signature for the loaded data and verify that the signatures match.

If XCALCSIG_FLAG_NON_ROAMABLE is specified, the hash is additionally signed by a key unique to the current Xbox console. The data will compute to a different signature on any other console, thus will only be valid on the initial console that first performs the calculation and writes the data. For example, if a save game was signed and the game and its signature written to an MU, and that MU taken to another Xbox console, and a player attempted to load the save game on the new console, the signature would not match and the game title should fail the load of that save game.

For more information about save games and save game signing, See the Save Game Signing section of XTL Services, the LoadSave Sample, the Xbox Technical Certification Game, and Save Games.

Content Signatures

To verify content downloaded through the Xbox Live service or from a game disc, use content signatures.

  • Specify XCALCSIG_FLAG_CONTENT in dwFlags to compute a content signature over the content,
  • Use XLoadContentSignatures to retrieve a signature from the content's signed metadata file, and then
  • Compare the computed signature with the retrieved signature.

The XCalculateContentSignature function computes the same signature that is computed when XCALCSIG_FLAG_CONTENT is used. XCalculateContentSignature computes the signature over an in-memory data set with a single call.By contrast, XCalculateSignatureBegin permits repeated calls to XCalculateSignatureUpdate, over smaller portions of the data set, so that the entire data set does not have to be in memory at one time. Signatures can thus be computed over files that are larger than the amount of available physical memory.

For more information, see Content Signatures and Downloading Content from Xbox Live.

Xbox Live Signatures

Content or data that will ever be uploaded to the Xbox Live service after first being written to the hard disk requires an extra level of security beyond that used for save game signing. Xbox Live signatures are similar to save game signatures in that they are used to sign data that will be written to storage, but differ in that the signature is stronger and contains additional information. Xbox Live signatures:

  • Use an Xbox Live key that is available only to titles while they are connected to Xbox Live.
  • Contain information about the title, Xbox, users, and the time at which the data was signed.
  • Can be expired or denied by the signature service at some time after the signature was issued (in response to a security issue, for example).

To sign data that will be written to the hard disk:

  • Use XCALCSIG_FLAG_ONLINE to calculate a signature over the data.
  • Save the calculated signature to the hard disk along with the data.

XCalculateSignatureBegin begins the calculation process, XCalculateSignatureUpdate updates the running digest, and XCalculateSignatureEnd computes the signature over the digest. A running digest can be computed in memory as the data is being written to the hard disk, and, when the data is complete, the digest signed and written to disk alongside the data.

Using XCALCSIG_FLAG_ONLINE requires that:

  • The title is currently logged onto the Xbox Live service.
  • The XONLINE_SIGNATURE_SERVICE service ID was specified when the title called XOnlineLogon, and a connection was successfully made to the signature service.
  • The title linked with xonlines.lib.

Before using previously stored data or uploading that data to the Xbox Live service, verify the data:

  • Load the data and the Xbox Live signature saved with the data.
  • Use XCALCSIG_FLAG_DIGEST to calculate a digest over the data.
  • Verify that the signature and data are valid by calling XOnlineSignatureVerify with the Xbox Live signature and the calculated digest.

Data read from volatile storage (the hard disk or memory units) may be uploaded to the Xbox Live service or used online only if that data was initially stored with an Xbox Live signature, and then verified as valid, at the time of use or upload, with XOnlineSignatureVerify.

Note that it may be desirable to sign data using both a save game signature and an Xbox Live signature, so that the data could be used both offline and online. Data signed only with an Xbox Live signature would be usable only while online, because the signature service is required in order to verify the signature with XOnlineSignatureVerify.

For more information, see Xbox Live Signatures.

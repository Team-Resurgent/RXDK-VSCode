Advertising Games to Other Players

When a title is ready to make its session available to other players on Xbox Live, it calls the XOnlineMatchSessionCreate function to begin advertising the session on the matchmaking service. This call provides the matchmaking service with an initial set of attribute values and initial player information. After the network task for this call is complete, the title can obtain the session information by calling XOnlineMatchSessionGetInfo with the task handle returned by XOnlineMatchSessionCreate. In addition to identifying the online task, the task handle also identifies the Xbox Live game session for the call to XOnlineMatchSessionGetInfo. XOnlineMatchSessionGetInfo uses the handle to return an ID for the session. This ID is used to identify the session to other matchmaking functions. XOnlineMatchSessionGetInfo also returns a key-exchange key. Use the session ID and the key-exchange key in a call to XNetRegisterKey to register a secure networking session.

The XOnlineMatchSessionDelete function removes any information about a gaming session from the matchmaking service. Therefore, titles call the XOnlineMatchSessionDelete function when they no longer need the matchmaking service. This could occur when either: 1) the online-game session is finished, or 2) the title is no longer accepting new players for the session.

Note that for certain games (join-in-progress games, for example), the title could continue to advertise the session even though the session is full. That way, if a player leaves the game (creating an opening for a new player), the title could update the session to reflect this fact by calling XOnlineMatchSessionUpdate.

The following table provides a brief summary of the matchmaking functions used to create, update, and delete game sessions.

Function Description Memory Use
XOnlineMatchSessionCreate Advertise a session. Yes - allocates memory for an Xbox Live task handle.
XOnlineMatchSessionGetInfo Retrieve information for an advertised session. No.
XOnlineMatchSessionUpdate Update a previously advertised session. No.
XOnlineMatchSessionDelete Stop advertising a session. No.

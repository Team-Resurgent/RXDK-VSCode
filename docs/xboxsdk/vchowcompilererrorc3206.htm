Compiler Error C3206

'function' : invalid template argument for 'arg', missing template argument list on class template

A template function is defined as taking a template type argument. However, a template template argument was passed

The following example generates C3206:

// C3206.cpp
template <class T>
void f()
{
}

template <class T>
struct S
{
};

void f1()
{
   f<S>();   // C3206
   // try the following line instead:
   // f< S<int> >();
}

This error can also be generated by using class templates as template type arguments. Although such usage worked in Visual Studio .NET, it is invalid C++ and is no longer allowed by the compiler. The following sample compiles in Visual Studio .NET but will fail in Visual Studio .NET 2003:

// C3206b.cpp
template <class T>
struct S
{
};

template <class T>
void func()
{   // takes a type
   T<int> t;
   // T t;
}

int main()
{
   func<S>();   // C3206 S is not a type.
   // To resolve, use the following line and uncomment T t above
   // func> S>int> >();
}

If a template template parameter is necessary, then the resolution for the error that is valid in both the Visual Studio .NET 2003 and Visual Studio .NET versions of Visual C++ requires you to wrap the function in a template class that takes a template template parameter.

// C3206c.cpp
template <class T>
struct S
{
};

template< template<class> class TT>
struct X
{
   static void func()
   {
      TT<int> t1;
      TT<char> t2;
   }
};

int main()
{
   X<S>::func();
}

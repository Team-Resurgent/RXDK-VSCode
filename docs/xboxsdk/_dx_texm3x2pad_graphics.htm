texm3x2pad

Used in combination with other texture address operations to perform 2×3 matrix multiplies.

texm3x2pad   tDest, tSrc0

Registers

tDest
Destination register, holding the result of the operation.
tSrc0
Temporary source register, specifying the input argument.

Remarks

This instruction is used to represent stages where only the texture coordinate is used. These corresponding stages have not textures bound and no sampling will occur. The input argument, t0, should still be specified.

The following example shows how the texm3x2pad instruction might be used.

tex t0             ; Define t0 as a standard 3-vector.
texm3x2pad t1, t0  ; Perform fist row of matrix multiply.
texm3x2tex t2, t0  ; Perform second row of matrix multiply to get a
                   ; 2-vector with which to sample texture 2.

Note that this instruction has an unnamed input that is the assigned texture that corresponds to this texture stage.

This instruction actually performs a dot product between the texture coordinates of the current stage and the output value from the specified input stage. Color values are unsigned integers in the range 0x0 to 0xff, but through the use of input modifiers, these values can be interpreted as signed numbers.

If the _bx2 input modifier is applied to the specified input argument, the color values are mapped to signed numbers using the formula

( ColorValue – 0x80 ) / 127.0f

So,

  • 0x0 is mapped to –1.007874016f (–128 / 127),
  • 0x1 is mapped to –1.0f,
  • 0x80 is mapped to 0.0f, and
  • 0xff is mapped to 1.0f.

If there is no input modifier applied to the specified input argument, the color values are mapped to the range zero to one using the formula

ColorValue / 255.0f

So,

  • 0x0 is mapped to 0.0f, and
  • 0xff is mapped to 1.0f

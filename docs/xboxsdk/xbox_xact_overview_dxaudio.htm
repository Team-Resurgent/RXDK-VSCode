XACT Overview

This section offers a general overview of the versions, benefits, and constraints of the Xbox Audio Creation Tool (XACT). The information here is somewhat oriented toward a programming audience. If you are a composer or sound designer, you may want to proceed to the XACT Authoring Guide.

XACT Executables

There are two different executables for XACT (Xactbld.exe and Xact.exe). They differ as follows:

  • Xactbld.exe can be run from the command line, making it easier to integrate into a build process.
  • Unlike Xactbld.exe, Xact.exe provides a graphical user interface, which makes it easier for audio engineers to create and manage wave banks.
  • Xact.exe enables you to preview audio content using the Audio Console Application on either the Xbox Development Kit (XDK) or Xbox Debug Kit.

Notes on the Japanese Version of XACT

The following information applies to the Japanese version of XACT.

  • The XDK setup program installs both the English and Japanese versions of XACT on Japanese versions of Windows. In this case, XDK Setup creates two shortcuts on the Windows Start Menu, one Japanese and the other English. The Japanese shortcut points to the Japanese version of XACT (XACT-JP.exe). The English shortcut points to the English version of XACT (XACT.exe). In previous versions of the XDK, only the English version of XACT was installed, and Setup created a Japanese shortcut that pointed to the English version.
  • Running the English version of XACT after running the Japanese version will map the .xap file association to use the English version. Therefore, it is important not run XACT.exe if you wish to maintain the original file association with XACT-JP.exe.
  • The Japanese version of XACT supports only 8-bit characters for the names of files, XACT sounds, and wave banks. In general, non-ANSI text is not supported in either version of XACT. This includes Unicode Kana.

Benefits

Following are some of the benefits provided by XACT.

  • XACT handles WMA, WAV, and AIFF files and properly parses loop points in WAV and AIFF files.
  • XACT can combine several audio files into a single wave bank (.xwb) file.
  • XACT supports real-time conversion of 16-bit PCM data to 8-bit PCM data in addition to real-time compression of 16-bit PCM data using the Xbox ADPCM CODEC.
  • XACT provides a way to preview audio content on the Xbox hardware when the Audio Console Application is running on an Xbox Development Kit or Xbox Debug Kit (Xact.exe only).
  • The IDirectSoundBuffer8::SetBufferData method can be applied directly to XACT wave banks.
  • XACT's individual audio files within a wave bank can be accessed and played using the IDirectSoundBuffer8::SetPlayRegion and IDirectSoundBuffer8::SetLoopRegion methods.
  • XACT simplifies audio data management. Rather than being required to download several different audio files, titles can now create a single wave bank (or a set of wave banks) that holds the audio data necessary for a particular part of the game, and then simply download that single file.
  • XACT provides a powerful feature called zero-latency streaming that solves the problem of fitting sounds within allocated memory. This feature is discussed in more detail in XACT Concepts.

Constraints

When using the XACT tool, keep the following points in mind.

  • XACT's output files are stamped with version information based on the version of the XACT GUI being used by the sound designer. As new versions of XACT are released, older versions of XACT projects can be loaded, but will be converted to the new version. Developers must link to the XACT library files that match the version of the files being generated by the sound designer.
  • Only in-memory wave banks are limited to 8 MB (only 8 MB of Xbox memory can be used for sounds.) Streamed wave banks, because they don't reside in memory, can include up to 1.8 GB of data.
  • XACT does not handle wave files compressed using ACM (Audio Compression Manager) CODECs other than the Microsoft ADPCM CODEC, but it does handle WMA files (which are compressed with a CODEC other than the ADPCM CODEC.)
  • When looping WMA files, the XACT engine flushes the internal WMA decoder between loops. This is an asynchronous operation that may take several milliseconds and will generate a stream starvation message. Silence will be played between loops.
  • Version 8 WMA files are not currently supported by the Xbox WMA decoder.
  • Stereo and other multichannel sound sources must be played on the default sound source.
  • The Audio Console Application limits the number of concurrent streams coming from the hard disk to 50. The actual resource limit depends on disk layout, the sound format, and other disk activity, but is set to a fixed number for consistency.

Known Issues

The following are known issues with XACT.

  • Auditioning an ADPCM wave with a length 1 loop region at its end causes XACT to fault.
  • Looping very small streaming waves does not work properly; do not loop waves in a streaming wave bank smaller than two packets, as set by the dwPacketSizeInMilliSecs member of the XACT_WAVEBANK_STREAMING_PARAMETERS structure.
  • Sound banks with parameter-controlled variation playlists must have at least one variation with a maximum weight of 100.00. Otherwise, auditioning the sound causes XACT to disconnect from AudConsole.
  • If a new sound is added to a sound group in the XACT GUI, the volume of the sound group is not immediately added to the volume of the new sound. To work around this issue, adjust the volume of the sound group slightly after adding the new sound.
  • The XACT GUI does not correctly display the default value for the max distance 3D sound property. The XACT GUI displays a value of 100,000, but the actual default value is 1,000,000,000. However, if the user changes any of the values on the 3D sound properties page, the XACT GUI adjusts the actual value down to 100,000, thereby making the displayed value correct.

  • The following actions should not be performed during in-game auditioning. Performing them may cause undefined behavior or crashes:

    • Deleting sounds or cues
    • Deleting wave banks
    • Renaming sound banks or wave banks
    • Deleting waves in wave banks
    • Changing wave bank types (for example, from streaming to in-memory)
    • Setting the Parameter Control slider for a game-initiated slider-controlled sound

    Over time, the XACT GUI may be updated to specifically prohibit these actions.

  • In the June 2003 and later XDK releases, if the IXACTWmaPlayList::Add method is used with the XACTWmaPlayListAdd_Directory value to add a directory that contains only one WMA soundtrack, the cue for the WMA playlist does not loop. However, if the directory contains more than one soundtrack, the cue loops correctly.
  • In the June 2003 and later XDK releases, if a wave variation's run-time parameter control (RPC) is set to a value that is outside the ranges specified for that wave variation, the system leaks 64 bytes of memory. The memory leak occurs only with wave variations, and only if the sound cue is primed using the Prepare method before playing it.
  • If a track uses marker events inside of a loop defined by a loop start event, the time stamps for the marker events must be less than the duration of the wave associated with that track. The marker events may not occur at or after the wave has stopped. If these conditions are not met, XACT may hang when the wave is played.

  • Copying a sound with a 3D preset from its current sound bank to a new sound bank will cause the project file to become unusable. To work around this problem, follow these steps:

    1. Open the .XAP file in a text editor.
    2. Go to the line of the error.
    3. Within the sound definition where the error occurred, delete the line beginning with "3D Preset".

    The project should then be able loaded as expected. The sound will have default 3D settings without a preset.

  • Wave variations added to a sound while the sound is playing will cause a crash when the sound is stopped. To avoid this situation, only add wave variations to sounds that are not currently playing.

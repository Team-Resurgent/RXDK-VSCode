What Is XLAST?

The Xbox Live Authoring and Submission Tool provides a visual and interactive environment for authoring submissions to Xbox Live. The submissions can be either content packages, which includes both online games and game components, or game stats updates, which can be made visible on Xbox Live and on the Web.

XLAST automates many of the tasks associated with authoring submissions. This has many advantages for you when you are preparing your submissions—for example:

  • XLAST simplifies the process of submitting Xbox Live game components.
  • It validates the submission before you send it. As a result, XLAST significantly reduces the number of times you need to submit your Xbox Live updates to Microsoft.
  • It provides you with a single tool for multiple submission types.
  • It enables you to build content packages for offline delivery without building associated online offering information.
  • It removes the need for you to edit metadata files when submitting content. It automatically incorporates the required metadata into the final submission.

Walkthrough: Authoring a Submission for Xbox Live

Suppose you are developing a content update for your massively multiplayer game called "Slugmasters of Gardenia" for Xbox Live, and you're ready to begin testing it. In this scenario, you would follow these steps.

  1. Compile the game's source code.
  2. Run XLAST to put the submission together for testing on your local development PC.
  3. Use the New Content Wizard to create a content package.
  4. Use the Document View to specify the folders and files that go into the content package.
  5. Build the submission.
  6. Test the submission and repeat this process from step 1 until the submission works as planned.
  7. Submit the package to PartnerNet for online testing. Repeat the process from step 1 as needed.

    8. Note    You will receive an email notifying you when your submission to PartnerNet is available for testing. This may take up to 24 hours.

  8. Download the content package from PartnerNet, just as a user would from Xbox Live, and test it to ensure it works as expected. Repeat from step 1 as needed.
  9. Rebuild the content package for online use and submit it to Microsoft for certification.

Project Files

The XLAST tool views each submission as a project. It keeps information about a given submission in a project file. The primary method of modifying the contents of a project file is through the XLAST Document View.

Project files are saved in XML format. Therefore, it is possible for you to edit them with a text editor. However, it is strongly recommended that you do not do so.

Note    Although XLAST allows you to save multiple projects in the same directory, this is an unsupported configuration. It can lead to projects improperly sharing data files. To prevent this, each project should be saved into its own directory.

Document View

The Document View provides you with a listing of the contents of your submission. In general, the Document View works essentially the same way no matter what type of submission you are creating. However, it does have specializations for particular submission types. These are discussed in later sections of the XLAST documentation.

The Document View displays two panes in its window. The left pane shows a tree view of the project file. When you click on any individual item in the tree, the right pane displays all of the properties of that item. If a given item has no properties, the right pane of the Document View displays a message indicating that fact.

As with most Windows programs, right-clicking on an item in the tree invokes a context-sensitive menu. This menu contains all of the actions that can be performed on that item.

Output Window

In addition to the Document View window, XLAST also displays the Output window. When you build your project, XLAST displays messages about the status of the build in the Output window.

Creating, Saving, and Loading Submissions

To create a project file for a new submission, click New on the File menu. XLAST displays a dialog box that asks what type of project you want to create. Select the project type and click OK. XLAST then starts the New Project Wizard for the selected type of project.

The New Project Wizard asks you for the information required to create the project. Using this information, it then creates a project file for you. Upon completing the New Project Wizard, you can begin to add files to your project or edit its properties using the Document View window.

Project files can be saved by clicking Save or Save As on the File menu. This saves the project file in XML format. Click Open on the File menu to reload an existing project file.

Basic UIX Customization Through Skinning

Titles can customize the Xbox Drop-in UI through the use of a skin file. A skin file contains strings, images, and layout information for each Xbox Drop-in UI screen. Content developers can work make changes to the skin file without rebuilding the title's executable (.xbe) file.

Customize the skin file by changing the string, images, sounds, layout information, and icons for any language setting. Information describing these assets is stored in the text skin description file (.inx). The SkinBld tool uses the skin description file and the assets it describes to create a binary skin file (.uix). You then include the resulting .uix file on your title's game disc.

If your title supports multiple screen resolutions, you should create separate skin files for each resolution. The title specifies the appropriate skin file for a given resolution when calling the UIXCreateLiveEngine function. Using the default 640×480 skin file when the screen is set to 1080 produces a small and poorly aligned user interface. Your game uses the same skin files for interlaced and progressive output.

UI plug-ins use the methods of IPluginSupport to obtain skin file information from the engine. For titles using the default ITitleUIPlugin interface created with the UIXCreateUIPlugin function, that retrieve and handle that information automatically without additional code.

SkinBld

The SkinBld tool compiles a Unicode text skin description file (.inx) and associated resources into a binary skin file (.uix) usable by the Xbox Drop-in UI engine. You must include the skin file on the title's game disk and specify it in the call to UIXCreateLiveEngine.

[/header] [/pre] [/rdf] input1.inx [input2.inx ...] output_file

USAGE: skinbld [/header] [/pre] [/rdf] input1.inx [input2.inx ...] outputfile.uix [/help sectionname]
Value Description
/header Generate a C/C++ header file with resource IDs.
/pre Generate a merged input file for reference.
/rdf Generate a merged input file for reference.
inputfile.inx The name of the input skin description file (Unicode text).
outputfile.uix The name of the output compiled skin file.

Skin Description File Sections

The skin description file contains the following sections: Image, Audio, String, POPUP_SCREEN, LOGON_1_USER_SCREEN, LOGON_2_USER_SCREEN, LOGON_4_USER_SCREEN, FRIENDS_SCREEN, NOTIFICATION_SCREEN, PLAYERS_DEFAULT_SCREEN, PLAYERS_DEPARTED_ONLY_SCREEN, and PLAYERS_CURRENT_ONLY_SCREEN. Within each section, values are given for the images, string, sounds, or UI elements that are contained within the section.

For UI element colors, values are A8R8G8B8 D3DCOLOR values specified as 32-bit hexadecimal numbers (for example, 0x88FF00AA). For UI elements, some values have defaults that are used when a value is not supplied. The default TextColor is 0xFFFFFFFF, the default DisabledTextColor is 0xFF808080, and the default FontHeight is 16.

For all sections, a Language= value can be given that specifies the language for that section. For example, Language="JA" specifies that the section contains values for Japanese. The language value determines which sections in the skin file are used by the engine, based on the language that was specified when the engine was created with UIXCreateLiveEngine. Sections that match the specified language are used if found. Otherwise, default language sections (sections with no language value specified) are used.

The two-letter language identifiers used in the skin file map to the language identifier value passed to UIXCreateLiveEngine. The built-in codes to support languages used in the Dashboard can also be specified. Note that "BR", Brazil, maps to the Dashboard language value XC_LANGUAGE_PORTUGUESE. Numeric data is also supported for Language. If the title supports a language other than a Dashboard language, a decimal number can be specified. Titles can display languages that the Dashboard cannot.

The following language codes are supported.

Value Description
EN English
JA Japanese
DE German
FR French
ES Spanish
IT Italian
KO Korean
TW Traditional Chinese
BR Brazilian Portuguese

[Image]

The [Image] section contains images used by the various Xbox Drop-in UI feature screens, as well as images embedded in strings specified in the [String] section.

The [Image] section contains the following images: IMG_A, IMG_B, IMG_X, IMG_Y, IMG_FRIEND_ONLINE, IMG_FRIEND_ONLINE_ON, IMG_FRIENDS_REQUEST_RECEIVED, IMG_FRIENDS_REQUEST_RECEIVED_ON, IMG_FRIENDS_REQUEST_SENT, IMG_FRIENDS_REQUEST_SENT_ON, IMG_GAME_INVITE_RECV, IMG_GAME_INVITE_RECV_ON, IMG_GAME_INVITE_SENT, IMG_GAME_INVITE_SENT_ON, IMG_PASSCODE_BLANK, IMG_PASSCODE_FILLED, IMG_BLANK, IMG_VOICE_HEADSET, IMG_VOICE_HEADSET_ON, IMG_UP_ARROW, IMG_DOWN_ARROW, IMG_LIST_HIGHLIGHT. IMG_SPIN_1, IMG_SPIN_2, IMG_SPIN_3, and IMG_SPIN_4. Additionally, images embedded in strings in the [String] section can be defined as desired.

For each image, the file name of the source bitmap image is specified. The following additional values are valid for each image:

Value Description
AlphaSource Alpha image bitmap file name.
Format Bundler format for image (the default is D3DFMT_DXT3).

[Audio]

The [Audio] section contains the event to sound name mappings. The events correspond to UI events, and will trigger the associated sound names. If using the default audio plug-in, the sound names must match sound names in the XACT sound bank specified in UIXCreateAudioPlugin. Otherwise, the sound names must be recognized by the custom audio plug-in, and appropriate sounds played in response to the UI events.

The current UI events that trigger sound are AUDIO_PRESS_A, AUDIO_PRESS_B, AUDIO_ERROR, AUDIO_LIST_ITEM_CHANGED, AUDIO_POPUP_MESSAGE, AUDIO_PASSCODE, and AUDIO_NOTIFICATION.

[String]

The [String] section contains the strings used for display strings, button names, error messages, and other notifications. A string can contain embedded icons before, after, or in the middle of the string. Embedded icons are contained within braces. For example "Press {IMG_A} to join" would embed into the display string the image specified as IMG_A in the [Image] section.

Strings beginning with STRP are parameterized strings containing a '%s' at the location within the string where a run-time text value is inserted.

For a list of strings, see the example language file, below, or the output of 'SkinBld /help String'.

[POPUP_SCREEN]

The [POPUP_SCREEN] sections contains the screen layout for the general pop-up screen that displays error messages or other messages to the user. The pop-up screen contains the following UI elements: SCREEN, TITLE, MESSAGE, MENU, ACTIONBUTTON, BACKBUTTON.

AlphaSource Screen background image alpha.
Format Screen background image Bundler format.
FontHeight Vertical size of the font, in pixels.

For the TITLE, MESSAGE, MENU, ACTIONBUTTON, and BACKBUTTON elements, the following values are valid:

Value Description

Y Vertical position of the object (top).

[LOGON_4_USER_SCREEN]

The [LOGON_4_USER_SCREEN] section contains the screen layout for the four-player logon screen. In addition to the elements in the one-player logon screen, the four-player screen contains the following additional UI elements, representing the three additional quadrants ("regions") representing each player: USERLIST1, USERLIST2, USERLIST3, STATUS1, STATUS2, STATUS3, PASSCODE1, PASSCODE2, and PASSCODE3. The four-player screen contains a JOINBUTTON element instead of the one-player CONTINUEBUTTON element.

The valid values for the USERLIST, STATUS, and PASSCODE elements are the same as for the corresponding elements in [LOGON_1_USER_SCREEN]. JOINBUTTON has the same valid values as BACKBUTTON.

IDirect3DDevice8::RunPushBuffer

Runs the push-buffer.

HRESULT RunPushBuffer(
  IDirect3DPushBuffer8 *pPushBuffer, 
  IDirect3DFixup8 *pFixup
);

Parameters

pPushBuffer
[in, out] Pointer to the push-buffer to run.
pFixup
[in] Pointer to the IDirect3DFixup8 object that will be applied to the push-buffer.

Return Values

If the method succeeds, the return value is S_OK.

Remarks

Before IDirect3DDevice8::RunPushBuffer is called, the caller is responsible for setting Direct3D in exactly the same state as when the push-buffer was originally recorded. The only exceptions to this are any render states that are marked in D3DRENDERSTATETYPE as being processed immediately, and any state that is "fixed-up."

If a push-buffer records a reference to any resource that is freed and then re-created before playback, any references to that object will have to be explicitly fixed-up to account for the potentially new physical address. Note that this implies that for push-buffer resources that are pre-built, all resource references within the push-buffer will have to be fixed up.

The fix-up can be assumed to persist (it's never necessary to apply the same fix-up twice in a row).

IDirect3DDevice8::RunPushBuffer has no means of referencing any objects, so all object lifetimes must be controlled via other means, such as IDirect3DDevice8::InsertFence.

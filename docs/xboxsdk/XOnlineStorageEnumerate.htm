XOnlineStorageEnumerate

Enumerates files that are stored on global storage.

HRESULT XOnlineStorageEnumerate(
    DWORD dwFacility,
    DWORD dwUserIndex,
    LPCWSTR wszStorageEnumerationPath,
    DWORD dwStartingIndex,
    DWORD cMaxResultsToReturn,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwFacility

[in] Indicates storage facility. Must be the following:
Value Description
XONLINESTORAGE_FACILITY_PER_TITLE Title managed global storage.

For a complete list of facility values, see the XONLINESTORAGE_FACILITY topic.

dwUserIndex
[in] Controller index (zero-based) of the player for whom to download the attachment.
wszStorageEnumerationPath

[in] The "*" wildcard can be used in this path. If a wildcard is not specified, only exact matches are returned (will not recurse subdirectories).

This path can be created using XOnlineStorageCreateServerPath.

dwStartingIndex
[in] Index of file to start enumerating from (this index is zero based). Files are returned in order from most to least recently modified.
cMaxResultsToReturn
[in] Maximum number of files to return.
hWorkEvent
[in, optional] Handle to an event that is signaled when there is work to do on the asynchronous task returned by the phTask parameter. This parameter can be NULL if no such synchronization is required.
phTask
[out] Pointer to a variable of type XONLINETASK_HANDLE that receives a task handle. Use this handle in calls to XOnlineTaskContinue to process the file enumeration.

Return Values

If XOnlineStorageEnumerate succeeds, the return value is S_OK.

If XOnlineStorageEnumerate fails, the return value is one of the following error codes:

Value Description
XONLINE_E_STORAGE_INVALID_REQUEST The request is invalid.
XONLINE_E_STORAGE_INVALID_ACCESS_TOKEN The access token signature is invalid.
XONLINE_E_STORAGE_CANNOT_FIND_PATH Name resolution failed.
XONLINE_E_STORAGE_INVALID_STORAGE_PATH Invalid storage path.
XONLINE_E_STORAGE_INVALID_FACILITY Invalid facility code.

Remarks

This function will allow a title to check whether files are stored in global storage. A title will have to handle gracefully the case where a file or all files are not present in global storage.

Using a trunk storage path, this API kicks off an async enumerate to get all files available under this path. Subpaths, if present, are not recursively traversed and returned. If such a hierarchy exists, the title must handle the recursion.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

OpenMutex

The OpenMutex function opens an existing named mutex object. 

HANDLE OpenMutex(
  DWORD dwDesiredAccess,  // access
  BOOL bInheritHandle,    // inheritance option
  LPCSTR lpName           // object name
);

Parameters

dwDesiredAccess
[in] Ignored.
bInheritHandle
[in] Ignored; use FALSE.
lpName
[in] Pointer to a null-terminated string that names the mutex to be opened. Name comparisons are case insensitive. The name can contain any character except the backslash character.

Return Values

If the function succeeds, the return value is a handle to the mutex object.

If the function fails, the return value is NULL. To get extended error information, call GetLastError.

Remarks

The OpenMutex function enables multiple open handles to the same mutex object. The function succeeds only if the mutex has already been created with the CreateMutex function. The returned handle can be used in any function that requires a handle to a mutex object, such as the wait functions.

The handle can be duplicated by using the DuplicateHandle function. Use the CloseHandle function to close the handle. The mutex object is destroyed when its last handle has been closed.

Example

For an example that uses OpenMutex, see Using Named Objects.

Requirements

Header: Declared in Winbase.h, for include use Xtl.h.

Win32 Deltas

The dwDesiredAccess parameter is ignored. All handles have full access. You can pass MUTEX_ALL_ACCESS to this parameter if desired.

The bInheritHandle parameter is ignored. Child processes cannot be created. There is nothing that could inherit the handle.

The lpName parameter is not limited to 255 characters. Name comparisons are case insensitive.

Implemented as ANSI only.

To make a call that will also work under Win32:

  • Specify MUTEX_ALL_ACCESS for the dwDesiredAccess parameter.
  • For lpName, consider case sensitivity and use strings with less than 255 characters.

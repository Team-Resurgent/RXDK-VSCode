Compiler Warning (level 1) C4002

too many actual parameters for macro 'identifier'

The number of parameters in the macro exceeds the number of formal parameters in the macro definition. The preprocessor collects the extra parameters but ignores them during macro expansion.

The following example generates C4002.

// C4002.cpp
// compile with: /W1
#define test(a) (a)

int main() {
   int a = 1;
   int b = 2;
   a = test(a,b);   // C4002
   // try...
   a = test(a);
}

This error can also be generated if there are extra commas in a macro. Past versions of Visual Studio accepted extra commas. However, for code to be valid in both the Visual Studio .NET 2003 and earlier versions of Visual C++, remove the extra commas.

// C4002b.cpp
// compile with: /W1
#define F(x,y)
int main()
{
   F(2,,,,,,3,,,,,,)   // C4002
   // Try the following line instead:
   // F(2,3)
}

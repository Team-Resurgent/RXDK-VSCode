IDirectMusicPerformance8::GetNotificationPMsg

Retrieves a pending notification message.

HRESULT GetNotificationPMsg(
  DMUS_NOTIFICATION_PMSG** ppNotificationPMsg
);

Parameters

ppNotificationPMsg
Address of a variable that receives a pointer to a DMUS_NOTIFICATION_PMSG structure. The application retrieving this message is responsible for calling IDirectMusicPerformance8::FreePMsg on it.

Return Values

If the method succeeds, the return value is S_OK, or S_FALSE if there are no more notification events to return.

If it fails, the method can return E_POINTER.

Remarks

Notification performance messages expire two seconds after they are triggered. If your main thread will be busy for more than two seconds (loading a level, for example), you may lose these messages. To avoid losing messages, your title can do one of the following:

  • Create a thread that polls the notifications to retrieve them while the main thread is busy.
  • Choose to call DirectMusicDoWork yourself and always check for notifications before the call to DirectMusicDoWork.

Notifications are removed as part of the work handled by DirectMusicDoWork, so they can't go away until DirectMusicDoWork is called. However, DirectMusicDoWork must be called regularly for DirectMusic to function properly.

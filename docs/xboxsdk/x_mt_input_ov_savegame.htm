Save Games

Enumerating Saved Games

The code example in the previous topic calls a simple EnumSavedGames() function that outputs to the debugger a list of the games on the MU that the title saved. In a game, this function could build up a list of available save games and display them to the user.

//
// Enumerate saved games on the specified device.
//
void EnumSavedGames( char chDrive )
{
    HANDLE hFind;
    XGAME_FIND_DATA xgfd;
    char szDrive[] = "X:\\";


    *szDrive = chDrive;
    hFind = XFindFirstSaveGame( szDrive, &xgfd );

    if( INVALID_HANDLE_VALUE != hFind )
    {
        do {
            OutputDebugStringW( xgfd.szSaveGameName );
            OutputDebugString( "\r\n" );
        } while( XFindNextSaveGame( hFind, &xgfd ) );
        XFindClose( hFind );
    }

    return;
}

Creating a Save Game

After a memory unit has been inserted into the controller and has been mounted to get a mapped drive letter, a save game can be written to that MU. To save a game on the memory unit, use the XCreateSaveGame function. XCreateSaveGame creates a save game container directory on the memory unit and returns the directory path where the data for that save game can be stored. The save game data file can then be written in that directory.

Suppose you implement a simple save game design where all that your title saves is the current level in the game that the player is on. This can be represented with a single number written to the save game data file, so the save game function might be as simple as the following example. Its parameters are the drive letter for the storage region (a memory unit or the hard disk) where the game will be saved, the save game name (typically specified by the user), and the level the player is on.

LPCSTR g_pszSaveFile = "\\save.dat";
LPCSTR g_pszImagePath = "d:\\saveimage.xbx";
LPCSTR g_pszImageMeta = "saveimage.xbx";

//
// Write a save game to the specified device using the specified game name.
//
BOOL SaveGame( char chDrive, LPCWSTR pwszGameName, DWORD dwSaveInfo )
{
    DWORD dwRes = FALSE, dwWritten;
    XCALCSIG_SIGNATURE xsig;
    char szDrive[] = "X:\\";
    char szPath[MAX_PATH];
    LPSTR pszPathEnd;
    DWORD dwSaveSize;
    HANDLE hFile;
    HANDLE hSig;

    // Create (or open an existing) save game with the specified name.

    *szDrive = chDrive;

    if( ERROR_SUCCESS == XCreateSaveGame(
                            szDrive,
                            pwszGameName,
                            OPEN_ALWAYS,
                            0,
                            szPath,
                            MAX_PATH ) )
    {
        // Find the end of the path string and append the save game name.
        pszPathEnd = szPath + strlen( szPath );
        strcpy( pszPathEnd, g_pszSaveFile );

        // Create (or open) our save game file.
        hFile = CreateFile(
                    szPath,
                    GENERIC_WRITE,
                    0,
                    NULL,
                    OPEN_ALWAYS,
                    FILE_ATTRIBUTE_NORMAL,
                    NULL );

        // Write the save game data to the file.
        if( INVALID_HANDLE_VALUE == hFile ) {
            return FALSE;
        }

        dwSaveSize = sizeof(DWORD);

        // Write the actual save info.
        dwRes = WriteFile(
                    hFile,
                    &dwSaveInfo,
                    dwSaveSize,
                    &dwWritten,
                    NULL );

        // Write the signature.
        hSig = XCalculateSignatureBegin( 0 );
        if( INVALID_HANDLE_VALUE != hSig ) {
            XCalculateSignatureUpdate( hSig, (BYTE *)&dwSaveInfo, dwSaveSize );
            XCalculateSignatureEnd( hSig, &xsig );
            dwRes = WriteFile(
                        hFile,
                        &xsig,
                        XCALCSIG_SIGNATURE_SIZE,
                        &dwWritten,
                        NULL );
        } else {
            dwRes = FALSE;
        }               

        SetEndOfFile( hFile );
        CloseHandle( hFile );

        // Provide the metadata image file.
        // Append the image file name to the end of the save game path name,
        // and copy our image file to the save game directory.
        strcpy( pszPathEnd, g_pszImageMeta );
        CopyFile( g_pszImagePath, szPath, FALSE );
    }

    return dwRes;
}

XCreateSaveGame returns the directory path of the save game in the szPath parameter. The code appends the save game name (stored in g_pszSaveFile) to that path and creates the save game file by calling CreateFile. Then it is simply a matter of writing the level (a single DWORD) out to the file using WriteFile, followed by the signature calculated over the save data. When loading a save game, the code should calculate the signature over the save data again and compare it to the signature at the end of the save file. See XCalculateSignatureBegin and the Save Game Signing section of XTL Services for more information. Also, for more sample code that demonstrates creating save games, see the LoadSave Sample and the Xbox Technical Certification Game topics.

The code now copies the image file for the save game from the title's launch directory (or DVD root). The image file is 64x64 24-bit color with a single alpha bit in the DXT1 format (created using the Bundler Tool). The code copies the image to the "saveimage.xbx" file in the save game directory, after which the image will show up in the Dashboard when players are browsing save games. For more information on the steps required to create this image file, see the Dashboard Metadata section. After this step, the save game is done.

You may want to provide custom images for each save game—for example, a partial screen shot of the player's location in the game when the save was done. Rather than simply copying from a previously created image file (as shown in the example above), an image in the proper format can be created directly within the game. Use IDirect3DDevice8::CreateTexture to create a texture in the D3DFMT_DXT1 format, copy the desired image into that texture using D3DXLoadSurfaceFromSurface, and then save that texture to a file with the XGWriteSurfaceOrTextureToXPR function. This full process is demonstrated in the LoadSave reference UI sample and the CXBSavedGame::SaveImage samples framework method.

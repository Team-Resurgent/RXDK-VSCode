XInputOpen

The XInputOpen function is used to acquire the handle necessary to use an input device attached to the Xbox console.

HANDLE XInputOpen(
  PXPP_DEVICE_TYPE DeviceType,
  DWORD dwPort,
  DWORD dwSlot,
  PXINPUT_POLLING_PARAMETERS pPollingParameters
);

Parameters

DeviceType
[in] Contains the Xbox device type of the device a title needs to open. If this type does not match the type of the device connected to the port and slot specified, the function will fail. This parameter must be one of the following values:
ValueDescription
XDEVICE_TYPE_GAMEPADAn Xbox Controller.
XDEVICE_TYPE_DEBUG_KEYBOARDA debug keyboard. Do not use in shipping titles. See Remarks below.
XDEVICE_TYPE_DEBUG_MOUSEA debug mouse. Do not use in shipping titles. See Remarks below.

Note   All game controllers (for example, wheels, joysticks, lightguns, and gamepads) use XDEVICE_TYPE_GAMEPAD.

dwPort
[in] Specifies the port number of the device the title wishes to open. This parameter must be one of the following values:
Port Description
XDEVICE_PORT0 The leftmost controller port (port 0).
XDEVICE_PORT1 The second controller port from the left (port 1).
XDEVICE_PORT2 The third controller port from the left (port 2).
XDEVICE_PORT3 The rightmost controller port (port 3).
dwSlot
[in] Specifies the slot number of the device to be opened. For devices located in controller slots, dwSlot selects the slot desired. If the device is not contained in a controller slot (for example, the controller itself), dwSlot should be XDEVICE_NO_SLOT. This parameter must be one of the following values:
Value Meaning
XDEVICE_NO_SLOT The device is not contained in a controller slot.
XDEVICE_TOP_SLOT The top slot of the controller specified in dwPort.
XDEVICE_BOTTOM_SLOT The bottom slot of the controller specified in dwPort.
pPollingParameters
[in] [optional] Pointer to an XINPUT_POLLING_PARAMETERS structure that specifies parameters for polling. If NULL, default parameters (dependent on device type) will be used.

Return Values

If the function succeeds, the return value is a device handle to the specified input device.

If the function fails, the return value is NULL. To get extended error information, call GetLastError.

Remarks

Before calling XInputOpen, XInitDevices must be called to initialize the Xbox peripherals.

The XInputOpen function gets a handle to the specified device and prepares the device for use by a title. These preparations include setting the polling parameters, allocating necessary memory, and reserving bandwidth on the bus. The default polling parameters are device type dependent; see XINPUT_POLLING_PARAMETERS for details. This function can fail due to insufficient bus bandwidth or insufficient memory. (Between about 260 and 320 bytes of memory are required, depending on the polling parameters.) At this time, opening even the maximum set of supported peripherals should not exceed the reservable bandwidth. Titles should use the XInputClose function to free the allocated resources once the device is no longer needed.

Handles remain valid only as long as the device remains connected to the system. Once disconnected (detected with XGetDeviceChanges), the handle represents the allocated resources only and not the device. The title should close the handle with XInputClose.

To use XDEVICE_TYPE_DEBUG_KEYBOARD, #define DEBUG_KEYBOARD before including Xtl.h. Do not use XDEVICE_TYPE_DEBUG_KEYBOARD in shipping retail game titles.

To use XDEVICE_TYPE_DEBUG_MOUSE, #define DEBUG_MOUSE before including Xtl.h. Do not use XDEVICE_TYPE_DEBUG_MOUSE in shipping retail game titles.

Requirements

Header: Declared in Xbox.h, for include use Xtl.h.
Import Library: Use Xapilib.lib

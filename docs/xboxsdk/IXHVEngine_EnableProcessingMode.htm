IXHVEngine::EnableProcessingMode

HRESULT EnableProcessingMode(
    XHV_PROCESSING_MODE processingMode
);

Parameters

processingMode

[in] An XHV_PROCESSING_MODE constant. Can be one of the following:
Value Description
XHV_INACTIVE_MODE This mode is already enabled by default.
XHV_LOOPBACK_MODE Links XHV loopback code into the game title.
XHV_SR_MODE Links XHV speech recognition code into the game title.
XHV_VOICECHAT_MODE Links XHV voice chat code into the game title.
XHV_VOICEMAIL_MODE Links XHV voice mail code into the game title. XHV_VOICECHAT_MODE must be enabled before enabling XHV_VOICEMAIL_MODE.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

This function determines which processing modes will be usable by the XHV engine and by the local talkers registered with the engine. The library code supporting a mode will not be linked in with the title unless the title specifically enables that processing mode. For example, the XHV code for speech recognition is linked into the title only if EnableProcessingMode is called with XHV_SR_MODE.

Because a title must call EnableProcessingMode before using any functions of the specified mode, the title typically calls the function in the initialization code.

A title must enable a specific processing mode before using SetProcessingMode to set a local talker to that mode. Also, once the title has registered a local talker with RegisterLocalTalker, further processing modes may not be enabled in the engine.

As an example, the order of calls for setting XHV_SR_MODE is:

  1. EnableProcessingMode with XHV_SR_MODE
  2. RegisterSpeechRecognitionBank
  3. SelectVocabulary
  4. SetProcessingMode with XHV_SR_MODE

Requirements

Header: Declared in Xhv.h.

Import Library: Use Xvoice.lib.

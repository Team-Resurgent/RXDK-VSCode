Using XMVDecoder::GetNextFrame

XMVDecoder::GetNextFrame provides an alternative method of playing back XMV files. For a given XMV file, titles should use either XMVDecoder::GetNextFrame or XMVDecoder::Play. Using both will result in undefined behavior.

The example below uses overlays to demonstrate the steps necessary to play back a file with XMVDecoder::GetNextFrame.

  1. Create and initialize IDirect3Device8 and IDirectSound8 objects. 
    IDirectSound8 *pSound;
IDirect3D8 *pD3D;
IDirect3DDevice8 *pDevice;

DirectSoundCreate( NULL, &pSound, NULL );

pD3D = Direct3DCreate8( D3D_SDK_VERSION );

// Setup the presentation parameters.
D3DPRESENT_PARAMETERS d3dPP;
ZeroMemory( &d3dPP, sizeof( D3DPRESENT_PARAMETERS ) );
d3dPP.BackBufferWidth                 = 640;
d3dPP.BackBufferHeight                = 480;
d3dPP.BackBufferFormat                = D3DFMT_LIN_A8R8G8B8;
d3dPP.BackBufferCount                 = 1;
d3dPP.SwapEffect                      = D3DSWAPEFFECT_DISCARD;
d3dPP.FullScreen_RefreshRateInHz      = 60;
d3dPP.FullScreen_PresentationInterval = D3DPRESENT_INTERVAL_ONE;
d3dPP.EnableAutoDepthStencil          = TRUE;
d3dPP.AutoDepthStencilFormat          = D3DFMT_D24S8;

pD3D->CreateDevice( D3DADAPTER_DEFAULT, 
                    D3DDEVTYPE_HAL,
                    0,
                    D3DCREATE_HARDWARE_VERTEXPROCESSING,
                    &d3dPP,
                    &pDevice );

  2. Next, create the decoder and load the XMV file with XMVDecoder_CreateDecoderForFile.

    XMVDecoder *pDecoder = NULL;
XMVDecoder_CreateDecoderForFile( 0, "d:\\media\\MyVideo.xmv", &pDecoder );

  3. In order to play back audio, you need to enable at least one audio stream. While it is possible to embed multiple audio streams into an XMV file, typically you only want to play one at at time. If you are trying to enable multiple streams, you can get the number of audio streams in the XMV file by calling XMVDecoder::GetVideoDescriptor. This example enables only the first stream by calling XMVDecoder::EnableAudioStream.

    pDecoder->EnableAudioStream( 0, 0, NULL, NULL );

  4. Because this example uses overlays to display the video, you need to create a surface using the D3DFMT_YUY2 format. The surface needs to have the same dimensions as the video stream. Because the hardware scans the data directly out of the overlay as it draws the screen, you can't change the contents of that overlay or you will get tearing. To prevent this, we will create two surfaces and alternate between them.

    IDirect3DTexture8 *pTextureShow;
IDirect3DTexture8 *pTextureDraw;

XMVVIDEO_DESC VideoDesc;
pDecoder->GetVideoDescriptor(&VideoDesc);

pDevice->CreateTexture( VideoDesc.Width, 
                        VideoDesc.Height, 
                        0,
                        0,
                        D3DFMT_YUY2,
                        0,
                        &pTextureShow );
                        
pDevice->CreateTexture( VideoDesc.Width, 
                        VideoDesc.Height, 
                        0,
                        0,
                        D3DFMT_YUY2,
                        0,
                        &pTextureDraw );
                        
IDirect3DSurface8 *pSurfaceShow;
IDirect3DSurface8 *pSurfaceDraw;
IDirect3DSurface8 *pSurfaceSubmit = NULL; 
pTextureShow->GetSurfaceLevel( 0, &pSurfaceShow );
pTextureDraw->GetSurfaceLevel( 0, &pSurfaceDraw );

  5. Enable overlays with IDirect3DDevice8::EnableOverlay. Perform at least one call to either IDirect3DDevice8::Present or IDirect3DDevice8::Swap before updating the overlay. Although not required, the example also clears the two render buffers.

    pDevice->Clear( 0, NULL, D3DCLEAR_TARGET, 0x00000000, 1.0f, 0L );
pDevice->Swap( D3DSWAP_DEFAULT );
pDevice->Clear( 0, NULL, D3DCLEAR_TARGET, 0x00000000, 1.0f, 0L );
pDevice->EnableOverlay( TRUE );

  6. Set up the destination and source rectangles for the overlay.

    RECT srcRect, dstRect;
srcRect.top = srcRect.left = dstRect.top = dstRect.left = 0;
srcRect.right = VideoDesc.Width;
srcRect.bottom = VideoDesc.Height;
dstRect.right = 640;
dstRect.bottom = 480;

  7. Finally, keep calling XMVDecoder::GetNextFrame and IDirect3DDevice8::UpdateOverlay until the video is done.

    GetNextFrame must be called often—25 times per second should be more than enough. Calling the function too often has minimal impact; however, calling it too infrequently will result in discontinuities in the audio stream.

    IDirect3DDevice8::GetOverlayUpdateStatus is called to determine if an overlay update is pending.

    XMVRESULT xr = XMV_NOFRAME;
while( xr != XMV_ENDOFFILE && xr != XMV_FAIL ) {

    // has pending overlay update completed? 
    if( pSurfaceSubmit && pDevice->GetOverlayUpdateStatus() ) {
        pSurfaceDraw = pSurfaceShow;
        pSurfaceShow = pSurfaceSubmit;
        pSurfaceSubmit = NULL;
    }

    // decode and check for new frame
    pDecoder->GetNextFrame( pSurfaceDraw, &xr, NULL );
    if( XMV_NEWFRAME == xr ) {
        pDevice->UpdateOverlay( pSurfaceDraw, &srcRect, &dstRect, FALSE, 0 );
        pSurfaceSubmit = pSurfaceDraw;
        pSurfaceDraw = NULL;
    }

    pDevice->Swap( D3DSWAP_DEFAULT );
}

pDevice->EnableOverlay( FALSE );
pDecoder->CloseDecoder();

    While an overlay update is pending, neither overlay frame can be written (decoded) to. While in this state, GetNextFrame can be called with pSurface set to NULL to perform background processing that doesn't require a writeable surface.

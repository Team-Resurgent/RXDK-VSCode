Adding Constants

As your title runs a compitition, it performs operations on the three tables in competition's database. Periodically, it needs to compare the values in a table's attributes (columns) with constant values. To make this easier to do, XLAST defines several constants for you, as shown in the following table.

Constant Description
c_status_pre_init Registration for the competition event has started.
c_status_active The competition event is currently active and joinable.
c_status_complete The competition event has finished.
c_status_cancelled The competition event was cancelled.
c_status_entrant_registered The entrant has registered for the event.
c_status_entrant_playing The entrant is currently playing in a competition event.
c_status_entrant_forfeit The entrant forfeited.
c_status_entrant_ejected The entrant was ejected from the event.
c_status_entrant_eliminated The entrant lost and has been eliminated from the competition.

Note    Under no circumstances should you change the names, Hex IDs, or values of any of the constants that XLAST defines for you.

XLAST also enables you to define your own title-specific constants. To do so, follow the steps below.

  1. Select the table that the constant will be added to by clicking either the Competitions, Entrants, or Events tab.
  2. Click the Add button. In the dialog box that appears, enter the name of the constant. The name may contain spaces and alphanumeric characters. In the code that XLAST generates, it replaces the spaces in the constant names with underscores.

  3. Choose the type of the constant. It can be either Int, String, or Binary. Constants of type Binary are generated as strings containing hexadecimal digits.

    Note    XLAST does not put a limit on the amount of data that you can store in a constant of type Binary. However, it is best to keep it under 256 digits.

  4. Enter the value of the constant. If the constant is of type Binary, you can load its value from a text file by clicking the Load From File button.

To edit an existing constant, highlight it in the list and select Edit. In the dialog box that appears, change the constant's name, type, or value, as needed.

If you no longer use a constant, you should delete it. Do so by highlighting it in the list and clicking the Delete button.

Note    Deleting a constant also deletes all filters, sort operators, and return values in the competition's queries that reference the constant.

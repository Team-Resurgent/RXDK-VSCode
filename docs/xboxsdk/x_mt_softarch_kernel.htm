Xbox Kernel Overview

The Xbox kernel is essentially new code written specifically to harness the power of the Xbox and to make that power fully available to Xbox titles. The design requirements (for example, no user mode, multiple process, or multiple processor support) allow a more streamlined and efficient kernel implementation. A fraction of the kernel code (less than 20%) is similar to code in the Windows 2000 kernel scheduler, one of the earliest, most efficient, and stable components of the Windows NT kernel. The Xbox kernel is also efficient, and very fast, with a code size of less than 250 KB, a total memory footprint of less than 512 KB, and a boot time of less than 1 second.

The kernel resides in ROM on every Xbox console. When the Xbox is turned on, the ROM is copied into RAM for faster execution, and the boot-up code in the kernel begins executing. The complete boot-up sequence is described later, but, briefly, after initializing the hardware and playing the boot-up animation while spinning up the DVD drive, the system checks for the existence of a disc in the DVD drive. If there is a movie DVD in the drive, the Xbox Dashboard is loaded and the DVD player component of the Dashboard plays the movie. If there is a game DVD in the drive, the game is executed. If there is no disc in the DVD drive or the media is unrecognizable, the Xbox Dashboard is loaded from the hard disk and executed.

The kernel is responsible for booting the system, setting up the hardware busses, and initializing the chip sets. It provides the virtual page management services, schedules thread execution, and is the basis for the hardware drivers, interrupt hooks, common namespaces, interrupt request routines, and DPCs (deferred procedure calls) on the Xbox. The kernel also provides stable, fast, reliable file systems for the hard disk and the DVD drive. All game titles can use these file systems, ensuring that games have reliable access to those storage devices and that they don't interfere with each other's storage data.

The kernel contains the encryption routines, hash algorithms, and certificate validation code that provide the essential security features of the Xbox. Most of these routines are used internally by the kernel when validating and loading games. However, some support is exposed to titles in order to enable save game signing, discussed later.

Services provided by the kernel can be accessed through functions in the Xbox Title Libraries (XTL), discussed in more detail in the XTL Overview and related topics. Kernel components are discussed in more detail in the following topics:

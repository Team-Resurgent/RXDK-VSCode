What Is Autoupdate?

The autoupdate feature provides you with a method of updating users' copies of your title after it has shipped. You can use this capability to provide players with fixes to security problems, bugs, and so forth.

You can use autoupdate to deliver a new version of any file that ships on your game disk. You can also use autoupdate to add data files that were not orginally shipped on the disk. However, an autoupdate cannot add new XBE files. If you update non-XBE files, you will need to make sure your title checks specifically for the updated files and uses content protection to ensure that they have not been tampered with.

The files that make up an autopdate are called an autoupdate package. Like your orginal game, autoupdate packages must pass certification. If you build your autoupdate more than a few months after the original game release, you will probably get a warning indicating that some of the XTLs you're using are not approved. This occurs because the XTLs used by both the original game and the autoupdate must have the same major version numbers, and the major version numbers expire after a few months. Getting this warning does not prevent you from building your autoupdate. However, it is strongly recommended that you use the ImageCert tool to verify that this is the reason that you are getting the warning. If ImageCert indicates that the XTLs your autoupdate uses are expired, it can still pass certification. If ImageCert gives any other reason that your XTLs are not approved, the autoupdate won't pass certification. It's best to address this problem before submission.

Because each Xbox has a unique serial number, autoupdates can be configured to work only on specific Xbox consoles. The Xbox Certification team uses this feature to test autoupdates before making them available to everyone. Once the autoupdate package has passed testing and certification, it is made global and applied to all Xbox consoles on Xbox Live that have your game.

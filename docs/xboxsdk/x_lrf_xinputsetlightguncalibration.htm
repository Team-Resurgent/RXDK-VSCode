XInputSetLightgunCalibration

Allows a game to set lightgun calibration offset values for two well-known calibration points: the upper-left corner and the middle of the TV screen.

DWORD XInputSetLightgunCalibration(
    HANDLE hDevice,
    PXINPUT_LIGHTGUN_CALIBRATION_OFFSETS pCalibrationOffsets
);

Parameters

hDevice
[in] Handle to the lightgun device.
pCalibrationOffsets
[in] Pointer to an XINPUT_LIGHTGUN_CALIBRATION_OFFSETS structure that contains calibration offset values for the lightgun.

Return Values

This function returns ERROR_SUCCESS if the calibration values are successfully set for the lightgun. The function returns ERROR_INVALID_HANDLE if the lightgun handle is invalid.

Remarks

The offsets are the difference between the defined location of the calibration points and the values reported by the lightgun when the game player pointed and shot at the calibration point. For example, if the player shot at the center calibration point (0,0), but the values measured by the lightgun were (40,30), the offset values sent back to the gun for calibration would be (−40,−30).

Note    Calibration data will vary depending on the state of the XINPUT_LIGHTGUN_LINE_DOUBLER and XINPUT_LIGHTGUN_FRAME_DOUBLER flags returned by the lightgun. Because there are four possible combinations of these flags, the game may keep up to four sets of calibration data. The lightgun, however, will only store one set of calibration data at a time.

The fixed location values for the calibration points are provided in the following constants:

// Upper-left corner:
#define XINPUT_LIGHTGUN_CALIBRATION_UPPERLEFT_X -25000
#define XINPUT_LIGHTGUN_CALIBRATION_UPPERLEFT_Y  25000

// Center:
#define XINPUT_LIGHTGUN_CALIBRATION_CENTER_X    0
#define XINPUT_LIGHTGUN_CALIBRATION_CENTER_Y    0

The following table lists the screen coordinate equivalents of the calibration point locations. The target reticle should be centered on these points.

Screen resolution Center calibration point Upper-left calibration point
640 × 480 320, 240 76, 57
720 × 480 360, 240 85, 57
1280 × 720 640, 480 152, 85
1920 × 1080 960, 540 227, 128
640 × 576 320, 288 76, 68
720 × 576 360, 288 85, 68

Note    Calling XInputSetLightgunCalibration blocks for about two or three milliseconds.

Calibration Process

Either the calibration process or restoration of previously measured calibration settings must be performed whenever the lightgun is reset (for example, at initial console power-on, a physical disconnect of the lightgun from the console, or static discharge into the lightgun, all of which appear to the system as a hot-plug of the controller).

Assuming basics such as detecting and opening a lightgun device have already been done, the following steps describe how a game should properly calibrate a lightgun:

  1. Provide a way for the user to enter calibration mode. For example, an option can be provided from the game's main menu.
  2. Draw the background of the calibration screen. The background should not be all black as the gun needs non-black pixels on screen to detect that it is pointing at the screen. The recommended colors for all calibration screen backgrounds are:
    • Either grey screen settings of R: 243 G: 243 B: 243 with H: 160, S: 0, L: 229
    • Or yellow screen settings of R: 253, G: 253, B: 0 with H: 40, S: 240, L: 119.
  3. Provide a message on-screen that tells the user to point the gun at the screen.
  4. Wait for the XINPUT_LIGHTGUN_ONSCREEN bit to be set on.

    Note    When the gun is first pointed at the TV screen, it will take approximately 20 frames for it to determine whether the TV is using line or frame doubling. The gun will not set the XINPUT_LIGHTGUN_ONSCREEN bit until this information has been determined. Once detected, the values of line and frame doublers will be latched and the XINPUT_LIGHTGUN_ONSCREEN bit will be updated virtually instantaneously from that point on when the gun detects on/off screen conditions.

  5. Once the XINPUT_LIGHTGUN_ONSCREEN bit is set, the game may paint the first target at the center calibration position and prompt the user to shoot at the target. The game should ignore any trigger pulls from the gun until the XINPUT_LIGHTGUN_ONSCREEN bit is set. The game will need to compute the correct center calibration location in actual screen coordinates based on the resolution of the screen.
  6. The game should wait for a trigger pull from the gun and then read the X,Y values returned by the gun when it is detected.
  7. The game should take these values and compute the offset values to calibrate the gun such that a trigger pull pointed at the same target would produce the X,Y values of the actual calibration point. For example, for the center calibration point (0, 0), if the gun reports (40, −30) for the trigger pull, the offset values should be (0 − 40, 0 − (−30)) or (−40, 30).
  8. The game may choose to decide that the calibration values are too far off to be acceptable (for example, the user is pointing at the bottom right rather than the center) and prompt the user to try again before proceeding.
  9. The game should send the computed calibration offsets for the center point to the gun using the XInputSetLightgunCalibration call. For example, using the above example values: 
    XInputSetLightgunCalibration(myLGHandle, -40, 30, 0, 0);
  10. The game may paint the first target at the upper-left calibration position and prompt the user to shoot at the target. The game should ignore any trigger pulls from the gun until the XINPUT_LIGHTGUN_ONSCREEN bit is set. The game will need to compute the correct upper-left calibration location in actual screen coordinates based on the resolution of the screen.
  11. The game should wait for a trigger pull from the gun and then read the X,Y values returned by the gun when it is detected.
  12. The game should take these values and compute the offset values to calibrate the gun such that a trigger pull pointed at the same target would produce the X,Y values of the actual calibration point. For example, for the upper-left calibration point (−25000, 25000), if the gun reports (−25010, 24500) for the trigger pull, the offset values should be (−25000 − (−25010), 25000 − 24500) or (10, 500).
  13. The game may choose to decide that the calibration values are too far off to be acceptable (for example, the user is pointing at the bottom right rather than the upper-left corner) and prompt the user to try again before proceeding. For the upper-left corner calibration point, in some video modes it is possible that the point reported could be a value on the upper-right side of the screen. The game should be able to handle this wraparound case.
  14. The game should send the computed calibration offsets for both the center point and the upper-left point to the gun using the XInputSetLightgunCalibration call. For example, using the above example values: 
    XInputSetLightgunCalibration(myLGHandle, -40, 30, 10, 5000);
  15. The game can place a crosshair on the screen that follows where the gun is pointed on screen to show users where they are pointing.
  16. The game should allow the user to recalibrate either calibration point until the user is satisfied with the results. As each new calibration setting is measured, the total cumulative offset for that point should be sent to the gun.
  17. Once satisfied, the user can choose to exit calibration mode and play the game with the current calibration settings.
  18. Outside of the calibration screen, the game must monitor the XINPUT_LIGHTGUN_FRAME_DOUBLER flag to ensure that its state does not change. There are TV sets that allow this mode to be changed dynamically. If the flag changes state, the game should lead the user to calibrate the gun or apply appropriate calibration settings measured and stored previously.
  19. (OPTIONAL) The game may choose to save the calibration settings either temporarily during the current game session or permanently in the title area. These settings may be silently reapplied when a lightgun is reset (for example, when the console is first turned on, the lightgun gets disconnected, or the lightgun is zapped by ESD such that it reset). To ensure that valid calibration values get applied to a gun, calibration settings should be stored and reapplied based on the following criteria:
    • Game title (implicit since the data is to be stored in the title area)
    • Line doubler state
    • Frame doubler state
    • Screen resolution
    • Lightgun vendor
    • Lightgun product
    • Lightgun version
    • (Optional) Port the lightgun was installed in when the calibration settings were measured
    If there are no settings stored (temporarily or permanently) for a lightgun that meet all the criteria, a normal lightgun calibration sequence should be used to measure them.
  20. If there is more than one lightgun, the calibration process should be followed for each in turn.

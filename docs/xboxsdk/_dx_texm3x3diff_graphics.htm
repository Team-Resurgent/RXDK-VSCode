texm3x3diff

Determines the diffuse color of a fragment.

texm3x3diff   tDest, tSrc0, tSrc1

Registers

tDest
Destination register, holding the result of the operation.
tSrc0
Source register, specifying the input argument.
tSrc1
Source register, specifying the input argument.

Remarks

The texm3x3diff texture mode is similar to the texm3x3spec and texm3x3vspec modes except that an additional texture lookup is performed to determine the diffuse color (non-reflected) of the fragment. A normalization cubemap is set on stage 0. The normal is then transformed to eye space in stages 1 through 3. A cubemap that contains the diffuse color for the surface is set on stage 2. Another cubemap that contains the specular color for the surface is set on stage 3. The resulting diffuse color can be read from t2, and the specular color can be read from t3.

This instruction can be used as shown below:

tex t0	                // bump normal map
texm3x3pad t1, t0	// Perform first row of transform.
texm3x3diff t2, t0	// Perform second row of transform, specify diffuse cubemap.
texm3x3spec t3, t0	// Perform last row of transform, specify specular cubemap.

This instruction actually performs a dot product between the texture coordinates of the current stage and the output value from the specified input stage. Color values are unsigned integers in the range 0x0 to 0xff, but through the use of input modifiers, these values can be interpreted as signed numbers.

If the _bx2 input modifier is applied to the specified input argument, the color values are mapped to signed numbers using the formula

( ColorValue – 0x80 ) / 127.0f

So,

  • 0x0 is mapped to –1.007874016f (–128 / 127),
  • 0x1 is mapped to –1.0f,
  • 0x80 is mapped to 0.0f, and
  • 0xff is mapped to 1.0f.

If there is no input modifier applied to the specified input argument, the color values are mapped to the range zero to one using the formula

ColorValue / 255.0f

So,

  • 0x0 is mapped to 0.0f, and
  • 0xff is mapped to 1.0f.

This texture mode is an Xbox extension.

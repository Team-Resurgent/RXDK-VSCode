texm3x3spec

Performs specular reflection and environment mapping.

texm3x3spec   tDest, tSrc0, tSrc1

Registers

tDest
Destination register, holding the result of the operation.
tSrc0
Source register, specifying the input argument.
tSrc1
Source register, specifying the input argument.

Remarks

This operation takes the resulting post-transformed vector and uses it as the normal to reflect an eye-ray vector. This eye-ray vector is always the constant color c0. The instruction uses the result as an address to look up in a cube texture set at that stage. A specular environment map would be preloaded.

tex t0                  ; Define t0 as a standard 3-vector.
texm3x3pad t1, t0       ; Perform first row of matrix multiply.
texm3x3pad t2, t0       ; Perform second row of matrix multiply.
texm3x3spec t3, t0, c0  ; Perform third row of matrix multiply, do
                        ; reflection calculation, and sample texture 3.
mov r0, t3              ; Copy final result to output color.

A bump (normal) map should be set on stage 0, and cube texture should be set on stage 3. Any textures set at stages 1 or 2 are ignored.

Texture coordinate set 0 positions the bump map. Texture coordinate sets 1, 2, and 3 are the rows of the 3×3 matrix.

Note that this instruction has an unnamed input that is the assigned texture that corresponds to this texture stage.

This instruction actually performs a dot product between the texture coordinates of the current stage and the output value from the specified input stage. Color values are unsigned integers in the range 0x0 to 0xff, but through the use of input modifiers, these values can be interpreted as signed numbers.

If the _bx2 input modifier is applied to the specified input argument, the color values are mapped to signed numbers using the formula

( ColorValue – 0x80 ) / 127.0f

So,

  • 0x0 is mapped to –1.007874016f (–128 / 127),
  • 0x1 is mapped to –1.0f,
  • 0x80 is mapped to 0.0f, and
  • 0xff is mapped to 1.0f.

If there is no input modifier applied to the specified input argument, the color values are mapped to the range zero to one using the formula

ColorValue / 255.0f

So

  • 0x0 is mapped to 0.0f, and
  • 0xff is mapped to 1.0f.

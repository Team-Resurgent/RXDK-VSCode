D3DPRESENT_PARAMETERS

Describes the presentation parameters.

typedef struct _D3DPRESENT_PARAMETERS_
{
    UINT                BackBufferWidth;
    UINT                BackBufferHeight;
    D3DFORMAT           BackBufferFormat;
    UINT                BackBufferCount;
    D3DMULTISAMPLE_TYPE MultiSampleType;
    D3DSWAPEFFECT       SwapEffect;
    HWND                hDeviceWindow;
    BOOL                Windowed;
    BOOL                EnableAutoDepthStencil;
    D3DFORMAT           AutoDepthStencilFormat;
    DWORD               Flags;
    UINT                FullScreen_RefreshRateInHz; 
    UINT                FullScreen_PresentationInterval;
    D3DSurface         *BufferSurfaces[3];
    D3DSurface         *DepthStencilSurface;

} D3DPRESENT_PARAMETERS;

Members

BackBufferWidth and BackBufferHeight
Width and height of the new screen. These must equal the width and height of one of the enumerated display modes found through IDirect3D8::EnumAdapterModes.
BackBufferFormat
Member of the D3DFORMAT enumerated type.
BackBufferCount
This value can be 0, 1, or 2. Note that 0 is treated as 1.

One back buffer is considered the minimum number of back buffers. The method call fails if 1 back buffer cannot be created.

MultiSampleType
Member of the D3DMULTISAMPLE_TYPE enumerated type.
SwapEffect
Describes how the swap chain is handled. This value can be any member of the D3DSWAPEFFECT enumeration
; however, D3DSWAPEFFECT_FLIP is treated the same as D3DSWAPEFFECT_DISCARD.
hDeviceWindow
Ignored; use zero.
Windowed
Must be FALSE. On Xbox, all applications run full-screen.
EnableAutoDepthStencil
If this value is TRUE, Microsoft® Direct3D® will manage depth buffers for the application. The device will create a depth-stencil buffer when it is created. The depth-stencil buffer will be automatically set as the render target of the device. When the device is reset, the depth-stencil buffer will be automatically destroyed and recreated in the new size.

If EnableAutoDepthStencil is TRUE, then AutoDepthStencilFormat must be a valid depth-stencil format.

AutoDepthStencilFormat
Member of the D3DFORMAT enumerated type. The format of the automatic depth-stencil surface that the device will create. This member is ignored unless EnableAutoDepthStencil is TRUE.
Flags
This member can be set to 0, or to one or more of the following flags ORed together:
D3DPRESENTFLAG_LOCKABLE_BACKBUFFER
This flag is ignored.
D3DPRESENTFLAG_WIDESCREEN
Set this flag if the title is supporting widescreen mode.
D3DPRESENTFLAG_INTERLACED
Set this flag if the title is rendering to an interlaced display. This flag cannot be ORed with D3DPRESENTFLAG_PROGRESSIVE.
D3DPRESENTFLAG_PROGRESSIVE
Set this flag if the title is rendering to a progressive display. The only progressive displays that are supported are 480p and 720p HDTV modes. This flag cannot be ORed with D3DPRESENTFLAG_INTERLACED.
D3DPRESENTFLAG_FIELD
Set this flag if the title is using field rendering.
D3DPRESENTFLAG_10X11PIXELASPECTRATIO
Set this flag to center the frame buffer on the display. On a TV that is 704 pixels across, this would leave 32 pixels of black border on the left and 32 pixels of black border on the right.
D3DPRESENTFLAG_EMULATE_REFRESH_RATE
Set this flag to allow emulation of refresh rates other than the native display modes refresh rates. For example, to emulate a 50 Hz refresh rate in an NTSC mode, set this flag and set FullScreen_RefreshRateInHz to 50. This flag does not work with 1080i and does not work with overlays.
FullScreen_RefreshRateInHz
The rate at which the display adapter refreshes the screen. This value must be one of the refresh rates returned by IDirect3D8::EnumAdapterModes or the following value.
D3DPRESENT_RATE_DEFAULT
The run-time chooses the presentation rate.
FullScreen_PresentationInterval
Bit mask of values representing what presentation swap intervals are available. This value must be D3DPRESENT_INTERVAL_DEFAULT or one of the following values:
D3DPRESENT_INTERVAL_IMMEDIATE
The driver supports an immediate presentation swap interval.
D3DPRESENT_INTERVAL_ONE
The driver supports a presentation swap interval of every screen refresh.
D3DPRESENT_INTERVAL_ONE_OR_IMMEDIATE
The driver supports a presentation swap interval every screen refresh but it will flip immediately if the rendering has gone over a frame. It otherise behaves like D3DPRESENT_INTERVAL_ONE.
D3DPRESENT_INTERVAL_TWO
The driver supports a presentation swap interval of every second screen refresh.
D3DPRESENT_INTERVAL_TWO_OR_IMMEDIATE
The driver supports a presentation swap interval every second screen refresh but will flip immediately if the rendering has gone longer than two screen refreshes. It otherwise behaves like D3DPRESENT_INTERVAL_TWO.
D3DPRESENT_INTERVAL_THREE
The driver supports a presentation swap inteval every third screen refresh.
D3DPRESENT_INTERVAL_THREE_OR_IMMEDIATE
The driver supports a presentation swap interval every third screen refresh but it will flip immediately if the rendering has gone longer than three screen refreshes. It otherwise behaves like D3DPRESENT_INTERVAL_THREE.
D3DPRESENT_INTERVAL_FOUR
Not supported on Xbox.

Neither IDirect3DDevice8::SetVerticalBlankCallback nor IDirect3DDevice8::BlockUntilVerticalBlank are tied to these intervals.

BufferSurfaces and DepthStencilSurface
If non-NULL, Direct3D will not allocate its own color and Z surfaces, but rather will use the specified surfaces. BufferSurfaces[0] will be used as the back-buffer, and BufferSurfaces[1] (and BufferSurfaces[2] if BackBufferCount is 2) will be used as the front-buffers. If antialiasing, BufferSurfaces[0] will be the "big" buffer and can be a different size and format from BufferSurfaces[1]. If not antialiasing, BufferSurfaces[0] must be the same size and format as BufferSurfaces[1]. If BackBufferCount is 2, BufferSurfaces[2] must be non-NULL and the same size and format as BufferSurfaces[1]. DepthStencilSurface must be at least as large as BufferSurfaces[0].

All surfaces should be created in tiled views using IDirect3DDevice8::SetTile, and the Z-buffer should have a tiling mapping with D3DTILE_FLAGS_ZBUFFER set. D3D_AllocContiguousMemory, SetTile, XGSetSurfaceHeader, and Register are the preferred ways to create the surfaces in this case, but CreateImageSurface can also be used before calling CreateDevice by using a NULL device interface pointer:

((D3DDevice*) NULL)->CreateImageSurface(...)

Direct3D performs no reference counting on any of the surfaces used in BufferSurfaces or DepthStencilSurface. It is the application's responsibility to ensure that it does not free the contiguous memory until all rendering is done and and the graphics hardware is no longer busy. The state of the graphics hardware can be determined using either IDirect3DDevice8::IsBusy or IDirect3DDevice8::BlockUntilIdle.

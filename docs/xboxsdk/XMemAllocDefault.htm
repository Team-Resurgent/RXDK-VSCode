XMemAllocDefault

Default memory replacement function that allocates memory.

LPVOID WINAPI XMemAllocDefault(
    SIZE_T dwSize,
    DWORD dwAllocAttributes
);

Parameters

dwSize
[in] Desired size, in bytes, of the memory to be allocated.
dwAllocAttributes
[in] An XALLOC_ATTRIBUTES value that specifies the desired attributes of the memory to allocate.

Return Values

If the function succeeds, the return value is the base address of the allocated memory.

If the function fails, the return value is NULL.

Remarks

XMemAllocDefault is the default memory callback function used by the XTL libraries for allocations. All memory allocations performed by the XTL libraries call XMemAlloc. If the title does not provide an overriden XMemAlloc, XMemAllocDefault is used. A title can provide an overriden XMemAlloc, and within its implementation, call XMemAllocDefault in order to perform the appropriate actions. Thus the title can track calls without needing to fully implement XMemAlloc functionality.

The CustomMemoryAllocator sample in the XDK demonstrates overriding the memory callback funtions while calling the default functions to perform some allocation tasks.

Note    XMemAlloc, XMemFree, and XMemSize can be reliably overridden only by functions defined in the main executable project, not in a library (.lib file). Otherwise, XDK libraries may pull in the default implementations before the application-defined functions are detected. The application-defined functions must be in an .obj file used to generate the title's .xbe, rather than in a .lib that is linked in with the title. Alternatively, adding the .lib in Additional Dependencies in Linker/Input may resolve the issue.

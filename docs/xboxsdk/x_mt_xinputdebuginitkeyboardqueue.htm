XInputDebugInitKeyboardQueue

The XInputDebugInitKeyboardQueue function initializes the debug keyboard and prepares a keyboard input queue for the title's use.

To use this function, you must define the DEBUG_KEYBOARD compiler define before including xtl.h. Do not use this function in actual shipping retail game titles.

DWORD XInputDebugInitKeyboardQueue(
  PXINPUT_DEBUG_KEYQUEUE_PARAMETERS pParameters
);

Parameters

pParameters
[in] Pointer to an XINPUT_DEBUG_KEYQUEUE_PARAMETERS structure that specifies configuration and notification options for keyboard input. If pParameters is NULL, default settings are used.

Return Values

If the function succeeds, the return value is ERROR_SUCCESS.

If the function fails, the return value is an error code defined in Winerror.h. The function does not use SetLastError to set the calling thread's last-error code.

Remarks

XInputDebugInitKeyboardQueue initializes debug keyboard input options and prepares a queue to detect and report keyboard events. After the keyboard input queue has been initialized, the title must enumerate peripheral devices to find a connected keyboard. Call XGetDevices with XDEVICE_TYPE_DEBUG_KEYBOARD to retrieve a bitmask of connected keyboards. If a connected keyboard is found, call XInputOpen with XDEVICE_TYPE_DEBUG_KEYBOARD for the device type, and with the port number returned by XGetDevices as the port to open.

Once the keyboard is successfully opened, key events can be retrieved with the XInputDebugGetKeystroke function.

Any standard USB keyboard should work correctly, but testing has been limited primarily to Microsoft keyboards. The keyboard can only be used on development consoles, and must be plugged into one of the Xbox controller ports using a USB-to-Xbox port adapter.

The debug keyboard does not toggle the keyboard lights when corresponding keys are pressed (for example, caps lock).


The following code example demonstrates a simple use of the debug keyboard.

//------------------------------------------------------------------------------ 
// debugkbd.cpp
//
// Example use of the debug keyboard functions.
// Initialize the debug keyboard and send keystrokes to the debugger.
//------------------------------------------------------------------------------ 

#define DEBUG_KEYBOARD
#include <xtl.h>


void main( void )
{
    XINPUT_POLLING_PARAMETERS xpp = { TRUE, TRUE, 0, 32, 32, 0 };
    XINPUT_DEBUG_KEYQUEUE_PARAMETERS xdkp = {
        XINPUT_DEBUG_KEYQUEUE_FLAG_KEYDOWN     |
         XINPUT_DEBUG_KEYQUEUE_FLAG_KEYREPEAT  |
         XINPUT_DEBUG_KEYQUEUE_FLAG_ASCII_ONLY,
        50,
        500,
        70 };
    XDEVICE_PREALLOC_TYPE xdpt[] = {
        {XDEVICE_TYPE_GAMEPAD, 4},
        {XDEVICE_TYPE_MEMORY_UNIT, 2},
        {XDEVICE_TYPE_DEBUG_KEYBOARD, 1} };
    XINPUT_DEBUG_KEYSTROKE ks;
    DWORD dwIns, dwRem, dwPort = 0;
    CHAR szOut[2] = "X";
    HANDLE hKeyboard;


    // Initialize the peripherals.
    XInitDevices( sizeof(xdpt) / sizeof(XDEVICE_PREALLOC_TYPE), xdpt );

    // Create the keyboard queue.
    if( ERROR_SUCCESS == XInputDebugInitKeyboardQueue( &xdkp ) )
    {
        // Wait for the device to show up.
        dwIns = XGetDevices( XDEVICE_TYPE_DEBUG_KEYBOARD );
        while( !dwIns )
        {
            XGetDeviceChanges( XDEVICE_TYPE_DEBUG_KEYBOARD, &dwIns, &dwRem );
        }

        // Convert the bitmask into a port number.
        while( !(dwIns & (1 << dwPort)) )
        {
            dwPort++;
        }

        // Open the keyboard device.
        if( hKeyboard = XInputOpen(
                            XDEVICE_TYPE_DEBUG_KEYBOARD,
                            dwPort,
                            XDEVICE_NO_SLOT,
                            &xpp ) )
        {
            // Loop waiting for keyboard input.
            while( TRUE )
            {       
                if( ERROR_SUCCESS == XInputDebugGetKeystroke( &ks ) )
                {
                    *szOut = ks.Ascii;
                    OutputDebugString( szOut );
                }
            }       
        }   
    }    

    // Don't return from main.
    Sleep( INFINITE );
}

The example code above has been simplified to demonstrate the minimum requirements for reading from the debug keyboard. It does not properly handle device removal or re-insertion, and is not architected to handle multiple device types. For a more complete treament of device interaction, including sample code, see Using Xbox Peripherals.

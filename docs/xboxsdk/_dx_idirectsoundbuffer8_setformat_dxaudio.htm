IDirectSoundBuffer8::SetFormat

Changes the data format of the buffer without releasing and recreating any objects.

HRESULT SetFormat(
    LPCWAVEFORMATEX pwfxFormat
);

Parameters

pwfxFormat
Address of a WAVEFORMATEX or WAVEFORMATEXTENSIBLE structure that describes the new format for the DirectSound buffer.

Return Values

Returns DS_OK if successful, an error code otherwise.

Remarks

The SetFormat method has the following restrictions:

  • This function stops the voice if it is playing.
  • If the number of hardware voices has to change (1 hardware voice per 2 channels), the voice resources are freed and reallocated. Because of this, SetFormat may fail due to insufficient voices or buffer SGEs.
  • This function cannot be called on MIXIN or FXIN buffers.

  • SetFormat always resets the mixbin assignments. Calling SetFormat with a WAVEFORMATEXTENSIBLE structure that has its dwChannelMask member filled in will reset the mixbins to the channels supplied in the format.

    Because SetFormat resets the mixbin assignments, it should be called before calling SetOutputBuffer or SetMixBins. Calling SetOutputBuffer before calling SetFormat will cause SetFormat to add the default speaker mixbins to the original buffer. Similarly, calling SetMixBins before calling SetFormat will reset the mixbins to the default speaker mixbins.

  • 3D voices must be mono.

Because SetFormat stops the voice, this function can potentially block. Calls to SetFormat should be proceeded by a call to IDirectSoundBuffer8::Stop, and then spin on IDirectSoundBuffer8::GetStatus. For more information, see Freeing Buffers on the Understanding Direct Sound page.

While changing the channel count is possible, calling either SetBufferData or SetFormat with a different channel count is inefficient. For example, when converting from stereo to 6-channel, a better solution is to free the stereo voice and reallocate it as a native 6-channel voice.

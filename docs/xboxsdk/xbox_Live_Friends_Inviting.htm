Inviting Friends to Play a Game

Xbox Live titles must allow a player to invite users from his or her friends list to participate in an online-game session. Such invitations are one method users can find out about available online-game sessions. Another method is by means of matchmaking.

Use the following functions to manage invitations to players on the friends list.

Function Description Memory Use
XOnlineFriendsGameInvite Invite members of the friends list to join a game. Titles can also use XOnlineGameInviteSend, which allows the title to include voice attachments with the game invite. Yes—allocates memory for a request to the service.
XOnlineFriendsAnswerGameInvite Responds to an invitation generated by XOnlineFriendsGameInvite. No
XOnlineFriendsRevokeGameInvite Revokes a previously generated game invitation. No
XOnlineFriendsGetAcceptedGameInvite Creates a task to retrieve a game invitation stored locally on the Xbox console. No. Also, this function does not require that the XOnlineFriendsStartup task be active.

Sending Invitations

Xbox Live titles use the XOnlineFriendsGameInvite function to invite a player on the friends list to participate in an online-game session. To process the invitation, after calling XOnlineFriendsGameInvite, the title should call XOnlineTaskContinue with the handle returned by XOnlineFriendsStartup.

The XOnlineFriendsGameInvite function takes as a parameter the session ID of the online-game session to which the receiving players are invited. This session ID is available from the title that is hosting the session. That title obtains the session ID by calling XOnlineMatchSessionCreate, followed by XOnlineMatchSessionGetInfo. For more information about obtaining session IDs, see the reference pages for these functions and the Matchmaking Overview.

Note    The Xbox Live TCRs require all new games to support voicemail attachments for game invitations and friend requests.

Receiving Invitations

Online titles must regularly check for game invitations, even during gameplay, and notify the player when a new invitation has arrived. Display of the invite notification during gameplay is expected to be unobtrusive. Titles should signal players to pause the game when convenient, at which point the title can display more complete notification information and further UI.

The recipient of an invitation receives a notification of type XONLINE_NOTIFICATION_TYPE_GAMEINVITE in his or her notification queue. The title running on the Xbox console of the receiving player should periodically call the XOnlineGetNotification function for each of the players on that console to check for the receipt of such notifications. The title should then inform the player or players of the pending invitation.

To determine the identity of the player who sent the invitation, the receiving title enumerates the friends list of the receiving user. The user who sent the invitation is present in the receiving player's friends list, with a friend state of XONLINE_FRIENDSTATE_FLAG_RECEIVEDINVITE. (The receiving player is also present in the inviting player's friends list, with a friend state of XONLINE_FRIENDSTATE_FLAG_SENTINVITE.) The friend state is stored in the Friendstate member of the XONLINE_FRIEND structure.

The title on the receiving player's Xbox console should call XOnlineFriendsAnswerGameInvite with an appropriate value from XONLINE_GAMEINVITE_ANSWER_TYPE. In order to process the response to the request, after calling XOnlineFriendsAnswerGameInvite, the title should call XOnlineTaskContinue with the handle returned by XOnlineFriendsStartup.

If the invitee accepts the invitation, the title should call XOnlineFriendsAnswerGameInvite and send the value XONLINE_GAMEINVITE_YES. The inviter receives a notification of XONLINE_NOTIFICATION_GAME_INVITE_ANSWER, and the dwFriendState member of the invitee's XONLINE_FRIEND structure contains the value XONLINE_FRIENDSTATE_FLAG_INVITEACCEPTED.

If the player chooses to decline the invitation, the response should be XONLINE_GAMEINVITE_NO. The inviter receives a notification of XONLINE_NOTIFICATION_GAME_INVITE_ANSWER, and the dwFriendState member of the invitee's XONLINE_FRIEND structure contains the value XONLINE_FRIENDSTATE_FLAG_INVITEREJECTED.

The receiving player (the invitee) also has the option to decline the invitation and remove the inviting player from his or her friends list. In this case, the title should use the value XONLINE_GAMEINVITE_REMOVE in the call to XOnlineFriendsAnswerRequest. The inviting player receives a notification of XONLINE_NOTIFICATION_TYPE_FRIENDREQUEST. However, the title should not display this notification to the inviting player. Each player should silently disappear from the other's friends list.

Receiving Player's Action Notification Received by Inviting Player Receiving Player's dwFriendState in Inviting Player's Friends List
Receiving player accepts XONLINE_NOTIFICATION_TYPE_GAMEINVITEANSWER XONLINE_FRIENDSTATE_FLAG_INVITEACCEPTED
Receiving player rejects XONLINE_NOTIFICATION_TYPE_GAMEINVITEANSWER XONLINE_FRIENDSTATE_FLAG_INVITEREJECTED

Revoking Invitations

If a player has not enumerated their friends list since receiving an invitation, the inviting player has the option of revoking the invitation. To revoke the invitation, the title calls XOnlineFriendsRevokeGameInvite. The task returned by the XOnlineFriendsEnumerate function must be active at the time the title calls XOnlineFriendsRevokeGameInvite. After calling XOnlineFriendsRevokeGameInvite, the title calls XOnlineTaskContinue with the handle returned by XOnlineFriendsStartup to process the revocation.

Setting the sessionID parameter to zero in a call to XOnlineNotificationSetState automatically causes all previously sent invitations to expire, regardless of whether they have been enumerated by the receiving player.

Connecting to the Xbox Live Game Session

Once a user has accepted an invitation to join an Xbox Live game session, the title must connect to the Xbox console hosting the session. Follow these steps to establish the connection:

  1. Obtain the session ID of the online-game session from the sessionID member of the XONLINE_FRIEND structure of the player who sent the invitation.
  2. Call XOnlineMatchSessionFindFromID with this session ID to obtain an XONLINE_MATCH_SEARCHRESULT structure for the session.
  3. To verify that the session still has available private slots, check the value of the dwPrivateAvailable member of the XONLINE_MATCH_SEARCHRESULT structure.
  4. To register a secure-networking session, call the XNetRegisterKey function with the SessionID and the key exchange key from the XONLINE_MATCH_SEARCHRESULT structure.
  5. To obtain the IN_ADDR of the Xbox console that is hosting the session, call the XNetXnAddrToInAddr function with the SessionID and the Host Address from the XONLINE_MATCH_SEARCHRESULT structure.
  6. Lastly, send a join request to the hosting Xbox console. For more information about join requests, see Xbox Secure Network Library.

Cross-Title Invitations

It is possible that the player who receives the invitation is not playing the same title as the player who sent the invitation. The title on the receiving player's Xbox console can determine whether the title is different by calling XOnlineTitleIdIsSameTitle. The title can call XOnlineTitleIdIsSameTitle with the value of the dwTitleID member of the XONLINE_FRIEND structure of the player who sent the invitation. If the title is different and the invitee accepts the invitation, the currently running title should prompt the player to insert the disk for the new title. The currently running title can retrieve the name of the new title by calling XOnlineFriendsGetTitleName with the title ID. The currently running title should shut itself down, and prepare for the Xbox console to reboot.

It is also possible for the player to join a game session from a different title if the title is being played by a member of the player's friends list. To do so, the currently running title should call XOnlineFriendsJoinGame passing in the address of the XONLINE_FRIEND structure for the friend playing the new title. The current title should then prompt the player for the new title disk as though it was accepting a cross-title invitation.

When the new title boots up, it should call XOnlineFriendsGetAcceptedGameInvite to check if an invitation to an online-game session is pending. All online-enabled titles should call this function when they first boot, even before calling XOnlineLogon or XOnlineFriendsStartup.

If there is no invitation or join pending, XOnlineFriendsGetAcceptedGameInvite returns S_FALSE. If a game invitation is pending, XOnlineFriendsGetAcceptedGameInvite returns an XONLINE_ACCEPTED_GAMEINVITE structure for the game invitation. This structure contains an XONLINE_FRIEND structure for the player who sent the invitation. It also contains an array of XUID structures for the users who accepted the invitation. As described earlier in this section, the title can use the information in these structures to establish a connection to the Xbox that is hosting the online-game session.

IDirect3DDevice8::SetBackBufferScale

SetBackBufferScale allows the title to dynamically scale the size of the back-buffer rendering in order to potentially improve fill rate.

HRESULT SetBackBufferScale(
  float x,
  float y
);

Parameters

x
[in] The horizontal scaling factor. This value must be larger than 0.0f.
y
[in] The vertical scaling factor. This value must be larger than 0.0f.

Return Values

If the method succeeds, the return value is S_OK.

Remarks

The Quincunx and Guassian filters are not compatible with back-buffer scaling and their use will generate debug spew.

SetBackBufferScale causes all transforms to be scaled in screen space by the specified factor. Both IDirect3DDevice8::Present and IDirect3DDevice8::Swap automatically adjust the source rectangle size of the copy to account for the scaling.

A scale factor of (1.0f, 1.0f) always means that the full back buffer is getting used, regardless of any antialiasing. The scale factors can be larger than 1.0 only if the title supplied its own buffer surfaces in the D3DPRESENT_PARAMETERS and those surfaces are large enough.

If there is no antialiasing being done, and if the SwapEffect member of D3DPRESENT_PARAMETERS is D3DSWAPEFFECT_FLIP, calling SetBackBufferScale will automatically convert the SwapEffect to D3DSWAPEFFECT_COPY in order to accomplish the necessary scaling.

Calling SetBackBufferScale essentially invalidates the current contents of the back buffer. If it is being used at all, it should be called before any rendering, right after the previous Swap or Present.

SetBackBufferScale can be with Swap and a device with a BackBuffer of two and whose FullScreen_PresentationInterval is D3DPRESENT_INTERVAL_IMMEDIATE in order to do all scene rendering at the scaled resolution, and then to draw a HUD or text in the final step at the non-scaled resolution.

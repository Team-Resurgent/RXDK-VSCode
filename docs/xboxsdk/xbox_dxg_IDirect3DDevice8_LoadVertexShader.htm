IDirect3DDevice8::LoadVertexShader

Loads a vertex shader into a section of the 136 available addresses on the GPU.

HRESULT LoadVertexShader(
  DWORD Handle,
  DWORD Address
);

Parameters

Handle
[in] The handle of the vertex shader to be loaded.
Address
[in] The location on the GPU to insert the vertex shader, in the range 0 through 135.

Return Values

If the method succeeds, the return value is D3D_OK.

If the method fails, the return value can be D3DERR_INVALIDCALL.

Remarks

The GPU used by the Xbox supports the loading of multiple vertex shaders into a set of 136 available addresses on the chip. Note however that the calling application must keep track of the allocation of addresses per vertex shader. No error checking for overwriting a previously allocated set of addresses is provided.

IDirect3DDevice8::SelectVertexShader is used to select which of the vertex shader programs loaded by LoadVertexShader to use as the current vertex shader program.

In general, a subsequent call to IDirect3DDevice8::SetVertexShader will invalidate any vertex shaders loaded by LoadVertexShader, and require them to be reloaded; however, if SetVertexShader is called using D3DFVF_XYZRHW, only instruction slots 0 through D3DVS_XBOX_RESERVEDXYZRHWSLOTS - 1 will be destroyed. This means that if a title is using programable vertex shaders in conjunction with fixed-function vertex shaders, it is recommended that the first programmable vertex shader should be loaded, beginning at address D3DVS_XBOX_RESERVEDXYZRHWSLOTS, and each subsesquent programmable shader should be added at its end.

Through careful experimentation, it is possible to discover which instruction slots are invalidated by the fixed function pipeline and therefore load vertex shaders into addresses that will not be invalidated through the use of the fixed-function pipeline.

The size of a vertex shader can be acquired using IDirect3DDevice8::GetVertexShaderSize.

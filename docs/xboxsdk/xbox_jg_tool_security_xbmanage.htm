Security Tool (XbManage)

The XbManage security tool allows you to administer access permissions on an Xbox Development Kit. Permissions are determined by the name of the PC attempting to access the Xbox, and have nothing to do with any Windows permissions or usernames.

USAGE: xbmanage [-x xboxname] [options]
Option Description
-lock Locks the target Xbox and gives full permission to the host machine.
-unlock Unlocks the target Xbox, restoring full global access.
-list Lists all machines that have permissions on the target Xbox.
-host:ComputerName [permissions] For the machine specified in ComputerName, this grants or revokes permissions on the target Xbox.
-setpassword Prompts the user to enter a password that will enable administration of the target Xbox from hosts without privileges.
-nopassword Removes the administrative password.
-prompt [permissions] Forces a prompt for the administrative password on the target Xbox. This is generally used with other options to change permissions on the target Xbox, even if the local host machine doesn't have manage permissions.
-x Xbox name or Xbox IP. Specifies a particular target Xbox.
-? Display a complete list of available options.

The following permissions may be combined in any order and are used with the -prompt and -host options above to administer access permissions. Using "+" grants a permission and "-" revokes the permission.

Permissions Description
[+ | -]all Grants/revokes all permissions.
[+ | -]read Grants/revokes file and directory read permission.
[+ | -]write Grants/revokes file write permission.
[+ | -]configure Grants/revokes permission to do ROM flashes and run remote configurations using XbSetCfg.
[+ | -]manage Grants/revokes permission for remote permission management.
[+ | -]control Grants permission to do those actions not covered by the other permisions. Actions include debugging, screen captures, reboots, and so on.

If the XbManage tool is used without any options, it will list the name of the Xbox target system, its security status, and the current permissions.

To enable security on an Xbox, it is necessary to first lock it using the -lock option. After security has been enabled, it is recommended that you set a remote administrative password using -setpassword. If you forget the administrative password and do not have management privileges, the only way to unlock an Xbox is to run a full recovery on the target Xbox.

See the (Remote) Target System Tools topic for more information about this and other remote target system tools.

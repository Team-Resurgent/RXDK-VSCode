Guidelines for Providing Additional Materials with Xbox Game Submissions

This document is intended for use by Xbox game producers and submission materials preparers. Its purpose is to help you avoid unnecessary delays in the game certification process.

So that we can properly test your game, game saves and cheats are required with all certification submissions. The exact needs for game saves and cheats that are provided with each submission are unique for each game genre and each game. Providing insufficient game saves or cheats that do not give sufficient access to the game will cause certification delays while we wait for more effective materials to be submitted. This document provides guidelines to help you establish the number and type of game saves and cheats required for your game specifically.

The Xbox Game Quality Group will use the game saves and cheats provided with your submission to access all game areas and all game states during the limited period of the certification process. Some games have a very linear structure, requiring many hours of play before access to advanced areas, characters, and so on is possible. Some games have very challenging gameplay, requiring many hours to develop specific skills before access to advanced areas, characters, and so on is possible. Game submitters should therefore provide game saves or cheats in all cases where access is limited by (1) time required playing through to the desired game state, and (2) difficulty in playing through to the game state.

When creating your list of submission game saves, keep in mind that the three testing teams in the Xbox Game Quality Group (Game Evaluation, Functional Testing, Compliance and Compatibility Testing) need access to the following:

•         Each area of the game world that is at any time and in any way accessible by the player, to test, among other reasons, geometry faults throughout the game world.

•         Each player-character/vehicle/weapon/tool that is at any time and in any way accessible by the player, to test the functionality of these things.

•         Each character-state and game-state (such as a Level 60 Wizard or second round of playoffs in a sports game) that is at any time and in any way accessible by the player.

•         Each gameplay mode (such as the playoffs in a sports title) accessible to the player, to test the functionality and gameplay of these modes of play.

•         Each in-game cinema scene (non-interactive video or in-game pre-scripted sequence) that is at any time and in any way accessible by the player, both the first time that it is viewed and in subsequent (possibly skippable) viewings, to test the code path.

Also keep in mind that, due to the limited time period of the certification process, the Game Quality Group has the following expectations with regard to your game saves and cheats:

•         None of the game elements listed above will take time-consuming gameplay to access the element from game start or a game save point provided. Possibly, provide a code to assist the testers in getting through any gameplay. (Generally, five minutes of play is acceptable. But we require one minute or less for all of the cinema scenes.)

•         None of the game elements listed above will require above-average game skills to access from game start or a game save point provided. Possibly, provide cheats such as invulnerability to get testers through any gameplay.

The following appendixes present general guidelines for game saves required for each major genre. General examples are provided to help you identify which genre guidelines are most appropriate for your game, while specific examples are included to show the range of problems that can be encountered when trying to meet these requirements. Note that while a game may be considered to be of a particular genre, that does not necessarily mean that the guidelines for that exact genre apply. Each game design is unique and may include aspects that result in atypical game save requirements.

Appendix I: General Examples

These are broad categories that generalize which types of games are typically found in any particular category. Individual games may vary from the standards of the genre that best categorizes the game.

Few saves/cheats needed Genres commonly found in this category are sports games, some Xtreme sports games, some racing games, and fighting games.

Games in which the environment(s)/modes are all accessible from one save. Games in which the player unlocks features throughout the game, which are saved. Games in which players progress through very similar events (for example, football games) to win the season. Games in which most cinema scenes are quickly and easily encountered during the standard gameplay experience (for example, fouling, scoring, and end-of-match cinemas in a sports title).

Best case for needed materials: One save in which all of the environments, characters, hidden features and modes have been made accessible to the player from the main menu.

Example games: Madden 2001, Tony Hawk 2, Soul Caliber, Gran Turismo 2, Smuggler's Run, and 4×4 Evolution.

Level-based saves/cheats Genres commonly found in this category are most racing games, linear platform games, most Xtreme sports games, and mission-based strategy/first-person shooters.

Games in which the environments are partitioned into discrete segments of linear gameplay that can be accessed only from playing from the beginning of the segment. Games in which linear gameplay can only be saved at certain discrete points (for example, save points). Also, games in which items are collected solely for opening new or hidden areas (for example, stars in Mario 64).

Best case for needed materials: One save at the beginning of each segment of gameplay. Each save should have all of the items from that level that are needed to open further levels in the game.

Examples:

•         A save before each circuit in a racing game.

•         A save at the beginning of each world, level, or save point in a platform game.

•         Cheats that allow a player to quickly run through a level without needing to fight or interact with the game (for example, invulnerability).

Example games: Metropolis Street Racer, Spiderman, Starcraft, Unreal Tournament, Rumble Racing, Midnight Club, Twisted Metal Black, NBA Streetz, and Time Splitters.

Open-ended games Genres commonly found in this category are linear RPG or adventure games, item-collecting platform games, and build-up strategy or shooter games.

Games in which the player collects optional items that offer better abilities (for example, special weapons, new spells, extra hearts). Games in which the player collects scattered items to open hidden areas.

Examples of needed materials: Several saves throughout the game, spaced so that each boss, area, weapon/item, or play mode is available to the testers by playing for a few minutes from each save. Cheats that give all of the items in the game to the character(s) or cheats that can grant levels, one by one, to the character(s).

Suggestion: When creating the walk-throughs that are needed for this type of game, make saves at regular intervals and list each of these saves right in the walk-through document.

Example: After crossing the desert (save 24), travel north into the village. Talk with the elder to get the next location placed on the world map. Talk with the blacksmith to get a side quest in which you get special ore from the caves to the east and return to him to get the Wonder Blade (save 25).

This is the first of the three chase areas of the game. The objective is to avoid hitting the shores, but cut the corners as tightly as possible to pull ahead. If you keep up for the whole race you land on an island.

Example games: Skies of Arcadia, Banjo Kazooee, Legend of Zelda: Ocarina of Time, most Final Fantasy games, Winback, Resident Evil, Onimusha, and Escape from Monkey Island.

Complex open-ended games (games with divergent gameplay paths) Genres commonly found in this category are nonlinear RPG or adventure games, and very open-ended strategy games.

Games in which the player can go almost anywhere or do things in such an open order that predicting a course is problematic. Games in which the player must build up their faction throughout the course of the game in order to beat it. Games in which the player may custom create characters of diverse nature. Games with different endings depending on the sum of different events throughout the game.

Examples of needed materials: Several saves through multiple passes through the game. Each pass through the game should attempt to show different styles of play, characters or player selected goals. Each set of saves for these passes should show cinemas, items, and environments that are not only relevant to a particular pass through the game but also cover the full range of these elements in the game.

Example Games: Legend of Zelda: Majora's Mask, Baldur's Gate II, Black & White, Civilization, Chrono Cross, Shadow of Destiny, Roller Coaster Tycoon, and Sim Theme Park.

Appendix II: Specific Cases

The Sims

The Sims is an extremely open-ended game in which the player's goal is self-determined. For saves from this game to be of any use in testing, they must come with notes describing what the situation is leading up to the save and what the player goals were that led to this point in time for each save.

Ideally there would be three or four walk-throughs to demonstrate some of the choices the player can make that lead to the developed and complex situation that make this game so popular (its fourth add-on, involving going on dates in the city, is currently being developed). In each walk-through, saves can be made at regular intervals, or just before significant events, so that evaluators have a chance to view the rewards that come through gameplay without playing the game for weeks at a time.

In addition, "fully built-up" saves that show the full grandeur of the house-building and demonstrate all of the objects that the player can buy for his Sims would allow testing of the functionality of all aspects of the game.

There are no cinema scenes, although there are in-engine events (like weddings). Fortunately for testing purposes, it is possible to save the game at any time, allowing examination of each of these events without significant gameplay; although testers will need to do some playing from the save point to ensure that the event takes place. To fulfill this need of our testing groups, all that would be needed is a list of each event that each save represents and what actions need to be done to trigger it.

Ogre Battle

Ogre Battle is a strategy game in which the player can build several different types of forces, and the game provides different endings depending upon the choices made at several points in the game. There are a few cinema scenes at the beginning and end of the game, and one before each chapter, but any save before each is simple in this game.

Capturing all of the unit possibilities is only somewhat problematic, because the player is allowed to have 16 units (of three to five characters each), allowing two or three saves per chapter to include all of the units and character possibilities. And with three endings, three separate gameplay paths could be used for both demonstrating the unit possibilities and leading to each ending.

Final Fantasy IX

Final Fantasy IX is a more-or-less linear game with large amounts of open-ended content. While a set of standard saves matched to a walk-through will capture most of the game for the testers, there is a lot of side material in this game that will need additional saves to place the testers right at the doorstep of alternate game activity (mini-games) and hidden objects.

Grandia II

Grandia II is a fairly linear game, although many items can be collected and there should be some variation between different players' saves taken at the same point in the game.

Two sets of saves may be enough for this game: a standard save at each critical point in the game with the characters equipped as a typical player would have them, and a maxed out save, with all of the gear the characters could have at that point in the game, at those same points.

Bust-a-Move

While only one save before each puzzle is needed for Bust-a-Move, there are a lot of puzzles in the game. On a game of this nature, it may be possible to give a save for every three or four puzzles, as long as complete directions on how to do each puzzle are included.

Mario Kart

In the well-known racing game Mario-Kart, the game would automatically save only at the end of each circuit, not each race. This means that testers will need to race through a minimum of three races from each save to see the end-of-circuit cinema scene. Due to the limitation built into the game on saving, one save for each circuit is the best that can be done for this game. In addition to this, a means by which to successfully beat a race instantly would effectively allow the Compliance and Compatibility Team to quickly access all of the cinemas in the game.

Tekken Tag Tournament

Because Tekken Tag Tournament does very little in the way of saving information, and there is no way in which to start play near the end of a series of matches, the most effective way of helping testers get past the challenging fights is to use cheats that either skip matches (marking them as won) or to provide some sort of invulnerability. This could allow testers to jump fairly quickly to each of the end-of-tournament cinemas with little work and also fight their way to those cinemas a second time to verify that they are skippable upon subsequent viewings. In addition, the testers can easily test the step-up in challenge as the tournament progresses without having to spend hours or days increasing their skill with the game.

SSX

In SSX, the player must finish in first place on a race to open up more content. All of the end-of-race cinema scenes are the same (except that the character might be different), so there is little need to make special saves to capture all of the cinemas. This is an example of a game where one save with all of the content opened up may suffice for testing purposes. (But remember, the Xbox Compatibility and Compliance Team will need to see every cinema the first time it is reached and on all further viewings to test the code path for skipping cinema scenes.)

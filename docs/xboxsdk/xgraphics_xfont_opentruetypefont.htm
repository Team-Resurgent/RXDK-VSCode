XFONT_OpenTrueTypeFont

Loads a TrueType font into the scan converter.

HRESULT __stdcall XFONT_OpenTrueTypeFont (
  LPCWSTR wszFileName,
  unsigned uCacheSize,
  XFONT **ppFont
);

Parameters

wszFileName
[in] File name of the font.
uCacheSize
[in] Size of the cache, in bytes.
ppFont
[out] Address of a pointer to the returned XFONT interface, representing the opened font.

Return Values

On success, the function returns S_OK.

Remarks

Calling XFONT_OpenTrueTypeFont will use more memory that just the font cache because the TrueType scan converter needs a bit of workspace to process the font. If memory is a concern, use a bitmap font instead.

For performance reasons, applications should avoid using TrueType fonts, if possible. Support for TrueType fonts is not enabled by default. To enable this support, the title must #define XFONT_TRUETYPE before including xfont.h.

XCreateSaveGame

The XCreateSaveGame function creates a new instance of a save game container in the specified user data storage region. The location of the save game (hard disk, memory unit, and so on) is dependent on the storage region specified.

DWORD XCreateSaveGame(
  LPCSTR lpRootPathName,
  LPCWSTR lpSaveGameName,
  DWORD dwCreationDisposition,
  DWORD dwCreateFlags,
  LPSTR lpPathBuffer,
  UINT uSize
);

Parameters

lpRootPathName
[in] Pointer to a null-terminated string that specifies the root directory of the user data storage region where the save game will be created. The parameter must contain only a drive letter, colon, and trailing backslash; a subdirectory cannot be specified.
lpSaveGameName
[in] Pointer to a null-terminated string that specifies the name of the save game. The name is limited to MAX_GAMENAME characters.
dwCreationDisposition
[in] Value that indicates whether a new save game is being created or an existing save game is being opened. This parameter must be one of the following values:

Value name Meaning
CREATE_NEW Create a new save game. If the specified save game already exists, the function fails.
OPEN_EXISTING Open an existing save game. The function fails if the save game does not exist.
OPEN_ALWAYS Open the save game if it exists. If it does not exist, the function creates the save game as if dwCreationDisposition were CREATE_NEW.

dwCreateFlags
[in] Can be zero or XSAVEGAME_NOCOPY. If XSAVEGAME_NOCOPY is specified, the save game is marked as not copyable—the Xbox Dashboard will not allow the user to copy the save game to a different location. XSAVEGAME_NOCOPY is valid only if dwCreateDisposition is CREATE_NEW or OPEN_ALWAYS.
lpPathBuffer
[out] Pointer to a buffer that receives the directory path of the save game container. The buffer must be large enough to contain the returned directory path string, including a terminating null character. The directory path returned is the directory of the save game container. This directory will contain the data and metadata for the specified save game.
uSize
[in] The size, in characters, of the buffer pointed to by the lpPathBuffer parameter.

Return Values

If the function succeeds, the return value is ERROR_SUCCESS.

If the function fails, the return value is an error code defined in Winerror.h. The function does not use SetLastError to set the calling thread's last-error code.

Remarks

The storage region specified in lpRootPathName can be the title's user data region on the hard disk (U:\), a mapped user data region for an alternate title (X:\), or the title's user data region of a mounted memory unit (F:\ through M:\).

If the buffer specified in lpPathBuffer is not large enough to contain the returned directory path, the path will be truncated and the function will indicate success.

The directory returned in lpPathBuffer is the directory that will hold the data and metadata files containing the information representing the saved game. After creating a save game container, the title uses the standard file I/O functions (CreateFile, WriteFile, ReadFile, and so on) to create, write, and manipulate files in that directory.

The XSAVEGAME_NOCOPY flag of the dwCreateFlags parameter is valid only if dwCreateDisposition is CREATE_NEW or OPEN_ALWAYS. The non-copyable property can be applied only to save games being newly created. If XSAVEGAME_NOCOPY and OPEN_ALWAYS are specified and the save game already exists, the XSAVEGAME_NOCOPY flag is ignored (in debug builds, a warning message is output to the debugger).

A call to XCreateSaveGame to create a new save game or overwrite an existing one (by specifying either CREATE_NEW or OPEN_ALWAYS for dwCreationDisposition) updates the time stamp for the save game container directory. Specifying OPEN_EXISTING for dwCreationDisposition does not update the time stamp. The Xbox Dashboard displays available save games in an MRU (most recently used) order.

For more information on save games and save game signing, See the Save Game Signing section of XTL Services, the LoadSave Sample, the Xbox Technical Certification Game, and Save Games.

BuildOffer Tool (BuildOffer)

The BuildOffer tool is used to build secure Xbox content packages (.xcp). See Downloading Content from Xbox Live for more information.

Usage

USAGE: buildoffer [switches] [N|L|X] package metadata [@list] [files]...
SwitchDescription
-? Display usage.
-r Recurse subdirectories.
-u Disable file compression.
-v Run in verbose mode, producing detailed package information.
-x:<DLL name> Loads the specified signature calculation DLL. You can specify as many signature DLLs as you wish by repeating the -x option.
-D Use debug mode (see description below).
-O Create an offline package, which may be installed offline.
-S Create packages for multiple continents (see description below).
N Create a new package.
L List the contents of a package.
X Extract the contents of a package.
package A required parameter that specifies the path and file name of the resulting package. If only a file name is specified, the package will be generated in the current directory.
metadata A required parameter that specifies the path and file name of a file that contains the catalog-referral metadata about this package. This program does not validate the metadata contained in the file. It is the responsibility of the content publisher to make sure the metadata is valid.

The BuildOffer tool must be run from the root directory where the content to be packaged resides.

The list of files to be packaged is specified through a combination of file names (which may contain wildcards), and/or list files. File names are relative to the current directory from which this application is run. List files may contain file names (one name per line) that are relative to the directory from which this program is run.

The -r Switch

By default, wildcard matches are non-recursive. Recursive matches can be enabled by specifying the -r option. The -r option is global and thus affects all wildcard matches. For example, to create a package that contains all files under the current directory hierarchy, one would do:

buildoffer -r N test.xcp .\metadata.xbx *.*

The directory hierarchy will be preserved when the package is installed on the Xbox hard disk. This allows the content publisher to lay out the content exactly as it should appear on the Xbox, and then build a package the preserves the whole layout.

The -D Switch

A debug mode can be specified using the -D option, which will output the raw cabinet file without the secure packaging. This allows the content publisher to list and extract the files to verify that the cabinet layout is correct. The raw cabinet file will have a .cab file extension rather than the .xcp extension that you would expect for a secure Xbox content package. The -S Switch

If content is offered on multiple continents, specify the -S option when invoking the BuildOffer tool. This option instructs BuildOffer to create multiple packages, one for each continent. The name of each package file will have a prefix that identifies which continent that package is for: NA for North America, EMEA for Europe, JA for Japan. When using the -S option, you must ensure that the bits in the Offering Identifier that specify the continent are all zero. BuildOffer sets these bits to the correct value for each package. For more information, see the Offering Packages topic.

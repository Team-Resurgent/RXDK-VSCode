XOnlineThrottleSet

Sets the current throttle settings for the given throttle type.

HRESULT XOnlineThrottleSet(
    DWORD dwServiceID,
    LPCSTR szThrottleTag,
    DWORD dwThrottleFlags,
    DWORD dwDelay
);

Parameters

dwServiceID

[in] Identifier for the service to set. May be one of the following:
Service Identifier Description
XONLINE_STATISTICS_SERVICE Refers to the statistics service functions
XONLINE_QUERY_SERVICE Refers to the query service functions
XONLINE_ARBITRATION_SERVICE Refers to the arbitration service functions
XONLINE_MESSAGING_SERVICE Refers to the messaging service functions
XONLINE_TEAM_SERVICE Refers to the team service functions

A complete listing of these flags and their relation to the szThrottleTag parameter can be found in the Remarks section below.

szThrottleTag
[in] String which indicates the specific method to set. See the Remarks section below for a complete listing of valid throttle tag values and their relation to the dwServiceID parameter.
dwThrottleFlags

[in] Flag values returned. May be the following:
Value Description
XONLINE_THROTTLE_FLAG_DELAY Delay starting each new operation until the throttle delay period expires.
XONLINE_THROTTLE_FLAG_FAIL Return XONLINE_E_TASK_THROTTLED for new tasks that are started during the throttle delay period.
XONLINE_THROTTLE_FLAG_CANCEL_PREVIOUS Force the previous task of the same type to fail with XONLINE_E_TASK_ABORTED_BY_DUPLICATE when possible.
XONLINE_THROTTLE_FLAG_RIP RIP in debug.
XONLINE_THROTTLE_FLAG_IGNORE_SERVER Ignore any changes in the delay or flags that the server requests (debug only).

dwDelay
[in] Delay time, in seconds.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

This function is a debug-only method. It is useful to temporarily work around the debug RIP that occurs when the throttle limits are exceeded, or to observe how the title behaves with different throttle settings.

XOnlineGetServiceInfo

Indicates whether a specified service has been successfully logged on, and optionally retrieves additional information about the service.

HRESULT XOnlineGetServiceInfo(
    DWORD dwServiceID,
    PXONLINE_SERVICE_INFO pServiceInfo
);

Parameters

dwServiceID
[in] Service identifier of the service to query.
pServiceInfo
[out] Pointer to an XONLINE_SERVICE_INFO structure that receives information about the specified service. This parameter can be NULL if this information is not required.

Return Values

The return value indicates the connection status of the specified service, and can be one of the following values:

Value Description
S_OK A successful connection to the service has been made.
XONLINE_E_INTERNAL_ERROR An unspecified error occurred. For example, the specified service does not exist in the service database.
XONLINE_E_LOGON_SERVICE_NOT_REQUESTED The service was not requested in the call to XOnlineLogon.
XONLINE_E_LOGON_SERVICE_NOT_AUTHORIZED The logged on users are not authorized to access the service.
XONLINE_E_LOGON_SERVICE_TEMPORARILY_UNAVAILABLE The service is temporarily unavailable. The title can proceed with other online functionality that does not use this service. If the player attempts to perform functions related to this service, the title should inform the player that the service is temporarily unavailable.

Remarks

XOnlineGetServiceInfo is used to determine the status of service logons specified in XOnlineLogon, and to retrieve additional information about logged on services. XOnlineGetServiceInfo should be called only after the logon task has returned XONLINE_S_LOGON_CONNECTION_ESTABLISHED.

The asynchronous logon task created with XOnlineLogon must remain open (the handle not closed) in order for XOnlineGetServiceInfo to be called.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

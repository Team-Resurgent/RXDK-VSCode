ILiveEngine::DoWork

Performs work within the Xbox Drop-in UI engine, and informs the title of further steps to perform.

HRESULT DoWork(
    DWORD *pDoWorkFlags
);

Parameters

pDoWorkFlags

[out] Pointer to a DWORD variable that receives flags indicating further actions that must be performed by the title. Can be a combination of the following values.
Value Description
UIX_DOWORK_NEED_TO_RENDER The engine requires rendering; call ILiveEngine::Render.
UIX_DOWORK_NOTIFICATIONS The engine has received in-game notifications that must be processed by the title; call ILiveEngine::GetNotifications. These notifications are returned only if the title has explicitly requested that notifications be handled by the title rather than the engine. See Remarks below.
UIX_DOWORK_NEED_TO_REBOOT The engine needs to reboot (to perform an auto-update, for example, or to boot to the Xbox Dashboard so the player can create a new Xbox Live account). The title should initiate a reboot by calling ILiveEngine::Reboot.
UIX_DOWORK_FEATURE_EXIT A feature has exited; call ILiveEngine::GetExitInfo. This flag is also set when the network connection is lost, though no feature is active or exited.
UIX_DOWORK_PROCESSING_INPUT Indicates that a feature is actively processing player input. If this flag is set, controller input is going to an active feature and should not be processed by the title. Otherwise, the title should process controller input normally.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

Titles should call DoWork once per frame.

Normally, the Xbox Drop-in UI engine handles in-game notifications automatically. The UIX_DOWORK_NOTIFICATIONS flag will be returned only if the title has disabled automatic notification handling by calling ILiveEngine::SetProperty and setting UIX_PROPERTY_DISPLAY_NOTIFICATIONS to FALSE. Otherwise, titles do not need to handle the UIX_DOWORK_NOTIFICATIONS flag.

Requirements

Header: Declared in Uix.h.

Import Library: Use Uix.lib.

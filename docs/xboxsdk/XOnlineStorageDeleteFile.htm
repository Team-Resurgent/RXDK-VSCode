XOnlineStorageDeleteFile

Allows a title to delete files per user title managed storage.

HRESULT XOnlineStorageDeleteFile(
    DWORD dwFacility,
    DWORD dwUserIndex,
    LPCWSTR wszStorageFileName,
    HANDLE hWorkEvent,
    XONLINETASK_HANDLE *phTask
);

Parameters

dwFacility

[in] Indicates storage facility. May be one of the following:
Value Description
XONLINESTORAGE_FACILITY_TEAMS Teams storage facility.
XONLINESTORAGE_FACILITY_PER_USER_TITLE Per title, per user storage.

For a complete list of facility values, see the XONLINESTORAGE_FACILITY page.

dwUserIndex
[in] Controller index (zero-based) of the player for whom to delete the file.
wszStorageFileName
[in] File name of the file to be deleted.
hWorkEvent
[in] Handle to an event that is signaled when there is work to do on the asynchronous task returned by the phTask parameter. This parameter can be NULL if such synchronization is not required.
phTask
[out] Pointer to a variable of type XONLINETASK_HANDLE that receives a task handle. Use this handle in calls to XOnlineTaskContinue to process the file upload.

Return Values

If XOnlineStorageDeleteFile succeeds, the return value is S_OK.

If XOnlineStorageDeleteFile fails, the return value is one of the following error codes:

Value Description
XONLINE_E_STORAGE_INVALID_REQUEST The request is invalid.
XONLINE_E_STORAGE_ACCESS_DENIED The client doesn't have sufficient permissions to access the file.
XONLINE_E_STORAGE_INVALID_ACCESS_TOKEN The access token signature is invalid.
XONLINE_E_STORAGE_CANNOT_FIND_PATH Name resolution failed.
XONLINE_E_STORAGE_INVALID_STORAGE_PATH Invalid storage path.
XONLINE_E_STORAGE_INVALID_FACILITY Invalid facility code.
XONLINE_E_STORAGE_FILE_NOT_FOUND The file to delete was not found.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

Setting the Loader's Search Directory

By default, the loader looks for objects in the current directory unless a full path is specified in the wszFileName member of the DMUS_OBJECTDESC structure describing the object being sought. By using the IDirectMusicLoader8::SetSearchDirectory method, you can set a different default path for the IDirectMusicLoader8::GetObject, and IDirectMusicLoader8::LoadObjectFromFile methods. This default path can apply to all objects, or only to objects of a certain class.

The following code example sets the search path for all DirectMusic file types.

IDirectMusicLoader8 *pLoader;
DirectMusicCreateInstance( CLSID_DirectMusicLoader, NULL, 
                           IID_IDirectMusicLoader8, (VOID**)&pLoader );

HRESULT hr;

// Set the search directory for DirectMusic styles
hr = pLoader->SetSearchDirectory(
    GUID_DirectMusicAllTypes,
    "d:\\allmyaudio\\",
    FALSE):

// insert handling code for the HRESULT

After calling the code above, the title can load all of their DirectMusic content by file name, without including the full path.

The following code example sets different search paths for segment and style files.

IDirectMusicLoader8 *pLoader;
DirectMusicCreateInstance( CLSID_DirectMusicLoader, NULL, 
                           IID_IDirectMusicLoader8, (VOID**)&pLoader );

HRESULT hr;

// Set the search directory for DirectMusic styles
hr = pLoader->SetSearchDirectory(
    CLSID_DirectMusicStyle,
    "d:\\myaudio\\styles\\",
    FALSE):

// insert handling code for the HRESULT                                   

// Set the search directory for DirectMusic segments
hr = pLoader->SetSearchDirectory(
    CLSID_DirectMusicSegment,
    "d:\\myaudio\\segments\\",
    FALSE):
    
// insert handling code for the HRESULT

After calling this function, the title can load a style or segment by file name, without including the full path. Also, the segments and styles are now organized in their own directories, which could make it easier to manage your audio content.

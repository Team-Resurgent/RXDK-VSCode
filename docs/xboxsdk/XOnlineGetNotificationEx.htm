XOnlineGetNotificationEx

Requests extended notification information for a given player.

BOOL XOnlineGetNotificationEx(
    DWORD dwUserIndex,
    PXONLINE_NOTIFICATION_EX_INFO pNotificationInfo,
    DWORD *pdwStateFlags
);

Parameters

dwUserIndex
[in] Controller index (zero-based) of the player making the request.
pNotificationInfo
[out] Pointer to an XONLINE_NOTIFICATION_EX_INFO structure to receive notification information.
pdwStateFlags
[out] Pointer to a DWORD to return additional information about the notification system:
Value Description
XONLINE_NOTIFICATION_STATE_FLAG_PENDING_SYNC Messaging is currently synchronizing with the server.
XONLINE_NOTIFICATION_STATE_FLAG_MORE_ITEMS There are still more unread notifications available.
XONLINE_NOTIFICATION_STATE_FLAG_OVERFLOW_ITEMS One or more notifications have been dropped and will not be reported.

Return Values

TRUE – A notification had been read, and the information is returned in pNotificationInfo. Additional state information is returned in pdwStateFlags.

FALSE – There are no notifications available to retrieve. The contents of pNotificationInfo will be zeroed out, and pdwStateFlags will be either 0 or XONLINE_NOTIFICATION_STATE_FLAG_PENDING_SYNC.

Remarks

The title may call this in a loop to return multiple notifications (if available) until the function returns FALSE. In case of multiple notifications, the function returns notifications in the order where the oldest notification is returned first.

If the title calls XOnlineGetNotificationEx infrequently enough, it is possible that the notification queue will overflow. This will result in loss of notification items. In this case, pdwStateFlags will contain XONLINE_NOTIFICATION_STATE_FLAG_OVERFLOW_ITEMS, and the title is encouraged to refresh its state by enumerating the messages using XOnlineMessageEnumerate.

In order to receive notifications of XONLINE_MSG_TYPE_TITLE_CUSTOM or XONLINE_MSG_TYPE_TEAM_RECRUIT messages generated using a family title ID, callers must have previously specified the family title ID to XOnlineMessageEnableReceivingFamilyTitleIDs.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

Uploading Game Clips

The following outline illustrates how to upload game clips to Xbox Live. The specific example given here is for uploading a ghost race. However, the same sequence of function calls can be applied to the process of uploading any type of game clip.

Note    If your game clip is stored on the console's hard drive, it must be signed before it is uploaded.

  1. As the user races, create a file of the replay and sign it with the Xbox Live key.

  2. When the user finishes the race, write the score to the server, as follows:

    1. Call XOnlineStatWrite. Store any game-specific information about the content as stat attributes, before initiating the upload. Here are some examples of the type of information to store:

      • Game region, if the attachment content does not work across multiple regions in the game.
      • Title version, to allow the game to block obsolete content after an auto-update.
      • Length of replay. If this is a high-score movie, a user may want to know how long it takes to watch.
      • The size of the file (compressed and uncompressed) is stored automatically by XOnlineStatWrite.
    2. Pump the write task handle with XOnlineTaskContinue.

    3. If you get the return value XONLINE_S_CAN_UPLOAD from XOnlineStatWrite, do the following:

      1. Call XOnlineStatWriteGetResult to get a remote file reference.
      2. Close the task with XOnlineTaskClose.
      3. Ask the player if he or she wants to upload. Alternatively, your title can begin the upload automatically.

      4. Upload the attachment file, as follows:

        1. Call XOnlineStorageUpload with the remote file reference retrieved by the call to XOnlineStatWriteGetResult.
        2. Pump the upload with XOnlineTaskContinue.
        3. Optionally call XOnlineStorageGetProgress to find out how the upload is going and to display a progress bar.
        4. XOnlineStorageGetProgress indicates that the upload is complete through the output parameters in its parameter list. Even if it is complete, there is no guarantee the content is on the specified leaderboard. The user could have lost his or her standing on the leaderboard during the upload. For example, at the time the upload started, Mary qualified for 9th place; by the time the upload completed, she had fallen to 15th place, but the leaderboard isn't configured to store attachments for that low a ranking.
        5. Close the task with XOnlineTaskClose.

XOnlineCompetitionTopology

Searches the topology of a competition.

HRESULT XOnlineCompetitionTopology(
    DWORD dwTemplate,
    ULONGLONG qwCompetitionID,
    DWORD dwPage,
    DWORD dwResultsPerPage,
    DWORD dwStartingEventTopologyID,
    DWORD dwEndingEventTopologyID,
    DWORD dwNumResultAttributeSpecs,
    const XONLINE_ATTRIBUTE_SPEC *pResultAttributeSpecs,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwTemplate
[in] ID of the dataset associated with the competition.
qwCompetitionID
[in] Unique ID of competition.
dwPage
[in] Page number of the results being requested. Pages start with 0.
dwResultsPerPage
[in] Number of results per page. Maximum is 255.
dwStartingEventTopologyID
[in] First event ID in the range of events being requested.
dwEndingEventTopologyID
[in] Last event ID in the range of events being requested.
dwNumResultAttributeSpecs
[in] Number of entries in the array specified by the pResultAttributeSpecs parameter.
pResultAttributeSpecs
[in] Array of XONLINE_ATTRIBUTE_SPEC structures containing title-specific attributes associated with the competition. This array must start with an integer attribute with the dwType member set to XONLINE_QUERY_ENTITY_ID. This must be followed by the list and size of the return values configured for the Topology query in XLAST.
hWorkEvent
[in, optional] Handle of associated an event you want to associate with this task. If you do not want to assign an event, set this parameter to NULL.
phTask
[out] Pointer to an XONLINETASK_HANDLE type. The handle is associated with the task started by XOnlineCompetitionTopology. This handle can be used in calls to XOnlineTaskContinue to perform work associated with this task.

Return Values

Returns S_OK if successful, an error code otherwise. These error codes can include the following.

Error Code Description
XONLINE_E_QUERY_SPEC_COUNT_MISMATCH The number of attributes listed in pResultAttributeSpecs does not match the list of return values on the server, which was configured using XLAST.
XONLINE_E_QUERY_ATTRIBUTE_TOO_LONG A string or BLOB (binary block of data) type attribute listed in pSearchAttributes or pResultAttributeSpecs specifies a length that is smaller than the data stored on the server.
XONLINE_E_QUERY_DATASET_NOT_FOUND The specified dwTeamplate does not correspond to the template configured in XLAST.

Remarks

The topology, or bracket, of a competition is the set of relationships between the competition's individual events. Each event generally corresponds to a match or a series of matches. An event has two competitor slots. A competitor can be an individual player or a team of players. All competitors need to be assigned to an available slot in an event. In cases where there are not enough competitors to fill the available slots, the sole competitor in the event is given an automatic win. This automatic win is called a bye.

Competitions also contain rounds. Rounds are groupings of events that simplify the presentation of competition information.

Topology queries return a group of results, each identified by an event ID. Event IDs consist of the zero-based round index in bits 16–32, and the zero-based event index in bits 0–16. You can create an event ID with the follwing expression. 

(roundIndex << 16) + eventIndex

To obtain the results of the topology query, call XOnlineCompetitionTopologyGetResults.

Single elimination competitions should not use this function. Instead, call XOnlineCompetitionTopologySingleElimination.

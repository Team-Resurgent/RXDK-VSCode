Directly Defined Pixel Shaders

Xbox offers an alternative to the pixel shader assembler language. Applications written on Xbox need to deal with only one hardware configuration, so it is possible for them to define pixel shaders in the native hardware format supported by the Xbox GPU. The D3DPIXELSHADERDEF structure encapsulates all of the hardware states required to define a pixel shader on Xbox:

typedef struct _D3DPixelShaderDef
{
   DWORD    PSAlphaInputs[8]; 
            
   // Other member fields
   // ...   
   
   DWORD    PSC0Mapping;               
   DWORD    PSC1Mapping;               
   DWORD    PSFinalCombinerConstants;  
} D3DPIXELSHADERDEF;

All but the last three DWORDs of the structure are loaded into the GPU hardware to define the pixel shader. The order of these fields is defined to make loading them into the hardware as efficient as possible. The last three DWORDs are used to define how Direct3D8 pixel shader constants map to the constant registers in each combiner stage. They are used by the Direct3D run-time software but not by the hardware.

Each member of D3DPIXELSHADERDEF, used by the hardware, has a corresponding Direct3D render state. For example, PSAlphaInputs[0] corresponds to the D3DRS_PSALPHAINPUTS0 render state. When a pixel shader has been set using the IDirect3DDevice8::SetPixelShader method, these render states can be used to modify the pixel shader on the fly. See D3DRENDERSTATETYPE for a complete listing of the render states.

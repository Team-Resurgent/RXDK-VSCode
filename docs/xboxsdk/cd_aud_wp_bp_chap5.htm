Xbox Audio Content Development Tools

By Scott Selfon, Audio Content Consultant, Content and Design Team, Xbox Advanced Technology Group

Introduction

As part of our continuing efforts to make development on the Xbox® video game system from Microsoft artist-centric and accessible to composers and sound designers, we have been steadily releasing a series of tools and samples that cover various aspects of audio production for the Xbox. This chapter provides an overview of each of the currently available tools as well as a preview of tools we hope to provide in the future. In most cases, rudimentary command-line tool documentation can be found by typing the name of the application followed by "-?" at a command prompt. For instance:

xgpimage -?

Note also that to use command-line applications that send data to the Xbox (for instance, xbsetcfg), you will need to point your Microsoft® Windows® 2000 development machine at the proper Xbox by using the "-x [xbox name]" switch. This needs to be done only once (unless of course you're switching between machines or you change your Xbox machine name from the XDK launcher).

xgpimage -x "AudioBox1"

will point future commands to the Xbox named "AudioBox1".

As always, if you think a specific area of content creation could use a tool, or if you have suggestions for one of our existing tools, please contact content@xbox.com.

Xbox Audio Creation Tool (xact.exe)

Description: Tool for creating wave banks and sound banks. Can be used in conjunction with XACT runtime engine for more complex content authoring, realtime auditioning, and abstraction of sound events for developers.

Installation: Installed with XDK at C:\Program Files\Microsoft Xbox SDK\xbox\bin\xact.exe.

Comments:

  • XACT can be used to create in-memory and streaming wave banks (.xwb) as well as sound banks (.xsb).
  • An optional header (.h) file can be generated for runtime use of wave banks without the XACT engine. For examples of this, see the WaveBank and WaveBankStream samples in the XDK.
  • Provides some basic "compression" options—ADPCM compression or 8-bit downsampling on individual waves (in addition to option to retain PCM).

  • Support for .wav, AIFF, and WMA file formats.
  • Can be used to audition cues, sounds, or waves on an XDK running the Audio Console Application, including 4- and 6-channel waves.
  • For more details on XACT, see Chapter 9 of the Audio Best Practices Guide, which focuses on content creation, and Chapter 10 of the Audio Best Practices Guide, which looks at XACT and the developer's role.

Xbox ADPCM Codec (xbadpcm.acm)

Description: Xbox ADPCM code is an Audio Compression Manager (ACM) codec that allows you to create and play back Xbox adaptive pulse code modulation (ADPCM) content on your Windows 2000/XP development machine.

Installation: Installed with Xbox Development Kit (XDK). Can also be manually installed from the XDK Tools page. Note that there is a known issue with the Voxware Compression Toolkit, which will need to be disabled before the Xbox ADPCM codec will function properly:

  1. From the desktop, click Start, Settings, and then Control Panel.
  2. In the Control Panel window, double-click Sounds and Multimedia. The Sounds and Multimedia Properties dialog box will appear.
  3. Click the Hardware tab.
  4. In the Devices pane, click Audio Codecs, and then click Properties. The Audio Codecs Properties dialog box will appear.
  5. Click the Properties tab.
  6. Double-click Voxware Compression Toolkit and click Do not use this audio codec. Once the Voxware codec is disabled, the Xbox ADPCM codec will function properly; however, any applications that require the Voxware codec will cease to work correctly.

Comments:

  • Because the Xbox ADPCM codec is an ACM codec, all applications (including most wave editing and effects software) that support ACM can use it.
  • Author ADPCM loop points on 64-sample boundaries (36 bytes).
  • Make sure when using ADPCM waves that the developer reads in the entire ADPCM header structure.
  • The codec supports 16-bit mono or stereo waves as source content.
  • Compression is roughly 3.5:1.

Xbox ADPCM Encoder (XbADPCMEncode.exe)

Description: The Xbox ADPCM encoder is a command-line tool for ADPCM encoding a pulse code modulation (PCM) wave file

Installation: Source code is installed with XDK in C:\Program Files\Microsoft Xbox SDK\Tools\Win32\Sound\xbadpcmencode. Must be built by developer to use.

Comments:

  • Provided to demonstrate how your own encoder could be written for batch purposes or to store custom format information.

  • Demonstrates saving of loop points within ADPCM format.

DSP Builder (dspbuilder.exe)

Description: DSP Builder is a graphical tool used for creating Global Processor DSP programs, which are effects images used for real-time effects creation and manipulation. The DSP Builder tool currently provides a subset of the functionality of XGPImage but in a more visual and accessible manner.

Installation: Installed with XDK in C:\Program Files\Microsoft Xbox SDK\xbox\bin\dspbuilder.exe.

Comments:

  • For more details about using DSP Builder and how to audition effects programs, see the white paper Authoring and Auditioning Effects Programs.
  • DSP Builder can be used in conjunction with titles that use the instrumented XACT library, or with the Audio Console application to audition, update, and swap effects in real time on the Xbox audio hardware.

XGPImage (xgpimage.exe)

Description: XGPImage is a command-line tool for creating Global Processor DSP programs, which are effects images used for real-time effects creation and manipulation.

Installation: Installed with XDK in C:\Program Files\Microsoft Xbox SDK\xbox\bin\xgpimage.exe.

Comments:

  • An effects image authored in DSP Builder can be exported to XGPImage for lower level manipulation (generally not necessary though).

  • Developer/content creator authors an .ini file (using DSPBuilder, WordPad, or other text editor) and then runs XGPImage on it to generate an image file (.scr) as well as a header file (.h) that the developer can use to alter effect parameters in real time once the image is running.

  • For details on the proper syntax for authored .ini files, see the XDK topic GP DSP Image Builder (XGPImage).

WavMerge (wavmerge.exe)

Description: WavMerge is a command-line Windows tool for creating multichannel (four- or six-channel) wave files from mono sources that Xbox can play back to the speakers of a 5.1 audio system.

Installation: Installed with XDK in C:\Program Files\Microsoft Xbox SDK\xbox\bin\wavmerge.exe. Also available online here.

Comments:

  • Channel order for six-channel ("5.1") is left, right, center, LFE, left surround, right surround.
  • Channel order for four-channel is left, right, left surround, right surround. Note that this is not Dolby Surround-style data (which you can either create as a pre-encoded stereo file or mix down manually from a four- or six-channel stream).
  • Xbox currently supports six-channel interleaved PCM waveformatex or waveformatextensible files for any "pre-rendered" 5.1 audio you wish to create. Note that this is not Dolby Digital-encoded streams, but rather a .wav file with six channels.
  • Some multichannel audio authoring tools already support waveformatextensible. If yours does not, save out separate mono wave files and use this tool to merge them into a four- or six-channel wave file.

DirectMusic Producer for Xbox (DMusProd.exe)

Description: DirectMusic Producer for Xbox is a graphical tool for authoring, managing, and auditioning Xbox audio content, including Microsoft® DirectX® Audio scripts, Downloadable Sounds (DLS) collections, and Microsoft® DirectMusic® content.

Installation: Available online here.

Comments:

  • Supplementary DirectMusic demo content can be installed from X:\DMusProdX\DemoContent\DMPDemoContent.exe, or online here.

  • See Chapter 4—Introductory Guide to DirectMusic Producer for Xbox (DMPX) for more information about DirectMusic Producer for Xbox, or White Papers on Audio Designer's Corner for authoring scenarios and concepts.

DirectMusic Audio Analyzer (AudAnalyze.exe)

Description: Command-line tool that scans one or more DirectMusic content files, and then creates an object factory that limits the included DirectMusic library size to object types used by the content, reducing compiled and in-memory code size.

Installation: Installed with XDK in C:\Program Files\Microsoft Xbox SDK\xbox\bin\audanalyze.exe.

Comments:

  • Audio Analyzer scans your content (typically segment files), and then creates a .cpp file containing a function called MyFactory(), which a developer can use to replace DirectMusicDefaultFactory(). This function contains a list of DirectMusic objects needed to play the specified content. If game code passes MyFactory() instead of DirectMusicDefaultFactory() to DirectMusicInitializeEx(), the linker will exclude unneeded DirectMusic objects from the compiled Xbox title, decreasing its size. (DirectMusicDefaultFactory() includes ALL the objects, whether needed or not.)

  • For more information on options and usage, type audanalyze at a command prompt.

  • Most of the DirectMusic samples provided with the XDK now take advantage of Audio Analyzer to create a custom object factory based on the content they use.

Audio Console Application (audconsole.xbe)

Description: The Audio Console Application is designed for composers and sound designers to audition their content on the Xbox hardware. It is used in conjunction with PC-based audio tools, such as XACT, DirectMusic Producer for Xbox, and DSP Builder, which manage content and effects program downloading to the Xbox, as well as playback of that content.

Installation: Provided on XDK recovery CD. After recovery, make sure that your content development machine is attached to the XDK either via a network hub or via an Xbox System Link Cable.

Comments:

  • For more information about using the Audio Console app in conjunction with DirectMusic Producer for Xbox, see Chapter 4—Introductory Guide to DirectMusic Producer for Xbox (DMPX).

Audio Samples

Description: Audio samples demonstrate to developers a variety of techniques that can also be used (often with minor modifications) to test content integration scenarios. Some of the samples are purely programmatic and will not be discussed in detail here. In the case of the DirectMusic examples, content found in the \media\sounds subdirectory can be loaded in DirectMusic Producer for closer examination. And as mentioned above, Audio Analyzer was run on most of the DirectMusic samples to optimize the included objects based on the content used.

  • Play3DSound—I3DL2 occlusion and obstruction; 3D positioning on an object; I3DL2 reverb presets
  • Envelope Generator—real-time manipulation of DLS-2 amplitude and filter cutoff envelopes on sounds
  • SetFilter—exposes filter block settings (DLS-2 low pass and parametric EQ) for real-time experimentation with effects on looped sounds
  • GlobalEffects—use of chained effects in a program; real-time parameter modification
  • WaveBank—demonstrates use of a wave bank on Xbox
  • WaveBankStream—demonstrates optimized asynchronous non-buffered streaming from hard disk or DVD, using a streaming wave bank (authored with XACT)
  • XACTWMAPlayList—demonstrates preferred method for integration and playback of user soundtracks and WMA files supplied with a title using XACT

    DirectMusic Samples

  • DM3DScript—DirectMusic playback onto a 3D audiopath (which can then be positioned in real time) using scripting
  • DMAudiopath—DirectMusic playback onto a 3D audiopath (which can then be positioned in real time)
  • DMGrooveLevel—style-based playback and interaction between groove level and pattern selection from a style
  • DMScript—scripting example of self-modifying music, variability, queuing, and dialog concatenation

Installation: Source code is installed with XDK installation. Audio samples are placed at C:\Program Files\Microsoft Xbox SDK\Samples\Xbox\Sound and can be compiled by a developer to run on an Xbox.

Comments:

  • In many cases, it is fairly trivial to swap in your own content for the existing sample content. Some samples do have limitations-all content must be the same format, additional code must be written to support ADPCM, and so on.

xbsetcfg (xbsetcfg.exe)—no longer supported for audio

Description: xbsetcfg is a command-line Windows tool for setting flags on the Xbox hardware, bypassing the Xbox Dashboard.

Installation: Installed with XDK in C:\Program Files\Microsoft Xbox SDK\xbox\bin\xbsetcfg.exe.

Comments:

  • Beta "enhanced" (S-video or HDTV) A/V adapters did not always properly report their support for digital output. At this point, you should only be using final A/V adapters, and only change your audio configuration from the Xbox Dashboard.

Wrap-Up

There are quite a few resources available to aid with audio content creation, testing, auditioning, and implementation for Xbox. If there is an area not addressed here that you would like to see presented as a sample, or if a particular process is blocking your progress on a project, please don't hesitate to let us know at content@xbox.com.

For more information about topics presented here, please consult other chapters of the Xbox Audio Best Practices Guide, the Xbox Development Kit documentation, or the white papers on Audio Designer's Corner, or send an e-mail message to content@xbox.com.

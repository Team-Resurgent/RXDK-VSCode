XOnlineOfferingPurchase

Creates an asynchronous task to purchase the specified online offering for the specified user.

HRESULT XOnlineOfferingPurchase(
    DWORD dwUserIndex,
    XOFFERING_ID OfferingId,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwUserIndex
[in] Controller index (zero-based) of the user who will be purchasing the offering. The user must have been successfully logged on with a previous call to XOnlineLogon.
OfferingId
[in] Unique identifier associated with the offering to be installed. This identifier is obtained from an XONLINEOFFERING_INFO structure when available offerings are enumerated.
hWorkEvent
[in, optional] Handle to an event that will be signaled when there is work to be done on the purchase task. This parameter can be NULL if no such event is required.
phTask
[out] Address of an XONLINETASK_HANDLE that receives the offering purchase task. This task sends the purchase request to the offering service and receives the response. This handle must be pumped to completion by XOnlineTaskContinue. Call XOnlineTaskClose to close the task.

Return Values

If the function succeeds, the return value is S_OK.

Remarks

XOnlineOfferingPurchase returns an asynchronous task that must be pumped with XOnlineTaskContinue until that function returns XONLINETASK_S_SUCCESS, indicating that the purchase has completed, or until an error code is returned, indicating that the purchase has failed. The task handle should then be closed with XOnlineTaskClose. Calling XOnlineTaskClose before the task completes or indicates an error will not succeed in closing the task.

Among the error codes that XOnlineTaskContinue may return for the purchasing task are those that indicate specific billing or provider errors, such as the user's purchase being denied. These errors code are defined in the xonline.h header, and are in the XONLINE_E_OFFERING family.

After a content (non-subscription) offering is purchased, it must still be downloaded and installed using the XOnlineContentInstall function.

XOnlineOfferingPurchase need not, and should not, be called for offerings that are free. The XOnlineOfferingIsFree macro, given an offering identifier (of type XOFFERING_ID), will indicate whether that offering is free or not.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

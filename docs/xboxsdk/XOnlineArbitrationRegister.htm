XOnlineArbitrationRegister

Registers an item with the arbitration system.

HRESULT XOnlineArbitrationRegister(
    const XONLINE_ARB_ID *pArbID,
    WORD wMaxRoundSeconds,
    DWORD dwFlags,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

pArbID
[in] Pointer to an XONLINE_ARB_ID structure that contains the arbitration session ID.
wMaxRoundSeconds
[in] Maximum length of the round, in seconds. However, the actual length of the round may be reset by the XOnlineArbitrationExtendRound function.
dwFlags

[in] DWORD composed of zero or more of the following flags combined using a bitwise OR operation.
Flag Value Description
XONLINE_ARB_REGISTER_FLAG_USER_COMPETITION 0x00000001 Arbitrated round is part of a player-organized competition.
XONLINE_ARB_REGISTER_FLAG_PUBLISHER_COMPETITION 0x00000002 Arbitrated round is part of a publisher-organized competition.
XONLINE_ARB_REGISTER_FLAG_TIME_EXTENDABLE 0x00000004 Arbitrated round duration can be extended with the XOnlineArbitrationExtendRound function.
XONLINE_ARB_REGISTER_FLAG_HOST_FORWARDS_PACKETS 0x00000008 The session forwards game data packets during the arbitrated round.
XONLINE_ARB_REGISTER_FLAG_TEAMS 0x00000010 Arbitrated round includes team participants. This flag should be used only when the player is a formal member of a permanent team. It is required for all registrants if team results need to be recorded.
XONLINE_ARB_REGISTER_FLAG_FFA 0x00000020 Arbitrated round is free-for-all style of gameplay.

hWorkEvent
[in, optional] Handle to an event that will be signaled when there is work to be done on this task. This parameter can be NULL if no such event is required.
phTask
[out] Address of a XONLINETASK_HANDLE variable. Upon return, this variable contains the handle of the task initiated by this function.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

This function registers the current Xbox and all users on it for participation in an arbitrated event. Upon completion of the event, the list of currently registered participants can be retrieved by calling the XOnlineArbitrationRegisterGetResults function. An Xbox can safely register for the same round multiple times if an updated list of registrants is needed.

When the last Xbox has registered, participants should ensure that all necessary connections to other registrants have been established, and that any users who have not successfully registered will not be participating in the round.

Every time a player registers for an arbitrated session, the user's DisconnectRating may be affected. The DisconnectRating, which is the ratio of disconnects to arbitrated sessions played, is returned by XOnlineLogon in the XUID structure.

Your title should not use this function for competitions. Instead, it should call the XOnlineCompetitionSessionRegister function.

Requirements

Import Library: Use Xonline.lib.

Finding Games Sessions to Join

When finding game sessions for a player to join, your title typically displays a list of sessions matching the player's search criteria. Before it can obtain the data to fill that list, it must allocate memory for the data. To find out how much memory to allocate, your title calls XOnlineMatchSearchResultsLen. In the XOnlineMatchSearchResultsLen function's dwNumResults parameter, specify the maximum number of search results that your title is to display. XOnlineMatchSearchResultsLen returns the number of bytes your title needs to allocate to contain the specified number of results.

After allocating memory for the results, your title can search for game sessions. Call XOnlineMatchSearch to start the search. This creates an asynchronous task that must be "pumped" with calls to XOnlineTaskContinue. When the results are ready, call XOnlineMatchSearchGetResults to obtain them. Your title then parses the results into individual attributes with the XOnlineMatchSearchParse function.

Your title can enable players to invite other players to join games. When a player sends an invitation, your title should append the ID of the game session the recipient is being invited to join. The recipient's copy of your title can then call XOnlineMatchSessionFindFromID to obtain information on the game session. It then displays the game session information and asks the user whether he/she wants to join.

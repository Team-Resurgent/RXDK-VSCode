What Is Xbox Live?

Xbox LiveTM is an online service provided by Microsoft to enhance the functionality of Xbox game titles. It enables multiplayer action through Internet connectivity. Xbox Live includes the following services:

  • Content Download
  • Friends List
  • Matchmaking
  • Statistics
  • Voice and Voice Messages
  • Title Managed Storage
  • Family Title IDs
  • Teams
  • Competitions
  • Messaging

Client-side API support for these services is provided by the Xbox Live Title library, XOnline.lib.

Because the Xbox Live services are delivered across the Internet, subscribers must have a broadband connection to the Internet. The broadband connection must be either a Digital Subscriber Line (DSL), Cable Modem, or Fiber To The Home (FTTH) fiber optic connection.

Xbox consoles and the Xbox Live service mutually authenticate using the Kerberos Protocol. The connection between a given Xbox console and the Xbox Live service is secured using Internet Protocol Security (IPSec).

As a service to game developers in the process of creating titles that support Xbox Live, the Xbox Live team hosts a datacenter of secure online servers called PartnerNet. PartnerNet enables developers to test the online functionality of their games in a simulated online environment.

Xbox Live User Accounts

To play an Xbox game title that uses Xbox Live, a player must sign in to the service. To do so, the player must have an account. Players create Xbox Live accounts using the Xbox Dashboard. When creating an account, a player enters a text string that serves as his or her identity on the service. This text string identity is known as a "Gamertag." A player may also optionally create a pass code, which is very much like a personal identification number (PIN) used with bank ATMs. The pass code prevents individuals other than the user from signing into the player's account.

Account information for each player is stored on either the Xbox console hard disk or in Xbox Memory Units (MUs). The hard disk can store information for up to eight players. Each MU can store account information for a single player. Because each game controller can support two MUs, a single Xbox console can store account information for up to 16 players.

Xbox Live Implies Multiple Xbox Consoles

Xbox Live provides an infrastructure for gaming to take place across multiple Xbox consoles connected to the Internet. The Xbox Live TCRs specify the features that Xbox Live titles must support. An Xbox game title may use some features, such as content download, without having to support the other Xbox Live services. In this case, however, the game title is not technically considered an Xbox Live title. In general, true Xbox Live titles enable multiple players to play cooperatively or competitively across an Internet connection. This usually means support for such features as messaging, friends lists, voice chats and so forth.

In addition, Xbox Live enables developers to create dynamic web sites that display player and game data from Xbox Live through the use of Xbox Web Services. For more information, download the Web Services documentation from the Xbox Central web site.

Xbox Live Tools

The following tools are particularly recommended for Xbox Live development.

  • NetSim2
  • NetMon
  • Xbox Live Authoring and Submission Tool (XLAST)

NetSim2

Most games for Xbox Live are developed on local area networks (LANs). Unfortunately, LANs do not provide a realistic reflection of network gaming across the Internet. It is difficult to tell how gracefully a game for Xbox Live handles the type of real-world problems it might encounter when used on the Internet. The NetSim2 tool simulates real-world network conditions to enable developers to accurately test their games. NetSim2 can simulate limited bandwidth, network latencies, packet loss, and authentication delays. With the NetSim2 tool's ability to simulate these conditions, you can test your game's ability to communicate from one Xbox to another and from an Xbox to the Xbox Live servers. This enables you to test your games for compliance with the Xbox Live TCRs.

NetMon

The Network Monitor tool (NetMon) allows developers and testers to capture and analyze Xbox network traffic. The XDK includes Xbox extensions that properly interpret and display Xbox network packets. Netmon can be used to evaluate patterns of network traffic and determine game packet frequency, packet size, and packet overhead. It is available for download in the Tools section of Xbox Central; it is not included as a part of the XDK.

Xbox Live Authoring and Submission Tool (XLAST)

The Xbox Live Authoring and Submission Tool (XLAST) provides a visual and interactive environment for authoring submissions to Xbox Live. The submissions can be either content packages, which include both online games and game components, or game stats updates, which can be made visible on Xbox Live and on the Web. XLAST automates many of the tasks associated with authoring submissions—for example:

  • XLAST simplifies the process of submitting Xbox Live game components.
  • It validates the submission before you send it. As a result, XLAST significantly reduces the number of times you need to submit your Xbox Live updates to Microsoft.
  • It provides you with a single tool for multiple submission types.
  • It enables you to build content packages for offline delivery without building associated online offering information.
  • It removes the need for you to edit metadata files when submitting content.
  • It automatically incorporates the required metadata into the final submission.

Required Files for Xbox Live Executables

All titles that connect to the Xbox Live service, including those that use Downloader.xbe, must also include the autoupdate executable, Update.xbe, and the most recent Xbox Dashboard, DashUpdate.xbe. These executables are distributed in the XDK in the redist directory. They must also be placed in the root directory of the game disc.

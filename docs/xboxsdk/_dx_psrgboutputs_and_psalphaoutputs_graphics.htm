PSRGBOutputs[0-7] and PSAlphaOutputs[0-7]

The PSRGBOutputs[0-7] and PSAlphaOutputs[0-7] fields specify the outputs for the RGB and alpha portions of each combiner stage. The PS_COMBINEROUTPUTS(ab,cd,mux_sum,flags) macro can be used to specify the three outputs as well as some additional flags to set the output mapping and other options. The macro is defined as below:

#define PS_COMBINEROUTPUTS(ab,cd,mux_sum,flags) (((flags)<<12)|((mux_sum)<<8)|((ab)<<4)|(cd))

The ab, cd, and mux_sum parameters are the destination registers for the outputs referred to as d0, d1, and d2. The flags parameter is used to specify various output modifiers.

Destination Registers

The following registers are valid destination registers:

PS_REGISTER_DISCARD
Discard the result. This register may be specified multiple times in each combiner stage.
PS_REGISTER_V0
Fragment diffuse color.
PS_REGISTER_V1
Fragment specular color.
PS_REGISTER_T0
Texture 0 output.
PS_REGISTER_T1
Texture 1 output.
PS_REGISTER_T2
Texture 2 output.
PS_REGISTER_T3
Texture 3 output.
PS_REGISTER_R0
Temporary register 0. The output of the pixel shader is written to this register in the final active combiner stage.
PS_REGISTER_R1
Temporary register 1.

The mux_sum output must be set to PS_REGISTER_DISCARD if a dot product operation is selected for the ab or the cd output.

Flags

The flags field contains one or more flags logically ORed together. Only one flag should be selected from each flag type.

These flags specify the output mapping for all three outputs. In the descriptions, x is the value calculated in the combiner stage and y is the value actually written to the destination register.

PS_COMBINEROUTPUT_IDENTITY
y = x
PS_COMBINEROUTPUT_BIAS
y = x - 0.5
PS_COMBINEROUTPUT_SHIFTLEFT_1
y = x * 2
PS_COMBINEROUTPUT_SHIFTLEFT_1_BIAS
y = (x - 0.5) * 2
PS_COMBINEROUTPUT_SHIFTLEFT_2
y = x * 4
PS_COMBINEROUTPUT_SHIFTRIGHT_1
y = x / 2

The following flags specify whether the mux_sum output is the result of an addition or a mux operation:

PS_COMBINEROUTPUT_AB_CD_SUM
mux_sum = ab + cd
PS_COMBINEROUTPUT_AB_CD_MUX
If the mux bit is set (mux behavior is determined by PS_COMBINERCOUNT_MUX_LSB and PS_COMBINERCOUNT_MUX_MSB), mux_sum = cd, otherwise, mux_sum = ab.

Note that if the mux operation is selected, either the least significant bit (lsb) or most significant bit (msb) of r0.a is used to select the output. Register r0 does not have to be an input to the stage.

The following flags specify the operation to perform on the ab side of the RGB combiner. These flags are valid for the RGB portion of the combiner only.

PS_COMBINEROUTPUT_AB_MULTIPLY
ab = a * b
PS_COMBINEROUTPUT_AB_DOT_PRODUCT
ab = a · b

If this is specified, the third output (mux_sum) is not available and must be set to PS_REGISTER_DISCARD.

This group of flags specifies the operation to perform on the cd side of the RGB combiner. These flags are valid for the RGB portion of the combiner only.

PS_COMBINEROUTPUT_CD_MULTIPLY
cd = c * d
PS_COMBINEROUTPUT_CD_DOT_PRODUCT
cd = c · d

If this is specified, the third output (mux_sum) is not available and must be set to PS_REGISTER_DISCARD.

Each of the last two flags may be specified with the other flags.

PS_COMBINEROUTPUT_AB_BLUE_TO_ALPHA
Causes the value from the blue channel of the result to be written to the alpha channel of the destination register. This is the only case where the RGB portion of the combiner can write to the alpha channel of the destination register. Note that this does not necessarily mean that the alpha portion output is discarded. The alpha portion could write to the alpha channel of an entirely different destination register.
PS_COMBINEROUTPUT_CD_BLUE_TO_ALPHA
Causes the value from the blue channel of the result to be written to the alpha channel of the destination register. This is the only case where the RGB portion of the combiner can write to the alpha channel of the destination register. Note that this does not necessarily mean that the alpha portion output is discarded. The alpha portion could write to the alpha channel of an entirely different destination register.

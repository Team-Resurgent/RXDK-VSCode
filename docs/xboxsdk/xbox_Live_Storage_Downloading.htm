Downloading from Storage

Before downloading, your title can find whether files are available for download by calling the XOnlineStorageEnumerate function. This starts an asynchronous task whose handle must be "pumped" by calling XOnlineTaskContinue. When XOnlineTaskContinue indicates that results are ready, your title obtains them by calling XOnlineStorageEnumerateGetResults.

Note    The task started by XOnlineStorageEnumerate does not operate recursively through the server's directory structure. Therefore, if the directory contains subdirectories and your title needs the content of those subdirectories, your title must enumerate their contents.

Downloading files require a server path that specifies the file name. You must either obtain the file name by enumeration or by hard-coding it into your title. Either way, your title must "know" the file name before it can perform the download. It is recommended that your title uses predictable file names when uploading to storage so that they can be easily found and downloaded. For example, if your title allows players to upload custom maps, you might use a file naming scheme along the lines of Player001.map, Player002.map, and so on. When creating your server path, you can use wildcards such as Player*.map.

To retrieve a file from server storage, call the XOnlineStorageDownload or XOnlineStorageDownloadToMemory function. XOnlineStorageDownload can download game clips, global data, per-team data, or per-user data. XOnlineStorageDownloadToMemory downloads items that were previously uploaded from memory with the XOnlineStorageUploadFromMemory function. Both XOnlineStorageDownload and XOnlineStorageDownloadToMemory begin an asynchronous task that must be pumped with calls to XOnlineTaskContinue. If your title uses XOnlineStorageDownloadToMemory, it must call XOnlineStorageDownloadToMemoryGetResults when XOnlineTaskContinue indicates that results are ready.

There is no mechanism for maintaining partial downloads and resuming from previous checkpoints. As most storage areas are limited greatly in size, downloads are not expected to take enough time to require resume functionality. Publishers expecting to release extremely large downloads to players should consider packaging these as content download packages rather than using global title-managed storage.

XOnlineMatchSessionUpdate

Updates the information associated with an online-game session.

HRESULT XOnlineMatchSessionUpdate(
    XNKID SessionID,
    DWORD dwPublicFilled,
    DWORD dwPublicOpen,
    DWORD dwPrivateFilled,
    DWORD dwPrivateOpen,
    DWORD dwNumAttributes,
    PXONLINE_ATTRIBUTE pAttributes,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

SessionID
[in] XNKID structure that identifies the online-game session. After the session is created using XOnlineMatchSessionCreate, you can obtain this identifier by calling XOnlineMatchSessionGetInfo.
dwPublicFilled
[in] New number of public slots that are currently occupied in this game session.
dwPublicOpen
[in] New number of unoccupied public players in this game session.
dwPrivateFilled
[in] New number of private slots that are currently occupied in this game session.
dwPrivateOpen
[in] New number of unoccupied private slots in this game session.
dwNumAttributes
[in] Number of attributes that will be advertised for this session. This number is equal to the number of XONLINE_ATTRIBUTE structures pointed to by the pAttributes parameter.
pAttributes

[in, out] Pointer to an array of XONLINE_ATTRIBUTE structures for the attributes for this session.

If an attributes value has changed since the call to XOnlineMatchSessionCreate or the previous call to XOnlineMatchSessionUpdate, set the value of the fChanged member of the XONLINE_ATTRIBUTE structure to TRUE. Also, set fChanged to TRUE for new attributes that weren't in the call to XOnlineMatchSessionCreate or a previous call to XOnlineMatchSessionUpdate.

The new attributes and attribute values should be compatible with any previously advertised attributes so as not to create inconsistencies in the online-game session.

hWorkEvent
[in, optional] Optional. Handle to a caller-created event object. This object will be signalled when there is work to do by calling XOnlineTaskContinue. The caller may pass in NULL instead, indicating that the caller will poll XOnlineTaskContinue itself.
phTask
[out] Address of an XONLINETASK_HANDLE variable that receives a valid handle for this session.

Return Values

If XOnlineMatchSessionUpdate succeeds, the return value is S_OK.

If XOnlineMatchSessionUpdate fails, the return value is one of the following error codes.

Value Description
E_OUTOFMEMORY Insufficient memory. Free some memory.
WSAEADDRNOTAVAIL See Winsock Error Codes for more information.
WSACONNRESET See Winsock Error Codes for more information.
Other See Online Errors for information about other possible errors.

The following table lists the possible values that XOnlineTaskContinue may return when called with the handle for the task created by XOnlineMatchSessionUpdate.

Value Description
XONLINE_E_MATCH_INVALID_SEARCH_REQ The request is invalid. The wrong number of attributes was passed to a stored procedure. More detailed error information is returned by XOnlineTaskContinue when the search completes. For debugging purposes, call XOnlineTaskContinue to find out what went wrong.
XONLINE_E_MATCH_STRING_TOO_LONG The attribute string for the create, update, or search operation is longer then 400 characters, or the string wasn't terminated with a null character. Limit string length to 400 wide characters, and ensure that the string is properly terminated.
XONLINE_E_MATCH_BLOB_TOO_LONG The BLOB for the create, update, or search operation is longer then 800 bytes. Limit BLOB length to 800 bytes.
XONLINE_E_MATCH_INVALID_ATTR_TYPE The attribute type for the create, update, or search operation is something other than player or session.
XONLINE_E_MATCH_INVALID_SESSION_ID The specified session ID is not a valid session ID returned by XOnlineMatchSessionGetInfo or XOnlineMatchSessionFindFromID.
XONLINE_E_MATCH_INVALID_OFFSET One of the attribute/parameter offsets in the request was invalid. The attribute ID was OR'd with an invalid data type. Ensure that the attribute ID is OR'd with a valid attribute data type mask (X_ATTRIBUTE_DATATYPE_INTEGER, X_ATTRIBUTE_DATATYPE_STRING, or X_ATTRIBUTE_DATATYPE_BLOB)
XONLINE_E_MATCH_INVALID_PARAM The data type that was passed to the stored procedure is not valid. Ensure that the data type sent to the stored procedure is the expected type.
XONLINE_E_MATCH_INVALID_TITLE_ID The title is not properly registered on the Matchmaking server. The attribute ID is invalid. Check the XMS file generated by MatchSim to ensure that the correct attribute ID is OR'd with the correct data type mask.
XONLINE_E_MATCH_INVALID_SESSION_ID The specified session ID is invalid.

Remarks

Use the XOnlineMatchSessionUpdate function to send the server updated information about the online-game session. The session must have been previously created using the XOnlineMatchSessionCreate function.

A private player is a player who is specifically invited to join a particular session. A public player is a player who finds the session through a matchmaking search. Matchmaking searches only return sessions that have openings for public players.

Attributes are represented by the XONLINE_ATTRIBUTE structure. The type of an attribute is either integer, string, or binary object. An integer occupies eight bytes (ULONGLONG). A string has a maximum size of 400 wide characters (800 bytes). Strings are NULL terminated. A binary object has maximum size of 800 bytes and is not NULL terminated.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

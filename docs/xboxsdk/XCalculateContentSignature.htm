XCalculateContentSignature

Calculates a signature over the specified data. The signature will match automatic content signatures created by the XBldSigs tool.

BOOL XCalculateContentSignature(
    LPBYTE pbData,
    DWORD dwDataSize,
    LPBYTE pbSignature,
    LPDWORD pdwSignatureSize
);

Parameters

pbData
[in] Pointer to a buffer that contains the data over which the signature is calculated.
dwDataSize
[in] Specifies the size, in bytes, of the Data buffer.
pbSignature
[out] Pointer to a buffer that receives the calculated signature.
pdwSignatureSize
[in, out] Pointer to a DWORD variable that specifies the size, in bytes, of the buffer pointed to by Signature. On return, SignatureSize receives the number of bytes written to the Signature buffer.

Return Values

If the function succeeds, the return value is TRUE.

If the function fails, the return value is FALSE. To get extended error information, call GetLastError.

Remarks

XCalculateContentSignature calculates a signature over the specified data, using the same algorithm used by the XBldSigs tool to calculate automatic signatures. To verify installed content data (both after initial installation and before each subsequent use), a signature can be calculated over the data with XCalculateContentSignature, and then compared with the signature for the same data as returned by either the XLocateSignatureByName function or the XLocateSignatureByIndex function.

If the content signatures are not automatic signatures created by XBldSigs, then, rather than calling XCalculateContentSignature, the title must use whatever title-specific algorithm it initially used to create the content signatures.

To determine the buffer size needed to hold the signature (without actually performing the signature calculation), specify zero for SignatureSize. The required size will be returned in SignatureSize.

Content signatures calculated by XCalculateContentSignature are equivalent to content signatures calculated by XCalculateSignatureBegin with XCALCSIG_FLAG_CONTENT, but different than signatures calculated by XCalculateSignatureBegin with XCALCSIG_FLAG_SAVE_GAME, XCALCSIG_FLAG_NON_ROAMABLE, or XCALCSIG_FLAG_ONLINE. Content signatures are used to verify installed content by comparing data signatures with those created by XBldSigs or BuildOffer and stored in the content's signature data. Each individual signature is a hash over the content data, and the collection of signatures, as a whole, is signed for security. Save game and Xbox Live signatures, intended for signing save games and other dynamic data, perform the additional step of signing each data hash with a title-specific or Xbox Live key, and returning the signed hash as the signature. These methods are not interchangeable. XCalculateContentSignature, and XCalculateSignatureBegin with XCALCSIG_FLAG_CONTENT, are specifically designed and optimized for use in verifying installed content.

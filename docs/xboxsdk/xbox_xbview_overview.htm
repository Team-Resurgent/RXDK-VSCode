Xbox Viewer (XbView) Overview

The Xbox Viewer (XbView) is an Xbox application that reads Xbox binary resource files (.xbr) and displays the graphical content on the Xbox as it would appear in the game engine. When the XbView application is launched, either from Right Hemisphere or the XDK Launcher, it loads the XbRC_Out.XBR file from the media subdirectory of the directory containing XbView.xbe.

The Different Views

XbView has four distinct views:

  • Model View
  • Animation View
  • Statistics View
  • Resource View

After loading the file, XbView starts in Model View mode, allowing the user to view the model, change the camera, lights, and antialiasing mode. The Animation view lets the user see the animation of the model. The Statistics view shows the user basic statistics about the current model. Resource View mode allows the user to view each of the resources used the create the final model. Pushing the White button cycles between the different views. XbView gets its background color and image from Deep Exploration.

Debug Mouse Support

XbView supports a debug mouse for viewing. Plug a USB mouse into any available Xbox Controller port using the Debug Keyboard adapter. The mouse can be plugged in even while the application is running. In both the Animation and Model views, the mouse behavior is nearly identical to the mouse behavior in Deep Exploration. Holding the left mouse button down and moving the mouse rotates the camera around the object. Holding the right mouse button down and moving the mouse dollies the camera in and out. Holding down both the left and right mouse buttons or holding down the mouse wheel button and moving the mouse translates the camera up/down and left/right. Rotating the mouse wheel dollies the camera in and out. Holding down the left extra button and moving the mouse rotates the directional light source.

Updating XbView

Deep Exploration will update the version of XbView if the version in the Deep Exploration directories is newer than the version on the Xbox. It will reboot the Xbox and start XbView automatically even if something else is running on the Xbox. It will not restart XbView if XbView is already running. It simply reloads the new model or bitmap and resets the camera and animation settings as described above, if it needs to. This means that if XbView is running and a newer version gets copied over to the Xbox, the user will not see the new version until you exit XbView and XbView restarts.

Customizing XbView

XbView is fully customizable. For details, see XbView Architecture.

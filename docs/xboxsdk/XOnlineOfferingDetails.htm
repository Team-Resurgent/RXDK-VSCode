XOnlineOfferingDetails

Creates an asynchronous task to retrieve the details associated with a specified online offering.

HRESULT XOnlineOfferingDetails(
    DWORD dwUserIndex,
    XOFFERING_ID OfferingId,
    DWORD dwLanguage,
    DWORD dwDescriptionIndex,
    PBYTE pbBuffer,
    DWORD dwBufferSize,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwUserIndex
[in] Controller index (zero-based) of the user who would be purchasing the offering. The user must have been successfully logged on with a previous call to XOnlineLogon.
OfferingId
[in] Unique identifier associated with the offering to be installed. This identifier is obtained from an XONLINEOFFERING_INFO structure when available offerings are enumerated.
dwLanguage

[in] Value that specifies the language used to determine the formatting of prices in the details results. dwLanguage must be one of the following values:
Value Description
XC_LANGUAGE_ENGLISH English
XC_LANGUAGE_JAPANESE Japanese
XC_LANGUAGE_GERMAN German
XC_LANGUAGE_FRENCH French
XC_LANGUAGE_SPANISH Spanish
XC_LANGUAGE_ITALIAN Italian
XC_LANGUAGE_KOREAN Korean
XC_LANGUAGE_TCHINESE Chinese (Traditional)
XC_LANGUAGE_PORTUGUESE Portuguese

To specify the language currently in use by the Xbox console (in other words, the language set by the user through the Dashboard), pass the return value of XGetLanguage as dwLanguage.

dwDescriptionIndex
[in] The details BLOB that should be retrieved for this offering. This is a title-specific index into the ordered list of detail BLOBs that the title submitted for the specified offering. Typically, this value will be used to differentiate between the available description languages for the offering, but the specific use of this value is up to the title.
pbBuffer
[out, optional] Pointer to a buffer that will be used by the function to store details results for the operation. The details results are obtained with a call to XOnlineOfferingDetailsGetResults rather than read directly from this buffer. If this parameter is NULL, the function will automatically allocate a buffer.
dwBufferSize
[in] Size, in bytes, of the buffer pointed to by pbBuffer. If pbBuffer is NULL, this parameter specifies the size of the results buffer that the function will automatically allocate. The XOnlineOfferingDetailsMaxSize function should be used to determine the buffer size needed to process the details information, including the details BLOB data.
hWorkEvent
[in, optional] Handle to an event that will be signaled when there is work to be done on the details retrieval task. This parameter can be NULL if no such event is required.
phTask
[out] Address of an XONLINETASK_HANDLE that receives the offering details task. This task sends the details request to the offering service and receives the response. This handle must be pumped to completion by XOnlineTaskContinue. After the task has completed, call XOnlineOfferingDetailsGetResults to extract the offering details. XOnlineTaskClose must then be called to close the task.

Return Values

Returns S_OK if the function succeeds.

Remarks

XOnlineOfferingDetails returns an asynchronous task that must be pumped with XOnlineTaskContinue until that function returns XONLINETASK_S_RESULTS_AVAIL, indicating that details results are available. XOnlineOfferingDetailsGetResults can then be called to retrieve the details results.

Once the results have been retrieved, XOnlineTaskContinue will return XONLINETASK_S_SUCCESS for the task, indicating that the task has completed and can be closed with XOnlineTaskClose.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

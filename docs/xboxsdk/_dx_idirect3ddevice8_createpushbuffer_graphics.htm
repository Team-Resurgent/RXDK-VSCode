IDirect3DDevice8::CreatePushBuffer

Creates an empty push-buffer for use in recording between an IDirect3DDevice8::BeginPushBuffer/IDirect3DDevice8::EndPushBuffer bracket.

HRESULT CreatePushBuffer(
  UINT Size,
  BOOL RunUsingCpuCopy,
  D3DPushBuffer **ppPushBuffer
);

Parameters

Size
[in] Size of the push-buffer in bytes, which must be a multiple of four bytes. In order to record into the push-buffer you must create it with a size of at least 4096 bytes and the size must be at least as large as any single command recorded into the push-buffer. (For example, if IDirect3DDevice8::DrawPrimitiveUP is called while recording, Size must be at least as large as all the vertex data and the call overhead.) If Size is not as large as any single command recorded into the push-buffer, Direct3D will overrun the end of the buffer while recording. (No check is done on retail builds for this case so as not to slow down any of the Direct3D methods while not recording a push-buffer.) Typically, the size should be at least 1100 bytes.

Size should also be at least as large as the total size of the push-buffer to be recorded. However, even on retail builds, Direct3D is guaranteed not to overrun the end of the buffer while recording a push-buffer that is too big (subject to the constraint that no single command is larger than the size). This permits GetSize to be called on a too-small push-buffer to get the size it would have been if it were large enough to hold the data. However, it is not always possible to use GetSize, call CreatePushBuffer with exactly that size, and then record the same data again. Certain push-buffer commands require an extra margin at the end while they are being recorded. Adding 512 bytes to GetSize should be sufficient.

RunUsingCpuCopy
[in] If TRUE, the push-buffer resource data is created in non-contiguous, cached memory, and D3DPUSHBUFFER_RUN_USING_CPU_COPY is set in the Common field of IDirect3DPushBuffer8. If FALSE, the push-buffer resource data is created in physically contiguous, write-combined memory.
ppPushBuffer
[out, retval] Address of a pointer to a IDirect3DPushBuffer8 interface, representing the created push-buffer.

Return Values

If the method succeeds, the return value is S_OK.

If there is insufficient memory, the method returns E_OUTOFMEMORY.

Remarks

In general, RunUsingCpuCopy should be FALSE when the push-buffer is large, because the push-buffer can be directly executed in-place by the GPU and so save in CPU time and system bandwidth. When RunUsingCpuCopy is FALSE, the GPU's push-buffer contains a JUMP to the resource data and then a JUMP back. The resource data is not copied to the GPU. However, RunUsingCpuCopy should be TRUE for small push-buffers, because the per-call overhead of IDirect3DDevice8::RunPushBuffer is extremely high for the direct execution case. When RunUsingCpuCopy is TRUE, no JUMPs are necessary since the resource data is copied to the GPU.

As with any Direct3D resource, Release should be called when done with the push-buffer resource.

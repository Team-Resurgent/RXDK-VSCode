XNetStartupParams

Contains the initialization settings for the Xbox Secure Network Library (SNL).

typedef struct {
    BYTE cfgSizeOfStruct;
    BYTE cfgFlags;
    BYTE cfgPrivatePoolSizeInPages;
    BYTE cfgEnetReceiveQueueLength;
    BYTE cfgIpFragMaxSimultaneous;
    BYTE cfgIpFragMaxPacketDiv256;
    BYTE cfgSockMaxSockets;
    BYTE cfgSockDefaultRecvBufsizeInK;
    BYTE cfgSockDefaultSendBufsizeInK;
    BYTE cfgKeyRegMax;
    BYTE cfgSecRegMax;
    BYTE cfgQosDataLimitDiv4;
} XNetStartupParams;

Members

cfgSizeOfStruct
Size of this structure in bytes.
cfgFlags
Configuration flags. This value must be between 0 and 255 inclusive. The default value is zero. Can be a combination of the following values:
XNET_STARTUP_BYPASS_DHCP
Skip DHCP and use auto-IP only. Only relevant on the Xbox development kit. This flag is silently ignored when linking to the secure versions of the library.
XNET_STARTUP_BYPASS_SECURITY
Allow unsecured communications. Only relevant on the Xbox development kit. This flag is silently ignored when linking to the secure versions of the library.
cfgPrivatePoolSizeInPages
Maximum size, in pages (4,096 bytes per page), of the private pool for the network stack. Valid values for this member must be in the range of 4-255 inclusive. The default size is 12 pages (48 KB). The private pool is used in the following situations:
  • Responding to ARP/DHCP/ICMP messages.
  • Responding to certain TCP control messages.
  • Allocating incoming TCP connection request sockets.
  • Buffering outgoing data until it is transmitted (UDP) or until it is acknowledged (TCP).
  • Buffering incoming data on a socket that does not have a sufficiently large overlapped read pending.

The private pool is used instead of the system pool in order to have completely deterministic memory behavior. In other words, all memory allocation occurs only when a socket function is called. No allocation happens asynchronously in the background.

The network stack will not allocate more than this amount of memory except during network function calls by the title. If the entire private pool is already allocated and an asynchronous network event that requires more memory occurs, that network event will be discarded silently.

cfgEnetReceiveQueueLength
Length of the Ethernet receive queue (in packets) allocated for the network card's DMA buffer. Each packet in the Ethernet receive queue takes up about 2 KB of physically contiguous memory. Valid values for this member must be in the range of 4-255 inclusive. The default is 8 packets (16 KB).
cfgIpFragMaxSimultaneous
Maximum number of IP datagrams that can be in the process of reassembly at the same time. Valid values for this member must be in the range of 1-255 inclusive. The default value is 4 packets.
cfgIpFragMaxPacketDiv256
Maximum size of an IP datagram (including header) that can be reassembled in units of 256 bytes each. Valid values for this member must be in the range of 4-255 inclusive. The default value is 8 units (2,048 bytes).

Note   Be careful when setting this parameter to a large value, because it opens up a potential denial-of-service attack by consuming large amounts of memory in the fixed-size private pool.

cfgSockMaxSockets
Maximum number of sockets that can be opened. This includes child sockets that are spawned from listening sockets due to pending incoming connections. Also remember that a TCP socket may continue to be active for some period of time even after closesocket is called to close the socket gracefully. Valid values for this member must be in the range of 1-232 inclusive.The default is 64 sockets.
cfgSockDefaultRecvBufsizeInK
Default receive buffer size for a socket in kilobytes. Valid values for this member must be in the range of 1 KB - 63 KB inclusive. The default is 16 KB.
cfgSockDefaultSendBufsizeInK
Default send buffer size for a socket in kilobytes. Valid values for this member must be in the range of 1 KB - 63 KB inclusive. The default value is 16 KB.
cfgKeyRegMax
Maximum number of XNKID/XNKEY pairs that can be registered at the same time by calling XNetRegisterKey. Valid values for this member must be in the range of 1-255 inclusive. The default is 4 key pair registrations.
cfgSecRegMax
Maximum number of security associations that can be registered at the same time. Valid values for this member must be in the range of 1-255 inclusive. The default is 32.

Security associations are created for each unique XNADDR/XNKID pair passed to XNetXnAddrToInAddr. Security associations are also implicitly created for each secure host that establishes an incoming connection with this host on a given registered XNKID. Note that there will be only one security association between a pair of hosts on a given XNKID no matter how many sockets are actively communicating on that secure connection.

cfgQosDataLimitDiv4
Maximum amount of Qos data that can be supplied to a call to < a href="xbox_XNetQosListen_xsnl.htm">XNetQosListen, or returned in the result set of a call to XNetQosLookup or XNetQosServiceLookup. Valid values for this member must be in the range of 1-255 inclusive. The default value is 64 (256 bytes).

Remarks

XNetStartupParams should be passed only to XNetStartup when the title needs to change one of the default network settings (like using the XNET_STARTUP_BYPASS_SECURITY to allow insecure connections). Otherwise, the title should just pass NULL to XNetStartup.

By default, the Xbox SNL operates in secure mode, which means that communications with untrusted hosts (like personal computers) is not allowed. However, the developer kit version of the Xbox SNL (Xnet.lib and Xnetd.lib) allows you to bypass this security by specifying the XNET_STARTUP_BYPASS_SECURITY flag in the cfgFlags member of the XNetStartupParams structure. The following example demonstrates an easy way of doing this.

XNetStartupParams xnsp;
memset(&xnsp, 0, sizeof(xnsp));
xnsp.cfgSizeOfStruct = sizeof(XNetStartupParams);
xnsp.cfgFlags = XNET_STARTUP_BYPASS_SECURITY;
INT err = XNetStartup(&xnsp);

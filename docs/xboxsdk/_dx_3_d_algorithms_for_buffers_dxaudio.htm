3D Algorithms for Buffers

The Xbox Media Communications Processor (MCP) has the ability to perform sophisticated 3D audio processes, including multichannel panning, HRTF (Head Related Transfer Function) 3D for speakers or headphones, and multichannel wave format playback.

There are three ways to make sounds come from multiple speakers:

  • Create a 3D HRTF buffer and a mono format data source, and use the DirectSound3D libraries to control (x,y,z) positions of source and listener. This is done by setting the DSBCAPS_CTRL3D flag when creating a DirectSound buffer or stream.
  • Create a non–3D buffer and a mono datasource, and use SetMixBins and SetMixBinVolumes to explicitly set the speakers and volumes of the sound.
  • Create a multichannel DirectSound buffer (nChannels >= 2) and an appropriate multichannel datasource (a .wav format with nChannels >= 2), and play the DirectSound buffer.

Definitions

There is often confusion between interactive 3D sound and multichannel sound. We will use the terms as follows:

  • Interactive 3D Sound
    Interactive 3D sound is created by taking a mono sound source and applying HRTF and/or multispeaker panning to create the illusion of a sound somewhere in 3D space around the listener (the sound source and listener each require their position to be specified using (x,y,z) coordinates). Effects such as Doppler and rolloff are also included in "3D audio processing." Xbox also includes effects for occlusion, obstruction, and "environmental reverb" using the I3DL2 specification, which can be found under the link "3D Audio Rendering Guidelines Level 2" on the http://www.iasig.org Web site.
  • Multichannel Sound
    Multichannel sound results from delivering an N-channel file to M speakers. Examples include:
    • A stereo wave file played to the main left and right speakers.
    • A linear 5.1 wave file played to 5.1 speakers.
    • A linear 4.0 wave file played and downmixed to the left and right speakers.

The Xbox MCP can play 256 simultaneous voices. Of these 256, 64 are connected to HRTF-based 3D audio processors. When a DirectSound buffer is created with the DSPCAPS_CTRL3D flag set, one of these 64 HRTF voices is allocated. The HRTF audio processors will output either two-speaker HRTF output, four-speaker HRTF output, or headphone-based HRTF output.

HRTF Coefficient Sets

Xbox supports two different sets of HRTF coefficients: one "full 360" set, which provides HRTF coefficients for a full 360-degree HRTF effect, and one "azimuth-only" set, which provides azimuth-only processing in the horizontal plane. The full set requires a working set of approximately 70 KB; the azimuth-only set requires approximately 7 KB. This allows an application to make a trade-off between more realistic HRTF processing and code size requirement. To use the full HRTF set, call the DirectSoundUseFullHRTF function after initialization. If a game wants to reduce its memory footprint and use the azimuth-only HRTF set, the DirectSoundUseLightHRTF function should be called after initialization.

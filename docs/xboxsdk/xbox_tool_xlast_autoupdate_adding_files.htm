Adding Files to an Autoupdate Package

After you create an autoupdate project with the New Autoupdate Wizard, the Document View presents you with a tree-style display of the current contents of the project. In addition to the root node, the tree contains a folder called Updated Files. All of the content for the autoupdate goes into the Updated Files folder.

To add files or folders to the Updated Files folder, right-click it. From the menu that appears, select Add File or New Folder. The files that XLAST sees as available are those stored in the path to the updated game binaries that you specified in the New Autoupdate Wizard. New folders that you create in the project should also exist in the original game.

After you add files or folders, you can click on them in the tree view to see their properties. For example, selecting a file in the project tree displays the properties File Name, Xbox Relative Path, Source Path, and Full Source Path. The Xbox Relative Path property specifies the path in the autoupdate folder on the Xbox. Both Source Path and Full Source Path give the path to the project directory on the development PC. However, Source Path is the relative path, while Full Source Path displays the absolute (fully qualified) path.

There are known issues with the autoupdate portion of XLAST. The first is that the tool incorrectly allows users to configure two folders or two files with the same name. In actuality, however, this is not allowed.

Also, XLAST allows users to rename XBEs in the original game bits directory. However, XBEs that exist in the original game bits directory should not be renamed.

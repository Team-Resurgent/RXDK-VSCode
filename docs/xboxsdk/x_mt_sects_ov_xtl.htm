XTL XBE Sections

Many of the functions available in the Xbox Title Libraries have been organized into sections based on their functional group. For example, networking code is contained in the XNET section. A title that only occasionally uses networking functions could mark this section as delayed load, and only load it as necessary when using the networking functions. Other sections can be treated similarly to further reduce a title's memory footprint. The following XTL sections are defined:

SectionFunctionality
D3DDirect3D classes and functions
D3DXDirect3DX classes and functions
DSOUNDDirectSound classes and functions
XGRPHXbox graphics functions in xgraphics.lib (for example, XGSwizzleBox)
XPPXbox peripherals functions (including XInitDevices)
XNETXbox networking functions in xnet.lib

These sections can be marked as delayed load sections and then explicitly loaded before their functions are called. Currently, it is recommended that these sections, once loaded, remain loaded (in other words, do not call XFreeSection or XFreeSectionByHandle on them). For more detailed information, use link -dump -symbols on a .lib file to determine the sections that individual functions are located in.

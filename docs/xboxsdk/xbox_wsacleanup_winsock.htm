WSACleanup

Terminates use of the Xbox Winsock layer.

int  WSACleanup (void);

Parameters

This function has no parameters.

Return Values

The return value is zero if the operation was successful. Otherwise, the value SOCKET_ERROR is returned, and a specific error number can be retrieved by calling WSAGetLastError.

WSACleanup terminates Winsock operations for all threads.

Error code Meaning
WSANOTINITIALISED A successful WSAStartup call must occur before using this function.
WSAESYSNOTREADY The network subsystem has failed or has not been initialized.

Remarks

The title must perform a successful WSAStartup call before it can use Winsock services. When it has completed the use of Winsock, the title must call WSACleanup to deregister itself from the Winsock implementation and allow the implementation to free any resources allocated on behalf of the title. Any pending blocking or asynchronous calls issued by any thread in this process are canceled without posting any notification messages or without signaling any event objects. Any pending overlapped send and receive operations (WSASend/WSASendTo/WSARecv/WSARecvFrom with an overlapped socket) issued by any thread in this process are also canceled. In this case, the pending overlapped operations fail with the error status WSAECANCELLED.

Sockets that were open when WSACleanup was called are reset and automatically deallocated as if closesocket were called; sockets that have been closed with closesocket but that still have pending data to be sent can be affected—the pending data can be lost if Winsock is unloaded from memory. To ensure that all pending data is sent, call shutdown to close the socket connection, then wait until the close completes before calling closesocket and WSACleanup.

There must be a call to WSACleanup for every successful call to WSAStartup. Only the final WSACleanup does the actual cleanup; the preceding calls simply decrement an internal reference count in Winsock.

Requirements

Header: Declared in Winsockx.h, for include use Xtl.h.

Import Library: Use Xnet.lib for titles that only support system link play, XOnline.lib for titles that support online play. Use the secure versions of the library file (Xnets.lib or XOnlines.lib) for final testing.

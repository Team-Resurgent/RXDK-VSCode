DSP Builder Overview

The DSP Builder tool (DSPBuilder) enables users to quickly assemble a DSP effects image using a graphical user interface. It creates the DSP effects image, the .ini file (which could be used with XGPImage later), and the .h file, without the user having to manually create and/or edit the .ini file used by XGPImage.

While XGPImage provides equal or greater flexibility than DSPBuilder, DSPBbuilder is much easier to use and should allow you to start developing your own images more quickly.

Features

The following list contains the features provided by DSPBuilder:

  • Provides a graphical interface to the information needed to build a DSP effects image.
  • Generates the DSP effects image (.bin), .h file for the image, and an .ini file that can be used with XGPImage.
  • Indicates bad mixbin assignments (red for invalid routings).
  • Allows you to easily insert the Crosstalk cancellation effect needed by DirectSound.
  • Allows you to easily insert the I3DL2 reverberation effect needed by DirectSound, and lets you select a default environment for it.
  • When a DSP Builder image is generated, the header file contains fixed names for I3DL2 and crosstalk-effect indices. These are referenced as GraphI3DL2_I3DL2Reverb and GraphXTalk_XTalk respectively, and are used with the DSEFFECTIMAGELOC structure.
  • Provides an optional mix to be sent to the LFE.
  • Automatically manages the temporary mixbins needed for your DSP effects image.
  • Abstracts many of the complexities in the .ini file used by the XGPImage tool (such as outputs to temporary mixbins needed by some effects (the I3DL2 reverb, for instance)).
  • Allows you to adjust the parameters of certain effects.
  • Allows you to preview changes using the Audio Console application.
  • Allows real-time updates of high-level parameters (so you can listen to the results of your changes on Xbox in real time).
  • Allows you to choose whether an effect overwrites or mixes its output.
  • Allows you to add voice on FX send channels 16 through 19.

Differences with XGPImage

The following list contains differences between XGPImage and DSPBuilder:

  • DSPBuilder allows you to adjust the parameters of some effects.

Auditioning DSP Builder Effects Images: DSP Server

Rather than having to generate and save the DSP image, giving it to the audio developer, and waiting for a rebuild of the game with the new image, there is an alternative for rapid auditioning of the content: DSP Server. DSP Server source code is provided with the XDK and can be integrated into a title with minimal coding. It is also used by the Audio Console Application (audconsole.xbe), an Xbox audio audition tool provided on XDK recovery discs and available for separate download from Xbox Central.

DSP Server allows the application to accept updated effects images in real time as transmitted from DSP Builder. So, while playing the title, the current effects program can be swapped out, effect parameters altered, and so on.

Developers only need to do the following:

  • Add DSPServer.cpp and DSPServer.h to the project tree. These files can be found in the source\DSound\DSPServer subdirectory of the XDK installation.
  • Include DSPServer.h in the application source file.
  • Start the DSP Server thread somewhere in the vicinity of the game's initialization routines by calling CDSPServer::Run. This is a static method. There are several optional parameters to this call, described in both DSPServer.cpp and DSPServer.h.
  • To stop the DSP Server thread before the game exits, call CDSPServer::Stop. This is also a static method.

An Xbox application that uses DSP Server can take advantage of DSP Builder's transmit option, accessible from both the Xbox menu as Transmit Image and the Transmit Image icon on the toolbar. When applied, the current effects program is downloaded to the Xbox from the PC.

DSP Builder Generated Header File

The DSP Builder generated header file contains a unique set of names for the effect constants (as long as the effects' names are not changed to be identical). Effects that you add will appear in the header in the following formats:

  • For effect indices, the format is <UserEffectName>_<EffectName>.
  • For state structures, the format is <UserEffectName>_FX0_<EffectName>_STATE.

In the examples above, <UserEffectName> will be replaced by the name you enter for the effect. <EffectName> is the actual name of the effect (for example, IIR2), or if there are duplicate effects (with the same default names), the name will be appended with a unique integer, or the user-specified name.

Note   Supported characters for user names include [A–Z, a–z, 0–9, _]. All other characters are deleted.

DSP Builder added effects (that is, those selected for the Build Options menu under Tools), will be named the following:

// Effect Indices (the names will never change):
GraphI3DL2_I3DL2Reverb                 // The default reverb in the image - even if added by and/or renamed by the user
GraphXTalk_XTalk                       // The default XTalk in the image
GraphXTalkToLFE_XTalkToLFE_A           // XTalk to LFE send A
GraphXTalkToLFE_XTalkToLFE_B           // XTalk to LFE send B
GraphXTalkToLFE_XTalkToLFE_C           // XTalk to LFE send C
GraphVoice_Voice_0                     // Voice on mixbin 16
GraphVoice_Voice_1                     // Voice on mixbin 17
GraphVoice_Voice_2                     // Voice on mixbin 18
GraphVoice_Voice_3                     // Voice on mixbin 19

// State Structures (the names will never change):
GraphI3DL2_FX0_I3DL2Reverb_STATE
GraphXTalk_FX0_XTalk_STATE
GraphXTalkToLFE_FX0_XTalkToLFE_A_STATE
GraphXTalkToLFE_FX1_XTalkToLFE_B_STATE
GraphXTalkToLFE_FX2_XTalkToLFE_C_STATE
GraphVoice_FX0_Voice_0_STATE
GraphVoice_FX1_Voice_1_STATE
GraphVoice_FX2_Voice_2_STATE
GraphVoice_FX3_Voice_3_STATE

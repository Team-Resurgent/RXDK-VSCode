XOnlineStorageDownloadToMemoryGetResults

Retrieves the storage download results from a call to the XOnlineStorageDownloadToMemory function.

HRESULT XOnlineStorageDownloadToMemoryGetResults(
    XONLINETASK_HANDLE hTask,
    PBYTE *ppbReceivedData,
    DWORD *pcbReceivedData,
    DWORD *pcbDataTotal,
    ULONGLONG *pqwOwnerPuid,
    FILETIME *pftCreationDate
);

Parameters

hTask
[in] Task handle returned from XOnlineStorageDownloadToMemory.
ppbReceivedData
[out, optional] Pointer into which the data was received.
pcbReceivedData
[out, optional] Number of bytes actually received.
pcbDataTotal
[out, optional] Number of bytes available on the server (pcbReceivedData is equal to pcbDataTotal if everything is received).
pqwOwnerPuid
[out, optional] Player unique ID (PUID) of the owner of this storage item.
pftCreationDate
[out, optional] Date this file was created.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

This function should not be called until pumping of the download task has returned a value other than XONLINETASK_S_RUNNING.

If the download was successful, this function returns the buffer to the data received (which should be the same as pbReceiveBuffer in the original download call), the number of bytes actually received into the buffer, and the number of bytes that was in the server response. Typically, if the receive buffer is large enough, the cbReceivedData and pcbTotal should match. If pcbTotal is larger than pcbReceivedData, the supplied buffer is not large enough to hold the whole response.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

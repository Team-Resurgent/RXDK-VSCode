IDirectMusicPerformance8::InitAudioX

Initializes the performance and optionally sets up a default audiopath. This method must be called before the performance can play using audiopaths.

HRESULT InitAudioX(
    DWORD dwDefaultPathType,
    DWORD dwPChannelCount,
    DWORD dwVoiceCount,
    DWORD dwFlags
);

Parameters

dwDefaultPathType
DWORD value that specifies the default audiopath type. Can be zero if no default path type is wanted. For a list of defined values, see IDirectMusicPerformance8::CreateStandardAudioPath.
dwPChannelCount
Value that specifies the number of performance channels to allocate to the path, if dwDefaultPathType is not zero.
dwVoiceCount
Ignored. Do not use.
dwFlags
Value that contains the flags used to specify how the performance should be initialized. Can be set to DMUS_INITAUDIO_NOTHREADS, which specifies that DirectMusic not use any threads and requires the title to call DirectMusicDoWork.

Return Values

Returns S_OK if successful, an error code otherwise. Can return one of the following error values:

E_NOINTERFACE
E_OUTOFMEMORY
E_POINTER

Remarks

This method can be called only once. It cannot be used to retrieve an existing IDirectMusic8 interface.

The number of performance channels specified by dwPChannelCount does not cause any allocation of hardware resources. Hardware resources are allocated on the fly as they are needed.

The performance must be terminated by using the IDirectMusicPerformance8::CloseDown method before being released.

Remarks

DirectMusic's allocation of 3D voices does not fall under the number set in dwVoiceCount, since it can be tightly controlled by the publisher by limiting the number of 3D audiopaths.

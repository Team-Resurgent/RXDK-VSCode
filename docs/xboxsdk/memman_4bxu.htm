IsBadCodePtr

The IsBadCodePtr function determines whether the title has read access to the memory at the specified address.

BOOL IsBadCodePtr(
  FARPROC lpfn   // memory address
);

Parameters

lpfn
[in] Pointer to an address in memory.

Return Values

If the title has read access to the specified memory, the return value is zero.

If the title does not have read access to the specified memory, the return value is nonzero. To get extended error information, call GetLastError.

Remarks

If the application is compiled as a debugging version and the title does not have read access to all bytes in the specified memory range, the function causes a first-chance exception, which may break into the debugger. Leaving the debugger, the function continues as usual and returns a nonzero value. This behavior is by design, as a debugging aid. The exception will cause a debugger break only if the debugger is configured to break on first-chance exceptions as well as unhandled exceptions.

IsBadCodePtr checks the read access only at the specified address and does not guarantee read access to a range of memory.

In a preemptive multitasking environment, it is possible for some other thread to change the access to the memory being tested. Even when the function indicates that the title has read access to the specified memory, you should use structured exception handling when attempting to access the memory. Use of structured exception handling enables the system to notify the title if an access violation exception occurs, giving the title an opportunity to handle the exception.

Requirements

Header: Declared in Winbase.h, for include use Xtl.h.
Import Library: Use Xapilib.lib

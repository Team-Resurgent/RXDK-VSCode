IDirect3DPushBuffer8

Applications use the methods of the IDirect3DPushBuffer8 interface to handle push-buffer resources.

The methods of the IDirect3DPushBuffer8 interface can be organized into the following groups:

Fix-ups CopyRects
  EndVisibilityTest
  Jump
  RunPushBuffer
  SetModelView
  SetPalette
  SetRenderState
  SetRenderTarget
  SetTexture
  SetVertexBlendModelView
  SetVertexShaderConstant
  SetVertexShaderInput
  SetVertexShaderInputDirect
Management BeginFixup
  EndFixup
  GetSize
Verification Verify

The IDirect3DPushBuffer8 interface inherits the following IDirect3DResource8 methods, which can be organized into these groups.

Devices GetDevice
Information GetType
Private Surface Data FreePrivateData
Miscellaneous Register
  BlockUntilNotBusy
  IsBusy
  MoveResourceMemory

  AddRef
  Release

Push-buffers that are pre-created via IDirect3DPushBuffer8::Register must be located in physically contiguous write-combined memory unless D3DPUSHBUFFER_RUN_USING_CPU_COPY is specified. More specifically, IDirect3DDevice8::RunPushBuffer needs to have the CPU write the fix-ups in a manner coherent with the GPU, so the memory can be neither cached nor read-only.

The LPDIRECT3DPUSHBUFFER8 and PDIRECT3DPUSHBUFFER8 types are defined as pointers to the IDirect3DPushBuffer8 interface.

XGSetVolumeTextureHeader

Manually creates a volume texture resource.

DWORD WINAPI XGSetVolumeTextureHeader(
  UINT Width,
  UINT Height,
  UINT Depth,
  UINT Levels,
  DWORD Usage,
  D3DFORMAT Format,
  D3DPOOL Pool,
  IDirect3DVolumeTexture8 *pVolumeTexture,
  UINT Data,
  UINT Pitch
);

Parameters

Width
[in] Width of the top-level of the volume texture, in pixels. This value must be a power of two. The pixel dimensions of subsequent levels will be the truncated value of half of the previous level's pixel dimension (independently). Each dimension clamps at a size of 1 pixel. Thus, if the division by two results in 0 (zero), 1 will be taken instead.
Height
[in] Height of the top-level of the volume texture, in pixels. This value must be a power of two. The pixel dimensions of subsequent levels will be the truncated value of half of the previous level's pixel dimension (independently). Each dimension clamps at a size of 1 pixel. Thus, if the division by 2 results in 0 (zero), 1 will be taken instead.
Depth
[in] Depth of the top-level of the volume texture, in pixels. This value must be a power of two. The pixel dimensions of subsequent levels will be the truncated value of half of the previous level's pixel dimension (independently). Each dimension clamps at a size of 1 pixel. Thus, if the division by 2 results in 0 (zero), 1 will be taken instead.
Levels
[in] The number of levels in the volume texture. If this is zero, Microsoft® Direct3D® will generate all texture sub-levels down to 1×1×1 pixels.
Usage
[in] Currently not used, set to 0.
Format
[in] Member of the D3DFORMAT enumerated type, describing the format of all levels in the volume texture.
Pool
[in] Ignored.
pVolumeTexture
[in, out] Address of the IDirect3DVolumeTexture8 interface, representing the volume texture resource which is to be filled.
Data
[in] Offset to the data held by this resource.
Pitch
[in] Specifies the pitch of the texture.

Return Values

Returns the number of bytes required by the resource, not counting the header.

Remarks

All resources, including volume texures, include a header that contains all of the information needed by the driver to process the resource and a data allocation that contains information used by the GPU. The resource headers are used only by the driver and so should be allocated from normal cached memory. The header contains all of the information needed by the driver to set up the hardware so it can use the information stored in the data allocation.

Unless the title is manually creating a resource, it is recommended that the title use IDirect3DDevice8::CreateVolumeTexture.

After calling XGSetVolumeTextureHeader, it is necessary to call IDirect3DResource8::Register in order to register the memory address of the resource.

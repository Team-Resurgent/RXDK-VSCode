XOnlineFriendsRequestEx

Sends a request to another player asking that the two players be placed on each other's friends lists. Use this function rather than the XOnlineFriendsRequest function when a voice attachment to the request is desired.

HRESULT XOnlineFriendsRequestEx(
    DWORD dwUserIndex,
    XUID xuidToUser,
    XONLINE_MSG_HANDLE hMsg,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwUserIndex
[in] Controller index (zero-based) of the player making the request. The player must have been successfully logged on with a previous call to the XOnlineLogon function.
xuidToUser
[in] XUID structure that uniquely identifies the recipient of the friend request.
hMsg
[in] An XONLINE_MSG_HANDLE handle of an XONLINE_MSG_TYPE_FRIEND_REQUEST message to be sent with the request. If this parameter is NULL, a default message with no voice attachment is generated.
hWorkEvent
[in, optional] Handle to a caller-created event object. This object is signaled when there is work to do by calling the XOnlineTaskContinue function. The caller may pass in NULL instead, indicating that the caller will poll XOnlineTaskContinue.
phTask
[out] Pointer to a variable of type XONLINETASK_HANDLE that receives a task handle representing the verification task. Pump this task to completion using XOnlineTaskContinue. When the task is complete, call the XOnlineTaskClose function to close the task.

Return Values

If XOnlineFriendsRequest succeeds, the return value is S_OK.

If XOnlineFriendsRequest fails, the return value is one of the following error codes:

Value Description
E_OUTOFMEMORY Insufficient memory. Free some memory.
XONLINE_E_NOTIFICATION_NO_TASK The friends list startup task is not active. You must first call the XOnlineFriendsStartup function to create the startup task.
XONLINE_E_NOTIFICATION_SELF The player requested to add himself or herself to his or her own friends list. Do not allow this. If you receive this error, display an error or generate a sound to indicate that this is not a valid action.
XONLINE_E_NOTIFICATION_TOO_MANY_REQUESTS Too many pending requests to the service exist. To avoid this error, pump the handle returned by XOnlineFriendsStartup more frequently. If you get this error during a game, pump the handle returned by XOnlineFriendsStartup, and then call XOnlineFriendsRequest again.
XONLINE_E_NOTIFICATION_LIST_FULL The player's friends list is full. An appropriate way to handle this return code is to display this message: "Unable to add the specified friend because your friends list is full. Press A to continue." This error should occur only if the title is currently enumerating the friends list. You can avoid this error by verifying that the player has fewer than 100 friends before requesting a new one. There are situations in which you do not receive this error even though the list is full.
XONLINE_E_NOTIFICATION_USER_ALREADY_EXISTS The player is already in the list. An appropriate way to handle this return code is to display this message: "This player is already on your friends fist. Press A to continue." Alternatively, you can do nothing. If the player forgot that a particular friend was on the list and invites that friend again, it will simply seem as though the invited player accepted immediately. This error should occur only if the title is currently enumerating the friends list.
Other For information about other possible errors, see Online Errors.

Remarks

Use XOnlineTaskContinue to pump the task returned by XOnlineFriendsStartup in order to send the request. Use XOnlineTaskContinue to pump the task handle returned by XOnlineFriendsEnumerate to update the friends list with the state of the request.

The title should not destroy the message object it specified through the hMsg parameter until the XOnlineFriendsRequestEx operation is complete.

The player receiving the request indicates his or her response to the receiving title, which then calls the XOnlineFriendsAnswerRequest function to respond to the request.

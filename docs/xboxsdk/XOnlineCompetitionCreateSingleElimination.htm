XOnlineCompetitionCreateSingleElimination

Creates a single-elimination competition.

HRESULT XOnlineCompetitionCreateSingleElimination(
    DWORD dwUserIndex,
    DWORD dwTemplate,
    ULONGLONG qwTeamID,
    const XONLINE_COMP_SINGLE_ELIMINATION_ATTRIBUTES *pDefaultAttributes,
    DWORD dwNumAdditionalAttributes,
    const XONLINE_ATTRIBUTE *pAdditionalAttributes,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwUserIndex
[in] Controller index (zero-based) of the player. The player must have been successfully logged on with a previous call to XOnlineLogon.
dwTemplate
[in] ID of the template describing the competition.
qwTeamID
[in] ID of the team on which the player competes. This parameter is used only when the fTeamCompetition member of the structure pointed to by the pDefaultAttributes parameter is TRUE. Otherwise, qwTeamID should be 0.
pDefaultAttributes
[in] Address of an XONLINE_COMP_SINGLE_ELIMINATION_ATTRIBUTES structure that describes the competition.
dwNumAdditionalAttributes
[in] Number of additional title-specific attributes contained in the array whose starting address is contained in the pAdditionalAttributes parameter.
pAdditionalAttributes
[in] Starting address of an array of title-specific attributes associated with the competition.
hWorkEvent
[in, optional] Handle of an event you want to associate with this task. If you do not want to assign an event, set this parameter to NULL.
phTask
[out] Pointer to an XONLINETASK_HANDLE type. The handle is associated with the task started by XOnlineCompetitionCreateSingleElimination. The handle can then be used in calls to XOnlineTaskContinue to perform work associated with this task.

Return Values

Returns S_OK if successful, or an error code otherwise. The error codes can include the following.

Error Code Description
E_INVALIDARG A required attribute is missing, or one or more of the parameters or attributes specified is invalid.

Remarks

XOnlineCompetitionCreateSingleElimination is used to create a new single-elimination competition. Use XLAST to configure competitions. XLAST assigns a new template ID that must be specified in the dwTemplate parameter. Competitions consist of a combination of standard attributes, specified in the structure pointed to by the pDefaultAttributes parameter, and an optional list of custom attributes pointed to by the pAdditionalAttributes parameter.

If the fIsTeamCompetition flag of the structure pointed to by the pDefaultAttributes parameter is FALSE, the competition is "owned" by the player logged in at the controller index specified in dwUserIndex. Otherwise, it is owned by the team specified in qwTeamId. The owner of a competition has special rights, including the abiltiy to cancel the competition or eject entrants. For a team-based competition, anyone on the team may perform the functions of an owner.

Note    The player specified in dwUserIndex (or the team specified in qwTeamId) is not automatically enrolled in the competition. After the call to XOnlineCompetitionCreate completes successfully, the application should call XOnlineCompetitionManageEntrant to enroll the user or team.

You can also use XLAST to define custom attributes. The order of the list of attribute structures pointed to by pAdditionalAttributes must match the order listed in XLAST.

The dwPrivateSlots and dwPublicSlots members of the XONLINE_COMP_SINGLE_ELIMINATION_ATTRIBUTES structure pointed to by the pDefaultAttributes parameter specify the number of private and public entrants allowed in the competition, respectively. Private entrants are players or teams invited to join the competition, typically by the competition owner. For private entrants, the title must specify a dwAction of XONLINE_COMP_ACTION_JOIN_PRIVATE in the call to XOnlineCompetitionManageEntrant when joining the competition. If the number of private players exceeds the number specified in dwPrivateSlots, the private entrant is entered in a public slot. However, a public entrant cannot occupy a private slot.

If the number of private and public entrants in the competition is less than the number specified in the dwMinimumPlayers member of the XONLINE_COMP_SINGLE_ELIMINATION_ATTRIBUTES structure as of the time specified in ftRegistrationClose, the competition is automatically canceled.

The ftRoundOneStart member of the XONLINE_COMP_SINGLE_ELIMINATION_ATTRIBUTES structure designates the scheduled time for the first round of the competition to start. Entrants may play the event before this time; this is simply a suggested time for the match to take place.

The ftRoundOneEnd member of the XONLINE_COMP_SINGLE_ELIMINATION_ATTRIBUTES structure specifies the time by which all players should have submitted scores for the first round matches. Any players who did not check in or submit scores risk forfeiting from the match and the competition.

The time difference between the ftRoundOneStart and ftRoundOneEnd members of the XONLINE_COMP_SINGLE_ELIMINATION_ATTRIBUTES structure must be of sufficient time to play a match game.

Rounds following the first round are scheduled based on ftRoundOneStart and the values specified for Interval and UnitOrDayMask. The duration of each subsequent round is the same as the duration specified for round one.

After XOnlineTaskContinue completes, call XOnlineCompetitionCreateGetResults to obtain the competition ID for the newly created competition.

The Competition Name attribute must always be set as an additional attribute (it is not a member of the XLAST DefaultAttribute structure). It is used by the Alerts system to deliver the competition reminders. The template for Competition Reminder alert text requires that the competition name be specified. For example, where TitleName is the title name and CompetitionName is the competition name, your title might display this alert: "Your match in the TitleName competition, CompetitionName, starts soon."

The Competition Name attribute ID is 0x8011000E, which corresponds to the XLAST attribute att_competition_name. The corresponding variable name for this attribute in the competition code generated by XLAST is pwszDisplayName.

Xbox Console Setup

Your first step is setting up your Xbox console. This console runs the game titles that you develop or test. Code written for the Xbox will run only on the Xbox console and not on any other system.

Console Types

Two types of Xbox consoles are available with the Xbox Development Kit: Development consoles, which have a clear case, and Debug consoles, which have a transparent green case. Development consoles have two hardware features that Debug consoles do not:

  • A SCSI port. The SCSI port enables DVD emulation (see Xbox Game Disc Emulation System Setup and Upgrade).
  • A serial port. Though rarely used, the serial port enables kernel debugging when necessary, as well as the ability to receive some debug output when running in "retail" mode.

Debug consoles, which are otherwise identical to Development consoles, can be used for development as well as for debugging and testing.

Setting up Your Console

Follow these steps to prepare your Xbox console for use:

  1. Connect a TV to the Xbox console.

    To create the most accurate run-time environment for testing Xbox games, it is recommended that a TV be used for the Xbox console monitor. Connect the TV using the supplied A/V pack. The console is by default set to support a standard NTSC TV. If you are using a PAL TV, you can change the console display settings by pressing the X controller button from the XDK Launcher screen. In the settings screen that appears, you can change the type of television from NTSC to PAL. The display settings will not take effect until you reboot the console.

    Connecting a VGA monitor to the Xbox console is currently not supported.

    Note   If you have changed the display settings such that you can no longer see video from the console, you can restore the settings in either of two ways:

    • From the XDK Recovery or XDK Launcher section, use the following key combinations to reset the video settings:
      Left trigger + right trigger + D-pad right = NTSC-M (North America)
      Left trigger + right trigger + D-pad up = PAL-I
      Left trigger + right trigger + D-pad left = NTSC-J (Japan)
    • After you have established a network connection with the Xbox console (see Establishing a Network Connection Between the Xbox Console and the Development System), you can use the remote configuration tool, XbSetCfg. Open a command window on the development system and use XbSetCfg with the appropriate switch to reset the display settings. The change in display settings will not take effect until you reboot the console. For more information about the XbSetCfg command-line switches, see the XbSetCfg topic.
  2. Connect the controller.

    Connect the Xbox Controller to the console by inserting the plug into one of the four controller ports on the front of the Xbox console.

  3. Attach a network cable.

    You can connect the console to the development system in either of two ways:

    • Using a LAN connection.  Attach one end of a standard network cable (not provided) to the Xbox console and the other end to the network hub.
    • Using a peer-to-peer connection.  Connect the provided network cross-link cable between the Xbox console and the development system.

    Note   Although a serial cable was provided with previous releases of the XDK, the Microsoft® Visual C++® debugger cannot communicate with the console via a serial connection. Use the included network cable to connect the Xbox console to your network hub.

  4. Install the Xbox system software.

    There are two methods for installing or updating the Xbox system software on the Xbox console: remote recovery or a recovery disc. You must first download either one from the Xbox developer support web site (https://xds.xbox.com). Log on to the site, navigate to the release page, find the section titled XDK Recovery, and download the recovery tool or disc image. Save whichever one you choose onto your computer, and follow the instructions listed on the release page.

    The Xbox Remote Console Recovery Tool is an executable file that updates the console over a network connection. Not only is this tool more convenient to use than a recovery disc, it also gives you the option of leaving the data on your console's hard disk intact. However, if your console hard disk is corrupted, you may not be able to use the remote recovery tool to update your console's system software. If this is the case, you must restore the hard disk from an Xbox console recovery disc.

    Note   The remote recovery tool can only recover your Xbox console to development kit mode. To recover the console to retail mode for final game testing, you must burn a recovery DVD-R disc using the recovery ISO image in the XDK.

    To install or recover the Xbox system software via recovery disc:

    1. Power up the console and insert the console recovery disc into the console disc drive. You will be prompted to press any controller button.
    2. Recovery now allows you to recover a console as a development kit (as it has in the past) or as a retail console (a new option). Press the Y button on the gamepad when prompted to recover the console as a retail console. After recovery is complete, the console will behave much like a retail Xbox console that will be sold to customers, which can be useful for testing a game title prior to sending it in for certification. In this retail configuration, only 64 MB will be available, so you will need to load titles from the disc drive. Also, the XDK Launcher and samples will not be installed, the Xbox Dashboard will be the default title, and you will not be able to use the remote debugger or use any Xbox tools to communicate with the console.
    3. After the console is updated, the disc drive tray will open and you will be prompted to remove the disc and press any controller button to reboot.
    4. The XDK Launcher screen will be displayed in the Settings: Machine Name mode. Enter a name for your console for network use. The name may be up to 20 characters long; all characters on the Xbox virtual keyboard are valid. Click OK when you are finished.

      Warnings

      • Booting from the console recovery disc will erase the data on the console hard disk. Copy any data that you wish to retain to another system or storage device before booting from the recovery disc.
      • Because the recovery process updates the ROM on the console, it is critical that you not power off or reboot the console during the recovery process.

To install or recover the Xbox system software using the remote recovery tool:

  1. Power up the console.
  2. Run the Xbox Remote Console Recovery Tool. The tool is an executable named XDKRecoveryNNNN.exe, where NNNN is the build number of the XDK version you are installing.
  3. Click Next to continue past the welcome screen.
  4. Enter the name or IP address of the Xbox console you wish to recover and click Next. The remote recovery tool reboots your console in preparation for copying files.
  5. Next, the remote recovery tool asks if you wish to reset your console to default settings. If you click Yes, the tool reformats the console's hard disk, erasing all data. If you click No, the tool leaves the hard disk's data intact, installing only new kernel and system software. Either way, the tool then copies files to your Xbox console.

    Note   Do not turn off your Xbox console while the remote recovery tool is copying files.

  6. Finally, the remote recovery tool asks if you wish to update the console's ROM. Choose Yes to update the ROM or No to leave the ROM alone.

    Note   Do not turn off your Xbox console while the remote recovery tool is updating the ROM. Doing so may leave your console in an unusable state.

  7. Click Finish to exit the remote recovery tool.

Remote Recovery Command-Line Switches

The remote recovery tool supports command-line switches that allow you to fine tune how you recover an Xbox console. The /x, /r, and /u switches, when used in scripts or batch files, are useful for performing unattended recovery of multiple Xbox consoles at once. The switches are as follows:

Switch Meaning
/L langid Installs setup for the specified language ID.
/? Displays a usage message.
/f Forces installation over an existing newer installation.
/s Skips pre-setup checks.
/e "AlternateExeName" Performs setup using resources packed in the specified executable file.
/c "CustomDLLPath" Specifies an alternate custom DLL.
/x xboxname Specifies the name of the target console to recover.
/r Resets the Xbox console to its default state. This option is off by default.
/u Runs recovery in unattended mode, requiring no user interaction of any kind.

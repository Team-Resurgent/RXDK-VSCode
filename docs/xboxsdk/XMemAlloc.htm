XMemAlloc

XMemAlloc is an application-defined memory allocation callback function that replaces the memory allocator used by the XTL libraries.

LPVOID WINAPI XMemAlloc(
    SIZE_T dwSize,
    DWORD dwAllocAttributes
);

Parameters

dwSize
[in] Desired size, in bytes, of the memory to be allocated.
dwAllocAttributes
[in] An XALLOC_ATTRIBUTES value that specifies the desired attributes of the memory to allocate.

Return Values

If the function succeeds, the return value is the base address of the allocated memory.

If the function fails, the return value is NULL.

Remarks

If a title implements an XMemAlloc function, all allocations performed by the XTL libraries will instead call the title's XMemAlloc routine. The title must allocate and return the memory as requested. The title can simply use the XPhysicalAlloc and HeapAlloc functions to provide the underlying heap support, if desired. Alternatively, the title can implement its own heap routines and thus replace the default heap allocator.

XTL components that need to allocate memory (graphics, sound, networking, and so on) all call XMemAlloc instead of other allocation routines. The default XMemAlloc then calls XPhysicalAlloc or HeapAlloc as appropriate. A title can provide its own XMemAlloc function, thus overriding the default, and allowing the title to trace XTL allocations and to replace the heap allocator. The title should provide all of the allocator replacement functions (XMemAlloc, XMemFree, and XMemSize) or none at all. The allocator functions to use (either the default or title-defined versions) are determined at link time, so the title imposes no additional overhead by supplying its own versions.

The title can, if desired, call its own implementation of XMemAlloc for allocations as well, thus routing all title and XTL allocations through a single allocation routine.

If the dwHeapTracksAttributes member of XALLOC_ATTRIBUTES indicates that the heap is responsible for storing allocation attributes, then the XMemAlloc implementation should store the attributes within the heap allocation itself (for heap allocations only, not for physical allocations). A title using the HeapAlloc function rather than a custom heap can use the XSetAttributesOnHeapAlloc to store the attributes in the unused portion of each heap allocation.

If the heap is storing allocation attributes along with the allocations, XMemFree can verify that the attributes specified in dwAllocAttributes match those stored with the allocation (in order to detect allocation/deallocation mismatches).

Note    XMemAlloc, XMemFree, and XMemSize can be reliably overridden only by functions defined in the main executable project, not in a library (.lib file). Otherwise, XDK libraries may pull in the default implementations before the application-defined functions are detected. The application-defined functions must be in an .obj file used to generate the title's .xbe, rather than in a .lib that is linked in with the title. Alternatively, adding the .lib in Additional Dependencies in Linker/Input may resolve the issue.

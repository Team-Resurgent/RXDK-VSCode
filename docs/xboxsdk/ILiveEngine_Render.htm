ILiveEngine::Render

Renders active Xbox Drop-in UI screens to the specified surface.

HRESULT Render(
    IDirect3DSurface8 *pSurface
);

Parameters

pSurface
[in] Pointer to an IDirect3DSurface8 interface representing the surface to render to.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

Titles should call Render within their render loop after performing game rendering to the specified surface. Render need be called only when the ILiveEngine::DoWork method indicates that rendering is required.

Some advanced Direct3D states and features are not recognized or supported by Render. For example, the Xbox Drop-in UI cannot retrieve from Direct3D a vertex shader instruction state that was directly loaded. For these unsupported states, the title must save the state and reset the state back to defaults before calling Render. After the call to Render, the title can then restore the state as needed.

The following states and features are not recognized or supported by Render:

  • Direct vertex shader programs, reached by methods of the IDirect3DDevice8 interface (LoadVertexShader, LoadVertexShaderProgram, SelectVertexShader, SelectVertexShaderDirect, SetVertexShaderInput, and SetVertexShaderInputDirect)
  • Direct pixel shader programs, set by the IDirect3DDevice8::SetPixelShaderProgram method
  • Clip planes, set by IDirect3DDevice8 methods (SetDepthClipPlanes, SetTransform with the D3DTS_PROJECTION constant)
  • Back buffer scaling, set by the IDirect3DDevice8::SetBackBufferScale method
  • Direct3D pure devices (that is, devices created using the D3DCREATE_PUREDEVICE flag).

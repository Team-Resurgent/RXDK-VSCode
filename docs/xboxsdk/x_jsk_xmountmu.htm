XMountMU

The XMountMU function mounts the specified memory unit into the file system namespace and returns the drive letter that can then be used to access that memory unit.

DWORD XMountMU(
  DWORD dwPort,                  
  DWORD dwSlot,                  
  PCHAR pchDrive               
);

Parameters

dwPort
[in] The port number of the controller that contains the memory unit to be mounted. This parameter must be one of the following values:
Port Description
XDEVICE_PORT0 The leftmost controller port (port 0).
XDEVICE_PORT1 The second controller port from the left (port 1).
XDEVICE_PORT2 The third controller port from the left (port 2).
XDEVICE_PORT3 The rightmost controller port (port 3).
dwSlot
[in] The controller slot that the memory unit is located in. This parameter must be one of the following values:
Value Meaning
XDEVICE_TOP_SLOT The top slot of the controller specified in dwPort.
XDEVICE_BOTTOM_SLOT The bottom slot of the controller specified in dwPort.
pchDrive
[out] Pointer to a character that receives the drive letter through which the memory unit can now be accessed.

Return Values

If the function succeeds, the return value is ERROR_SUCCESS.

If the function fails, the return value is an error code defined in Winerror.h. The function does not use SetLastError to set the calling thread's last-error code.

Remarks

Before calling XMountMU, XInitDevices must be called to initialize the Xbox peripherals.

After a title detects connected memory units by calling either XGetDevices or XGetDeviceChanges, the available memory units must be mounted into the file system namespace before they can be read from or written to.

XMountMU mounts the specified memory unit and returns a drive letter that is mapped to the memory unit and that can be used to access the MU. The drive to mount cannot be chosen; it is predetermined based on slot location.

Once mounted into the file system, the normal file I/O functions (such as CreateFile) can be used on the memory unit.

When the title is finished interfacing with the MU, it should call XUnmountMU to unmount the MU and free up the associated resources. Attempting to mount an MU that is already mounted will fail.

Requirements

Header: Declared in Xbox.h, for include use Xtl.h.
Import Library: Use Xapilib.lib

XUID

Contains the user identifier and user-specific settings.

typedef struct{
    union {
        ULONGLONG qwUserID;
        ULONGLONG qwTeamID;
    } ;
    DWORD dwUserFlags;
} XUID;

Members

qwUserID
Value that contains the user identifier.
qwTeamID
Value that contains the team identifier. Note that Xbox Live team IDs start with the prefix 0xFeFe.
dwUserFlags

Value that contains settings associated with this player. Can be a combination of the following flags:
Value Description
XONLINE_USER_VOICE_NOT_ALLOWED This player is not allowed to use the voice chat feature.
XONLINE_USER_PURCHASE_NOT_ALLOWED This player is not allowed to make any purchases.
XONLINE_USER_NICKNAME_NOT_ALLOWED This player is not allowed to choose custom nicknames.
XONLINE_USER_SHARED_CONTENT_NOT_ALLOWED This player is not allowed to share content.

Additionally, the following masks indicate the bit offsets of additional data stored in the dwUserFlags member:

Value Description
XONLINE_USER_GUEST_MASK Mask for the bits that contain the player guest number. If this player is not a guest, the value is zero. If this player is a guest, the sponsoring player is specified in qwUserID. Up to three guest players (one for each additional controller) can be associated with a single logged on player.
XONLINE_USER_NOSHOW_RATING_MASK Mask for the bits that contain the player NoShowRating. The NoShowRating value represents the ratio of tournament match no-shows to tournament match sessions played.
XONLINE_USER_DISCONNECT_RATING_MASK Mask for the bits that contain the DisconnectRating. The DisconnectRating value represents the ratio of disconnects to arbitrated sessions played.
XONLINE_USER_COUNTRY_MASK Mask for the bits that contain the player country identifier, representing the country in which the player is registered. This value is one of the XONLINE_COUNTRY_ values in the XOnline.h file.

Remarks

For titles using arbitration, the service aggregates information about playing patterns. Specifically, arbitration service keeps a tally of the number of arbitrated sessions a player has engaged in, as well as the number of times a player disconnected from an arbitrated session prematurely. All players are divided into eight buckets based on the ratio of disconnects to the total number of sessions played.

Likewise, the service keeps track of the total number of tournament matches each player has registered for, as well as the number of matches each player failed to show for. All players are divided into eight buckets based on the ratio of no-shows to the total number of tournament matches played.

Titles can use this information for player profiles, matchmaking, unlocking content, trophies, and so on. Because the Disconnect and No-Show ratings are updated every time a player participates in a tournament or an arbitrated game session, they must not be persisted beyond the duration of a player session.

The NoShowRating and Disconnect Rating are returned by XOnlineLogon in the dwUserFlags field of the XUID structure.

Use the XOnlineUserDisconnectRating(dwUserFlags) macro to access the DisconnectRating and the XOnlineUserNoShowRating(dwUserFlags) macro to access the NoShowRating.

Use the following functions to extract additional user information from an XUID structure:

  • XOnlineAreUsersIdentical
  • XOnlineIsUserGuest
  • XOnlineIsUserNicknameAllowed
  • XOnlineIsUserPurchaseAllowed
  • XOnlineIsUserSharedContentAllowed
  • XOnlineIsUserVoiceAllowed
  • XOnlineUserCountryId
  • XOnlineUserGuestNumber

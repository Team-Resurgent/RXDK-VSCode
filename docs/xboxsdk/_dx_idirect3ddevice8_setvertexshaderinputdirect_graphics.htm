IDirect3DDevice8::SetVertexShaderInputDirect

Sets the byte offset from the start of the vertex buffer data. This function can work with push-buffer fix-ups. Unlike IDirect3DDevice8::SetVertexShaderInput, this function does not require a vertex shader handle.

HRESULT SetVertexShaderInputDirect(
    D3DVERTEXATTRIBUTEFORMAT *pVAF,
    UINT StreamCount,
    D3DSTREAM_INPUT *pStreamInputs
);

Parameters

pVAF
[in] Pointer to a D3DVERTEXATTRIBUTEFORMAT structure that is used to program the GPU's stream registers.
StreamCount
[in] Number of elements in the array pointed to by pStreamInputs.
pStreamInputs
[in] Pointer to an array of D3DSTREAM_INPUT structures used to specify vertex buffers as well as the offsets from the starts of those vertex buffers.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

Calling SetVertexShaderInputDirect does not set a vertex shader, it only sets the format that a vertex shader is expecting. The GPU is not set to use a vertex shader until SetVertexShader, SelectVertexShader, or SelectVertexShaderDirect is called.

When IDirect3DDevice8::SetVertexShaderInputDirect is called with a non-NULL pointer to a D3DVERTEXATTRIBUTEFORMAT structure, IDirect3DDevice8::SetVertexShaderInputDirect takes control of the GPU's stream registers until the function is called with a NULL pointer. This means that after calling SetVertexShaderInputDirect, any future calls to SelectVertexShader or SelectVertexShaderDirect will not change the vertex attribute format and calls to SetStreamSource will not set the stream inputs. After a call of SetVertexShaderInputDirect(NULL, 0, NULL), those functions will once again take effect.

This method doesn't support the notion of a nonzero IndexBase value.

The D3DTSS_TEXCOORDINDEX mapping for every texture stage must be identity.

Requirements

Header: Declared in D3d8.h.

Import Library: Use D3d8.lib.

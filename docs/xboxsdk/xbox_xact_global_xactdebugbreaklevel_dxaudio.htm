g_dwXactDebugBreakLevel

Determines whether a particular level of debug message will cause a break in the debugger.

DWORD g_dwXactDebugBreakLevel;

Remarks

XACT uses two global variables to control debug output: g_dwXactDebugLevel controls the amount of output displayed in the debugger and g_dwXactDebugBreakLevel determines whether a particular level of debug message will cause a break in the debugger.

Each time a DPF (debug printf) statement is made in XACT, a level is included. If g_dwXactDebugLevel is greater than or equal to this level, the statement appears in the debugger. If not, the DPF returns immediately.

If the debug level check succeeds and the statement is then sent to the debugger, the debug engine compares the level passed in with the DPF call to g_dwXactDebugBreakLevel. A debug break occurs if the following two conditions apply: g_dwXactDebugBreakLevel is greater than or equal to this level, and g_dwXactDebugBreakLevel is nonzero. This is useful for tracking down the source of a particular warning or error.

The debug break levels currently used are as follows:

Debug Break Level Description
0 Disregard level (no break)
1 Errors
2 Warnings
3 General information
4 Verbose

The default DPF level is 3 (general information) and the default debug break level is 1 (errors).

Requirements

Import Library: Use Xactengd.lib or Xactengid.lib.

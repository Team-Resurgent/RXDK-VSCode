OpenWaitableTimer

The OpenWaitableTimer function opens an existing named waitable timer object.

HANDLE OpenWaitableTimer(
  DWORD dwDesiredAccess,  // access
  BOOL bInheritHandle,    // inheritance option
  LPCSTR lpTimerName      // object name
);

Parameters

dwDesiredAccess
[in] Ignored.
bInheritHandle
[in] Ignored; use FALSE.
lpTimerName
[in] Pointer to a null-terminated string specifying the name of the timer object. Name comparison is case insensitive. The name can contain any character except the backslash character.

Return Values

If the function succeeds, the return value is a handle to the timer object.

If the function fails, the return value is NULL. To get extended error information, call GetLastError.

Remarks

The OpenWaitableTimer function enables multiple open handles to the same timer object. The function succeeds only if the timer has already been created with the CreateWaitableTimer function. The returned handle can be used in any function that requires the handle to a timer object, such as the wait functions.

The returned handle can be duplicated by using the DuplicateHandle function. Use the CloseHandle function to close the handle. The timer object is destroyed when its last handle has been closed.

Requirements

Header: Declared in Winbase.h, for include use Xtl.h.
Import Library: Use Xapilib.lib

Win32 Deltas

The dwDesiredAccess parameter is ignored. All handles have full access. You can pass TIMER_ALL_ACCESS to this parameter if desired.

The bInheritHandle parameter is ignored. Child processes cannot be created. There is nothing that could inherit the handle.

The lpName parameter is not limited to 255 characters. Name comparisons are case insensitive.

Implemented as ANSI only.

To make a call that will also work under Win32:

  • Specify TIMER_ALL_ACCESS for the dwDesiredAccess parameter.
  • For lpName, consider case sensitivity and use strings with less than 255 characters.

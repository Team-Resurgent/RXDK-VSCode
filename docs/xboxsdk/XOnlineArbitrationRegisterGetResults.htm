XOnlineArbitrationRegisterGetResults

Retrieves the results of the task begun by XOnlineArbitrationRegister.

HRESULT XOnlineArbitrationRegisterGetResults(
    XONLINETASK_HANDLE hTask,
    DWORD dwRegistrantsBufferCount,
    XONLINE_ARB_REGISTRANT *pRegistrants,
    DWORD *pdwNumRegistrants
);

Parameters

hTask
[in] Task handle obtained from XOnlineArbitrationRegister.
dwRegistrantsBufferCount
[in] Number of registrant entries in the array specified by the pRegistrants parameter.
pRegistrants
[in, out] Array of XONLINE_ARB_REGISTRANT structures. Each structure in the array contains information about a registrant (a player or a team) in a competition.
pdwNumRegistrants
[out] Actual number of registrants in the competition.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

This function returns the list of participants currently registered for an arbitrated round. The list includes only the participants who are registered at the time the XOnlineArbitrationRegister operation completed, in the order in which they registered.

Each list entry includes an identifier for the corresponding Xbox, a list of users on the Xbox, and a normalized value representing the perceived "reliability" of the Xbox. The least reliable participant in the list always has a reliability value of 0; all others have an arbitrary value at or above 0 based on their relative reliability.

Because the reliability value is relative and differs based on who is in the list of participants, titles are not expected to display the value to users. In fact, it is strongly discouraged. Rather, they may use it along with other information sources such as QoS results to determine which Xbox should perform round/game session host duties, and so forth.

Do not use this function for competitions. Instead, have your title call the XOnlineCompetitionSessionRegisterGetResults function.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

XOnlineStorageGetInstallLocation

Retrieves the path to the storage item file downloaded by XOnlineStorageDownload. 

HRESULT XOnlineStorageGetInstallLocation(
    DWORD dwFacility,
    LPCWSTR wszStoragePath,
    LPSTR szLocation,
    DWORD *pdwLocationSize
);

Parameters

dwFacility

[in] Indicates storage facility. May be one of the following:
Value Description
XONLINESTORAGE_FACILITY_STATS Game-clips (stats with attachments) facility.
XONLINESTORAGE_FACILITY_TEAMS Teams storage facility.
XONLINESTORAGE_FACILITY_PER_TITLE Title-managed global storage.
XONLINESTORAGE_FACILITY_PER_USER_TITLE Per title, per user storage.

For a complete list of facility values, see the XONLINESTORAGE_FACILITY page.

wszStoragePath

[in] The service's storage path for the storage item. For game clips, this path can be obtained by calling XOnlineStatRead. This path is stored in the lpString member of the XONLINE_STAT structure that has a wID member equal to XONLINE_STAT_ATTACHMENT_PATH. For all other facilities, this path can be constructed by using the XOnlineStorageCreateServerPath function.

szLocation

[out] Pointer to a buffer that receives the path to the downloaded attachment file. The size of this buffer should be XONLINESTORAGE_MAX_PATH.

pdwLocationSize

[in, out] Pointer to a DWORD variable.

On input, this variable specifies the size of the buffer pointed to by the szLocation parameter.

On output, this variable specifies the number of bytes written to the buffer including the terminating NULL character.

Return Values

If XOnlineStorageGetInstallLocation succeeds, the return value is S_OK.

If XOnlineStorageGetInstallLocation fails, the return value is one of the following error codes.

Value Description
XONLINE_E_STORAGE_INVALID_STORAGE_PATH Invalid storage path.
XONLINE_E_STORAGE_INVALID_FACILITY Invalid facility code.
HRESULT_FROM_WIN32(ERROR_INSUFFICIENT_BUFFER) The buffer pointed to by the szLocation parameter is not large enough to hold the path. The required length is returned in the DWORD pointed to by the pcbLocation parameter.
Other See Online Errors for information about other possible errors.

Remarks

If the buffer pointed to by the szLocation parameter is not large enough to hold the path, XOnlineStorageGetInstallLocation returns HRESULT_FROM_WIN32(ERROR_INSUFFICIENT_BUFFER), and the required length is returned in the DWORD pointed to by the pcbLocation parameter.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

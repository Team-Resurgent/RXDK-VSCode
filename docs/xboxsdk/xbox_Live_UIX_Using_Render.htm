UIX Render Loop

The render loop for most titles likely checks for input device changes, retrieves controller input, renders to the current render surface, and presents that surface on the screen. To use UIX, two additional steps need to be added to the render loop. The first is that the title's render loop must invoke ILiveEngine::SetInput. Second, the loop must call ILiveEngine::DoWork.

The ILiveEngine::SetInput method forwards controller input, in the form of an XINPUT_STATE structure, to the UIX engine. ILiveEngine::DoWork gives the engine an opportunity to perform the required processing such as calling Xbox Live by calling the appropriate functions and pumping asynchronous online task handles.

ILiveEngine::DoWork can return flags indicating that additional calls need to be made into the engine in order to perform subsequent tasks. The UIX_DOWORK_NEED_TO_RENDER flag indicates that render work must be done by the engine, and the title should call ILiveEngine::Render to render the UI to a Direct3D surface. The UIX_DOWORK_NEED_TO_REBOOT flag indicates that the engine needs to reboot, so the title should call ILiveEngine::Reboot. A reboot is typically required when the user needs to access the Xbox Dashboard (for account management, as an example) or when an auto-update must be performed. The UIX_DOWORK_FEATURE_EXIT flag indicates that an active feature has exited, and the title should call ILiveEngine::GetExitInfo to determine which feature exited, and why. The title can use the exit information to determine the next course of action. For example, after a successful logon by the logon feature, the title can continue with online game play.

ILiveEngine::DoWork also sets the UIX_DOWORK_FEATURE_EXIT when the network connection is lost or fails, even if the logon feature is not currently active and did not exit. In this case, the exit code returned by ILiveEngine::GetExitInfo will be UIX_EXIT_LOGON_FAILED.

If a title has requested that the Xbox Drop-in UI not automatically handle notifications (by setting the UIX_PROPERTY_DISPLAY_NOTIFICATIONS property to FALSE), the UIX_DOWORK_NOTIFICATIONS flag may be set. This flag indicates that the title must call ILiveEngine::GetNotifications to retrieve in-game user notifications and then must display those notifications appropriately. Normally, the engine handles detection and display of notifications automatically.

ILiveEngine::DoWork sets the UIX_DOWORK_PROCESSING_INPUT flag to indicate that the engine is processing player input. If this flag is set, controller input is going to an active feature and should not be processed by the title. Otherwise, the title should process controller input normally.

The title should perform its normal game rendering, and then perform Drop-in UI rendering (if indicated) to the same surface, and then display the resulting surface by updating the swap chain (using IDirect3DDevice8::Present or IDirect3DDevice8::Swap).

An example render loop, with the additional Xbox Drop-in UI calls, is shown here:

DWORD dwFlags, i;

// ... existing title code that renders to g_pSurface
// ... existing title code that detects input device changes

// Forward controller input
XINPUT_STATE InputState;
for( i = 0; i < XGetPortCount(); i++ ) {
    if( g_hInputDevice[i] ) {
        XInputGetState( g_hInputDevice[i], &InputState );
        pLiveEngine->SetInput( i, &InputState );
    } else {
        pLiveEngine->SetInput( i, NULL );
    }
}

// Call DoWork (and additional engine functions as indicated).
pLiveEngine->DoWork( &dwFlags );
if( dwFlags & UIX_DOWORK_NEED_TO_RENDER ) {
    pLiveEngine->Render( g_pSurface );
}
if( dwFlags & UIX_DOWORK_NEED_TO_REBOOT ) {
    pLiveEngine->Reboot();
}
if( dwFlags & UIX_DOWORK_FEATURE_EXIT ) {
    UIX_EXIT_INFO ExitInfo;
    pLiveEngine->GetExitInfo( &ExitInfo );
}

if( !(dwFlags & UIX_DOWORK_PROCESSING_INPUT) ) {
    // Process controller input normally.
}

// Present (or Swap)
g_pDevice->Present( NULL, NULL, NULL, NULL );

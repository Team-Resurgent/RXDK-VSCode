DmRegisterPerformanceCounter

Registers a performance counter on the Xbox console.

DmRegisterPerformanceCounter(
	LPCSTR szName, 
	DWORD  dwType, 
	PVOID  pv
	);

Parameters

szName
Name of the counter.
dwType
Type of counter, as defined in DM_COUNTINFO.
pv
This parameter is set as either a pointer to a function meeting the signature specified above, a pointer to a 32-bit value, a pointer to a 64-bit value, or a pointer to a DM_COUNTINFO structure.

Return Values

Returns XBDM_NOERR if successful, an error value otherwise.

Creating and Registering a Performance Counter

To write a performance counter, the developer first needs to make several decisions:

  1. Is the counter synchronous or asynchronous?
  2. Is the counter an event or value counter?
  3. If the counter is a value counter, is it an absolute counter or a percentage counter?

The answers to the questions determine how to write the resulting counter.

Synchronous Counters

A developer writing a synchronous counter must write a function with the following signature:

HRESULT (__stdcall *PDM_COUNTPROC)(PLARGE_INTEGER,PLARGE_INTEGER);

This function will then be called by DmQueryPerformanceCounter (in the case of value counters) or indirectly by our timing thread (in the case of event-based counters). Developers should write synchronous counters when they don't need to worry about capturing values at specific time intervals or in response to system events.

Asynchronous Counters

A developer writing an asynchronous counter supplies either a 32-bit address, 64-bit address, or an address to a DM_COUNTDATA structure, depending on the flags passed into DmRegisterPerformanceCounter. Developers should write asynchronous counters when they need to update their counter values at specific time intervals or in response to system events.

Event-based Versus Value-based

The decision of whether to author an event-based or value-based counter depends largely on what you are trying to measure. Any data that needs to be examined with respect to time should be an event-based counter. Counters that measure information at the instant they are called should be value-based counters.

Value-based Counters: Absolute Versus Percentage

An absolute counter returns only a raw value. Examples of value-based absolute counters are "Memory Available" or "Number of textures allocated".

A percentage counter returns a ratio between 0 and 1. An example of value-based percentage counter is "CPU usage".

Depending on which type of value counter you choose makes a difference in how you must return values in response to a DmQueryPerformanceCounter call. An absolute counter always returns a single 64-bit value for CountType; RateValue is undefined. A percentage counter always returns two 64-bit values, but only uses the bottom 32-bits of each.

Count Providers

When registering a performance counter, one of the following types must be specified:

  • DMCOUNT_ASYNC—pv points to a DM_COUNTDATA structure that will be returned at query time. The data provider is responsible for keeping the count data up-to-date.
  • DMCOUNT_SYNC—pv points to a PDM_COUNTPROC function, which for a PRATIO counter returns its numerator value in the first LARGE_INTEGER and its demoninator value in the second LARGE_INTEGER, and for all other types returns its data in the first LARGE_INTEGER.
  • DMCOUNT_ASYNC32—pv points to a DWORD which represents the value of the counter; for a PRATIO counter, pv points to an array of two DWORDs.
  • DMCOUNT_ASYNC64—pv points to a LARGE_INTEGER which represents the value of the counter; for a PRATIO counter, pv points to an array of two LARGE_INTEGERs.

For counters other than DMCOUNT_ASYNC, the low four bits specify the frequency, in 100-millisecond increments (from 100ms to 1sec) that the counter will be periodically queried to establish rate or progressive information. If the frequency portion is zero, the counter will not be periodically queried, and rate or progressive information will be unavailable.

Win32 Performance Monitor Example

#include "windows.h"
#include "xboxdbg.h"
#include <conio.h>

int main()
	{
	PDM_WALK_COUNTERS pdmwc;
	DM_COUNTINFO dmci;
	DM_COUNTDATA dmcd;

	// An example of how to enumerate the available counters
	pdmwc = NULL;
	while (DmWalkPerformanceCounters(&pdmwc, &dmci) != XBDM_ENDOFLIST)
		{
		printf("Counter '%s', Type 0x%04x\n", dmci.Name, dmci.Type);
		}
	DmCloseCounters(pdmwc);

	// An example of how to display the value of an event counter //
	while (!_kbhit())
		{
		DmQueryPerformanceCounter("Frames", DMCOUNT_EVENT |
                                                   DMCOUNT_FREQ1SEC |
                                                   DMCOUNT_PERSEC, &dmcd);

		printf("%g fps\n", (double)dmcd.CountValue.QuadPart /
                                  (double)dmcd.RateValue.QuadPart);

		Sleep(1000);
		}

	return 0;
	}

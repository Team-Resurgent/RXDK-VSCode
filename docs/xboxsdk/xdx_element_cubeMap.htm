cubeMap Element

Describes a cube map resource.

<cubeMap
   id = string
   filter = string
   format = string
   levels = integer
   size = integer
/>

Required Attributes

id

Texture identifier used for referencing from <textureState> elements. For more details, see XDX id Attribute.

Optional Attributes

filter

Specifies the filter used to resize the image.

If this value is specified, it must contain one of the following flags:

Value Description
D3DX_FILTER_NONE No scaling or filtering will take place. Pixels outside the bounds of the source image are assumed to be transparent black.
D3DX_FILTER_POINT Each destination pixel is computed by sampling the nearest pixel from the source image.
D3DX_FILTER_LINEAR Each destination pixel is computed by sampling the four nearest pixels from the source image. This filter works best when the scale on both axes is less than two.
D3DX_FILTER_TRIANGLE Every pixel in the source image contributes equally to the destination image. This is the slowest of the filters. This is the default filter if nothing is specified.
D3DX_FILTER_BOX Each pixel is computed by averaging a 2×2(×2) box of pixels from the source image. This filter works only when the dimensions of the destination are half those of the source, as is the case with mipmaps.

In addition, zero or more of the following flags may be ORed in with one of the flags above.

Value Description
D3DX_FILTER_WRAP Pixels will be wrapped, not mirrored. This is the default behavior.
D3DX_FILTER_MIRROR Specifying this flag is the same as specifying the D3DX_FILTER_MIRROR_U, D3DX_FILTER_MIRROR_V, and D3DX_FILTER_MIRROR_W flags.
D3DX_FILTER_MIRROR_U Pixels off the edge of the texture on the u-axis should be mirrored, not wrapped.
D3DX_FILTER_MIRROR_V Pixels off the edge of the texture on the v-axis should be mirrored, not wrapped.
D3DX_FILTER_MIRROR_W Pixels off the edge of the texture on the w-axis should be mirrored, not wrapped.
D3DX_FILTER_DITHER This flag indicates that the resulting image be dithered using a 4x4 ordered dither algorithm.
format

Format for the output texture. Must be a member of the D3DFORMAT enumeration; however, it cannot be a linear format or a depth/stencil format. If the output format is not specified, it will default to D3DFMT_A8R8G8B8.

levels

Number of mipmap levels to automatically generate. Must be between 0 and 13. To generate mipmaps all the way down to 1×1, use 0. This attribute should not be specified if the <texture> tag contains <mip> elements. For RGB textures, mipmaps are generated using a triangular filter with dithering. For UVL textures, mipmaps are generated using a point filter with dithering. The filtering used to generate the mipmaps cannot be changed using the XDX file.

size

The width and height in texels of each face. The size must be a power of two between 1 and 4096. If this attribute is not specified, size will be the largest width or height of the included faces. If the size specified or calculated is not a power of two, or does not equal the source image size, the image will be scaled (using a trianglular filter with no dithering) to the next largest power of two.

Element Information

Parent Elements XDX
Child Elements cubeFace

Remarks

A cube map texture consists of six textures, one for each face of the cube.

Examples

Specifying the cube faces

It is not necessary to specify every cube face. This example skips the negative z-face.

<cubeMap id="cubeMap1" size="256" format="D3DFMT_DXT4" >
   <cubeFace face="XP" source="textures/XP1.bmp" /> 
   <cubeFace face="XN" source="textures/XN1.bmp" />
   <cubeFace face="YP" source="textures/YP1.bmp" /> 
   <cubeFace face="YN" source="textures/YN1.bmp" />
   <cubeFace face="ZP" source="textures/ZP1.bmp" /> 
</cubeMap>

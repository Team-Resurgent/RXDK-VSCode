IDirect3DFixup8

Applications use the methods of the IDirect3DFixup8 interface to handle push-buffer resources.

The methods of the IDirect3DFixup8 interface can be organized into the following groups:

Information GetSize
  GetSpace
Management Reset

The IDirect3DFixup8 interface inherits the following IDirect3DResource8 methods, which can be organized into these groups.

Devices GetDevice
Information GetType
Private Surface Data FreePrivateData
Miscellaneous Register
  BlockUntilNotBusy
  IsBusy
  MoveResourceMemory

  AddRef
  Release

The IDirect3DFixup8 interface also has the following public data members:

DWORD Common;
DWORD Data;
DWORD Lock;
DWORD Run;

When IDirect3DDevice8::RunPushBuffer is called with a fix-up object specified, it will parse the fix-up data pointed to by Data and with a byte offset of Run.

The fix-up data is encoded as follows. The first DWORD is the size, in bytes, of the push-buffer fix-up to be modified. The second DWORD is the offset, in bytes, from the start of the push-buffer where the fix-up is to be modified. The subsequent DWORDS are the data to be copied. This encoding repeats for every fix-up to be done, until it terminates with a size value of 0xffffffff.

The offsets must be in an increasing order.

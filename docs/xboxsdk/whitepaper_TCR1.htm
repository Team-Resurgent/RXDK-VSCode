Xbox TCR Resources and Recommendations

By Pete Isensee, Development Lead, Xbox Advanced Technology Group

The Technical Certification Requirements (TCRs) for Xbox® games are designed to ensure that games meet a high quality and consistency level. One of the goals of the Xbox Game Quality team is to decrease the number of times that a game goes through the compliance certification phase. Reducing failure rates leads to faster game development turnaround time, which saves publishers and developers time and money.

This document lists resources and tools that development and QA test teams can use to more effectively pass Xbox certification. It also includes a list of general recommendations that Xbox developers have found useful in planning for and dealing with technical certification.

Resource List

Many resources are available to help developers understand Technical Certification Requirements. These resources include the following:

  • The Xbox Guide. The definitive resource for all Xbox TCRs. The Guide is available in the Certification section of this Web site.
  • Xbox Developer Support (xboxds@xbox.com). Developer support regularly fields certification issues. This email alias should be your primary point of contact if you have a certification question.
  • Xbox Game Quality Team (xboxcert@xbox.com). The game quality team can answer questions specific to certification. However, they do not answer general technical support issues. When in doubt, prefer the Developer Support email alias (xboxds@xbox.com).
  • xds.certification newsgroup. The certification newsgroup allows you to post specific questions about certification requirements to your peers. See the Xbox Newsgroups section to find out how to access the newsgroups. The newsgroups are primarily designed as a place where external developers and testers can share information with each other. The newsgroups are not an official support mechanism, and the newsgroups are not moderated. Contact Xbox Developer Support for urgent issues and for issues that require a guaranteed reply.
  • White papers. The main white paper index on the Xbox Central Web site contains multiple documents related to certification. These documents are updated on a regular basis. To find all related documents, sort by subject and view the certification section.
  • Technical Certification Game. A mini game with full source code is included in the XDK at Samples\Xbox\Certification\TechCertGame. This game meets all TCRs and explicitly documents where it meets specific requirements in the source code by using the commenting convention "// TCR x-y", where x-y is the requirement number.
  • Pre-Certification Report. After you submit your game for pre-certification, you will receive a report that includes a list of TCRs that your game is currently failing. This is a great time to focus on eliminating those issues so that they don't come back to haunt you when you'd much rather be shipping. If you have questions about the Pre-Certification report, contact xboxds@xbox.com.
  • Account Manager and Technical Game Manager. Your Xbox account manager (AM) and technical game manager (TGM) are familiar with certification issues and can be valuable resources.

Tools List

Many tools are available to help developers and testers verify compliance. All of these tools are used by the Game Quality team when they evaluate your title during certification. Additional information about these tools is included in the XDK Help documentation. These tools include the following:

  • XDK Launcher certification tools. To access these tools, press X from the XDK Launcher main menu on the Xbox Development or Debug Kit (XDK). The XDK Launcher certification tools, Live tools and memory unit tools allow you to do dozens of useful things like fill up the hard disk, fragment the disk and display the screen's safe area. These tools are available only on Development and Debug kits recovered in debug mode. The XDK Launcher is not available on kits recovered in retail mode.
  • XbCp. One of the most useful Xbox tools is also one of the simplest. XbCp allows you to copy files to and from an XDK console and PC. It has switches that allow you to recursively copy entire folders, or only copy files that have changed. XbCp is a command line tool that can be integrated into via batch files to perform all sorts of copy requests.
  • Xbox Neighborhood. The Xbox Neighborhood (also known as the Microsoft Windows Explorer Shell Extension for Xbox) is a GUI tool for easily accessing the Xbox hard disk from a PC. It provides a view of the Xbox hard disk much like the standard Windows Explorer. For example, a tester could use the Xbox Neighborhood to copy files from an Xbox to a PC in preparation for running the XbSGCorrupt tool.
  • ImageBld. ImageBld is used to create the game executable(s). Normally it runs automatically when the game is built within Visual Studio. However, you can also use ImageBld from the command line to see what Xbox Title Library (XTL) versions, whether they're approved for submission, as well as what XTL functions are being linked with the game. Run "ImageBld /dump default.xbe" to check all XBEs before final submission. If your game is segmented into multiple XBEs, be sure to check each XBE, not just the default. You should also verify any redistributable XBEs such as downloader.xbe.
  • XbWatson. Testers should run XbWatson on their PC while they test the game on Xbox. If the game crashes, XbWatson is invaluable in helping developers track down the source of the crash.
  • VerifierX Analyzer (VXA). When a game is executed on an Xbox development kit, it generates a log file reporting potential certification errors. These errors are also sent to the Visual Studio debugger output window. The VXA tool extracts this log and generates output that shows what certification requirements were tested and what the results were for each test. You can also report results on specific requirements. This allows developers and testers to quickly see any potential failure cases. VXA can't test all requirements. For instance, it can't verify UI. However, it can test requirements that are impossible to test any other way, including many of the system requirements in section C2 of the Xbox Guide.
  • Xbox Game Disc Authoring Tool (XbGameDisc). XbGameDisc is the tool used to author and layout the final Xbox DVD media. This tool also has some powerful features designed for QA teams. Use this tool to emulate the speed of final media, so you can better gauge actual load times and file seek times. You can also use the tool to inject read errors for particular files or groups of files. This allows you to simulate the effect of flawed DVD media, fingerprints and scratches and see how your title responds to those errors.
  • NetSim. The Network Simulator tool (NetSim) is designed to emulate Internet conditions that affect all online games, including Xbox-specific features like key exchange delay. It allows developers and testers to configure latency, packet loss and bandwidth conditions. It also allows users to load configuration files that match specific TCRs regarding bandwidth and latency requirements.
  • NetMon. The Network Monitor tool (NetSim) allows developers and testers to capture and analyze Xbox network traffic. The XDK includes Xbox extensions that properly interpret and display Xbox network packets. NetMon can be used to evaluate patterns of network traffic and determine game packet frequency, packet size, and packet overhead. NetMon is available in the Tools section for download; it's not included as a part of the XDK.
  • Xbox File Event Viewer. This is a component available in the Visual Studio .NET environment, selectable from the Tools menu. It tracks information about low-level file access on Xbox, much like the FileMon utility for the PC. Developers and testers can use this tool to determine what files are being accessed (and when) by the game.
  • XbSetCfg. This tool allows developers and testers to remotely configure the video, audio, language, time and other settings on an Xbox development kit. This can be useful when testing how the game works using other Dashboard language, video and audio parameters. This tool also allows the default XBE to be toggled between the Dashboard and the XDK Launcher, which is useful when testing that the game is launching the Dashboard properly.
  • Xbox File Corruption Tool (XbSGCorrupt). One of the prime points of entry for Xbox hackers is the hard disk and Xbox memory units. Therefore, there are a number of certification requirements specific to validating data from the hard disk and MUs. The XbSGCorrupt tool allows developers and testers to generate damaged versions of save games and other content files. Don't let the name fool you—XbSGCorrupt can corrupt any files, not just saved games. For instance, you can use XbSGCorrupt to damage downloaded content.

General Recommendations

  • Read the Xbox Guide. The Guide should be required reading for all members of your team-developers, testers, and producers.
  • Ensure your game design reflects the Technical Certification Requirements. Update the game design document to account for required UI and other required game elements.
  • If you're porting from another console, schedule time to redesign sections of UI and code to accommodate Xbox requirements regarding controllers, memory units and the hard disk.
  • Do not rely on shipped Xbox titles as a TCR resource. The Xbox Guide has changed over time, and the requirements for current titles may be different than for games that have already shipped.
  • Integrate TCR testing into your normal testing procedures. The Xbox Guide lists the test cases that your QA team can use to emulate the TCR tests that the Xbox Certification team will perform. The tools listed above can be used to automate significant portions of this process-make sure your QA team is aware of these tools.
  • Encourage your QA team to use XbWatson during testing so that the game developers can get the information they need to resolve crash bugs.
  • Read the Pre-Certification Compliance Report. This report should be required reading for all members of your team, especially developers and testers. If you don't act on the failure cases from this report, you're just delaying the inevitable, because those same failures will show up during final certification. Take the time to fix the issues early.
  • If you have questions about any TCR, contact Developer Support (xboxds@xbox.com). Feel free to loop in your Account Manager (AM) and Technical Game Manager (TGM). Although the TCRs are intended to be clear and unambiguous, there are cases where interpreting a requirement in light of your game can be complex. Don't wait until the week before shipping to get your TCR questions answered; get those questions answered early.
  • We encourage your questions and feedback. The Xbox team regularly reviews the TCRs to clarify intent, and your feedback is part of that process. Send feedback to xboxds@xbox.com.

For recommendations regarding specific TCRs, see the Xbox Core TCR Top Issues white paper.

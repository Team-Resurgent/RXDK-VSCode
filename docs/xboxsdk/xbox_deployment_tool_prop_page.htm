Xbox Deployment Tool

The Xbox Deployment Tool copies files to the Xbox. It is intended to be used from within the Visual Studio .NET IDE. The its configuration options are set through property pages from within the Visual Studio IDE.

The General Property Page

The General property page can be found by first loading an Xbox project file in the Visual Studio IDE. Next, right-click the name of the project in the Solution Explorer window and choose Properties in the menu that appears. In the dialog box that appears, click the Xbox Deployment folder, and then click General.

The General property page contains the following properties:

Excluded From Build
Specifies whether the Xbox Deployment Tool is excluded from the build for the current configuration.
Remote Path
Specifies the full path (on the Xbox) where the XBE will be copied to. This also determines the values of the RemotePath macros:
MacroDescription
$(RemoteDir) Directory (on the Xbox) where the primary output file from the build is copied to. This is defined as the drive and path and includes the trailing backslash '\'.
$(RemoteName) Base name (on the Xbox) of the primary output file from the build. This does not include the file extension.
$(RemotePath) Absolute path name (on the Xbox) where the primary output file from the build is copied to. This is defined as drive, path, base name, and file extension.
$(RemoteFileName) The file name (on the Xbox) of the primary output file for the build. This is defined as base name and file extension.
$(RemoteExt) The file extension (on the Xbox) of the primary output file for the build.

These macros can be used anywhere in an Xbox project's property pages dialog box where strings are accepted. These macros are not case sensitive.

Additional Files
Semicolon-separated list of additional files and directories to be deployed. Each item in the list may be in one of the following two formats:
  • A path, either relative or absolute, to a file or directory. In this situation, the path's status as relative or absolute, and the path's location in relation to the project directory, determine exactly what is copied and where it is copied to, according to the following rules:
    • If the path is a relative path, the source is relative to the project directory on the personal computer, and the destination is relative to the remote directory on the Xbox console. For example, if the project directory is C:\MySources\Game and the remote directory is XE:\Samples\Game, specifying a path of Media\File copies the file C:\MySources\Game\Media\File to XE:\Samples\Game\Media\File.
    • If the path is an absolute path and is a child of the project directory, then the source is the path specified, and the destination is calculated by extracting the child part of the path and appending it to the remote directory. For example, if the project directory is C:\MySources\Game and the remote directory is XE:\Samples\Game, specifying a path of C:\MySources\Game\Media\File2 copies the file C:\MySources\Game\Media\File2 to XE:\Samples\Game\Media\File2.
    • If the path is an absolute path but is not a child of the project directory, then the source is the path, and the destination is calculated by appending the last element of the path to the remote directory. For example, if the project directory is C:\MySources\Game and the remote directory is XE:\Samples\Game, specifying a path of C:\MySources\SharedMedia\File3 copies the file C:\Mysources\SharedMedia\File3 to XE:\Samples\Game\File3.
    In all of these cases, if the path specifies a directory name, the deployment tool recursively copies the directory and all its contents. Also, if the destination is a subdirectory whose parent directories don't exist, the deployment tool creates the parent directories. For example, if a file were to be copied to XE:\Samples\Game, and the XE:\Samples directory did not already exist, XE:\Samples would be created.
  • A dest=source pair, where the text to the left of the equal sign is the path to the destination file or directory, and the text to the right of the equal sign is the path to the source file or directory. If "dest" is a relative path, it is relative to the remote directory. If "source" is a relative path, it is relative to the project directory. For example, if the project directory is C:\MySources\Game and the remote directory is XE:\Samples\Game, the following are true:
    • A path of Media=testMedia copies all the files in C:\MySources\Game\TestMedia to XE:\Samples\Game\Media.
    • A path of Media=C:\Backup\OldMedia copies all the files in C:\Backup\OldMedia to XE:\Samples\Game\Media.
    • A path of XE:\Media=TestMedia copies all the files in C:\MySources\Game\TestMedia to XE:\Media.
    • A path of XE:\Media=C:\Backup\OldMedia copies all the files in C:\Backup\OldMedia to XE:\Media.
Suppress Startup Banner
Setting this to Yes will suppress the display of the startup banner and information messages.
Progress
Setting this to Yes displays status information about the progress of the deployment.
Force Copy
Setting this to Yes forces the deployment tool to always copy files, even if the destination file size and date match the source file.

Setting the Destination Xbox

Use the Remote Machine property to set the destination Xbox to which the Xbox Deployment tool copies the project after it is built. You can specify more than one destination Xbox by adding the names of additional Xbox consoles to the Remote Machine property. The names must be separated by semicolons. The Xbox Deployment tool copies the project to each destination Xbox in the order specified by the Remote Machine property. Note that the debugger uses the first destination Xbox in the list.

Alternatively, you can specify the destination Xbox by using the Xbox Neighborhood tool. This requires that you install the Microsoft Windows Explorer Shell Extensions for Xbox.

Using the XBECOPY_SUPPRESS_COPY Environment Variable

The XBECOPY_SUPPRESS_COPY environment variable is a feature added specifically for supporting nightly builds. If, for instance, you configure a computer to perform automated unattended nightly builds, the target Xbox systems may be turned off. Without setting this environment variable, the Xbox Deployment tool would return an error, causing the rest of the build process to stop. If the command interpreter environment variable XBECOPY_SUPPRESS_COPY exists and is set to a nonzero value, the Xbox Deployment tool simply prints out a warning message and exits immediately, allowing the build to continue.

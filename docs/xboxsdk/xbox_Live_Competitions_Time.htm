Time Advancement

Developing competitions presents unique challenges in dealing with the passage of time. In a single competition, for example, completing a series of rounds might take weeks, if not months, depending on how the competition is configured. Because it isn't practical to wait weeks for an automatic event such as a round progression, developers need some way to adjust the dates associated with a competition. Time advancement enables developers to "jump forward" in time with respect to an instance of a competition. By jumping forward, developers can trigger events or avoid waiting for specific windows of time (for example, registration open/close).

To adjust the time on a competition, call the XOnlineQuerySelect function with the following parameters.

Parameter Setting
dwPlayerIndex 0
dwTeamId 0
dwDatasetId Competition Template ID (the same value used for dwTemplate in the original call to XOnlineCompetitionCreate/XOnlineCompetitionCreateSingleElimination)
entityID The qwCompetitionID from XOnlineCompetitionCreateGetResults
dwAction XONLINE_COMP_ACTION_DEBUG_ADVANCE_TIME
dwNumAttributes 1
pAttributes Pointer to an array of XONLINE_ATTRIBUTE structures, with a single element. For the dwAttributeID member of this structure, specify XONLINE_COMP_ATTR_DEBUG_ADVANCE_TIME. The qwValue member is used to specify the time, in seconds, you wish to advance.

Call XOnlineTaskContinue to complete the task. After it's finished, the timing of various events associated with the competition will be adjusted by the specified time. This method simulates the advancement of time by moving any time values used on the service back by the specified time delta. For example, specifying a five-minute advancement would modify the competition to appear as though it had been created five minutes earlier than it was. This method also adjusts any scheduled events—for example, slotting or round progression. However, because these events are not instantaneous and run asynchronously, it is unlikely they will have completed immediately after the time adjustment. To enable these events to complete, a wait of at least one minute after the time adjustment or the competition start time (whichever occurred most recently) is recommended.

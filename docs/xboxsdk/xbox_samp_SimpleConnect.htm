SimpleConnect Sample

The SimpleConnect sample demonstrates how to form a security exchange manually rather than on the first sent packet. This allows much better control over the delays that NAT traversal and security exchange introduce. It also demonstrates how to attempt to recover when a security exchange is lost.

Overview

The first instance of SimpleConnect you run creates a 'host' and loops indefinitely. The second instance is the one you want to run with a debugger or XbWatson to see the output messages.

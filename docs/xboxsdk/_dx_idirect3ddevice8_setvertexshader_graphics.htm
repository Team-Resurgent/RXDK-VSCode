IDirect3DDevice8::SetVertexShader

Sets the current vertex shader to either a previously created vertex shader or to a flexible vertex format (FVF) fixed function shader.

HRESULT SetVertexShader(
  DWORD Handle
);

Parameters

Handle
[in] Handle to a vertex shader, specifying the vertex shader to set. The value for this parameter can be the handle returned by IDirect3DDevice8::CreateVertexShader, or an FVF code. An FVF code is a combination of flexible vertex format flags.

Return Values

If the method succeeds, the return value is D3D_OK.

Remarks

The effect of using an FVF code in place of the handle is to enable a fixed-function vertex shader, with an implicit declaration that matches the FVF code contents read from stream zero. Only stream zero is referenced when an FVF-specified shader is bound to the device.

In general, a subsequent call to IDirect3DDevice8::SetVertexShader will invalidate any vertex shaders loaded by IDirect3DDevice8::LoadVertexShader, and require them to be reloaded. However, if SetVertexShader is called using D3DFVF_XYZRHW, only instruction slots 0 through D3DVS_XBOX_RESERVEDXYZRHWSLOTS – 1 will be destroyed. This means that if a title is using programmable vertex shaders in conjunction with fixed-function vertex shaders, it is recommended that the first programmable vertex shader should be loaded, beginning at address D3DVS_XBOX_RESERVEDXYZRHWSLOTS, and each subsequent programmable shader should be added at its end.

Through careful experimentation, it is possible to discover which instruction slots are invalidated by the fixed function pipeline. With this information, you can then load vertex shaders into addresses that will not be invalidated through the use of the fixed-function pipeline.

A fixed-function vertex shader cannot be specified if the current constant mode is D3DSCM_192CONSTANTS as set by IDirect3DDevice8::SetShaderConstantMode.

Before a vertex shader that has been set on the device may be deleted and its associated resources freed, either a new vertex shader must be set, or zero must be passed in as the handle to SetVertexShader. The following code demonstrates how an active vertex shader may be deleted. (The code assumes that pDevice is a valid pointer to an IDirect3DDevice8 interface and that dwVertexShader is a valid handle to a vertex shader.)

pDevice->SetVertexShader( dwVertexShader );

// Use the vertex shader to render the scene.
// ...

// Delete the vertex shader.
pDevice->SetVertexShader( 0 );
pDevice->DeleteVertexShader( dwVertexShader );

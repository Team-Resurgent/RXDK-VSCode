Debugging your DirectMusic Code

To help you debug your title, the DirectSound title library provides the following functions:

  • DirectMusicMemCheck
  • DirectMusicMemDump
  • DirectMusicSetDebugLevel

These functions allow you to control the trace message levels that are written to the debug output, trace message levels that will cause a break in the debugger, and help you track down memory leaks.

Controlling Trace Messages

The DirectMusic title library provides you with the DirectMusicSetDebugLevel, which allows you to control the trace message levels that are displayed in the debug output and the trace message levels that cause a break in the debugger.

Checking for Memory Leaks

Forgetting to release a DirectMusic object is a common mistake that can cause memory leaks in your title. The DirectMusic title library provides two functions, DirectMusicMemCheck and DirectMusicMemDump, to help you track the DirectMusic objects being used by your title.

DirectMusicMemDump can be used to write out a list of all the objects currently allocated to the debug output.

To do comparisons or more selective checking, use the DirectMusicMemCheck. DirectMusicMemCheck allows you to specify an object identifier and retrieve the current number of items that are currently allocated of that type. Currently, the identifiers are subject to change, so applications should enumerate the objects by calling DirectMusicMemCheck until it returns –1.

The following example demonstrates a possible usage of the DirectMusicMemCheck function, where MyLog is a made-up function that can be replaced with your own code to track the object count.

LPSTR szObjectName;
DWORD dwObjectID = 0;
long nCount = DirectMusicMemCheck(dwObjectID,&szObjectName);

while(nCount!=-1)
{
    MyLog(dwObjectID, szObjectName, nCount);
    dwObjectID++;
    nCount = DirectMusicMemCheck(dwObjectID,&szObjectName);
}

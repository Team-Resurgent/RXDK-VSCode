XInputSetState

The XInputSetState function sends data to a connected controller peripheral. It is used, for example, to send data to a rumble pack or to other potential future controllers that require data or control from the console.

This function is the counterpart of XInputGetState.

Note   This function is always asynchronous. Do not use a local function (stack) variable for the pFeedback argument to this function. See the Remarks section below for more information.

DWORD XInputSetState(
  HANDLE hDevice,                  
  PXINPUT_FEEDBACK pFeedback     
);

Parameters

hDevice
[in] Handle to the specified device.
pFeedback
[in, out] Pointer to an XINPUT_FEEDBACK structure containing the data to send to the specified device. Once the initial call of XInputSetState has been completed, titles can poll dwStatus in the XINPUT_FEEDBACK_HEADER structure to monitor when the feedback action has been completed.

Return Values

If the function succeeds, it will return ERROR_IO_PENDING to signify that the action is in progress. When the action has been completed, the value of dwStatus will change to ERROR_SUCCESS or a failure code.

If the function fails, it will return a valid Win32 error code.

Remarks

Like XInputGetState, the Feedback parameter is a union of all possible feedback types.

Titles have two options for synchronizing to the completion of XInputSetState:

  1. A routine can poll dwStatus in the feedback header. Once the value changes from ERROR_IO_PENDING, the routine has finished with the requested action.
  2. If hEvent in the XINPUT_FEEDBACK_HEADER structure is a valid event handle, it will be set when the I/O completes.

The memory and data in the structure pointed to by pFeedback must be preserved by the title until the system has indicated (through one of the two methods given above) that the output request has been completed. Do not write new data into the structure, free the structure, or allow it to go out of scope until that time. For example, do not use a local stack variable for the XINPUT_FEEDBACK structure, as it can go out of scope and be overwritten before the asynchronous operation on the data is complete. Use a global variable or dynamically allocated memory instead.

XInputSetState should not be called again for the same device until the previous set state request has completed.

Requirements

Header: Declared in Xbox.h, for include use Xtl.h.
Import Library: Use Xapilib.lib

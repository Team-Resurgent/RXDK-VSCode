Stats and Xbox Live

The stats API exposes the functionality of the Xbox Live stats service to your game. Game titles use the stats service to store and retrieve statistical information about players of that game title. The type of statistical information stored depends on the nature of the title and is defined by the title developer. Stats collected by the title over a series of game sessions can be used to track the player's performance on the title over time. For example, the title might track the player's top five scores and the dates they were obtained, or the player's 10-day, 30-day, and 90-day moving averages. Titles can use this information to help in matching players of similar experience and abilities. Alternatively, they can display players' states to help give players "bragging rights" within the community that you build around your title.

The Xbox Live service stores all of the statistical information on network servers. All stats are accessible to instances of the game title running on any Xbox console on the network. Therefore, the player's stats remain with the player regardless of which Xbox console he or she is using. Unlike traditional rankings in which a game title can only compare players that have played on the same console, the stats service is able to rank players who have played the title on any network-enabled Xbox console. This enables the service to compare a given player's performance to that of all other players of your title.

Different titles require different types of statistical information. Game titles are allowed to collect and store whatever stats are appropriate for their game design. The stats service supports storing any number of different statistical values per player. The title developer decides which stats to collect. Collected stats are displayed in leaderboards. Although the stats service can collect any number of statistical values per player, a maximum of 64 of values can be displayed in a leaderboard at one time.

Note     Xbox Live titles should consider how they will protect against guests being used to artificially improve a player's measured performance. For example, in a deathmatch-style game, guests could be used to improve a player's stats by pre-arranging kills with the guest. Titles can protect against this type of cheating by adjusting their rules when guests are present.

Types of Stats

Statistical values must be numeric and belong to one of three possible data types. These types correspond to the values of the XONLINE_STAT_TYPE enumeration type:

Integer; 32-bits XONLINE_STAT_LONG
Big integer; 64-bits XONLINE_STAT_LONGLONG
Real number; 64-bits XONLINE_STAT_DOUBLE
Variable-length string XONLINE_STAT_LPCWSTR

Stats Defined by the Stats Service

In addition to the stats that the title defines, there are eight stats defined by the stats service itself.

Identifier Description Type
XONLINE_STAT_RANK Rank stat. Ranks have a type of XONLINE_STAT_LONG (32 bits).
XONLINE_STAT_RATING Rating stat. Ratings have a type of XONLINE_STAT_LONGLONG (64 bits).
XONLINE_STAT_NICKNAME Player nickname. Titles should not use this stat. Additionally, the XOnlineStatWriteEx function prevents this stat from being specified. Nicknames have a type of XONLINE_STAT_LPCWSTR (variable-length string).
XONLINE_STAT_LEADERBOARD_SIZE Number of entries on the leaderboard. The leaderboard size has a type of XONLINE_STAT_LONG (32 bits).
XONLINE_STAT_ATTACHMENT_PATH Game clip path. Attachment paths have a type of XONLINE_STAT_LPCWSTR (variable-length string).
XONLINE_STAT_ATTACHMENT_SIZE Size of game clip. The attachment size has a type of XONLINE_STAT_LONG (32 bits).
XONLINE_STAT_UNIT_ACTIVITY_COUNTER Not supported.
XONLINE_STAT_UNIT_LAST_ACTIVITY_DATE Not supported.

In the same way that a set of stats for a player is always associated with a particular leaderboard, the title-specific nickname for a player is also associated with a particular leaderboard. Because the nickname is handled as a stat, the nickname for a player on a leaderboard could change each time the player's stats are updated. Additionally, it is possible for a given player to be known by different nicknames on other leaderboards.

The Stats API

The following table summarizes the functions in the stats API:

Function Description
XOnlineStatWrite Stores stats for one or more players on specified leaderboards.
XOnlineStatWriteEx Sets statistical information for one or more specified players or Xbox Live teams.
XOnlineStatWriteGetResult Retrieves a handle to a remote file reference. Use this handle to upload file attachments to accompany player statistics written with the XOnlineStatWrite function.
XOnlineStatReset Resets (deletes) stats for a player on the specified leaderboard.
XOnlineStatRead Creates an asynchronous task to retrieve stats for one or more players from specified leaderboards.
XOnlineStatReadGetResult Obtains the results of the asynchronous task created by XOnlineStatRead.
XOnlineStatLeaderEnumerate Creates an asynchronous task to retrieve an ordered listing of players from a specified leaderboard.
XOnlineStatLeaderEnumerateGetResults Obtains the results of the asynchronous task created by XOnlineStatLeaderEnumerate.
XOnlineStatUnitEnumerate Not supported.
XOnlineStatUnitEnumerateGetResults Not supported.
XOnlineStatUnitRead Not supported.
XOnlineStatUnitReadGetResult Not supported.

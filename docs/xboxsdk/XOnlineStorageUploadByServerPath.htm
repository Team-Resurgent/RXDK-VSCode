XOnlineStorageUploadByServerPath

Uploads the contents of the specified directory to the specified storage facility as a single storage item.

HRESULT XOnlineStorageUploadByServerPath(
    DWORD dwFacility,
    DWORD dwUserIndex,
    LPCWSTR wszStorageFileName,
    FILETIME ftServerExpirationDate,
    LPCSTR szDirectory,
    DWORD dwUploadFlags,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwFacility

[in] Indicates storage facility. May be one of the following:
Value Description
XONLINESTORAGE_FACILITY_TEAMS Teams storage facility.
XONLINESTORAGE_FACILITY_PER_USER_TITLE Per title, per user storage.

For a complete list of facility values, see the XONLINESTORAGE_FACILITY page.

dwUserIndex
[in] Controller index (zero-based) of the player for whom to download the attachment.
wszStorageFileName
[in] File to be uploaded. This can be created using the XOnlineStorageCreateServerPath function.
ftServerExpirationDate
[in] File expiration date on the server. A value of zero means the file never expires.

Note    This must be in the UTC format.

szDirectory
[in] All contents under the specified directory will be packaged and uploaded. Subdirectories are included.
dwUploadFlags
[in] Flags that qualify the upload process. This value must be zero.
hWorkEvent
[in] Handle to an event that is signaled when there is work to do on the asynchronous task returned by the phTask parameter. This parameter can be NULL if such synchronization is not required.
phTask
[out] Pointer to a variable of type XONLINETASK_HANDLE that receives a task handle. Use this handle in calls to XOnlineTaskContinue to process the file upload.

Return Values

If XOnlineStorageUploadByServerPath succeeds, the return value is S_OK.

If XOnlineStorageUploadByServerPath fails, the return value is one of the following error codes:

Value Description
XONLINE_E_USER_NOT_LOGGED_ON User not logged on.
XONLINE_E_LOGON_INVALID_USER User is invalid
E_OUTOFMEMORY There is not enough memory to create the asynchronous task context.
XONLINE_E_STORAGE_INVALID_STORAGE_PATH Storage path cannot be created.
XONLINE_E_STORAGE_INVALID_FACILITY Invalid facility.
XONLINE_E_STORAGE_INVALID_REQUEST The request is invalid.
XONLINE_E_STORAGE_ACCESS_DENIED The client doesn't have sufficient permissions to upload the file.
XONLINE_E_STORAGE_INVALID_ACCESS_TOKEN The access token signature is invalid.
XONLINE_E_STORAGE_CANNOT_FIND_PATH Name resolution failed.
XONLINE_E_STORAGE_INVALID_STORAGE_PATH Invalid storage path.
XONLINE_E_STORAGE_INVALID_FACILITY Invalid facility code.

Remarks

This function is similar to XOnlineStorageUpload, except that instead of using a hServerFileReference HANDLE, it takes a storage file name.

Your title must start the signature service before it can call this function.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

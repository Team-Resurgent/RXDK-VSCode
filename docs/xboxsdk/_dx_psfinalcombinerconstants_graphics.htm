PSFinalCombinerConstants

The last field, PSFinalCombinerConstants, is very much like the PSC0Mapping and PSC1Mapping fields except that only two constant mappings are defined because there are only two constants in the final combiner (c0 and c1). The macro PS_FINALCOMBINERCONSTANTS(c0,c1,flags) may be used to set these constants. This macro is defined as below:

#define PS_FINALCOMBINERCONSTANTS(c0,c1,flags) (((DWORD)(flags) << 8) | ((DWORD)(c0)&0xf)<< 0) | (((DWORD)(c1)&0xf)<< 4)

The c0 and c1 parameters are used to set the associated constants in the final combiner.

There is an additional flag defined in this field that affects the behavior of the Direct3D run-time code. This flag is specified in the third parameter of the macro and can be selected from the following flags:

  • PS_GLOBALFLAGS_NO_TEXMODE_ADJUST
  • PS_GLOBALFLAGS_TEXMODE_ADJUST

In the Direct3D assembly language, it is not required to specify whether a texture operation is expected to operate on a 2D texture, a 3D texture, or a cubemap texture. The same tex instruction is used for all of these textures. However, the GPU hardware does have specific modes for each of these textures. If PS_GLOBALFLAGS_TEXMODE_ADJUST is set to adjust the texture modes, the Direct3D run-time code will make the following adjustments to the texture mode in each stage based on the type of texture that is set on that stage:

If the set texture is a cubemap then:

  • Change PS_TEXTUREMODES_PROJECT2D to PS_TEXTUREMODES_CUBEMAP.
  • Change PS_TEXTUREMODES_PROJECT3D to PS_TEXTUREMODES_CUBEMAP.
  • Change PS_TEXTUREMODES_DOT_STR_3D to PS_TEXTUREMODES_DOT_STR_CUBE.

If the set texture is a volume texture:

  • Change PS_TEXTUREMODES_PROJECT2D to PS_TEXTUREMODES_PROJECT3D.
  • Change PS_TEXTUREMODES_CUBEMAP to PS_TEXTUREMODES_PROJECT3D.
  • Change PS_TEXTUREMODES_DOT_STR_CUBE to PS_TEXTUREMODES_DOT_STR_3D.

If the set texture is neither a cubemap nor a volume texure:

  • Change PS_TEXTUREMODES_PROJECT3D to PS_TEXTUREMODES_PROJECT2D.
  • Change PS_TEXTUREMODES_CUBEMAP to PS_TEXTUREMODES_PROJECT2D.

Because this remapping is required to support the pixel shader assembly language, this flag is set by default by the assembler. Most directly defined pixel shaders will want to disable this automatic texture mode mapping because they have complete control over the texture mode that is set in each stage.

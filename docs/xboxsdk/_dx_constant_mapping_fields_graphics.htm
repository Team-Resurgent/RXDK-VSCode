Constant Mapping Fields

The constant mapping fields are not used directly by the hardware. They are used by the software to define how the 16 global Direct3D constants map to the constant registers in each combiner stage and the final combiner.

Constants are treated differently when using the pixel shader assembly language than they are when pixel shaders are defined directly. The pixel shader assembly language provides for 16 constant registers that can be read and used by any combiner stage. These constants are set using the IDirect3DDevice8::SetPixelShaderConstants method. The graphics processing unit (GPU) hardware does not have 16 global constant registers, it has 2 constant registers that are local to each stage. Each of these constants can be read only by the combiner stage to which they belong. The final combiner also has two constants that work the same way.

The constant mapping fields in the D3DPIXELSHADERDEF structure are used to make these "local" combiner constants behave as if they were global constants.

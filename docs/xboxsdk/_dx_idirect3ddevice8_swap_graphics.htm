IDirect3DDevice8::Swap

Swap is an alternative to IDirect3DDevice8::Present. It allows the title to either implement their own back-buffer to front-buffer copy routine, or to write to the front-buffer after the copy is done but before the flip is complete. This function is only useful for devices that are using either antialiasing or the D3DSWAPEFFECT_COPY mode.

DWORD Swap(
  DWORD Flags
);

Parameters

Flags
[in] Must be one of the following values:
Value Definition
D3DSWAP_DEFAULT Presents the contents of the next-in-sequence back-buffer owned by the device. Using D3DSWAP_DEFAULT is equivalent to calling IDirect3DDevice8::Present.
D3DSWAP_COPY This allows the title to render to the front-buffer after the copy from the back-buffer to the front-buffer, but before the front-buffer page-flip is initiated. D3DSWAP_COPY allows the title to do things like rendering its HUD or text after any back-buffer scaling and filtering is done. In this way, the title can avoid scaling artifacts in its HUD and text when IDirect3DDevice8::SetBackBufferScale is used with non-integer scaling factors, and avoid extra filtering from the antialiasing filter.

D3DSWAP_COPY is intended for use when BackBufferCount is 2. It can also potentially be used when the BackBufferCount is 1 and the FullScreen_PresentationInterval is not D3DPRESENT_INTERVAL_IMMEDIATE if the rendering is done fast enough to beat the CRT raster. Be aware that long interupts may interfere with this if the rendering is done near the top of the screen.

The title must call Swap with D3DSWAP_FINISH when done.
D3DSWAP_BYPASSCOPY This sets up the hardware to allow the title to implement their own back-buffer to front-buffer copy routine. Direct3D will swap the front-buffer structure pointers, if necessary, initiate synchronization with the page-flip handlers, set texture 0 the antialiasing source, and set the render target to the front-buffer destination.

The title needs to do its own copy using standard Direct3D routines and then call Swap with D3DSWAP_FINISH when done.
D3DSWAP_FINISH This sets the hardware to complete the synchronization with the page-flip handlers, handles gamma changes, and restores texture 0 and the render target to the value they held before using either D3DSWAP_COPY or D3DSWAP_BYPASSCOPY.

Return Values

The function returns the current swap count.

Remarks

A maximum of two swaps can be simultaneously enqueued in the push-buffer at one time. If a swap occurs while two swaps are already enqueued the push-buffer, Direct3D will do a BlockOnFence until one of the pending swaps is handled by the GPU.

Note   In addition to the swaps enqueued in the push-buffer, there may be up to BackBufferCount swaps which have already been handled by the GPU but which are pending their vertical blank before being shown.

Preparing Content for Autoupdate

The files sent to players' consoles as part of an autoupdate are sent in an autoupdate package. You must use the new XLAST autoupdate wizard to create an autoupdate package. After the package is created, you must add files to it, build it, and test it. When you're sure it's working properly, you must submit it for certification.

Creating an autoupdate package requires that you have a directory on your development PC that contains the certified game as it originally shipped. You also need a directory containing the files for the autoupdate package. If you update any of the XBE files in your game, you must update all XBEs that log on to Xbox Live. The reason for this is that all the files must have the same same title ID and version number. This includes redistributables from the XDK (if your game uses them) such as Downloader.xbe.

If your autoupdate package contains non-XBE data files, such as textures and sound files, the updated XBE files must be programmed to use the new data files rather than files on the game disk. These files reside on the hard disk drive. Specifically, they are located at T:\$u. The Xbox automatically loads XBEs that are part of an autoupdate from the proper redirected location (T:\$u). Access to additional non-XBE files is not automatically redirected. You must add it in yourself.

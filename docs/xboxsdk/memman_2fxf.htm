LocalAlloc

The LocalAlloc function allocates the specified number of bytes from the title's default heap.

Note  All Globalxxx and Localxxx memory management functions use the title's default heap, and provide less flexibility than the heap functions. Titles should use the heap functions instead.

HLOCAL LocalAlloc(
  UINT uFlags,     // allocation attributes
  SIZE_T uBytes    // number of bytes to allocate
);

Parameters

uFlags
[in] Specifies how to allocate memory. If zero is specified, the default is the LMEM_FIXED value. This parameter can be any combination of the following values.
Value Meaning
LMEM_FIXED Allocates fixed memory. The return value is a pointer to the memory object. This is the default, and memory is allocated as fixed whether you specify this flag or not.
LMEM_ZEROINIT Initializes memory contents to zero.
LPTR Combines LMEM_FIXED and LMEM_ZEROINIT.
NONZEROLPTR Same as LMEM_FIXED.

uBytes
[in] Specifies the number of bytes to allocate.

Return Values

If the function succeeds, the return value is a handle to the newly allocated memory block.

If the function fails, the return value is NULL. To get extended error information, call GetLastError.

Remarks

If the heap does not contain sufficient free space to satisfy the request, LocalAlloc returns NULL. Because NULL is used to indicate an error, virtual address zero is never allocated. It is, therefore, easy to detect the use of a NULL pointer.

If this function succeeds, it allocates at least the amount requested. If the amount allocated is greater than the amount requested, the process can use the entire amount. To determine the actual number of bytes allocated, use the LocalSize function.

To free the memory, use the LocalFree function.

When allocating memory blocks larger than one or two megabytes, it is more efficient to use the VirtualAlloc or VirtualAllocEx function instead.

Requirements

Header: Declared in Winbase.h, for include use Xtl.h.
Import Library: Use Xapilib.lib

Xbox Game Disc Format Utility (GDFUtil)

The Xbox Game Disc Format Utility (GDFUtil) is a command-line utility that converts between various game disc layout and submission formats. Game disc layouts are normally created with the Xbox Game Disc Authoring Tool (XbGameDisc), a tool with a graphical user interface.

GDFUtil mirrors some of XbGameDisc's functionality by enabling command-line creation and manipulation of layouts, and adds a new, readable layout format. Using GDFUtil, an automated build process can unpackage a PKG layout, modify the file contents and file layout, then repackage the layout.

For a detailed discussion of the game disc layout process, see the Xbox Game Disc Authoring Tool.

Note    GDFUtil should not be used to premaster game title submissions to Microsoft. Instead, please use the Xbox Game Disc Authoring Tool.

Layout Formats

GDFUtil works with the following game layout formats:

Format Description
XLO An Xbox game disc layout file, created or readable by XbGameDisc or GDFUtil.
LCF A layout control file. An LCF is an XML layout format that simply specifies the files in the layout, their order, and their layer.
ISO A DVD-R ISO format file. Currently not supported as an input file format.
PKG A package file created by XbGameDisc or GDFUtil; contains layout information as well as the game files and directories. This is an electronic submission format.
directory A directory can be specified as an input or output format; that directory is used as the source or destination of the file tree.

Note    TAPE is not supported as an input or output format in this release.

The XLO format file is the format file used by XbGameDisc and GDFUtil to save layout and authoring session information. This format does not contain the game files but does store the directory location that contains the game files.

The LCF format is an XML text file that contains layout information including the list of game files, their order, and their layer affinity. It does not contain the game files or information on the location of the game files. The LCF format provides an easily edited, modified, and generated format, suitable for automated build processes.

The ISO format is a disk image that can be used to burn a single layer DVD-R. This format contains the game files but does not contain layer affinity information.

The PKG format is output by XbGameDisc's Xbox Premastering Wizard and is suitable for certification submission. This format contains all of the files in the layout as well as layout information (including file order and layer). A PKG includes an embedded XLO that contains the layout information for the PKG.

Usage

USAGE: gdfutil [options] source destination
Option Description
/INPUTDIR:dir Specifies the location of the source format file tree. For source formats that already specify the file tree location (such as XLO), this option overrides that location. For the LCF source format, this option must be used to specify the location of the file tree. This option can be used, for example, to specify a different directory than is specified in the source XLO. This option is valid for source formats XLO, PKG, and LCF and is required for source format LCF.
/OUTPUTDIR:dir For source formats that contain or reference files, specifies the location where extracted files and folders should be written. This option is valid for destination formats XLO, PKG, and LCF. Files are written to the Temp folder if no directory is specified.
/PKGTEMPDIR:dir Specifies the directory that GDFUtil uses when extracting PKG source format embedded files. If this directory is not specified, the user's Temp directory is used and the temporary files are removed when the operation is complete. To process PKG input, GDFUtil must first extract the embedded files. This option is valid for source format PKG.
/PKGXLODIR:dir Specifies where a PKG's embedded XLO file should be extracted. If this directory is not specified, the user's Temp directory is used. This option is valid for source format PKG. If the layout for your package contains a file named main.xlo, you must specify an extraction location when extracting the package. Otherwise, the main.xlo file in the layout overwrites the main.xlo file that is generated during package creation.
/ALLOWMOD Specifies that GDFUtil should not stop processing if it determines that the files in the file tree differ from those referenced in the layout file. If this option is not specified, and if the file tree doesn't match the layout file, GDFUtil exits. This option is valid for source formats XLO and LCF.
/ADDNEWFILES Specifies that GDFUtil should attempt to incorporate new files found in the file tree into the layout. If this option is not specified, and if new files are found in the file tree, gdfutil exits. This option must be used with /ALLOWMOD. It is valid for source formats XLO and LCF.
/ACCURATEISO Specifies that GDFUtil should place files on the ISO in the location that best matches the layout. If this option is not specified, files are compacted to generate a smaller ISO. This option is valid for destination format ISO.
/ALLOWLARGEISO Enables the creation of a disk image even if it exceeds the size allowed for a standard ISO DVD-R. Cannot be used with /ACCURATEISO.

The source and destination arguments can specify either filenames or directories. If a filename is specified, the filename indicates the input or output file, and the extension specifies the layout format (which must match one of the supported layout formats, described above). If a directory is specified, that directory is used as the source or destination of the file tree.

GDFUtil converts the source format to the destination format, preserving as much information as is possible between the two formats.

The /ALLOWMOD and /ADDNEWFILES options modify GDFUtil's behavior when comparing the files of an input XLO (or LCF) and the file tree specified in the XLO (or specified with the /INPUTDIR option). If the file information in the XLO does not match the files in the file tree (this includes file size), GDFUtil exits with an error, by default. The /ALLOWMOD option instructs GDFUtil to continue processing despite these differences. Also, by default, new files found in the file tree that are not in the XLO (or LCF) are not included in the output format. The /ADDNEWFILES option instructs GDFUtil to include these new files in the output format.

The /ACCURATEISO option specifies that ISO output be accurate rather than compact. The two possible forms of ISO output are:

  1. Accurate—Output files are placed in the same layout as the input format. Because DVD-R media is single-layer, only the files from Layer 0 of the input layout are written to the ISO file. If the input layout contains files on Layer 1, these files are not included in the ISO file.
  2. Compact—Squeeze as many files as possible (including those on Layer 1) onto the output disc image, up to the capacity of a DVD-R disc. Compact layout ignores the layout specified in the input format, compacting files by removing gaps and placeholders.

Examples

The following example creates an LCF format output file using a PKG as input. For PKG input, the location of the extracted file tree can be specified with the /PKGTEMPDIR option.

gdfutil /pkgtempdir:c:\blaster blaster.pkg blaster.lcf

The LCF file and the file tree could then be modified (manually or automatically), adding or removing files as necessary.

The following example creates a PKG format output file using an LCF input file. For LCF input, the location of the file tree must be specified with the /INPUTDIR option.:

gdfutil /inputdir:c:\blaster blaster.lcf blaster.pkg

Example LCF File

A simple LCF file might look like the following:

<xbGameDiscLayoutTemplateFile>
  <Header>
    <SchemaVersion Major="1" Minor="0"/>
    <OldestSchemaCompatibleWith Major="1" Minor="0"/>
    <LastSave TimeMSSince1970="1043982343" ToolID="GDFUtil" Major="1" Minor="0" Build="5342" QFE="1"/>
  </Header>
  <Files Location="Layer0">
    <Object Type="File"      Name="\default.xbe"/>
    <Object Type="File"      Name="\level8.dat"/>
    <Object Type="Directory" Name="\"/>
    <Object Type="File"      Name="\cq\doggle.nam"/>
    <Object Type="Directory" Name="\cq"/>
  </Files>
  <Files Location="Layer1">
    <Object Type="File"      Name="\cq\spr\ginnad.lvl"/>
    <Object Type="Directory" Name="\cq\spr"/>
  </Files>
  <Files Location="EitherLayer">
  </Files>
  <Files Location="Ignore">
  </Files>
</xbGameDiscLayoutTemplateFile>

The <Files> element lists files on the media in order from fastest to slowest; the Location attribute specifies the layer.

The <Header> element contains information about the version of GDFUtil used to generate the LCF, as well as a timestamp.

Remarks

If both the input and output formats contain layer information, that information is preserved in the conversion and contained in the output format. In cases where layout information is a preference rather than a requirement, the preference is honored when possible.

GDFUtil compares the input format, output format, and any appropriate directories, and, if the formats allow, does an 'lba-exact' copy of files (meaning that it preserves gaps and placeholders).

GDFUtil tracks the versions of the tools used to save a layout, create and extract PKG files, and create the final media. This data is stored on the media and is used to track potential layout issues. This is also true for Primary Volume Descriptors in ISO formats.

Multi-title (Demo Disc) certification information is preserved between the PKG and XLO formats.

When creating ISO disk images, it is important to ensure that none of the files in your title are larger than the size of 1 DVD-X2 layer. If such a file exists, GDFUtil is unable to create the disk image. It outputs an error message and stops the process of disk creation. You can override this behavior by enabling the /ALLOWLARGEISO option. If you do, GDFUtil outputs a warning, rather than an error, when it encounters the large file.

Note    GDFUtil does not support individual files sizes larger than 1.02GB or layouts with more than 6.5 GB of data.

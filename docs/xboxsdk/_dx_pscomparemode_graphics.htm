PSCompareMode

The final texture field specified how the PS_TEXTUREMODES_CLIPPLANE mode works on each texture stage. This texture mode eliminates the current fragment depending on the sign of each texture address component (r, s, t, and q). The bits in this field specify the condition for each input that will cause the fragment to be eliminated. The PS_COMPAREMODE(t0,t1,t2,t3) macro is used to specify four flags for each texture stage. The macro is defined as below:

#define PS_COMPAREMODE(t0,t1,t2,t3) (((t3)<<12)|((t2)<<8)|((t1)<<4)|(t0))

Each flag specifies how the comparison works for each input. For example, specifying the PS_COMPAREMODE_S_LT for stage 0 means that if the PS_TEXTUREMODES_CLIPPLANE texture mode is set on stage 0, the fragment will be eliminated if the s input is less than zero. Likewise, the PS_COMPAREMODE_S_GE means that the fragment will be eliminated if the s input is greater than or equal to zero. Four flags should be specified for each stage that is using the PS_TEXTUREMODES_CLIPPLANE mode.

The following flags may be specified:

PS_COMPAREMODE_S_LT
Elimate the fragment for the specified stage if the s input is less than zero.
PS_COMPAREMODE_S_GE
Elimate the fragment for the specified stage if the s input is greater than or equal to zero.
PS_COMPAREMODE_T_LT
Elimate the fragment for the specified stage if the t input is less than zero.
PS_COMPAREMODE_T_GE
Elimate the fragment for the specified stage if the t input is greater than or equal to zero.
PS_COMPAREMODE_R_LT
Elimate the fragment for the specified stage if the r input is less than zero.
PS_COMPAREMODE_R_GE
Elimate the fragment for the specified stage if the r input is greater than or equal to zero.
PS_COMPAREMODE_Q_LT
Elimate the fragment for the specified stage if the q input is less than zero.
PS_COMPAREMODE_Q_GE
Elimate the fragment for the specified stage if the q input is greater than or equal to zero.

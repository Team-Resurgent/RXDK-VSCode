XInitDevices

The XInitDevices function initializes the peripheral control software on the console and also enables the title to specify the maximum number of devices of each device type that will ever be used simultaneously. The title must call this function before accessing or enumerating any peripherals (controllers, memory cards, voice units, and so on).

VOID XInitDevices(
  DWORD dwPreallocTypeCount,
  PXDEVICE_PREALLOC_TYPE PreallocTypes
);

Parameters

dwPreallocTypeCount
[in] Number of structures contained in the PreallocTypes array.
PreallocTypes
[in] Pointer to an array of XDEVICE_PREALLOC_TYPE structures. Each array element is a structure that specifies the maximum number of devices of a specified device type that will be used simultaneously.

Return Values

This function does not return a value.

Remarks

XInitDevices initializes the peripheral control software on the console. A title must call this function before accessing or enumerating any peripherals.

This function also informs the system software how many devices of each type will be simultaneously used by the title. The driver uses this information to determine the resources to allocate in order to handle interaction with the specified number of devices. Titles can use this functionality to limit the resources allocated by the driver to the minimum needed by the title.

The dwPreallocTypeCount parameter specifies the number of structures in the PreallocTypes array, or, in other words, how many device types are being initialized.

The PreallocTypes parameter is a pointer to an array of XDEVICE_PREALLOC_TYPE structures. Each element of the array is a structure that specifies a device type and how many devices of that type will be simultaneously used by the title. The DeviceType member of the structure must be a valid Xbox device type (for example, XDEVICE_TYPE_GAMEPAD). The dwPreallocCount member of the structure specifies the number of devices of that type that will be used.

The system pre-allocates the necessary resources to allow the specified number of devices to be used simultaneously. For example, if the DeviceType member of the XDEVICE_PREALLOC_TYPE structure is XDEVICE_TYPE_MEMORY, and the dwPreallocCount member is two, the title can then only mount two memory units at one time. The title can still access all of the eight potential memory units, but it must do so sequentially, mounting and unmounting the memory units in turn (with at most two mounted at a time).

The limit to the number of devices of a given type that can be used simultaneously does not limit the enumeration of devices of that type. Titles can still enumerate the full complement of attached devices, even if resources were only allocated for the use of fewer devices at a time.

Specifying at least one device of a certain type will cause the system to initialize the appropriate driver and allocate the requested resources, even if that device type is never subsequently used by the title.

If dwPreallocTypeCount is zero and PreallocTypes is NULL, the maximum limits for each device type used will apply. The defaults are determined by the control software for each device type, and are currently four game controllers, eight memory units, four voice units, one debug mouse, and one debug keyboard. When using this default behavior, only device types that are called from the title's code (thus causing their corresponding control software libraries to be linked in) will have resources allocated for their use. Using XInitDevices in this way is not recommended, because the title loses control of the exact amount of resources allocated.

This function limits the opening or mounting of peripherals using the XInputOpen, XMountMU, or XVoiceCreateMediaObject functions. Those functions will fail if a title attempts to open or mount more devices at a time than were specified in the call to XInitDevices.

XInitDevices must be called only once by the title, and is normally called in the title's initialization code. It takes some amount of time (which varies, but usually less than half a second) for devices to initialize after the function is called. The title should call XGetDevices, then continue to call XGetDeviceChanges as part of the game loop, until the attached devices show up. This way, the title can continue initialization without blocking and waiting for device startup. If the title is finished initializating and really wants to wait for complete device startup (for example, before displaying UI representing connected controllers), the title can call XGetDeviceEnumerationStatus to check enumeration status. See Using Xbox Peripherals for complete example code demonstrating device initialization and detection.


The following example shows a typical call to XInitDevices that might occur at title startup:

XDEVICE_PREALLOC_TYPE deviceTypes[] =
{
   {XDEVICE_TYPE_GAMEPAD, 4},
   {XDEVICE_TYPE_MEMORY_UNIT, 2}
};

XInitDevices(
    sizeof(deviceTypes) / sizeof(XDEVICE_PREALLOC_TYPE),
    deviceTypes );

The XInitDevices call above specifies that at most four gamepads, and at most two memory units, will be used simultaneously by the title.

Xbox Live Lessons and Best Practices

By Andre Vrignaud, Technical Evangelist, Xbox Online

So you want to make a better Xbox LiveTM title. Excellent...we'd love to help!

In the months since launch, the Xbox Live team has learned a number of lessons worth passing along. We've noticed some consistency in things that titles do well and areas that could be improved. This document is intended to communicate some of what we learned.

Below, you'll find a variety of high-level points that provide guidance as you design your Xbox Live title. In general, we don't go into too much depth because we want this document to be usable by everybody, from producers, testers, and developers all the way up to the publisher. If you have more in-depth questions on how to implement a suggestion or feature, please contact your account manager or your TGM. If you have comments or suggestions to add to this document, please email andre@xbox.com.

Overall Xbox Live Implementation

  • Start by looking at titles that have implemented Xbox Live particularly well—for example, Moto GP, Whacked!, and Midtown Madness 3. Also, see this excellent example of Xbox Live UI flow.
  • Make your title Xbox Live Aware—even your offline, single-player titles. Xbox Live Aware simply means the ability to sign in with your Gamertag at the beginning of the game so you can have access to your Friends list and game invitations while playing offline or single-player games. This feature helps keep Xbox Live gamers playing your game, and it enables them to stay connected to the community, see when people come online to play, and send and accept invites. With Xbox Live Aware titles, gamers don't have to be in a fully Xbox Live title if they want to play and keep an eye on who might be on the service.

    You can easily implement Xbox Live Aware in your title by using Drop-in UI (UIX). The recommended implementation is to silently sign in the last used Gamertag at game launch (if an Internet connection is available). Gamers can then set whether they want to enable this feature in the Xbox Dashboard. Note that if aren't using UIX, there is an API available to help you implement this silent sign-in.

    In sum, with the advent of UIX, no upcoming Xbox Live title should be released without the ability to be "Always Live." We also recommend Always Live for system link titles.

  • Don't forget to enable the Voice over Speakers option. (This is a TCR as of June 2003.) You want gamers to hear the action in your title. Headsets will be sold separately from Xbox Live kits as of Fall 2003.
  • Get a gamer who has accepted a cross-game invite into the game as quickly as possible. There's no reason for the gamer to have to watch through all of the start-up screens and cinematics if they're trying to get into a specific game session as quickly as possible. Whacked! is a great example of how to implement joining a game with a cross-game invite.
  • Consider your options for hosting larger-scale games.
    • Some games have created a Dedicated Host Mode (DHM) to enable larger-scale games. Games have been able to support hosting 16 players on a single Xbox when in DHM. Two good examples of DHM titles are Unreal Championship and Return to Castle Wolfenstein.
    • With DHM, the host doesn't play or render the game on screen, but acts as a dedicated host, with limited functionality available to users (scores, text description of action, ability to change game parameters, and so on). To enable the host to verbally inform users of imminent shutdown or game parameter changes, consider adding voice support for DHM.
    • Larger-scale games can use a PC server in a data center. PC servers can enable larger-scale games(more players) and must be certified along with the game. You can host with Microsoft or with your own ISP/data center. If using your own, be aware that the hosting provider must meet certain requirements. Talk to your account manager or TGM for details.

Downloadable Content

  • Downloadable content is the #1 requested feature of Xbox Live titles. As of July 2003, two out of three Xbox Live titles support downloadable content.
  • The XDK provides a simple API for enumerating and downloading content within the game.
  • You can also use the Xbox Content Download Manager (Downloader.xbe), a stand-alone executable that you can call to enable downloadable content in your game. To use Downloader.xbe
    • Place a Download Content entry on your main menu. When the user selects this entry, you leave your game XBE and load Downloader.xbe, which takes care of all the hard work—signing in, TCRs, enumerating content, handling premium content, and so on.
    • Return to the same point you left your game XBE from afterward. There's no reason for the gamer to have to watch through all of the start-up screens and cinematics if they're returning from downloading content.
    • Your game should be architected to enumerate available content at launch. It must also verify downloaded content before loading it.
    • Think about skinning Downloader.xbe. Mech Assault did a great job of this, and the result looks much better than the default Downloader skin. Implementing this is easy and takes just a couple of hours.

      Note     Downloader.xbe now supports premium content—that is, the ability to charge users for downloadable content that you offer.

    • Use XOnlineOfferingIsNewContentAvailable to check whether new content is available for download, If new content is available, highlight the Download Content item in the menu in some fashion to call it out to the user. Also, note that this API does not require a user to log in! This lets you silently check for new content in the background and only highlight the Download Content entry if there is something new.

User Interface (UI)

  • Consistent UI is key to game experience. Use A to advance in menus, B to go back—these are the defaults for most games; it's frustrating when games don't support them.
  • Make it easy to get to a Friends List. We recommend using an always-available single button (Y is common).
  • Allow the use of D-pad left/right to scroll through list boxes a page at a time.
  • Consider using the Xbox Drop-in UI (UIX), which can greatly ease Xbox Live development. UIX currently includes support for sign-in (one-player or four-player) and Friends List. Players List, Matchmaking, and two-player sign-in are coming soon. As additional features are enabled on Xbox Live, such as Teams and Tournaments, UIX will evolve to support them.
  • Take advantage of the Friends List—for example:
    • Use the default sorting provided by the Xbox Live client libraries. This automatically alphabetizes and sorts by priority/relevance. (Also, think about adding a divider between the "priority/relevance" items and just general Friends.)
    • Gamers appreciate an information-rich Friends List that includes data such as player on, playing what game, accepted invite—you get this information by default from presence queries.
    • XOnlineNotificationSetState gives you 32 bytes of data you can define to allow viewing time in game, laps left, game type, health, frags, and so on (not real-time, but a ~30 second delay). If you plan, this data can be visible in all of your titles. Also, if you version your schema, you can add information types in the future.

Voice

  • Always allow gamers to talk when they first enter the game. Enable a Voice channel by default.
  • "Push to talk" is not recommended. Several game press reviews have dinged this approach. It is okay to consider as an option—that is, push-to-talk to address all people in the game (taunts), while the default behavior (button not pushed) is just talking to your team.
  • Enable Voice during level-loads, transitions, and so on. Players should always be able to talk. It's frustrating to have a conversation cut off just because of a level-load.
  • "Voice over Speakers" is a TCR as of June 2003. The reason is free trials of Xbox Live will be available as of Fall 2003, and many trial users won't have headsets initially. Voice over Speakers enables them to hear what they're missing.
  • The simplest way to implement Voice over Speakers is to have the game check whether a Voice puck is inserted; if not, pump Voice through the speakers. Star Wars: The Clone Wars does this well (and adds a Voice over Speakers volume control option).
  • For more information, see the white paper Creating a Custom Voice Chat Engine.

Statistics

  • Show stats for your friends separately. Gamers want to gauge their progress against friends and the rest of the world. This is one of the most popular features of Xbox Live. Moto GP shows a great example.
  • Show gamer positions on the leaderboard by default, centered on their Gamertag with a few entries above and below for context.
  • Provide the ability to quick-jump to the top and bottom of the leaderboard. This saves network resources because many gamers want to see their position relative to the entire leaderboard.
  • List on-screen the total number of entries on the leaderboard. (This information can be returned with your stats query; you just need to display it.)
  • Use D-pad left/right to enable users to jump a screen at a time.
  • Enable gamers to quick-jump to their position on any leaderboard.
  • Punish griefers! Report stats often, or store the number of games started and number of games finished. The difference between these reveals the number of dropouts.
  • Try to create leaderboards such that any player could find something relevant to him or her.
  • Leaderboards should provide easy access to:
    • Top 10
    • My Ranking
    • My Friends Ranking
    • Total number of entries in leaderboard

Matchmaking/OptiMatch

  • One common complaint of gamers is finding themselves in a game where they aren't matched in skill with other players. This experience can be frustrating and intimidating. To address this, consider adding Matchmaking based on skill. Moto GP has a good implementation that allows you to consider leaderboard information when matching, and to race only against opponents close to you in lap times.
  • Consider only allowing players to use game objects that they've earned by unlocking in the single-player game. For example, add an OptiMatchTM option that filters against people who have played to a certain level, or who have access to a certain vehicle.
  • Consider adding a Matchmaking option based on whether gamers have Voice. Remember, as of Fall 2003, free Xbox Live trial offers will be available with millions of Xbox Live titles. Many gamers will not have Voice initially, and you want people to be able to filter on that aspect. Headsets will be available as stand-alone SKUs at retail.
  • Use QoS (Quality of Service) to refine Matchmaking results. Don't query and QoS large numbers of sessions (think 100) at a time. In such cases, the gamer has to wait too long and thinks something might be broken. Instead, query smaller sets and do QoS on those (10 at a time, for example). If you are QoS-probing a large set of hosts, set the maximum bandwidth up to 64Kbps (default is 32Kbps). When you are done, set it back to 32Kbps.
  • Use Proximity to find matches that are physically close to you. This is simply turned on in your Matchmaking definition file (also known as the .xms file).

Community Tools

  • Build a game lobby where gamers can meet, plan, break into teams, configure, launch, return to, and re-launch from. Implementing this can be very different depending on the type of game (team-based, one-on-one, join in-progress games, and so on). The key is to think about ways to allow gamers to continue playing the game easily with people they're already having fun with.
  • Nothing is more frustrating to gamers than having a great session and not being able to talk trash afterward. So think about scenarios where teams can talk to each other, then to everyone in the game (good for post-game banter).
  • Make sure gamers have the ability to change game options from the lobby and to re-launch games easily with the same players.
  • Think about enabling players to join lobbies while a game is in session, so they can wait for the next game (Moto GP does this nicely). Allowing players to watch a game in session is a nice bonus.
  • Introduce community control options where players can vote to kick out problem users, mute them, ban them from a server, and so on. Return to Castle Wolfenstein offers a great example of this.
  • Take advantage of Xbox Live Web services. We enable you to take Xbox Live service data such as leaderboards, Friends Lists, presence, and so on, and use these features on your own Web site to build community. We expect to allow you to write arbitrary data to a leaderboard via Web services that you can then display in-game (this is currently read-only).
    • We give you a binary block of data (BLOB); you crack it and display it on your own site that you build.
    • The Microsoft XSN Sports Web portal will be built using exactly these same Web services that are available to you.
    • See the white paper Xbox Live 2003 Web Services for more details.

Testing

  • Understand the Xbox Core TCR Top Issues.
  • Play across the Internet. This sounds obvious, but a lot of issues can be exposed when using home routers, home broadband connections (which are often limited in upstream bandwidth), and so on. If you can, send your testers home for at least some testing. We always see issues that are exposed by real-world testing.
  • Test with NetSim to simulate Internet conditions. NetSim will generally give better results than "real-world" testing. Use NetSim to roughly determine the best possible network conditions for your title.
  • Make sure your testers are using voice headsets to communicate, even if they are in the same room. We found issues where bandwidth wasn't stressed enough during testing because testers didn't use voice headsets to communicate.
  • Be aware that not all routers are equal. Some have firmware issues that cause a variety of issues in online titles. (You can find a list of approved routers on xbox.com.) Common symptoms include:
    • Some players see game movement while others don't.
    • Voice drop-outs: A can talk to B and C, while B and C can only talk to A and not to each other.
  • Encourage your development teams to play the game nightly, from home, for the last six to eight weeks of the development cycle. Teams have successfully found and characterized critical bugs this way.
  • Some test Gamertags should have pass codes, some shouldn't; some should have spaces, some shouldn't. Some test Gamertags should be the maximum length of 15 characters.

XBR::SetPixelShaderResource (XbView)

HRESULT SetPixelShaderResource( DWORD PixelShaderResourceIndex );

This function sets the pixel shader to the resource index passed in. It returns S_OK if the function succeeds. Otherwise, it returns the HRESULT of the function that fails. This function also uses pixel shader constants 0, 4, 5, 6, 7, and 8 to do the per-pixel lighting. At present, there is no way to stop the viewer from clobbering these pixel shader constants, and since there are only 16 pixel shader constants usable, this is a sizable portion of them. Since the Xbox Preview Pipeline doesn't allow the user to easily change or add pixel shader constants, this isn't a big problem. This is a good thing to keep in mind if you plan to modify the Preview Pipeline for your own usage.

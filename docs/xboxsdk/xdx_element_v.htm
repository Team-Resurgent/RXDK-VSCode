v Element

The <v> element is used to group all of the data for a given vertex.

<v
   index = integer
/>

Required Attributes

None.

Optional Attributes

index

Used for referencing from <indexBuffer> elements. If this is specified, it must be between 0 and 65535. If no index is given, the vertices will be stored in the same order as in the XDX file.

Element Information

Parent Elements vertexBuffer
Child Elements None.

Remarks

The format of the data within the <v> element is determined by the <declVB> element sharing the same parent <vertexBuffer>.

Examples

A. Vertex with position only

The vertices below describe a triangle with position data for each vertex.

<vertexBuffer id="vertexBuffer1" >
   <declVB>
      <vregVB name="pos" format="FLOAT3" />
   </declVB>

   <v>0 -0.5  0.866025</v>
   <v>0 -0.5 -0.866025</v>
   <v>0  1    0</v>
</vertexBuffer>
B. Vertex with position and color

The vertices below describe a triangle where the vertices are colored red, green, and blue.

<vertexBuffer id="vertexBuffer2" >
   <declVB>
      <vregVB name="pos" format="FLOAT3" />
      <vregVB name="col" format="D3DCOLOR" />
   </declVB>

   <v>0 -0.5  0.866025 0xffff0000</v>
   <v>0 -0.5 -0.866025 0xff00ff00</v>
   <v>0  1    0        0xff0000ff</v>
</vertexBuffer>

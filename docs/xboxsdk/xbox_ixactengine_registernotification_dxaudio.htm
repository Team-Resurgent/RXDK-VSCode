IXACTEngine::RegisterNotification

Registers for an event notification.

HRESULT RegisterNotification(
    PCXACT_NOTIFICATION_DESCRIPTION pNotificationDesc
);

Parameters

pNotificationDesc
[in] A pointer to an XACT_NOTIFICATION_DESCRIPTION structure that describes the notification to register.

Note    The pSoundCue parameter of XACT_NOTIFICATION_DESCRIPTION is reserved when calling this function and should be set to NULL.

Return Values

Returns DS_OK if successful; returns an error code otherwise.

Remarks

Before receiving notifications from the XACT engine, the type of notification must be registered. The following example illustrates how to register notification for when a particular sound cue index on a sound bank starts:

// This code assumes that g_pXACTEngine is a valid pointer to the
// XACT engine and pSoundBank is a valid pointer to an IXACTSoundBank.

XACT_NOTIFICATION_DESCRIPTION Desc;
DWORD dwSoundCueIndex;

pSoundBank->GetSoundCueIndexFromFriendlyName("CUE0", &dwSoundCueIndex);

memset(&Desc,0,sizeof(Desc));
Desc.wFlags = XACT_FLAG_NOTIFICATION_USE_SOUNDCUE_INDEX;
Desc.dwSoundCueIndex = dwSoundCueIndex;
Desc.u.pSoundBank = pSoundBank;
Desc.wType = eXACTNotification_Start;
		
g_pXACTEngine->RegisterNotification(&Desc);

After the notification has been registered, repeatedly call IXACTEngine::GetNotification to retrieve notifications as they occur. To clear pending notifications, call IXACTEngine::FlushNotification.

The behavior of the eXACTNotification_StreamDataAvailable notification varies depending on whether it is registered with the XACT_FLAG_NOTIFICATION_USE_WAVEBANK or the XACT_FLAG_NOTIFICATION_USE_SOUNDCUE_INDEX flag:

  • When registered with the XACT_FLAG_NOTIFICATION_USE_WAVEBANK flag, this notification fires as soon as all zero-latency waves in the wave bank have been primed. If the wave bank contains no zero-latency waves, then the notification will fire immediately, and will not wait for a call to IXACTSoundBank::Prepare on any or all of the waves in the wave bank. This behavior allows the application to wait until all the zero-latency waves have been automatically primed with a single API call.
  • Registering eXACTNotification_StreamDataAvailable with XACT_FLAG_NOTIFICATION_USE_SOUNDCUE_INDEX behaves slightly differently. The notification will not fire immediately for sound cues referencing non-zero-latency waves, rather IXACTSoundBank::Prepare must be called. Notification will fire for sound cues referencing zero-latency waves as soon as their wave banks have finished their automatic priming. The eXACTNotification_StreamDataAvailable notification will fire on all cues that reference streamed waves (regardless if the wave was zero-latency or not).

Since loading a wave bank is required before registering notifications on it, a wave bank may become primed before notifications are registered, in which case the notification will be available immediately. Notifications will not be missed if they are registered after the XACT Engine has primed the wave bank.

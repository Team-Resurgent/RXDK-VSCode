Adding Competition Templates

Titles use Competition Templates as a model for dynamically creating competitions.

Note    Competition Templates are different and distinct from Competition Leaderboard Templates, which are templates for leaderboards associated with a competition. You create and submit them in Creating Stats Configurations.

  • Adding a Competition Template to the Project
  • Query Service Configuration
  • Code Generation

Adding a Competition Template to the Project

After you create a project with the New Competitions Project Wizard, you must add competition templates to the project. To so do, use the following procedure.

  1. Right-click the root node of the project in the Project View's tree. Select New Competition Template.
  2. In the dialog box that appears, use the drop-down list to select the type of competition template you want to create. Currently, XLAST can create templates only for single-elimination competitions.
  3. After you click OK, XLAST creates a Competition Template of the select type and inserts it into the tree in the project's Project View window.

The default name of a new Competition Template is New Competition Type. You can rename it by clicking on the string New Competition Type and typing a new name.

Query Service Configuration

Every Competition Template must provide query service configuration information. This information is also used in code generation. To set the query service configuration information in your Competition Template, use the following procedure.

  1. Click on the template's node in the project tree. Use the tabbed page labeled Query Service Configuration in the right-hand pane of the Project View window.
  2. Choose the creator of the competition. Competitions and tournaments can be created by the publisher of a title, or by players. Selecting Publisher or Users in the Tournament Creators group makes this a template for either publisher-created or user-created competitions, respectively.
  3. If you selected Users in the previous step, you must set the maximum number of competitions that a single user can create at any one time. The system maximum is 3. However, you can change this value to 1 or 2, if you want.
  4. If you want leaderboards created for the competition, you must first create a Stats configuration and put a Competition Leaderboard Template into it. You can then enter the ID number of the Competition Leaderboard Template into the Leaderboards group. Also, you'll need to specify the number of leaderboards to create for each competition.

Code Generation

XLAST can generate code to help you create competitions and display information about them on the Web. To do so, follow the procedure below.

  1. Click the Code Generation tab in the right-hand pane of the Project View window.
  2. Enter the default display name. This name is displayed on the Web if your title does not supply another name at run time. You can also enter a formatting string, such as "Final Four Elimination for %s".
  3. Type in a default description. This description is used if your title does not supply another description at run time.
  4. Input the default URL for the competition.

    5. Note    The default name, description, and URL are inserted as Unicode strings in the file that XLAST generates. Do not use double quotation marks in any of these three fields.

  5. Specify the round recurrence.
    • Back-to-back If rounds are set to back-to-back, a given round is played after the previous one finishes. If you choose back-to-back rounds, you must specify the maximum length of the rounds in minutes.
    • Daily Daily rounds don't have to occur every day. You can specify which days they need to be played by. For example, putting a check in the boxes for Tuesday and Thursday means that two rounds are scheduled for the week. The first round must be played by Tuesday, and the second by Thursday.
    • Weekly If round recurrence is weekly, each round must be played before the week ends.
  6. Specify the type and number of entrants. By default, entrants are individuals. If you want this to be a templete for team competitions, put a check in the Entrants are teams, not individuals box.
  7. If you want, you can have the Xbox Live service send reminder messages to the participants a specified number of minutes before each round begins.

After you've entered the appropriate information, you can get XLAST to generate code for use in your title by selecting Action from the main menu. Next, click Build. XLAST builds your competitions configuration and generates the source code.

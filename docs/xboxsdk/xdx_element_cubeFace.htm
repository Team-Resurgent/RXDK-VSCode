cubeFace Element

Specifies one of the textures used in a cubemap.

<cubeFace
   face = string
   source = string
   alphaSource = string
   colorKey = string
   filter = string
/>

Required Attributes

face

Specifies which face of the cube map is getting defined. Must be one of the following values:

Value Description
XP Positive x-face (right side) of the cube map.
XN Negative x-face (left side) of the cube map.
YP Positive y-face (front face) of the cube map.
YN Negative y-face (back face) of the cube map.
ZP Positive z-face (top face) of the cube map.
ZN Negative z-face (bottom face) of the cube map.
source

Source image file. For more details, see Supported Image File Types. If there is an alpha channel in the source bitmap, that alpha channel is preserved and output. If there is an alpha channel in the source bitmap and the alphaSource attribute is specified, the alphaSource bitmap will replace the alpha channel given in the source bitmap.

Optional Attributes

alphaSource

Alpha channel source image file. For more details, see Supported Image File Types. The alpha channel is computed by converting the alphaSource image to a full 32-bit color image and taking the average of the red, blue, and green channels as the alpha channel.

colorKey

Used to specify the color key the compiler should use to determine what colors to convert to alpha. By default, color key alpha component is zero and no color key operations are performed.

filter

Specifies the filter used to resize the image.

If this value is specified, it must contain one of the following flags:

Value Description
D3DX_FILTER_NONE No scaling or filtering will take place. Pixels outside the bounds of the source image are assumed to be transparent black.
D3DX_FILTER_POINT Each destination pixel is computed by sampling the nearest pixel from the source image.
D3DX_FILTER_TRIANGLE Every pixel in the source image contributes equally to the destination image. This is the slowest of the filters. This is the default filter if nothing is specified.
D3DX_FILTER_LINEAR Each destination pixel is computed by sampling the four nearest pixels from the source image. This filter works best when the scale on both axes is less than two.
D3DX_FILTER_BOX Each pixel is computed by averaging a 2×2(×2) box of pixels from the source image. This filter works only when the dimensions of the destination are half those of the source, as is the case with mipmaps.

In addition, zero or more of the following flags may be ORed in with one of the flags above.

Value Description
D3DX_FILTER_WRAP Pixels will be wrapped, not mirrored. This is the default behavior.
D3DX_FILTER_MIRROR Specifying this flag is the same as specifying the D3DX_FILTER_MIRROR_U, D3DX_FILTER_MIRROR_V, and D3DX_FILTER_MIRROR_W flags.
D3DX_FILTER_MIRROR_U Pixels off the edge of the texture on the u-axis should be mirrored, not wrapped.
D3DX_FILTER_MIRROR_V Pixels off the edge of the texture on the v-axis should be mirrored, not wrapped.
D3DX_FILTER_MIRROR_W Pixels off the edge of the texture on the w-axis should be mirrored, not wrapped.
D3DX_FILTER_DITHER This flag indicates that the resulting image be dithered using a 4×4 ordered dither algorithm.

Element Information

Parent Elements cubeMap
Child Elements None.

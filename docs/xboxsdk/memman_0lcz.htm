GlobalReAlloc

The GlobalReAlloc function changes the size of the specified memory block. The size can increase or decrease.

Note  All Globalxxx and Localxxx memory management functions use the title's default heap, and provide less flexibility than the heap functions. Titles should use the heap functions instead.

HGLOBAL GlobalReAlloc(
  HGLOBAL hMem,   // handle to global memory object
  SIZE_T dwBytes, // new block size
  UINT uFlags     // how to reallocate object
);

Parameters

hMem
[in] Handle to the memory block to be reallocated. This handle is returned by either the GlobalAlloc or GlobalReAlloc function.
dwBytes
[in] Specifies the new size, in bytes, of the memory block.
uFlags
[in] Specifies how to reallocate the memory block.
Value Meaning
GMEM_MOVEABLE Allows the memory block to be moved to a new location if it is necessary to do so to satisfy the requested block size.
GMEM_ZEROINIT Causes the additional memory contents to be initialized to zero if the memory block is growing in size.

Return Values

If the function succeeds, the return value is a pointer to the reallocated memory block.

If the function fails, the return value is NULL. To get extended error information, call GetLastError.

Remarks

The return value is the address of the first byte of the memory block. To access the memory, simply cast the return value to a pointer.

If GlobalReAlloc fails, the original memory is not freed.

When allocating memory blocks larger than one or two megabytes, it is more efficient to use the VirtualAlloc or VirtualAllocEx function instead.

Requirements

Header: Declared in Winbase.h, for include use Xtl.h.
Import Library: Use Xapilib.lib

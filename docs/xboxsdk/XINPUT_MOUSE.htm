XINPUT_MOUSE

Contains mouse state information and positional changes.

To use XINPUT_MOUSE, define the DEBUG_MOUSE compiler define before including Xtl.h. Do not use XINPUT_MOUSE in shipping retail game titles.

typedef struct _XINPUT_MOUSE{
    BYTE bButtons;
    char cMickeysX;
    char cMickeysY;
    char cWheel;
} XINPUT_MOUSE, *PXINPUT_MOUSE;

Members

bButtons
Indicates the mouse buttons that are currently pressed. Can be a combination of the following flags:
Value Description
XINPUT_DEBUG_MOUSE_LEFT_BUTTON The left mouse button.
XINPUT_DEBUG_MOUSE_RIGHT_BUTTON The right mouse button.
XINPUT_DEBUG_MOUSE_MIDDLE_BUTTON The middle mouse button (mouse wheel button).
XINPUT_DEBUG_MOUSE_XBUTTON1 The first extra mouse button (typically on the left side of the mouse).
XINPUT_DEBUG_MOUSE_XBUTTON2 The second extra mouse button (typically on the right side of the mouse).
cMickeysX
X-position delta since the mouse state was last queried.
cMickeysY
Y-position delta since the mouse state was last queried.
cWheel
Mouse wheel delta since the mouse state was last queried.

Remarks

The bButtons member indicates the current absolute state of the mouse buttons.

The cMickeysX, cMickeysY, and cWheel members indicate relative deltas since the title last queried the mouse state with XInputGetState. Mouse position changes are aggregated by the driver and then returned as one set of deltas when the title calls XInputGetState. The delta values are capped at plus or minus 127, so if the title does not call XInputGetState frequently enough to detect all position changes, the maximum delta reported is plus or minus 127.

To use XINPUT_MOUSE, #define DEBUG_MOUSE before including Xtl.h. Do not use XINPUT_MOUSE in shipping retail game titles.

Debug mouse support is provided for development-time purposes (for example, clicking on characters in-game, bringing up context menus to set rendering modes, panning camera positions, and so on). The debug mouse will work in either debug or release builds of the title, but the feature is marked as unapproved at all times and thus will cause a certification failure if included in a title submitted for certification.

No user interface (a mouse pointer, for example) is provided for use with debug mouse support. As it does with other Xbox input devices, the title associates UI with the device as desired or as necessary.

There are no additional libraries (beyond XapiLib.lib or XapiLibd.lib) that a title must link with in order to obtain debug mouse support.

To use debug mouse support in your title, specify from one to eight XDEVICE_TYPE_DEBUG_MOUSE devices when calling XInitDevices, and then use the XGetDevices, XGetDeviceChanges, XInputOpen, XInputGetState, and XInputClose functions as you would with other Xbox device types. Do not use any other input device functions (such as XInputSetState or XInputGetCapabilities) with the debug mouse.

A debug mouse can be connected to a console port using a debug keyboard dongle, or to an open USB port on a connected debug keyboard. A debug mouse cannot be connected to an MU slot in a controller.

Compatibility testing has been done for most major wired and wireless USB mice. Some features (for example, the extra mouse buttons) may not work with all mice, especially wireless mice.

The /DUMP ImageBld switch will report whether a title is using the debug mouse feature. The feature will always be reported as unapproved.

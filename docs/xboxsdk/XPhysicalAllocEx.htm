XPhysicalAllocEx

Allocates a region of contiguous physical memory within a specified memory range, and returns a pointer to the newly allocated memory.

LPVOID XPhysicalAllocEx(
    SIZE_T dwSize,
    ULONG_PTR ulLowestAcceptableAddress,
    ULONG_PTR ulHighestAcceptableAddress,
    ULONG_PTR ulAlignment,
    DWORD flProtect
);

Parameters

dwSize
[in] Desired size, in bytes, of the memory to be allocated. This value is rounded up to the nearest page size.
ulLowestAcceptableAddress
[in] The lowest physical address that is acceptable for the allocation. This value is rounded down to the nearest system page (4 KB) boundary.
ulHighestAcceptableAddress
[in] The highest physical address that is acceptable for the allocation. This value is rounded down to the nearest system page (4 KB) boundary.
ulAlignment
[in] The desired alignment for the starting address returned for the allocated memory. This parameter must be a power of two. A value less than the system page size (4 KB) is rounded up to the system page size.
flProtect

[in] The access protection and optional memory type modifiers. One of the following access protection flags must be specified:
Value Description
PAGE_READONLY Enables read and execute access to the committed region of pages. An attempt to write to the committed region results in an access violation.
PAGE_READWRITE Enables read, write, and execute access to the committed region of pages.

In addition, one of the following flags may be specified to modify the characteristics of the allocated memory:

Value Description
PAGE_NOCACHE Allows no memory caching for memory in the committed region of pages. The hardware attributes for the physical memory will be treated as "no cache." This is useful for mapping a video frame buffer with no caching.
PAGE_WRITECOMBINE The allocated memory has a write-combining cache policy. This is useful for mapping a video frame buffer. Write combining is a special caching mode that batches writes to the same cache line so they can be transferred in a single bus clock. Write combining does not preserve ordering of the writes, a tradeoff that is usually acceptable for frame buffers.

Return Values

If the function succeeds, the return value is the base address of the allocated region of pages.

If the function fails, the return value is NULL. To get extended error information, call GetLastError.

Remarks

The XPhysicalAllocEx function allocates contiguous physical memory and returns a pointer to the allocated memory. If there is not enough contiguous physical memory available within the specified address range, the function will fail.

The returned address will be aligned according to ulAlignment. For example, if ulAlignment is 0x2000, the returned starting address will be aligned on an 8-KB boundary.

The allocated memory is not initialized to any known value. If zero-initialized memory is desired, the title must explicitly set the memory contents to zero.

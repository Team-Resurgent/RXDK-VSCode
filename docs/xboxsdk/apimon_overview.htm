API Monitor Overview

API Monitor is a tool that consists of two programs: the test suite execution application and the suite designer application. Together, these two programs provide Xbox application developers with the following capabilities.

  • Validate and Monitor Called Functions. The suite execution application validates that all of the API functions in the test suite are called by the specified Xbox program. In addition, it enables you to monitor the frequency and duration of calls to those functions.
  • Inject Faults. The API Monitor tool can intercept calls to functions in test suites and return the value specified by the suite. This enables you to inject faults into your Xbox program and thereby execute code paths intended for error handling.
  • Verify Compliance with Technical Certification Requirements (TCRs). The Microsoft Xbox TCRs specify that your Xbox application must call certain functions in the Xbox libraries. They also specify that your Xbox handle certain error codes gracefully. With the ability to monitor calls to API functions and to inject faults, you can verify that your program meets all relevant TCRs.
  • Design and Execute Test Suites. By splitting the API Monitor tool into two programs, Microsoft gives your developers the ability to provide test suites to testers that testers cannot alter. Developers, who may be more knowledgeable about TCRs and game internals, can design the tests. They can be executed by testers who may have considerably less background in software development.

To inject faults, test error handling, and verify compliance with TCRs, API Monitor uses call spoofing. With this technique, API Monitor intercepts calls to API functions and returns the value specified in the test suite. Testers can then validate that source code designed to handle error codes executes as specified by the TCRs.

To spoof a function call, select the desired Category and API, and then choose the appropriate Action from: Spoof Call Once, Spoof Call Always, Spoof Task Once, or Spoof Task Always. Selecting the Spoof Call Once option spoofs the Return Value only on the very first call to the function that appears in the API column. Spoof Call Always spoofs the specified Return Value for all calls to the function in the API column.

In addition to spoofing function calls, API Monitor can also spoof tasks. Task spoofing enables the testing of asynchronous calls that return task handles, namely the XOnline series of API functions. Because task spoofing is done on asynchronus calls, it does not cause the return value specified in a given test to be returned when the asynchronous function completes. Instead, the asynchronous function returns a task handle. The title then makes calls to "pump" the task handle for status for results.

Choosing Spoof Task Once for an XOnline function causes the specified Return Value to be returned on the first call to pump the task handle. Spoof Task Always spoofs the specified Return Value on every call that pumps the task handle. For example, invoking the function XOnlineLogon with the action Spoof Task Always causes XOnlineTaskContinue to return an error when it is passed the logon task handle obtained by XOnlineLogon. You do not need to add a spoof for the XOnlineTaskContinue function in this case; API Monitor handles this for you by passing a spoofed task handle to the original XOnlineLogon call.

Note    If you set API Monitor to Spoof Task Once, you will not get a specific message stating when the task was spoofed. In such cases, API Monitor always injects the return value you specify the first time your program pumps the task handle. The only indication you get that the call was spoofed is in the Calls column of the API Monitor Selected APIs box. If the value in the Calls column is greater than zero, the task was spoofed.

API functions are divided into three categories.

  • Winbase – Includes common functions such as GetLastError and GlobalAlloc.
  • Xbox – Lists functions in the Xbox library.
  • XOnline – Specifies functions related to Xbox Live.

Note    Before using API Monitor, it is highly recommended that you use the Remote Configuration tool (XbSetCfg) with the -t option to synchronize the time on your development Xbox with the time on your host development PC.

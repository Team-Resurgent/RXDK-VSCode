Deferred Settings

Every change to 3D sound buffer and listener settings causes a recalculation of the DirectSound 3D geometry, at the expense of CPU cycles. To minimize the performance impact of changing 3D settings, use the DS3D_DEFERRED flag in the dwApply parameter of any of the IDirectSound8 or IDirectSoundBuffer8 methods that change 3D settings. Then call the IDirectSound8::CommitDeferredSettings method to execute all of the deferred commands at once.

Note  Deferred settings are overridden by immediate settings. For example, if you set the listener velocity to (1.0, 0.0, 0.0) with the DS3D_DEFERRED flag and then set it to (2.0, 0.0, 0.0) with the DS3D_IMMEDIATE flag, the velocity becomes (2.0, 0.0, 0.0) and does not change when CommitDeferredSettings is called.

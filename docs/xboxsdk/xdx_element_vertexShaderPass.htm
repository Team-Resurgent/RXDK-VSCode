vertexShaderPass Element

Sets the vertex shader for current render pass.

<vertexShaderPass
   vertexShader = string
/>

Required Attributes

vertexShader

Name of the vertex shader to use for the render pass. This value must correspond to the id attribute of a previously defined <vertexShader>.

Optional Attributes

None.

Element Information

Parent Elements pass
Child Elements constantVS, declVS

Remarks

Any constants or declaration made using <constantVS> or <declVS> inside of a <vertexShaderPass> will add to, or overwrite, those made inside the shader referenced by the vertexShader attribute.

Examples

<vertexShader> and <vertexShaderPass>

This example shows how declarations and constants get combined. The vertex buffer (object1_VB) is getting set during the render pass. This allows multiple vertex buffers to be used with a given vertex shader. During the render pass, the vertex shader will use three constants: c11, c40, and c90. The c11 constant is set by VertexShader1. The values for c40 and c90 are determined by the <vertexShaderPass> that overwrites the c90 value set in VertexShader1.

<vertexShader id="VertexShader1">
   <declVS>
      <stream index="0"> ... </stream>
   </declVS>
   <asm> ... </asm>
   <constantVS index="11">1 1 1 1</constantVS>
   <constantVS index="90">0 0 -1 0</constantVS>
</vertexShader>
...
<frame id="Frm0">
   <model id="object1">
      <pass>
         <vertexShaderPass vertexShader="VertexShader1">
            <declVS>
               <stream index="0" vertexBuffer="object1_VB"/>
            </declVS>
            <constantVS index="40">1 2  3  4</constantVS>
            <constantVS index="90">0 0 -5 -5</constantVS>
         </vertexShaderPass>
         ...
      </pass>
   </model>
</frame>

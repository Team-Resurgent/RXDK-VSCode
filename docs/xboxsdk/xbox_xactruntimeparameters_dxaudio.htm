XACT_RUNTIME_PARAMETERS

Contains XACT audio engine initialization information.

typedef struct _XACT_RUNTIME_PARAMETERS{
    DWORD dwMax2DHwVoices;
    DWORD dwMax3DHwVoices;
    DWORD dwMaxConcurrentStreams;
    DWORD dwMaxNotifications;
    DWORD dwInteractiveAudioLookaheadTime;
} XACT_RUNTIME_PARAMETERS, *PXACT_RUNTIME_PARAMETERS;

typedef const XACT_RUNTIME_PARAMETERS *PCXACT_RUNTIME_PARAMETERS;

Members

dwMax2DHwVoices
The maximum number of 2D hardware voices/streams that the XACT engine will use. This can range from 0 to 188.
dwMax3DHwVoices
The maximum number of 3D hardware voices/streams that the XACT engine will use. This can range from 0 to 64.
dwMaxConcurrentStreams
Of the total voices specified by dwMax2DHwVoices, this is the maximum number of 2D voices that XACT may use for streaming from the hard disk and DVD. The remaining 2D voices, as specified by dwMax2DHwVoices, will be used for in-memory playback.
dwMaxNotifications
The maximum number of notifications. If zero is passed, the engine will use a default value.
dwInteractiveAudioLookaheadTime
The look-ahead time, in milliseconds, used by interactive audio. Defines the minimum transition time for non-immediate transitions. See Remarks below.

Remarks

The number of 2D voices that XACT can reserve must be less than or equal to 188. Any remaining hardware voices are available to be used by DirectSound and/or DirectMusic, if desired.

During development, the value for dwMaxNotifications can be tuned to suit the needs of your title. For a good beginning estimate, apply the following formula:

(Number of cue instances playing concurrently) * (number of notification types per cue) + (number of cues) * (number of notification types per cue)

When waves and sounds are auditioned, they use voices from the same pool the XACT engine uses for in-game sounds. It is important to provide enough voices to support both.

dwInteractiveAudioLookaheadTime is used only by XACT Interactive Audio. It defines the minimum time to wait before performing any type of transition other than Immediate to non-TimeAligned, and is the time that the engine will wait while the destination sound is being prepared or primed. This period is called the look-ahead time because it determines how far ahead the engine looks for loop ends and markers. The engine also uses this value when doing time-aligned transitions, by picking an effective transition time that is dwInteractiveAudioLookaheadTime milliseconds greater than the triggering transition time (again, in order to allow the destination sound time to prepare or be primed). Immediate to non-TimeAligned transitions ignore this value, making their transitions as soon as the destination sound is prepared. Titles should set this value based on the anticipated time needed to prepare or prime transitions. Zero milliseconds is definitely too short, and can cause dropouts when making transitions. For reference, the Audio Console Application auditioning tool (AudConsole) uses a value of 250 milliseconds.

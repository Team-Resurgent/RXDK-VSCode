Testing an Autoupdate Package

Selecting Test from the Action menu deploys the current autoupdate project to your development Xbox just as if the autoupdate was downloaded and installed. You can then test the package to ensure it works as expected.

Deployment for testing fails if the path to the original game build contains certification information that is not consistent with the autoupdate. This type of failure can be caused by the following:

  • Mismatched title IDs. The title IDs of the autoupdate and the original game must be the same. XLAST does not give a warning or error if the title IDs do not match.
  • Mismatched major version numbers. The major version numbers of the Xbox Title Libraries in both the original game and the autoupdate must match. The minor version numbers do not have to match.
  • No default.xbe. The file default.xbe is required.
  • Media Type bitmask is not set correctly. All updated XBEs must have the XBEIMAGE_MEDIA_TYPE_HARD_DISK bit set in the Media Type bitmask or Test Local fails. XLAST does not give a warning or error if the XBEIMAGE_MEDIA_TYPE_HARD_DISK bit is not set. For more information see the /TESTMEDIATYPES.

Note    The only time before certification that XLAST checks this certificate information is when deploying for testing.

After you have deployed the autoupdate package to your development Xbox, copy the original game build to the E:\ drive of the development Xbox. If you launch the program default.xbe from the original build and default.xbe from the autoupdate executes, the autoupdate was properly deployed. It is ready for testing.

When you are finished testing, you can delete the original build of the game and execute the Clean command from the XLAST Action menu. Both of these steps are optional.

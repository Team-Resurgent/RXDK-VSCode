Soundtrack Support on Xbox

The Xbox features the ability to allow users to rip tracks from their favorite CD's and store these tracks on a special section of the hard disk. Each track is stored as part of a soundtrack located in the Dashboard Music directory on the Xbox. These soundtracks can then be accessed via low-level API calls or high-level soundtrack support through the XACT engine.

Audio stored in tracks on the Xbox are in 44kHz, 128kbit WMA format.

  • Low-Level API Soundtrack Support
  • High-Level Soundtrack Support Using XACT

Low-Level API Soundtrack Support

The XDK features a set of low-level API calls that can be used to create and/or access existing user soundtracks from within an application. The following functions are available:

  • XFindFirstSoundtrack
  • XFindNextSoundtrack
  • XOpenSoundtrackSong
  • XGetSoundtrackSongInfo
  • XAddSoundtrack
  • XAddSongToSoundtrack

To play a soundtrack song, an XWmaFileMediaObject decoder must be created. The BackgroundMusic Sample and EnumSoundtrack Sample provide examples of how these APIs are commonly used.

High-Level Soundtrack Support Using XACT

XACT provides a high-level support mechanism when working with soundtracks. An IXACTSoundBank can be created by calling IXACTEngine::CreateSoundBank. In turn, a IXACTWmaPlayList may be created by calling IXACTSoundBank::CreateWmaPlayList. The IXACTWmaPlayList provides the following interfaces for manipulating soundtracks:

  • IXACTWmaPlayList::Add
  • IXACTWmaPlayList::GetCurrentSongInfo
  • IXACTWmaPlayList::Next
  • IXACTWmaPlayList::Previous
  • IXACTWmaPlayList::Remove
  • IXACTWmaPlayList::SetCurrent
  • IXACTWmaPlayList::SetPlaybackBehavior

To play a particular track, use the IXACTSoundBank::GetSoundCueIndexFromFriendlyName to retrieve the cue index, then call IXACTSoundBank::Play with that index.

See the XACTWMAPlayList Sample for an example of how to manage user-created soundtracks.

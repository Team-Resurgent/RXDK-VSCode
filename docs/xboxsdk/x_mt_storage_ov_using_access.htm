Accessing Storage Regions

Titles access the defined data regions through the namespace provided by the system. Because games running on the Xbox have essentially full control of the hardware, theoretically it is not possible to fully protect regions and portions of regions outside of a particular title's namespace. However, the region organization hierarchy, in combination with certification requirements, ensures that titles will not experience interference or adverse effects from other titles.

The basic method of maintaining title region separation is implemented using logical drives that are mapped to data regions specific to the title. The first time a title is launched on the console, subdirectories dedicated to that title are created within each of the global data regions that the title will eventually access. At initialization time, logical drives are mapped to the title's specific data region directories, and those drives are made available to the title. The root of the mapped logical drive points to the title's own area within that region. Thus, from the root of the mapped drive downward, the data area is dedicated to the title. Titles cannot see the data of other games, with the exception of those specifically allowed by the XMountAlternateTitle function.

The Storage Region Namespace

Each data region of the hard disk, each memory unit location, and the DVD player will be accessible to the title through specific logical drive letters defined by the system. Each drive letter is assigned a specific data region, and these assignments will not be changed. Those drive letters that do not have a region assignment are reserved for future use.

Drive Letter Region Assignments

The following table lists the drive letter data region assignments defined by the Xbox storage system.

Drive Letter Data Region
D: The DVD Player. On the development console only, if the title is launched from the hard disk, D:\ will be mapped to the directory that the title was launched from, rather than being mapped to the DVD player. Note that if a title launches another title located in a subdirectory of D:\, the mapping of D:\ will remain unchanged. It will point to the launch directory of the first title, not of the newly launched title.
E: This drive does not actually exist on the console. It is used solely by the Xbox Development Kit as the drive where development titles are copied to the Xbox development console for execution. This drive is visible only to the remote tools, not to titles.
F: — R: Dynamically allocated and/or reserved drives. Some drives in this region are used to map memory units.
S: On the development console only, the root directory of the persistent data region, under which each title's specific persistent data directory resides. This drive is visible only to the remote tools, not to titles.
T: The title's persistent data region.
U: The title's user data region on the hard disk.
V: On the development console only, the root directory of the user data region, under which each title's specific user data directory resides. This drive is visible only to the remote tools, not to titles.
W: Used to map another title's persistent data region.
X: Used to map another title's user data region (from disk or a memory unit).
Y: Unused
Z: The title's utility data region.

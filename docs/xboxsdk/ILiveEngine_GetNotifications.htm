ILiveEngine::GetNotifications

Determines pending notifications for a specified player.

HRESULT GetNotifications(
    DWORD Port,
    UIX_NOTIFICATION_PURPOSE Purpose,
    DWORD *pNotifications
);

Parameters

Port
[in] Controller port index (zero-based) of the player.
Purpose
[in] UIX_NOTIFICATION_PURPOSE value specifying the reason that notifications are being queried.
pNotifications

[out] Pointer to a DWORD variable that receives a set of flags indicating current notifications.
Value Description
UIX_DOWORK_NOTIFY_FRIEND_REQUEST A friends request has been received.
UIX_DOWORK_NOTIFY_GAME_INVITE A game invite has been received.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

GetNotifications returns the pending notifications for the specified player. Titles use this information to determine the notification icons that must be displayed in either of these places:

  • Next to the Friends option in the main start menu.
  • In the lower right corner of the screen for three seconds during gameplay.

UIX_NOTIFICATION_MENU notifications are cleared when the title calls the ILiveEngine::StartFeature method for the friends feature and the user views the Friends List.

UIX_NOTIFICATION_IN_GAME_FLASH notifications are cleared when the title retrieves them with GetNotifications. In-game notifications are not returned if the engine is handling them (which it does by default).

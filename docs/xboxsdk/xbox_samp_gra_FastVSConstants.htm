FastVSConstants Sample

This sample demonstrates techniques for performing fast vertex shading.

Overview

The FastVSConstants sample shows how to use the IDirect3DDevice8::BeginState and Direct3DDevice8::EndState functions to efficiently store calculated vertex shader constant data directly into the push-buffer. It also illustrates the importance of pre-fetching data in order to achieve maximum CPU performance.

Path

Source: (XDK Root)\Samples\Xbox\Graphics\VertexShader\FastVSConstants

Remarks

The sample draws 20,000 simple objects (tetrahedrons) per frame. The data needed to draw each object is pre-fetched to avoid cache misses. Pressing the A button toggles between the simple drawing loop and the optimized one. The optimized version is approximately 68 percent faster than the unoptimized version. Most of the speed increase (approximately 90 percent) is due to pre-fetching the data needed to draw each object. The rest (about 10 percent) can be attributed to inlining the matrix multiplication and writing the vertex shader constants directly into the push-buffer. Avoiding cache misses while writing data into write-combined memory such as the push-buffer is extremely important because a cache miss can interrupt the write combining resulting in additional bus cycles. The frame rate is 100 percent CPU bound when using the unoptimized drawing loop. The frame rate of the optimized drawing loop is GPU bound with the CPU usage at about 90 percent.

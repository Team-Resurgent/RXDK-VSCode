Samples

This document provides a brief description of each game sample for the Xbox® video game system from Microsoft®.


This section includes:

  • Base Framework Components – The base classes and helper functions for all samples.
  • Graphics – Samples that illustrate Microsoft® Direct3D® concepts.
  • Audio – Samples based on Microsoft® DirectSound®, Microsoft® DirectMusic®, and the Xbox Audio Creation Tool (XACT).
  • System – Samples that demonstrate XTL features.
  • Xbox Live and Networking – Samples that use Xbox Live and networking technology.
  • Debug Manager – Samples that show how to communicate with the target system via the debug channel.
  • Reference User Interface – Information about usability issues.
  • Streaming Media – Samples that use XMV to demonstrate video concepts.

Development System Versus Target System

Developing for the Xbox requires two systems: a development system and a target system (the console). Use your development system to run your development tools, such as the compiler and the debugger, and use your target system to run your executables.

Running the Samples

You can run the Xbox samples as soon as you receive your Xbox Development Kit (XDK). Six of the samples (Audio Console App, DolphinClassic, Gamepad, PlayField, Xbox Dashboard, and Xbox Live Setup Tool) and their required media are available from the XDK Launcher as soon as you use the recovery CD to install the latest Xbox System Software on your target system.

All Xbox samples are available on your development system, but you will need to build them (on the development system) before you can run them (on the target system). For details on installing and configuring your development and target systems, see Setting Up Your Console and Development System.

When you build the samples, they will be automatically copied over to the target system (via a call to XbCp in the post-build settings). Make sure to specify your target system (using "xbcp -x YourXboxName") before building any samples.

To exit a sample and return to the XDK Launcher, press and hold the left trigger and right trigger. Then press the Black key on the Xbox controller.

Note   The Xbox samples will not run on any machine other than the target system that ships with the XDK. The Xbox System Software has been specifically tailored to the XDK hardware.

"Xbox Game" Project Type

Once you have installed the Xbox SDK on your development system, the Microsoft® Visual C++® Integrated Development Environment (IDE) will offer an "Xbox Game" project type. When you select this new project type, the IDE generates an empty project for you. This empty project has the proper build settings to develop a game for Xbox using the Xbox SDK.

DirectX 8.0 Samples Versus Xbox Samples

There are several Xbox samples that were ported directly from the Microsoft® DirectX® 8.0 samples. The primary difference between the Xbox versions and the DirectX 8.0 versions are:

  • The Xbox samples don't use graphics device interface (GDI) code—no mouse, no menus, no GDI text.
  • The Xbox samples always run full-screen in one of the predefined Xbox video resolutions.
  • The keyboard and mouse code has been removed. All input for the Xbox samples is delivered via gamepads.
  • The sample code has been tweaked for the Xbox. All Xbox samples rely on the support code in the Common directory.
  • The Xbox samples are compiled to the Xbox executable (*.xbe) binary format and will run only on the XDK target system using the Xbox system software. The DirectX samples are compiled to standard Microsoft® Windows® executables (*.exe) and will run on any DirectX 8.0-enabled Windows system.
  • There may be minor visual improvements in some of the Xbox samples.

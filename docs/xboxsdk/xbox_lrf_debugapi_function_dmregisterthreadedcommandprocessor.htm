DmRegisterThreadedCommandProcessor

Registers a command handler and creates a new thread in which the command handler executes.

DMHRAPI DmRegisterThreadedCommandProcessor(
    LPCSTR szProcessor,
    PDM_CMDPROC pfn
);

Parameters

szProcessor
LPCSTR that represents the command processor.
pfn
Pointer to a command processor. Command processors must take the following form:
HRESULT __stdcall my_handler(LPCSTR szCommand, LPSTR szResponse, DWORD cchResponse, PDM_CMDCONT pdmcc);

Return Values

Returns XBDM_NOERR if successful, an error value otherwise.

Remarks

Command handlers registered by means of DmRegisterCommandProcessor cannot call CRT or XAPI functions. Threaded command handlers, registered by means of DmRegisterThreadedCommandProcessor, can call CRT and XAPI functions because they run in a thread separate from the global server thread. Threaded command handlers run only on XAPI threads, not on the global server thread.

When a title first opens a connection with DmOpenConnection, the title may use that connection only to invoke built-in commands or commands directed at non-threaded handlers. Any attempt to invoke a threaded handler results in the error XBDM_MUSTBEDEDICATED. To invoke a command directed at a threaded handler, the title must first dedicate that connection to a specific handler by calling DmDedicateConnection. The call to DmDedicateConnection should follow the title's call to DmOpenConnection.

Once a connection has been dedicated, the connection may be used only to invoke commands directed at the specific dedicated handler. Any attempt to use a dedicated connection to invoke either built-in commands or commands directed at non-threaded handlers results in the error XBDM_MUSTBEDEDICATED. A title may re-dedicate a connection to another handler at any time by calling DmDedicateConnection again. Passing NULL for the szHandler parameter to DmDedicateConnection re-dedicates the connection to the global server thread. Re-dedicating this connection allows the connection to once again be used for built-in commands.

See the Remarks section of DmRegisterCommandProcessor for further details on registering a command processor.

Messages with Attachments

Your title can attach nearly any type of data to messages using properties. Use the XOnlineMessageSetProperty function to store your attachment into a message property. The first parameter to XOnlineMessageSetProperty is the ID of the message. The wPropTag parameter contains a value indicating the type of data the property contains. The property data itself is passed in the pPropValue parameter. The data in the pPropValue parameter can be of any size up to the maximum size limit. Your game specifies the size of the data with the dwPropValueSize parameter.

Note    Properties over a certain threshold (4,096 bytes is the maximum size of the message details) should be stored on the storage server as attachments, and represented as storage URLs in the message details. These properties could contain things such as a large map, armor, or a ghost race. They are not retrieved along with other message details until the client explicitly requests them to be retrieved.

Examples of the uses of attachments include passing voice messages from player to player, passing text messages, sending competition information, or even exchanging game items (magic swords, ray guns, nanobot healing packs, and so on) between players. You are limited only by your imagination. If you want to pass your own custom types of attachments, you must create a custom property type. To do so, use the XONLINE_MSG_PROP_TAG macro with the XOnlineMessageSetProperty function. Note, however, that when attaching custom data to messages, you may find it easiest to just send it as a binary data block (a blob) using the value XONLINE_MSG_PROP_TYPE_BINARY with XONLINE_MSG_PROP_TAG. The content, use, and format of a binary blob is entirely up to you.

Use the XOnlineMessageDownloadAttachmentToDirectory or XOnlineMessageDownloadAttachmentToMemory function to download attachments. These are asynchronous tasks, so your title uses XOnlineTaskContinue to "pump" the message handle. To get the percentage of the download that has been completed, call XOnlineMessageDownloadAttachmentGetProgress. If the game is downloading to memory, your title calls XOnlineMessageDownloadAttachmentToMemoryGetResults to get the results when the download is finished. Close the task handle by calling XOnlineTaskClose.

Attachments and Voice Messaging

Your title can record audio data and attach it to a message in the same manner it attaches other types of data—with the XOnlineMessageSetProperty function. As a result, you have the ability to include voice messaging in your titles. Voice messaging makes it much simpler and easier for players to communicate messages to each other. It frees them from having to enter text for the message with an Xbox controller. Experience has shown that players find voice messaging more natural, quick, and convenient than sending text messages.

Note    The Xbox Live TCRs require all new games to support voice mail attachments for game invitations and friend requests.

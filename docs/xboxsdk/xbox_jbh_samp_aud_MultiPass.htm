MultiPass Sample

The MultiPass sample demonstrates how to use multipass audio by routing several mono DirectSound buffers to a 3D mixin buffer that is then positioned in 3D. One mono buffer is created for each of several sound effects, as well as one 3D mixin buffer. The mono buffers have their output set to be the 3D mixin buffer. Each mono buffer then has individual control over its volume and playback, but will be positioned along with all the other buffers at the position of the 3D buffer.

Overview

Sounds are created using DirectSound and the helper class in the XBSound.h and XBSound.cpp files. The helper class just provides easy access to the WAV file to get format information and read sample data from the file.

3D DirectSound buffers get sent to the crosstalk mixbins. Therefore, to get sound from those mixbins to the speakers, you must have a DSP image loaded that contains at least the crosstalk code. For information on how to do this, see the DownloadScratch function.

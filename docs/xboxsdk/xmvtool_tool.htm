Xbox Media Video Tool (XMVTool)

The XMVTool is used to import audio and video data from XMV, WMV, WMA, AVI, and WAV files and convert them into the Xbox Media Video (XMV) format.

To use XMVTool on Microsoft® Windows® 2000, you need to install Microsoft® Windows Media® 8 Encoding Utility (WM8EUTIL). You can download WM8EUTIL from the following location:

http://www.microsoft.com/windows/windowsmedia/WM8/encoding.aspx

You do not need to install Media Player to run XMVTool on Microsoft®Windows® XP.

USAGE: xmvtool [options]
Option Description
-d FileName Dumps information about the specific FileName. If this option is used, it must be the only option used.
-dl FileName Dumps detailed information about the specific FileName (long). If this option is used, it must be the only option used.
-s Dumps statistics about the resulting XMV file.
-p size Sets the packet size in kilobytes, from 4–512. size must be a multiple of four.
-t time Sets the minimum length of a packet in milliseconds.
-l time Sets the absolute length of the video in milliseconds. Using this option will cause the XMVTool to either truncate or pad all streams to achieve the desired movie length.
-i FileName Add all audio and video streams from FileName to the resulting XMV file.
-ic FileName Add all audio and video streams from from FileName to the resulting XMV file. All audio streams are compressed with ADPCM compression as they are added.
-v FileName Add only the video stream from FileName to the resulting XMV file.
-a FileName Add only the first audio stream from FileName to the resulting XMV file.
-ac FileName Input and compress the first audio stream from FileName.
-o FileName Create the resulting XMV file and name it FileName.
-? Displays a complete list of available options.

The -i, -v, and -a options can be used to build an XMV file out of multiple files. Specifying more than one video stream will result in an error. Multiple audio streams can be specified and they will be stored in the resulting XMV file in the same order in which they appear on the command line.

The -ic and -a options compress the audio streams using ADPCM compression and store it in the XMV file. These two switches have no effect on the video stream, which is always compressed.

The difference between -s and -d (or -dl) is that -s will only dump information about the resulting XMV file that would be produced. -d and -dl can be used to dump information about any file.

Only one output file can be specified using the -o option. If the output file is not specified, no XMV file will get created; however, the tool will display statistics about the file it would have created, just as though the -s option was used.

Below are several examples showing how the tool can be used.

Create an XMV file from a single source
xmvtool -i VideoSource.wmv -o MyVideo.xmv

The video and all of the audio streams from VideoSource.wmv will get repackaged into MyVideo.xmv.

Create an XMV file containing only video
xmvtool -v VideoSource.wmv -o MyVideo.xmv

Only the video stream from VideoSource.wmv will get repackaged into MyVideo.xmv.

Combine separate audio and video files into a single XMV file
xmvtool -v VideoSource.wmv -a AudioSource1.wma -a AudioSource2.wav -o MyVideo.xmv

This combines the video stream from VideoSource.wmv with the audio streams from two separate files, AudioSource1.wma and AudioSource2.wav. Only the first audio stream from each of the audio sources will get copied. Inside the destination XMV file, the audio streams will be numbered in the same order that they appeared on the command line. While only one video source can ever be specified, any number of audio streams can be added using the -a option.

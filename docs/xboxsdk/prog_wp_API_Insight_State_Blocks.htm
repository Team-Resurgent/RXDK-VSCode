API Insight: The Truth About State Blocks

by Mikey Wetzel, Software Design Engineer, Xbox Advanced Technology Group

Welcome to the next installment in the API Insight series that aims to expose some inner details of the more interesting Xbox API calls. This time around, we will be looking at state blocks and how they are implemented.

As the title implies, state blocks do have a dark side. The takeaway is that state blocks are implemented entirely in software, meaning that your application could do at least as good, if not better. Better because your application can make assumptions about your engine design, objects sorted by state, and so on, that would allow your state management code to easily exceed the performance of D3D state blocks.

The pseudo-code for state blocks is listed in Appendix A: Pseudo-code D3D8.H Header File and Appendix B: Pseudo-code State Blocks. The first listing is the public D3D8.h header file stripped down to code that is relevant only to state blocks. The second listing is for implementing state blocks, and will be referenced throughout this document. Please be well aware that the code provided is not actual source code, but is a close enough derivation of it to give you the gist of what's going on.

What Is a State Block?

Before we get too deep into the discussion, we need to make sure we all know what state blocks are and how they are used. The XDK documentation is the proper place for this, but I'm happy to give you the basics here.

In DirectX 8, state blocks are essentially objects that contain a snapshot of a large number of D3D states (render states, current textures, materials, lights, shaders, and so on). State blocks are created and destroyed with calls to D3DDevice::CreateStateBlock and D3DDevice::DeleteStateBlock.

When created, a state block allocates memory for space to save all the states and then proceeds to "capture" or record the system's current state. At any later time, the application can then call D3DDevice::ApplyStateBlock and all the previously set state will again be set. The advantage is having one API call versus many individual state settings calls. From an engine design point-of-view, you may already be thinking in terms of broader effects (like an alpha-only pass with alpha-blending on, z-testing on, and z-writes disabled) in which case it's more convenient to organize state into larger groupings.

An existing state block can be updated with a call to D3DDevice::UpdateStateBlock. This will replace the contents of the state block with a snapshot of the system state at the time of that call. This would be useful for dynamic state blocks where you are trying to do the following:

  1. Preserve state
  2. Set new state
  3. Render object
  4. Restore previous state

So far, you may be thinking that state blocks sound like a good thing. The problem is the rather brain dead way that state blocks are implemented and that they sometimes record the entire state of the device (literally hundreds of states) because they don't have the foresight to know which states will need to be restored later on.

Compiled State Blocks

Aside from the usual call to CreateStateBlock, another API set can be used to create and record into a state block. In order to use the other API, you need to add the following code:

#define D3DCOMPILE_BEGINSTATEBLOCK

before you include any Xbox header files. Take a look at the listing for the D3D8.h header file, and find any references to this preprocessor definition.

The definition enables a bunch of macros that update an internal array called D3D__StateBlockDirty[] whenever a state is changed. For performance reasons, the header file hides this work behind a conditional #ifdef so that the work can be avoided in the non–state block case.

The definition also exposes access to two functions: BeginStateBlock and EndStateBlock, which serve as an alternate way to create a state block. In between the two calls, your code would set a bunch of render states. Internally, this works with the macros mentioned above. The call to BeginStateBlock does little more than clear out this array. As your application makes calls to set various state, the macros update the D3D__StateBlockDirty[] array to record which states have been modified. The call to EndStateBlock then walks the list of dirty states and creates an actual state block. Afterwards, your application can call ApplyStateBlock, as usual.

The advantage of working with state blocks in this fashion is that just the minimum amount of state that you directly specify is included in the state block. The alternative of calling CreateStateBlock is a lot less optimal because it records the entire set of all device state...literally hundreds of states.

Deconstructing the Pseudo-Code

Let's now turn our attention to the pseudo-code. If you're reading this on a monitor, it'll really help here to have a separate printout for reference. To start, find the code for the D3DDevice_CreateStateBlock function.

Creating and Recording State Blocks

Inside CreateStateBlock, there are three code paths, depending on whether the caller wants to record vertex shader state, pixel shader state, or the entire set of states used by the device. Regardless, all code paths boil down to the same logic: for each relevant state, mark the state's entry in the D3D__StateBlockDirty[] array as dirty. The DIRTY_RENDERSTATE and DIRTY_TEXTURESTATE helper macros simply update this array. Note that lights are special cased, since there can be an arbitrary number of them.

The function ends by passing control to the internal RecordStateBlock function. You may want to compare this to the BeginStateBlock/EndStateBlock functions, which do mostly the same thing, except they rely on the caller to dirty a small number individual states rather than dirtying all the states at once.

RecordStateBlock is a lengthy function, but it breaks down into three easily digestible pieces:

  1. Walk through all dirty state, and calculate how much space is needed to store all the dirty states.
  2. Allocate an object with enough space to hold all the state information
  3. Walk through all dirty state again, but this time actually record the state into the object

The object mentioned in step 2 is an internal, variable length StateBlock structure, defined near the top of the pseudo-source code. The header part of this structure is just a count of the states to follow. For instance, if the state block recorded seven vertex shader constants, then that would be reflected in the dwNumVertexShaderConstants field. The latter part of the structure is a variable length Data array, which is filled with actual state values during step 3 above. Since this data is treated as one big DWORD array, the order that state is read from this array must match the order in which the array was written to.

Even with a small, efficient list of dirty states, it's clear that RecordStateBlock makes no assumptions about what to expect, and must traverse a large list of values to see what's dirty and what to record. Armed with pseudo-code in hand, I'm sure you could do better. Fortunately, recording into state blocks is typically a seldom-performed operation.

Capturing a State Block

Once a state block is created, you will likely need to update it at some later time. The UpdateStateBlock function is used for this, and it has the nice advantage of only updating states that were initially tagged as dirty during creation. Of course, if you called CreateStateBlock then this means all states were tagged. However, if you used BeginStateBlock/EndStateBlock, then your list of tagged state blocks is probably a bit more manageable.

CaptureStateBlock uses the count and existence information in the StateBlock structure's header to only consider the relevant states. Look at the pseudo-code for this function and you will see that the implementation is straightforward. For a concrete example, imagine that only the texture on stage 2 was set. Well, the dwNumTextures in this case would be one. The texture loop would then extract the stage and texture pointer from the Data field and apply the new texture, complete with reference counting. It's that simple, although the large number of conditional checks, pointer arithmetic, and reference counting is hardly optimal.

Note    The pseudo-code calls are making use of an internal g_pDevice object your code will not have access to. Don't be concerned by this, as this is just pseudo-code and was never intended to compile. The functionality should still be obvious, and that's all we care about.

Applying a State Block

Finally, let's look at the code to apply a state block. The code for ApplyStateBlock actually looks very similar to CaptureStateBlock. The logic flow and loops are exactly the same, the only difference being that CaptureStateBlock updates the state block with current state, while ApplyStateBlock just puts the state into action. Neither function does anything magical; you could easily write similar code.

Reasons to Use State Blocks

After all the state block bashing above, allow me to contradict myself and point out a few times where using state blocks may be justified. First and foremost is convenience. If state blocks help shrink your development time, and you really aren't interested in squeezing every bit of performance out of the Xbox, then state blocks are probably acceptable as they are. Second, if you are maintaining an engine that is cross-platform for both Xbox and the PC, then any amount of common code is probably a win. I would argue that state blocks are probably no better on the PC then on the Xbox, but if some PC hardware out there actually supported state blocks in hardware, then my argument would fall apart. Of course, I really don't like any of these answers, but at some point I have to acknowledge that game developers have a game to ship.

Conclusion

State blocks may come in handy and they definitely give the illusion of being simple and straightforward. There is no free lunch, though; the truth of the matter is that the implementation of state blocks involves a large number of conditional checks, pointer arithmetic, and reference counting that could easily be avoided by an application that had a better sense of what states it was interested in preserving.

Appendix B: Pseudo-code State Blocks

This appendix contains pseudo-code for how state blocks are implemented on the Xbox.

Please note that this code will not compile. It's only provided for educational purposes, and any other use may cause temporary insanity.

//-----------------------------------------------------------------------------
// File: Block.cpp
//
// Desc: State block implementation.
//
//       The whole goal of the Xbox state block implementation is to add as
//       little overhead to non-state-block functionality as is possible.
//       Specifically, we have gone out of our way to avoid adding any
//       run-time checks for 'record' mode to any of our state-setting
//       routines such as SetRenderState.
//
// Copyright (C) Microsoft Corporation.  All Rights Reserved.
//-----------------------------------------------------------------------------


//-----------------------------------------------------------------------------
// Name: D3D__StateBlockDirty[]
// Desc: An array of bytes that correspond to changed state between
//       BeginStateBlock() and EndStateBlock() calls.
//-----------------------------------------------------------------------------
BYTE D3D__StateBlockDirty[(D3DSBD_MAX + 3) & ~3];

// Make sure the macros we need are defined.
#ifndef D3DCOMPILE_BEGINSTATEBLOCK

    #define DIRTY_RENDERSTATE(state) \
    { D3D__StateBlockDirty[D3DSBD_RENDERSTATES + state] = TRUE; }

    #define DIRTY_TEXTURESTATE(stage, state) \
    { D3D__StateBlockDirty[D3DSBD_TEXTURESTATES + (state*D3DTSS_MAXSTAGES) + stage] = TRUE; } 

#endif



//-----------------------------------------------------------------------------
// Name: struct StateBlock
// Desc: Structure that we allocate to replay state-blocks when called by
//       ApplyStateBlock().
//-----------------------------------------------------------------------------
struct StateBlock
{
    // Note: Each of the following fields represent the count of their
    //       corresponding entries in the variably formatted 'Data' array.
    //
    // Note: The order of the variably formatted 'Data' array also
    //       matches the order in which these fields are listed.
    //
    // Note: Shaders require that the restore order be:
    //          1. Shader
    //          2. Shader constants
    //          3. Shader render states

    DWORD dwNumTextures;             
    BOOL  bHasPixelShader;
    BOOL  bHasVertexShader;
    BOOL  bHasIndexBuffer;
    DWORD dwNumVertexStreams;
    DWORD dwNumPixelShaderConstants;
    DWORD dwNumVertexShaderConstants;
    DWORD dwNumRenderStates;         
    DWORD dwNumTextureStates;   
    DWORD dwNumTransforms;           
    BOOL  bHasViewport;             
    BOOL  bHasMaterial;             
    BOOL  bHasBackMaterial;             
    DWORD dwNumLights;               
    DWORD dwNumLightEnables;         

    // Variable length, and variably formatted data, based on the above:
    DWORD Data[1];
};



//-----------------------------------------------------------------------------
// Name: D3DDevice_CreateStateBlock()
// Desc: Looks at the current state and captures a bunch of it.
// Note: This code does not record palette contents.
//-----------------------------------------------------------------------------
HRESULT WINAPI D3DDevice_CreateStateBlock( D3DSTATEBLOCKTYPE Type,
                                           DWORD* pToken ) 
{ 
    if( Type == D3DSBT_ALL )
    {
        // Dirty ALL the state-block dirty flags, so that everything
        // is recorded:
        FillMemory( &D3D__StateBlockDirty[0], sizeof(D3D__StateBlockDirty), 
                    0xffffffff );

        for( DWORD i = 0; i < g_pDevice->m_LightCount; i++ )
        {
             if( g_pDevice->m_pLights[i].Flags & LIGHT_SET )
             {
                 g_pDevice->m_pLights[i].Flags |= (LIGHT_STATEBLOCK_SET|
                                                   LIGHT_STATEBLOCK_ENABLED);
             }
        }
    }
    else if( Type == D3DSBT_PIXELSTATE )
    {
        // First reset all flags, so that no extraneous stuff gets into
        // our state block:
        ClearStateBlockFlags();

        // We have to record the current pixel shader program, all
        // pixel shader constants, and pixel shader related render
        // states and texture stage states.

        D3D__StateBlockDirty[D3DSBD_PIXELSHADER] = TRUE;

        FillMemory( &D3D__StateBlockDirty[D3DSBD_PIXELSHADERCONSTANTS],
                    D3DPS_CONSTREG_MAX_DX8, 0xffffffff );

        DIRTY_RENDERSTATE( D3DRS_ZENABLE );
        DIRTY_RENDERSTATE( D3DRS_FILLMODE );
        DIRTY_RENDERSTATE( D3DRS_SHADEMODE );
        DIRTY_RENDERSTATE( D3DRS_ZWRITEENABLE );
        DIRTY_RENDERSTATE( D3DRS_ALPHATESTENABLE );
        DIRTY_RENDERSTATE( D3DRS_SRCBLEND );
        DIRTY_RENDERSTATE( D3DRS_DESTBLEND );
        DIRTY_RENDERSTATE( D3DRS_ZFUNC );
        DIRTY_RENDERSTATE( D3DRS_ALPHAREF );
        DIRTY_RENDERSTATE( D3DRS_ALPHAFUNC );
        DIRTY_RENDERSTATE( D3DRS_DITHERENABLE );
        DIRTY_RENDERSTATE( D3DRS_FOGSTART );
        DIRTY_RENDERSTATE( D3DRS_FOGEND );
        DIRTY_RENDERSTATE( D3DRS_FOGDENSITY );
        DIRTY_RENDERSTATE( D3DRS_EDGEANTIALIAS );
        DIRTY_RENDERSTATE( D3DRS_ALPHABLENDENABLE );
        DIRTY_RENDERSTATE( D3DRS_ZBIAS );
        DIRTY_RENDERSTATE( D3DRS_STENCILENABLE );
        DIRTY_RENDERSTATE( D3DRS_STENCILFAIL );
        DIRTY_RENDERSTATE( D3DRS_STENCILZFAIL );
        DIRTY_RENDERSTATE( D3DRS_STENCILPASS );
        DIRTY_RENDERSTATE( D3DRS_STENCILFUNC );
        DIRTY_RENDERSTATE( D3DRS_STENCILREF );
        DIRTY_RENDERSTATE( D3DRS_STENCILMASK );
        DIRTY_RENDERSTATE( D3DRS_STENCILWRITEMASK );
        DIRTY_RENDERSTATE( D3DRS_TEXTUREFACTOR );
        DIRTY_RENDERSTATE( D3DRS_WRAP0 );
        DIRTY_RENDERSTATE( D3DRS_WRAP1 );
        DIRTY_RENDERSTATE( D3DRS_WRAP2 );
        DIRTY_RENDERSTATE( D3DRS_WRAP3 );
        DIRTY_RENDERSTATE( D3DRS_COLORWRITEENABLE );
        DIRTY_RENDERSTATE( D3DRS_BLENDOP );

        for( DWORD i = 0; i < D3DTSS_MAXSTAGES; i++ )
        {
            DIRTY_TEXTURESTATE( i, D3DTSS_COLOROP );
            DIRTY_TEXTURESTATE( i, D3DTSS_COLORARG1 );
            DIRTY_TEXTURESTATE( i, D3DTSS_COLORARG2 );
            DIRTY_TEXTURESTATE( i, D3DTSS_ALPHAOP );
            DIRTY_TEXTURESTATE( i, D3DTSS_ALPHAARG1 );
            DIRTY_TEXTURESTATE( i, D3DTSS_ALPHAARG2 );
            DIRTY_TEXTURESTATE( i, D3DTSS_BUMPENVMAT00 );
            DIRTY_TEXTURESTATE( i, D3DTSS_BUMPENVMAT01 );
            DIRTY_TEXTURESTATE( i, D3DTSS_BUMPENVMAT10 );
            DIRTY_TEXTURESTATE( i, D3DTSS_BUMPENVMAT11 );
            DIRTY_TEXTURESTATE( i, D3DTSS_TEXCOORDINDEX );
            DIRTY_TEXTURESTATE( i, D3DTSS_ADDRESSU );
            DIRTY_TEXTURESTATE( i, D3DTSS_ADDRESSV );
            DIRTY_TEXTURESTATE( i, D3DTSS_BORDERCOLOR );
            DIRTY_TEXTURESTATE( i, D3DTSS_MAGFILTER );
            DIRTY_TEXTURESTATE( i, D3DTSS_MINFILTER );
            DIRTY_TEXTURESTATE( i, D3DTSS_MIPFILTER );
            DIRTY_TEXTURESTATE( i, D3DTSS_MIPMAPLODBIAS );
            DIRTY_TEXTURESTATE( i, D3DTSS_MAXMIPLEVEL );
            DIRTY_TEXTURESTATE( i, D3DTSS_MAXANISOTROPY );
            DIRTY_TEXTURESTATE( i, D3DTSS_BUMPENVLSCALE );
            DIRTY_TEXTURESTATE( i, D3DTSS_BUMPENVLOFFSET );
            DIRTY_TEXTURESTATE( i, D3DTSS_TEXTURETRANSFORMFLAGS );
            DIRTY_TEXTURESTATE( i, D3DTSS_ADDRESSW );
            DIRTY_TEXTURESTATE( i, D3DTSS_COLORARG0 );
            DIRTY_TEXTURESTATE( i, D3DTSS_ALPHAARG0 );
            DIRTY_TEXTURESTATE( i, D3DTSS_RESULTARG );
        }
    }
    else if( Type == D3DSBT_VERTEXSTATE )
    {
        // First reset all flags, so that no extraneous stuff gets into
        // our state block:
        ClearStateBlockFlags();

        // We have to record the current vertex shader program, all
        // vertex shader constants, vertex shader related render
        // states and texture stage states, and lights:

        D3D__StateBlockDirty[D3DSBD_VERTEXSHADER] = TRUE;

        FillMemory( &D3D__StateBlockDirty[D3DSBD_VERTEXSHADERCONSTANTS],
                    D3DVS_CONSTREG_COUNT_XBOX, 0xffffffff );

        DIRTY_RENDERSTATE( D3DRS_SHADEMODE );
        DIRTY_RENDERSTATE( D3DRS_SPECULARENABLE );
        DIRTY_RENDERSTATE( D3DRS_CULLMODE );
        DIRTY_RENDERSTATE( D3DRS_FOGENABLE );
        DIRTY_RENDERSTATE( D3DRS_FOGCOLOR );
        DIRTY_RENDERSTATE( D3DRS_FOGTABLEMODE );
        DIRTY_RENDERSTATE( D3DRS_FOGSTART );
        DIRTY_RENDERSTATE( D3DRS_FOGEND );
        DIRTY_RENDERSTATE( D3DRS_FOGDENSITY );
        DIRTY_RENDERSTATE( D3DRS_RANGEFOGENABLE );
        DIRTY_RENDERSTATE( D3DRS_AMBIENT );
        DIRTY_RENDERSTATE( D3DRS_COLORVERTEX );
        DIRTY_RENDERSTATE( D3DRS_LIGHTING );
        DIRTY_RENDERSTATE( D3DRS_NORMALIZENORMALS );
        DIRTY_RENDERSTATE( D3DRS_LOCALVIEWER );
        DIRTY_RENDERSTATE( D3DRS_EMISSIVEMATERIALSOURCE );
        DIRTY_RENDERSTATE( D3DRS_AMBIENTMATERIALSOURCE );
        DIRTY_RENDERSTATE( D3DRS_DIFFUSEMATERIALSOURCE );
        DIRTY_RENDERSTATE( D3DRS_SPECULARMATERIALSOURCE );
        DIRTY_RENDERSTATE( D3DRS_VERTEXBLEND );
        DIRTY_RENDERSTATE( D3DRS_POINTSIZE );
        DIRTY_RENDERSTATE( D3DRS_POINTSIZE_MIN );
        DIRTY_RENDERSTATE( D3DRS_POINTSPRITEENABLE );
        DIRTY_RENDERSTATE( D3DRS_POINTSCALEENABLE );
        DIRTY_RENDERSTATE( D3DRS_POINTSCALE_A );
        DIRTY_RENDERSTATE( D3DRS_POINTSCALE_B );
        DIRTY_RENDERSTATE( D3DRS_POINTSCALE_C );
        DIRTY_RENDERSTATE( D3DRS_MULTISAMPLEANTIALIAS );
        DIRTY_RENDERSTATE( D3DRS_MULTISAMPLEMASK );
        DIRTY_RENDERSTATE( D3DRS_PATCHEDGESTYLE );
        DIRTY_RENDERSTATE( D3DRS_PATCHSEGMENTS );
        DIRTY_RENDERSTATE( D3DRS_POINTSIZE_MAX );
        DIRTY_RENDERSTATE( D3DRS_DEPTHCLIPCONTROL );

        for( DWORD i = 0; i < D3DTSS_MAXSTAGES; i++ )
        {
            DIRTY_TEXTURESTATE( i, D3DTSS_TEXCOORDINDEX );
            DIRTY_TEXTURESTATE( i, D3DTSS_TEXTURETRANSFORMFLAGS );
        }

        for( DWORD i = 0; i < g_pDevice->m_LightCount; i++ )
        {
            if( g_pDevice->m_pLights[i].Flags & LIGHT_SET )
            {
                g_pDevice->m_pLights[i].Flags |= (LIGHT_STATEBLOCK_SET|
                                                  LIGHT_STATEBLOCK_ENABLED);
            }
        }
    }
    else
        return E_INVALIDARG;

    // Finally, have RecordStateBlock do all the hard work of actually 
    // recording the state:
    return RecordStateBlock( pToken );
}



//-----------------------------------------------------------------------------
// Name: D3DDevice_DeleteStateBlock()
// Desc: 
//-----------------------------------------------------------------------------
VOID WINAPI D3DDevice_DeleteStateBlock( DWORD Token ) 
{ 
    StateBlock* pBlock = (StateBlock*) Token;
    DWORD*      pData  = &pBlock->Data[0];

    // Release any referenced objects:
    for( DWORD i = 0; i < pBlock->dwNumTextures; i++ )
    {
        DWORD           stage    = pData[0];
        D3DBaseTexture* pTexture = (D3DBaseTexture*)pData[1];
        pData += 2;

        if( pTexture )
            pTexture->Release();
    }


    if( pBlock->bHasPixelShader )
    {
        DWORD handle = pData[0];
        pData++;

        if( handle )
            // Note this is a reference-counted delete:
            g_pDevice->DeletePixelShader(handle);
    }

    if( pBlock->bHasVertexShader )
    {
        DWORD handle = pData[0];
        pData++;

        if( IsVertexProgram(handle) )
            // Note this is a reference-counted delete:
            g_pDevice->DeleteVertexShader(handle);
    }

    if( pBlock->bHasIndexBuffer )
    {
        D3DIndexBuffer* pIndexBuffer = (D3DIndexBuffer*)pData[1];
        pData += 2;

        if( pIndexBuffer )
            pIndexBuffer->Release();
    }

    for( i = 0; i < pBlock->dwNumVertexStreams; i++ )
    {
        D3DVertexBuffer* pVertexBuffer = (D3DVertexBuffer*)pData[2];
        pData += 3;

        if( pVertexBuffer )
            pVertexBuffer->Release();
    }

    // Free our state block memory:
    MemFree( pBlock );
}



//-----------------------------------------------------------------------------
// Name: D3DDevice_BeginStateBlock()
// Desc: Gets ready to start recording a state block.
//-----------------------------------------------------------------------------
VOID WINAPI D3DDevice_BeginStateBlock() 
{ 
    g_pDevice->m_StateFlags |= STATE_RECORDBLOCK;

    ClearStateBlockFlags();
}



//-----------------------------------------------------------------------------
// Name: D3DDevice_EndStateBlock()
// Desc: Here we run through our 'dirty' list and record any changed states into
//       a variable-length, variably formatted allocation.
//-----------------------------------------------------------------------------
HRESULT WINAPI D3DDevice_EndStateBlock( DWORD* pToken ) 
{
    g_pDevice->m_StateFlags &= ~STATE_RECORDBLOCK;

    return RecordStateBlock( pToken );
}



//-----------------------------------------------------------------------------
// Name: D3DDevice_CaptureStateBlock()
// Desc: Update the values in the block (weird lights being the exception)
//-----------------------------------------------------------------------------
VOID WINAPI D3DDevice_CaptureStateBlock( DWORD Token ) 
{ 
    StateBlock* pBlock = (StateBlock*)Token;
    DWORD*      pData  = &pBlock->Data[0];

    for( DWORD i = 0; i < pBlock->dwNumTextures; i++ )
    {
        DWORD           stage    = pData[0];
        D3DBaseTexture* pTexture = (D3DBaseTexture*)pData[1];

        if( pTexture )
            pTexture->Release();

        pTexture = g_pDevice->m_Textures[stage];
        if( pTexture )
            pTexture->AddRef();

        pData[1] = (DWORD)pTexture;
        pData += 2;
    }

    if( pBlock->bHasPixelShader )
    {
        DWORD handle = pData[0];

        if( handle )
            g_pDevice->DeletePixelShader(handle);

        handle = (DWORD)g_pDevice->m_pPixelShader;
        if( handle )
            g_pDevice->m_pPixelShader->AddRef();

        pData[0] = handle;
        pData++;
    }

    if( pBlock->bHasVertexShader )
    {
        DWORD handle = pData[0];

        if( IsVertexProgram(handle) )
            g_pDevice->DeleteVertexShader(handle);

        handle = g_pDevice->m_VertexShaderHandle;
        if( IsVertexProgram(handle) )
            g_pDevice->m_pVertexShader->AddRef();

        pData[0] = handle;
        pData++;
    }
    
    if( pBlock->bHasIndexBuffer )
    {
        D3DIndexBuffer* pIndexBuffer = (D3DIndexBuffer*)pData[1];

        if( pIndexBuffer )
            pIndexBuffer->Release();

        pIndexBuffer = g_pDevice->m_pIndexBuffer;
        if( pIndexBuffer )
            pIndexBuffer->AddRef();

        pData[0] = g_pDevice->m_IndexBase;
        pData[1] = (DWORD) pIndexBuffer;
        pData += 2;
    }

    for( DWORD i = 0; i < pBlock->dwNumVertexStreams; i++ )
    {
        DWORD            index         = pData[0];
        D3DVertexBuffer* pVertexBuffer = (D3DVertexBuffer*)pData[2];

        if( pVertexBuffer )
            pVertexBuffer->Release();

        pVertexBuffer = g_Stream[index].pVertexBuffer;
        if( pVertexBuffer )
            pVertexBuffer->AddRef();

        pData[1] = g_Stream[index].Stride;
        pData[2] = (DWORD) pVertexBuffer;
        pData += 3;
    }

    for( DWORD i = 0; i < pBlock->dwNumPixelShaderConstants; i++ )
    {
        memcpy( &pData[1], &g_pDevice->m_PixelShaderConstants[pData[0]][0],
                4*sizeof(FLOAT) );
        pData += 5;
    }
    
    for( DWORD i = 0; i < pBlock->dwNumVertexShaderConstants; i++ )
    {
        memcpy( &pData[1], &g_VertexShaderConstants[pData[0]][0], 4*sizeof(FLOAT) );
        pData += 5;
    }

    for( DWORD i = 0; i < pBlock->dwNumRenderStates; i++ )
    {
        pData[1] = D3D__RenderState[pData[0]];
        pData += 2;
    }

    for( DWORD i = 0; i < pBlock->dwNumTextureStates; i++ )
    {
        pData[2] = D3D__TextureState[pData[0]][pData[1]];
        pData += 3;
    }

    for( DWORD i = 0; i < pBlock->dwNumTransforms; i++ )
    {
        *((D3DMATRIX*) &pData[1]) = g_pDevice->m_Transform[pData[0]];
        pData += 1 + sizeof(D3DMATRIX) / 4;
    }

    if( pBlock->bHasViewport )
    {
        *((D3DVIEWPORT8*)&pData[0]) = g_pDevice->m_Viewport;
        pData += sizeof(D3DVIEWPORT8) / 4;
    }

    if( pBlock->bHasMaterial )
    {
        *((D3DMATERIAL8*)&pData[0]) = g_pDevice->m_Material;
        pData += sizeof(D3DMATERIAL8) / 4;
    }
    
    if( pBlock->bHasBackMaterial )
    {
        *((D3DMATERIAL8*)&pData[0]) = g_pDevice->m_BackMaterial;
        pData += sizeof(D3DMATERIAL8) / 4;
    }
    
    for( DWORD i = 0; i < pBlock->dwNumLights; i++ )
    {
        *((D3DLIGHT8*)&pData[1]) = g_pDevice->m_pLights[pData[0]].Light8;
        pData += 1 + sizeof(D3DLIGHT8) / 4;
    }
    for( DWORD i = 0; i < pBlock->dwNumLightEnables; i++ )
    {
        g_pDevice->GetLightEnable( pData[0], (BOOL*)&pData[1] );
        pData += 2;
    }

    ASSERT(pData == pBlock->pDataEnd);
}



//-----------------------------------------------------------------------------
// Name: D3DDevice_ApplyStateBlock()
// Desc: Plays back a state block.
//-----------------------------------------------------------------------------
VOID WINAPI D3DDevice_ApplyStateBlock( DWORD Token )
{ 
    StateBlock* pBlock = (StateBlock*) Token;
    DWORD*      pData  = &pBlock->Data[0];

    // Handle textures
    for( DWORD i = 0; i < pBlock->dwNumTextures; i++ )
    {
        DWORD           dwStage  = *pData++;
        D3DBaseTexture* pTexture = (D3DBaseTexture*)*pData++;
        g_pDevice->SetTexture( dwStage, pTexture );
    }

    // Handle pixel shader
    if( pBlock->bHasPixelShader )
    {
        DWORD handle = *pData++;
        g_pDevice->SetPixelShader( handle );
    }

    // Handle vertex shader
    if( pBlock->bHasVertexShader )
    {
        DWORD handle = *pData++;
        g_pDevice->SetVertexShader( handle );
    }

    // Handle indices
    if( pBlock->bHasIndexBuffer )
    {
        DWORD           dwIndexBase  = *pData++;
        D3DIndexBuffer* pIndexBuffer = (D3DIndexBuffer*)*pData++;

        g_pDevice->SetIndices( pIndexBuffer, dwIndexBase );
    }

    // Handle vertex streams
    for( DWORD i = 0; i < pBlock->dwNumVertexStreams; i++ )
    {
        DWORD            index         = *pData++;
        DWORD            stride        = *pData++;
        D3DVertexBuffer* pVertexBuffer = (D3DVertexBuffer*)*pData++;

        g_pDevice->SetStreamSource( index, pVertexBuffer, stride );
    }

    // Handle pixel shader constants
    for( DWORD i = 0; i < pBlock->dwNumPixelShaderConstants; i++ )
    {
        DWORD index         = *pData++;
        VOID* pConstantData = pData;
        pData += 4;

        g_pDevice->SetPixelShaderConstant( index, pConstantData, 1 );
    }

    // Handle vertex shader constants
    for( DWORD i = 0; i < pBlock->dwNumVertexShaderConstants; i++ )
    {
        DWORD index         = *pData++;
        VOID* pConstantData = pData;
        pData += 4;

        // In our state block code here, we refer to them as constants 0 to 191,
        // but SetVertexShaderConstant knows of them as -96 to 95:
        g_pDevice->SetVertexShaderConstant( index - 96, pConstantData, 1 );
    }

    // Handle render states
    for( DWORD i = 0; i < pBlock->dwNumRenderStates; i++ )
    {
        DWORD state = *pData++;
        DWORD value = *pData++;

        // On free builds, we do a small optimization where we don't bother
        // to call SetRenderState if the state is already set to the proper
        // value.  We don't do this on checked builds as a small little
        // sanity check to ensure that all of our default values would 
        // actually get through the SetRenderState parameter checking.

        if (D3D__RenderState[state] != value)
        {
            g_pDevice->SetRenderStateNotInline((D3DRENDERSTATETYPE) state, value);
        }
    }

    // Handle texture stage states
    for( DWORD i = 0; i < pBlock->dwNumTextureStates; i++ )
    {
        DWORD stage = *pData++;
        DWORD state = *pData++;
        DWORD value = *pData++;

        if( D3D__TextureState[stage][state] != value )
        {
            g_pDevice->SetTextureStageStateNotInline( stage,
                                         (D3DTEXTURESTAGESTATETYPE)state, value );
        }
    }

    // Handle transforms
    for( DWORD i = 0; i < pBlock->dwNumTransforms; i++ )
    {
        DWORD      state   = *pData;
        D3DMATRIX* pMatrix = (D3DMATRIX*)pData;
        pData += sizeof(D3DMATRIX) / 4;

        g_pDevice->SetTransform( (D3DTRANSFORMSTATETYPE)state, pMatrix );
    }

    // Handle viewport
    if( pBlock->bHasViewport )
    {
        D3DVIEWPORT8* pViewport = (D3DVIEWPORT8*)pData;
        pData += sizeof(D3DVIEWPORT8) / 4;

        g_pDevice->SetViewport( pViewport );
    }

    // Handle material
    if( pBlock->bHasMaterial )
    {
        D3DMATERIAL8* pMaterial = (D3DMATERIAL8*)pData;
        pData += sizeof(D3DMATERIAL8) / 4;

        g_pDevice->SetMaterial( pMaterial );
    }
    // Handle back material
    if( pBlock->bHasBackMaterial )
    {
        D3DMATERIAL8* pMaterial = (D3DMATERIAL8*)pData;
        pData += sizeof(D3DMATERIAL8) / 4;

        g_pDevice->SetBackMaterial( pMaterial );
    }

    // Handle lights
    for( DWORD i = 0; i < pBlock->dwNumLights; i++ )
    {
        DWORD      index  = *pData++;
        D3DLIGHT8* pLight = (D3DLIGHT8*)pData;
        pData += sizeof(D3DLIGHT8) / 4;

        g_pDevice->SetLight( index, pLight );
    }

    // Handle light enables
    for( DWORD i = 0; i < pBlock->dwNumLightEnables; i++ )
    {
        DWORD index  = *pData++;
        BOOL  enable = *pData++;

        g_pDevice->LightEnable( index, enable );
    }

    ASSERT(pData == pBlock->pDataEnd);
}



//-----------------------------------------------------------------------------
// Name: ClearStateBlockFlags()
// Desc: Clears all the 'dirty' flags we use to tell when a particular state
//       gets modified.
//-----------------------------------------------------------------------------
VOID ClearStateBlockFlags()
{
    ZeroMemory( &D3D__StateBlockDirty[0], sizeof(D3D__StateBlockDirty) );

    // Lights are special because there can be an arbitrary number of them,
    // so their 'modified' status is internally kept directly in their
    // structures:
    for( DWORD i = 0; i < g_pDevice->m_LightCount; i++ )
    {
        g_pDevice->m_pLights[i]->Flags & = ~( LIGHT_STATEBLOCK_SET | 
                                             LIGHT_STATEBLOCK_ENABLED );
    }
}



//-----------------------------------------------------------------------------
// Name: RecordStateBlock()
// Desc: Here we run through our 'dirty' list and record any changed states
//       into a variable-length, variably formatted allocation.
//-----------------------------------------------------------------------------
HRESULT RecordStateBlock( DWORD* pToken ) 
{ 
    (*pToken) = NULL;

    // We always do two passes.  The first pass determines the size of
    // the state block allocation we'll need, and the second pass fills
    // in the state block:





    //-------------------------------------------------------------------------
    // For the first pass, determine the size (in DWORDs) of the state block
    //-------------------------------------------------------------------------

    BYTE* pDirty = &D3D__StateBlockDirty[0];
    DWORD dwSize = 0;

    // Handle textures
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_TEXTURES]);
    for( DWORD i = 0; i < D3DTSS_MAXSTAGES; i++ )
    {
        if( *pDirty++ )
            dwSize += 2;
    }

    // Handle pixel shader states
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_PIXELSHADER]);
    if( *pDirty++ )
        dwSize += 1;

    // Handle vertex shader states
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_VERTEXSHADER]);
    if( *pDirty++ )
        dwSize += 1;

    // Handle indices
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_INDICES]);
    if( *pDirty++ )
        dwSize += 2;

    // Handle vertex streams
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_STREAMS]);
    for( DWORD i = 0; i < D3DVS_STREAMS_MAX_V1_0; i++ )
    {
        if( *pDirty++ )
            dwSize += 3;
    }

    // Handle the pixel shader constants
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_PIXELSHADERCONSTANTS]);
    for( DWORD i = 0; i < D3DPS_CONSTREG_MAX_DX8; i++ )
    {
        if( *pDirty++ )
        {
            // There's no point in saving the pixel shader constants if no
            // pixel shader is active:
            if( g_pDevice->m_pPixelShader )
                dwSize += 5;
        }
    }

    // Handle vertex shader constants
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_VERTEXSHADERCONSTANTS]);
    for( DWORD i = 0; i < D3DVS_CONSTREG_COUNT_XBOX; i++ )
    {
        if( *pDirty++ )
        {
            // We never save the vertex shader constants that map from -96
            // to -1, although we leave space for them in the 'dirty' array
            // for future expansion:
            if( i >= 96 )
                dwSize += 5;
        }
    }
 
    // Handle render states
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_RENDERSTATES]);
    for( DWORD i = 0; i < D3DRS_MAX; i++ )
    {
        if( *pDirty++ )
        {
            // If the fixed function pipeline is active, the shadowed
            // render states for the pixel shader registers are garbage,
            // so don't save those:
            if( (g_pDevice->m_pPixelShader != NULL) ||
                (i < D3DRS_PSALPHAINPUTS0) ||
                ( (i > D3DRS_PSINPUTTEXTURE) && (i != D3DRS_PSTEXTUREMODES) ) )
            {
                dwSize += 2;
            }
        }
    }

    // Handle texture stage states
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_TEXTURESTATES]);
    for( DWORD j = 0; j < D3DTSS_MAX; j++ ) // j = 0 .. 29
    {
        for( DWORD i = 0; i < D3DTSS_MAXSTAGES; i++ ) // i = 0 .. 3
        {
            // Note that in the 'dirty' buffer, we always have all the
            // stages for a particular state be consecutive

            if( *pDirty++ )
                dwSize += 3;
        }
    }

    // Handle transforms
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_TRANSFORMS]);
    for( DWORD i = 0; i < D3DTS_MAX; i++ )
    {
        if( *pDirty++ )
            dwSize += 1 + sizeof(D3DMATRIX) / 4;
    }

    // Handle viewport
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_VIEWPORT]);
    if( *pDirty++ )
        dwSize += sizeof(D3DVIEWPORT8) / 4;

    // Handle material
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_MATERIAL]);
    if( *pDirty++ )
        dwSize += sizeof(D3DMATERIAL8) / 4;

    // Handle back material
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_BACKMATERIAL]);
    if( *pDirty++ )
        dwSize += sizeof(D3DMATERIAL8) / 4;

    // Handle lights
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_MAX]);
    for( DWORD i = 0; i < g_pDevice->m_LightCount; i++ )
    {
        if( g_pDevice->m_pLights[i].Flags & LIGHT_STATEBLOCK_SET )
            dwSize += 1 + sizeof(D3DLIGHT8) / 4;
    }

    // Handle light enable states
    for( DWORD i = 0; i < g_pDevice->m_LightCount; i++ )
    {
        if( g_pDevice->m_pLights[i].Flags & LIGHT_STATEBLOCK_ENABLED )
            dwSize += 2;
    }
 
    //-------------------------------------------------------------------------
    // Allocate space for the state block
    //-------------------------------------------------------------------------

    StateBlock* pBlock = (StateBlock*)MemAllocNoZero( sizeof(DWORD) * dwSize );
    if( NULL == pBlock )
        return E_OUTOFMEMORY;

    // Zero just the header part of the block (all but the Data part)
    ZeroMemory( pBlock, sizeof(StateBlock)-sizeof(DWORD) );


    //-------------------------------------------------------------------------
    // Now do a second pass where we fill in the state block
    //-------------------------------------------------------------------------

    BYTE*  pDirty = &D3D__StateBlockDirty[0];
    DWORD* pData  = &pBlock->Data[0];

    // Handle textures
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_TEXTURES]);
    for( DWORD i = 0; i < D3DTSS_MAXSTAGES; i++ )
    {
        if( *pDirty++ )
        {
            D3DBaseTexture* pTexture = g_pDevice->m_Textures[i];

            pBlock->dwNumTextures++;
            *pData++ = i;
            *pData++ = (DWORD)pTexture;

            if( pTexture )
                pTexture->AddRef();
        }
    }

    // Handle pixel shader states
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_PIXELSHADER]);
    if( *pDirty++ )
    {
        PixelShader* pPixelShader = g_pDevice->m_pPixelShader;

        pBlock->bHasPixelShader = TRUE;
        *pData++ = (DWORD)pPixelShader;
        
        if( pPixelShader )
            pPixelShader->AddRef();
    }

    // Handle vertex shader states
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_VERTEXSHADER]);
    if( *pDirty++ )
    {
        DWORD handle = g_pDevice->m_VertexShaderHandle;

        pBlock->bHasVertexShader = TRUE;
        *pData++ = handle;
        
        if( IsVertexProgram(handle) )
            g_pDevice->m_pVertexShader->AddRef();
    }
 
    // Handle indices
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_INDICES]);
    if( *pDirty++ )
    {
        D3DIndexBuffer* pIndexBuffer = g_pDevice->m_pIndexBuffer;

        pBlock->bHasIndexBuffer = TRUE;
        *pData++ = g_pDevice->m_IndexBase;
        *pData++ = (DWORD)pIndexBuffer;

        if( pIndexBuffer )
            pIndexBuffer->AddRef();
    }

    // Handle vertex streams
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_STREAMS]);
    for( DWORD i = 0; i < D3DVS_STREAMS_MAX_V1_0; i++ )
    {
        if( *pDirty++ )
        {
            D3DVertexBuffer* pVertexBuffer = g_Stream[i].pVertexBuffer;

            pBlock->dwNumVertexStreams++;
            *pData++ = i;
            *pData++ = g_Stream[i].Stride;
            *pData++ = (DWORD) pVertexBuffer;
            
            if( pVertexBuffer )
                pVertexBuffer->AddRef();
        }
    }

    // Handle the pixel shader constants
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_PIXELSHADERCONSTANTS]);
    for( DWORD i = 0; i < D3DPS_CONSTREG_MAX_DX8; i++ )
    {
        if( *pDirty++ )
        {
            // There's no point in saving the pixel shader constants if no
            // pixel shader is active:
            if( g_pDevice->m_pPixelShader )
            {
                pBlock->dwNumPixelShaderConstants++;
                *pData++ = i;
                memcpy( pData + 1, &g_pDevice->m_PixelShaderConstants[i][0],
                        4*sizeof(FLOAT) );
            }
        }
    }

    // Handle vertex shader constants
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_VERTEXSHADERCONSTANTS]);
    for( DWORD i = 0; i < D3DVS_CONSTREG_COUNT_XBOX; i++ )
    {
        if( *pDirty++ )
        {
            // We never save the vertex shader constants that map from -96
            // to -1, although we leave space for them in the 'dirty' array
            // for future expansion:
            if( i >= 96 )
            {
                pBlock->dwNumVertexShaderConstants++;
                *pData++ = i;
                memcpy( pData, &g_VertexShaderConstants[i][0],
                        4*sizeof(FLOAT));
                pData += 4;
            }
        }
    }


    // Handle render states
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_RENDERSTATES]);
    for( DWORD i = 0; i < D3DRS_MAX; i++ )
    {
        if( *pDirty++ )
        {
            // If the fixed function pipeline is active, the shadowed
            // render states for the pixel shader registers are garbage,
            // so don't save those:
            if( (g_pDevice->m_pPixelShader != NULL) ||
                (i < D3DRS_PSALPHAINPUTS0) ||
                ( (i > D3DRS_PSINPUTTEXTURE) && (i != D3DRS_PSTEXTUREMODES) ) )
            {
                pBlock->dwNumRenderStates++;
                *pData++ = i;
                *pData++ = D3D__RenderState[i];
            }
        }
    }

    // Handle texture stage states
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_TEXTURESTATES]);
    for( DWORD j = 0; j < D3DTSS_MAX; j++ ) // j = 0 .. 29
    {
        for( DWORD i = 0; i < D3DTSS_MAXSTAGES; i++ ) // i = 0 .. 3
        {
            // Note that in the 'dirty' buffer, we always have all the
            // stages for a particular state be consecutive
            if( *pDirty++ )
            {
                pBlock->dwNumTextureStates++;
                *pData++ = i;   // Stage
                *pData++ = j;   // State
                *pData++ = D3D__TextureState[i][j];
            }
        }
    }

    // Handle transforms
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_TRANSFORMS]);
    for( DWORD i = 0; i < D3DTS_MAX; i++ )
    {
        if( *pDirty++ )
        {
            pBlock->dwNumTransforms++;
            *pData++ = i;
            *((D3DMATRIX*)pData) = g_pDevice->m_Transform[i];
            pData += sizeof(D3DMATRIX) / 4;
        }
    }

    // Handle viewport
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_VIEWPORT]);
    if( *pDirty++ )
    {
        pBlock->bHasViewport = TRUE;
        *((D3DVIEWPORT8*)pData) = g_pDevice->m_Viewport;
        pData += sizeof(D3DVIEWPORT8) / 4;
    }

    // Handle material
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_MATERIAL]);
    if( *pDirty++ )
    {
        pBlock->bHasMaterial = TRUE;
        *((D3DMATERIAL8*)pData) = g_pDevice->m_Material;
        pData += sizeof(D3DMATERIAL8) / 4;
    }



    // Handle back material
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_BACKMATERIAL]);
    if( *pDirty++ )
    {
        pBlock->bHasBackMaterial = TRUE;
        *((D3DMATERIAL8*)pData) = g_pDevice->m_BackMaterial;
        pData += sizeof(D3DMATERIAL8) / 4;
    }

    // Handle lights
    ASSERT(pDirty == &D3D__StateBlockDirty[D3DSBD_MAX]);
    for( DWORD i = 0; i < g_pDevice->m_LightCount; i++ )
    {
        if( g_pDevice->m_pLights[i].Flags & LIGHT_STATEBLOCK_SET )
        {
            pBlock->dwNumLights++;
            *pData++ = i;
            *((D3DLIGHT8*)pData) = g_pDevice->m_pLights[i]->Light8;
            pData += sizeof(D3DLIGHT8) / 4;
        }
    }

    // Handle light enable states
    for( DWORD i = 0; i < g_pDevice->m_LightCount; i++ )
    {
        if( g_pDevice->m_pLights[i].Flags & LIGHT_STATEBLOCK_ENABLED )
        {
            pBlock->dwNumLightEnables++;
            *pData++ = i;
            g_pDevice->GetLightEnable(i, (BOOL*)pData );
            pData++;
        }
    }

    // We're done!
    ASSERT(pData == pBlock->pDataEnd);

    (*pToken) = (DWORD)pBlock;
    return S_OK;
}

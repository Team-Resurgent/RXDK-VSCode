Implementation Information

The Xbox Title Libraries (XTL) provide an implementation of the C run-time libraries that is targeted specifically for game titles on the Xbox game system. These libraries are a subset of the Microsoft C run-time libraries and include the Standard C++ Library (which includes STL, the Standard Template Library) and the legacy iostream library.

Because the XTL C run-time libraries implementation is customized for the Xbox game system, changes to some C run-time functions are required to accurately reflect the underlying system features. Additionally, functions or groups of functions that are no longer applicable or that have limited usefulness have been removed.

For a list of C run-time functions that are not supported in the XTL, see Unsupported Functions.

Specific features of the Xbox game system that have influenced the C run-time implementation are discussed below.

Note   The C run-time libraries provide single-byte character as well as wide-character versions of many functions. Except as indicated, all references in this topic to a function's availability or behavior refer to both the single-byte and wide-character versions of that function.

Note   The XTL C run-time libraries feature set is not yet finalized. The information presented here will be updated as more details are determined.

Standard Handles

The Xbox system software does not define any standard input, standard output, or standard error handles. The C run-time streams stdin, stdout, and stderr are therefore not initialized as valid file streams. Operations on these streams will always return EOF, as will operations on the iostreams cout, cin, and cerr, and the file handles 0, 1, and 2.

Environment Variables

The Xbox system software does not define any environment blocks or environment variables. The C run-time functions that perform environment operations are not supported. These functions include getenv and _putenv. The _searchenv function will succeed only if the file name passed in is a fully-qualified path to a file that exists. It does not check the current working directory (there is no such thing on the Xbox game system), and it does not look in the named environment variable (because there are no environment variables) for a list of directories to search.

Command-Prompt Window

The Xbox game system does not have a command-prompt window. The C run-time functions that provide character-based input and output for the command-prompt window are not supported. These functions include _cgets, _cprintf, _cputs, _cscanf, _getch, _getche, _kbhit, _putch, and _ungetch.

Pipes

The Xbox system software does not support pipes, so the C run-time functions _pipe, _popen, and _pclose are not supported.

Process Execution

Only a single process—the game title—is running at any one time on the Xbox game system. Additional concurrent processes cannot be launched. All C run-time functions that provide process execution or control are not supported. These functions include the _exec and _spawn families, _cwait, system, and _getpid.

Current Directory

There is no concept of a current working directory or current drive for a game title on the Xbox game system. The C run-time functions that deal with setting and retrieving the current directory or drive are not supported. These functions include _getdrive, _chdrive, _chdir, _fullpath, _getcwd, and _getdcwd.

Error and Assert Output

Error and assert messages from the C run-time libraries are not displayed in a pop-up window on the Xbox game system. Instead, by default, those messages are directed to the debugger. If no debugger is attached, the messages are ignored.

The CRTDBG_MODE_WNDW report mode is not supported by _CrtSetReportMode. The default report mode for all report types (warnings, errors, and asserts) is CRTDBG_MODE_DEBUG, meaning that, by default, the CRT will send all report types to the debugger. Titles can change the report mode to CRTDBG_MODE_FILE if desired, but CRTDBG_FILE_STDOUT and CRTDBG_FILE_STDERR cannot be used when calling _CrtSetReportFile; a valid CRT file handle must be specified.

Command-Line and Environment Stub Functions

The C run-time libraries for processing the command line and the environment block are not used for game titles. Therefore, supplying the _setargv or _setenvp stub function to suppress the loading of those libraries is not necessary.

Heap Management

The XTL heap management libraries provide mechanisms for walking and compacting the heap, but only when the debug version of Xapilib (XapiLibD.lib) is linked to. The C run-time functions that rely on those capabilities, including _heapchk, _heapset, _heapmin, and _heapwalk, are available only when XapiLibD.lib is linked in with the title.

Other Functional Differences

The _stat function always returns –1 for the st_dev and st_rdev members of the _stat structure. If the path passed to the _stat function is a root directory name—"D:\", for example—the function fails and returns –1.

The _fstat function's handle parameter can only be a file handle, not a device handle. The function will fail and return –1 if the handle argument is not a file handle.

The _tzset function does not check for the existence of the TZ environment variable to determine time zone information; instead, it directly queries the Xbox system software.

setlocale cannot be used to change the current locale, only to query it.

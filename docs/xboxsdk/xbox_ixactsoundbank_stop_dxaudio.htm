IXACTSoundBank::Stop

Stops a sound cue. For manual release cues, frees associated resources.

HRESULT Stop(
    DWORD dwSoundCueIndex,
    DWORD dwFlags,
    PXACTSOUNDCUE pSoundCue
);

Parameters

dwSoundCueIndex
[in] Sound cue index to stop. Specify XACT_SOUNDCUE_INDEX_ALL to stop all sound cues. Use XACT_SOUNDCUE_INDEX_UNUSED to indicate that the cue is specified in pSoundCue rather than by index; otherwise, pSoundCue must be NULL.
dwFlags
[in] Flags affecting playback. Can be zero or the following:
Value Description
XACT_FLAG_SOUNDCUE_FLUSH Specifies that all queued cues on the same layer as the specified cue should be flushed. If a stop notification has been registered, all flushed cues will signal that they have stopped.
XACT_FLAG_SOUNDCUE_IMMEDIATE Specifies that the sound should terminate immediately, ignoring any envelope data. With the addition of envelopes, the default behavior is to have the sound play out through the end of the release envelope or waveform.
pSoundCue
[in] A pointer to an IXACTSoundCue to stop. If this parameter is specified, the dwSoundCueIndex parameter must be XACT_SOUNDCUE_INDEX_UNUSED.

Note    Once Stop has been called, this pointer is invalid and should not be used. Calling IXACTSoundBank::Play will provide a new pointer.

Return Values

Returns DS_OK if successful; returns an error code otherwise.

Remarks

When calling Stop, the default behavior is to play the audio to completion as specified by the sound designer. To stop the sound immediately, use the XACT_FLAG_SOUNDCUE_IMMEDIATE flag.

For Stop to work properly, cues need not be created as manual release cues. However, because cues are terminated or released during the XACTEngineDoWork call, the title should process stop notifications before calling XACTEngineDoWork. This will help ensure that Stop is called only on cues that are valid and have not yet been released.

Titles must call Stop to release sound cue pointers returned by PlayEx, Play, PrepareEx, or Prepare. For sound cues that play to completion, call Stop when the cue's stop notification arrives. Sound cue pointers are not returned (and need not be released) if XACT_FLAG_SOUNDCUE_AUTORELEASE was specified when the cue was prepared or played.

For non-zero latency streaming sounds, you must wait for the notification eXACTNotification_StreamDataAvailable on a playing sound cue before calling Stop to stop that sound cue.

Note that when a Stop notification arrives, the XACT_FLAG_NOTIFICATION_SOUNDCUE_DESTROYED flag in the XACT_NOTIFICATION_DESCRIPTION, if set, indicates that the cue has already been freed by XACT, and no calls can be made using the pSoundCue pointer returned in that structure.

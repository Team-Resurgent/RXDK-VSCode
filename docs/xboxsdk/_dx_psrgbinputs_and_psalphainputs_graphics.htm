PSRGBInputs[0-7] and PSAlphaInputs[0-7]

These two fields specify the input registers for the RGB and alpha portions of each combiner stage. It is most convenient to use the PS_COMBINERINPUTS(a,b,c,d) macro to specify the inputs for a stage. The macro is defined as follows:

#define PS_COMBINERINPUTS(a,b,c,d) (((a)<<24)|((b)<<16)|((c)<<8)|(d))

Each input consists of a register name, an input modifier, and a channel designator, all ORed together.

Registers

The register is specified from the following list and must be ORed together with an input modifer and a channel designator:

PS_REGISTER_ZERO
All components are zero.
PS_REGISTER_C0
Constant c0 for this stage.
PS_REGISTER_C1
Constant c1 for this stage.
PS_REGISTER_FOG
Fog register. Only the RGB value of this register may be read by the eight combiner stages. It contains the color of the fog. The Alpha value of the fog register may be read by the final combiner. It contains the fog factor that specifies the amount of fog color for the current fragment.
PS_REGISTER_V0
The diffuse color of the fragment. Note that this value may have been modified by a previous stage from the value calculated by the vertex shader.
PS_REGISTER_V1
The specular color of the fragment. Again, this value may have been modified by a previous stage from the value calculated by the vertex shader.
PS_REGISTER_T0
The output of texture stage 0, possibly modified by a previous combiner stage.
PS_REGISTER_T1
The output of texture stage 1, possibly modified by a previous combiner stage.
PS_REGISTER_T2
The output of texture stage 2, possibly modified by a previous combiner stage.
PS_REGISTER_T3
The output of texture stage 3, possibly modified by a previous combiner stage.
PS_REGISTER_R0
Temporary register 0. If the final combiner is not used (PSFinalCombinerInputsABCD and PSFinalCombinerInputsEFG both set to 0), the final color and alpha output of the pixel shader should be stored in the r0 register of the final active combiner stage.
PS_REGISTER_R1
Temporary register 1.

Input Modifers

The input modifer must be one of the following. It must be ORed together with a register and an channel designator. In the descriptions, x stands for the input register and y is the value presented to the register.

PS_INPUTMAPPING_UNSIGNED_IDENTITY
if(x<0) y = -x else y = x
PS_INPUTMAPPING_UNSIGNED_INVERT
y = 1 - x
PS_INPUTMAPPING_EXPAND_NORMAL
y = 2*max(0,x) - 1
PS_INPUTMAPPING_EXPAND_NEGATE
y = 1 - 2*max(0,x)
PS_INPUTMAPPING_HALFBIAS_NORMAL
y = max(0,x) - 1/2
PS_INPUTMAPPING_HALFBIAS_NEGATE
y = 1/2 - max(0,x)
PS_INPUTMAPPING_SIGNED_IDENTITY
y = x
PS_INPUTMAPPING_SIGNED_NEGATE
y = -x

For convenience, combinations of the zero register with several input modifiers are defined as convenient constants:

PS_REGISTER_ONE = PS_REGISTER_ZERO | PS_INPUTMAPPING_UNSIGNED_INVERT
PS_REGISTER_NEGATIVE_ONE = PS_REGISTER_ZERO | PS_INPUTMAPPING_EXPAND_NORMAL
PS_REGISTER_ONE_HALF = PS_REGISTER_ZERO | PS_INPUTMAPPING_HALFBIAS_NEGATE
PS_REGISTER_NEGATIVE_ONE_HALF = PS_REGISTER_ZERO | PS_INPUTMAPPING_HALFBIAS_NORMAL

Channel Designator

The channel designator needs to be one of the following values. This must be ORed together with a register and an input modifier.

PS_CHANNEL_RGB
Input comes from the RGB portion of the input register (valid for RGB portion only).
PS_CHANNEL_BLUE
Input comes from b portion of input register (valid for alpha portion only).
PS_CHANNEL_ALPHA
Input comes from alpha portion of input register (valid for both RGB and alpha portions).

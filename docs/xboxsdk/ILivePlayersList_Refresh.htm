ILivePlayersList::Refresh

Refreshes the players list displayed by the players feature.

HRESULT Refresh();

Parameters

None.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

The title should call Refresh to refresh the players list only when player information held by the title has changed and is being displayed. Refresh need not (and should not) be simply called every frame. Refresh should be called only while the players list is displayed, and only if the name returned by ITitlePlayersListItem::GetName has changed, or if the title wishes to change the sorted order of the players displayed in the list (if sorting was enabled when ILiveEngine::StartFeature was called).

For example, a player in a first-person shooter game brings up the players list. The title displays the score along with each player's name by appending the score to the player name and returning the result in ITitlePlayersListItem::GetName. As the game progresses, and while the players list is displayed, scores may change. Hence, the title will want to update the information displayed on the screen. In this case, if a score changes and the players list is displayed, the title would call Refresh.

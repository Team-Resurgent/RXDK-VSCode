Loading and Freeing XBE Sections

XLoadSectionByHandle or XLoadSection can be used to load a specified section into memory. If the section is not already loaded into memory, memory is allocated for the section, the section is loaded at its predetermined address, and the reference count for that section is set to one. No fixups are necessary when loading a section—all function references will either be internal to the title (in which case the function addresses will be predetermined) or be calls to the system software and thus redirected through the import address table. After a section is loaded, functions and data in that section can be referenced. If a title requests that a section be loaded, and that section is already loaded into memory, the reference count for the section is incremented by one and no further action is performed.

Each call to XLoadSection or XLoadSectionByHandle should be paired with a call to XFreeSection or XFreeSectionByHandle when the title no longer needs to use the section. When the title frees a section, the reference count of the section is decremented by one. If the reference count goes to zero, the section is unloaded and the associated memory is freed. Functions and data in that section can then no longer be referenced.

The reference count can be used to keep a section loaded as long as it is in use by any portion of the title. If a portion of code needs to access, for example, save game routines in a title that have been grouped into a section, the code can load the section, use the routines, and then free the section. If some other portion of code also needs to use the save game section, it can use the same procedure (load, use, then free the section) regardless of whether the save game section has been loaded or not. In either case, the section will be loaded if not already loaded, and will be freed when all users of the section have called XFreeSection or XFreeSectionByHandle so that the reference count drops to zero.

When a section is loaded, the address that the section is loaded at is returned. This address will be primarily used for files that were inserted as sections into the XBE image. For other sections (those created by the compiler or linker), calls to functions within those sections will already point to correct function addresses, so a title can call functions within those sections just as it would call any other function. No function tables or fixups are required, thus the load address of the section is not required. For files inserted as sections, however, the only way to reference information in that section is through the address returned when the section is loaded.

If a section is going to be loaded and unloaded multiple times, it is much more efficient to call XGetSectionHandle to get a handle to the section, and then to use that handle in calls to XLoadSectionByHandle and XFreeSectionByHandle. The by-handle functions load a section directly, whereas the functions that accept a section name must search through the section table and perform string comparisons to find the matching section.

Continuing the previous example, the following code loads the TESTSEG section and then uses the functions and data contained in that section.

// Restore code and data sections to normal.
#pragma code_seg()
#pragma data_seg()
#pragma bss_seg()
#pragma const_seg()

The section pragmas above restore the placement of code and data into their normal sections. Then the TESTSEG section is accessed:

void main( void )
{
    char szOut[50];

    // Load the section
    if( XLoadSection( "TESTSEG" ) ) {

        // Set value 2
        SetVal2( 2 );

        // Query the values
        wsprintf( szOut, "Values: %d, %d, %s.\r\n",
                  GetVal1(), GetVal2(), GetString() );
        OutputDebugString( szOut );

        // Free the section
        XFreeSection( "TESTSEG" );

    } else {
        wsprintf(szOut, "XLoadSection failed, error: %u.\r\n", GetLastError());
        OutputDebugString( szOut );
    }

    // Don't return from main
    Sleep( INFINITE );
}

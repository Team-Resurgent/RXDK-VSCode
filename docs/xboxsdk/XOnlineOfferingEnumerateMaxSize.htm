XOnlineOfferingEnumerateMaxSize

Determines the buffer size needed to contain a specified number of enumeration records. This buffer size is used in calls to XOnlineOfferingEnumerate.

DWORD XOnlineOfferingEnumerateMaxSize(
    const XONLINEOFFERING_ENUM_PARAMS *pEnumParams,
    DWORD dwTitleSpecificDataMaxSize
);

Parameters

pEnumParams
[in] Pointer to an XONLINEOFFERING_ENUM_PARAMS structure that specifies enumeration parameters. The wMaxResults member specifies the number of enumeration records desired per enumeration request.
dwTitleSpecificDataMaxSize
[in, optional] Maximum size of title specific data that the title will use. If zero is specified, the maximum allowed size (XONLINEOFFERING_ENUM_MAX_TITLE_DATA_SIZE) will be used in calculating the needed buffer size.

Return Values

The return value is the buffer size, in bytes, needed to contain the number of enumeration records specified in the wMaxResults member of the structure pointed to by the pEnumParams parameter.

Remarks

XOnlineOfferingEnumerateMaxSize is used to determine the buffer size needed to contain a specified number of enumeration results when calling XOnlineOfferingEnumerate. The enumeration parameters specified in pEnumParams should match those specified when XOnlineOfferingEnumerate is called.

In order to use less memory for the enumeration buffer, a value less than the maximum allowed BLOB size (XONLINEOFFERING_ENUM_MAX_TITLE_DATA_SIZE) can be specified for cbTitleSpecificDataMaxSize, indicating that the title will be using a smaller maximum BLOB size.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

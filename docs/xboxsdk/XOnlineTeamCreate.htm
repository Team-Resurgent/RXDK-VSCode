XOnlineTeamCreate

Creates a team and assigns the initial team member.

HRESULT XOnlineTeamCreate(
    DWORD dwUserIndex,
    const XONLINE_TEAM_PROPERTIES *pTeamProperties,
    const XONLINE_TEAM_MEMBER_PROPERTIES *pFirstTeamMemberProperties,
    DWORD dwMaxTeamMemberCount,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwUserIndex
[in] Controller index (zero-based) of the player creating the team.
pTeamProperties
[in] Address of an XONLINE_TEAM_PROPERTIES structure that describes the team information.
pFirstTeamMemberProperties
[in] Address of an XONLINE_TEAM_MEMBER_PROPERTIES structure that describes information about the initial team member.
dwMaxTeamMemberCount
[in] Maximum number of team members that can join the team.
hWorkEvent
[in, optional] Handle to an event.
phTask
[out] Pointer to an XONLINETASK_HANDLE returned.

Return Values

Returns S_OK if successful, an error code otherwise.

XOnlineTeamCreate returns the value XONLINE_E_ACCOUNTS_ACCOUNT_BANNED if a voice-banned player attempts to create a team.

The strings in the XONLINE_TEAM_PROPERTIES structure are automatically checked to ensure they do not contain offensive text. Titles do not need to call the XOnlineStringVerify function before calling XOnlineTeamCreate. If a string does have inappropriate content, the error XONLINE_E_TEAMS_NAME_CONTAINS_BAD_WORDS, XONLINE_E_TEAMS_DESCRIPTION_CONTAINS_BAD_WORDS, XONLINE_E_TEAMS_MOTTO_CONTAINS_BAD_WORDS, or XONLINE_E_TEAMS_URL_CONTAINS_BAD_WORDS is returned from XOnlineTaskContinue, depending on the offending string.

Remarks

When the team is created, the player automatically joins the team and is made leader of the team. The team name must be unique or the function fails.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

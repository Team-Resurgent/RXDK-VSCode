ConvertThreadToFiber

The ConvertTheadToFiber function converts the current thread into a fiber. You must convert a thread into a fiber before you can schedule other fibers.

LPVOID ConvertThreadToFiber(
  LPVOID lpParameter  // fiber data for new fiber
);

Parameters

lpParameter
[in] Specifies a single variable that is passed to the fiber. The fiber can retrieve this value by using the GetFiberData macro.

Return Values

The return value is the address of the fiber.

Remarks

The ConvertThreadToFiber function converts the current thread into a fiber. The fiber is now the currently executing fiber and can create and schedule other fibers. Fibers cooperatively schedule each other explicitly by calling SwitchToFiber. Only fibers can schedule other fibers. If a thread needs to schedule a fiber, it must first call ConvertThreadToFiber to become a fiber.

Fibers have very low overhead. Fiber state information is stored at the top of a fiber's stack, so there is no memory cost for a fiber other than its stack.

The fiber data specified in lpParameter is stored as part of the fiber state information.

Requirements

Header: Declared in Winbase.h, for include use Xtl.h.
Import Library: Use Xapilib.lib

Dashboard Video Settings

The user has the option to set a number of video settings in the Xbox Dashboard. These video settings influence the set of video modes that the game can select. The possible settings exposed in the Dashboard vary based on which Audio Video (AV) pack the video region of the console was set to, either North America, Japan, or rest of the world.

Normal, Letterbox, and Widescreen

The user will be able to choose one of the following settings, regardless of the installed AV pack:

  • Normal
  • Letterbox
  • Widescreen

These settings are used to specify whether the user's TV is a widescreen display. The Letterbox setting is used only for DVD playback. For game purposes, it is treated the same as Normal. If either Normal or Letterbox is selected, a game should not attempt to select a widescreen display mode. If Widescreen is selected, a game may choose to enable a widescreen video mode, but it is not required to do so. A game selects a widescreen video mode by setting the D3DPRESENTFLAG_WIDESCREEN flag in the flags field of the D3DPRESENT_PARAMETERS structure that is specified when the Direct3D device is created. If widescreen is not enabled by the user, attempting to create the device with this display mode will fail.

A widescreen video mode activates a special signal to tell the TV to switch into widescreen mode. Some TVs do not recognize this signal, and the user will be required to manually switch the TV. If your game selects a widescreen mode, it will also have to adjust its transform matrix to provide a 16:9 display aspect ratio rather than the more common 4:3 aspect ratio.

480p, 720p, and 1080i

If the Xbox HDTV AV Pack is installed, the user may enable the following HDTV settings:

  • Enable 480p
  • Enable 720p
  • Enable 1080i

These settings are not mutually exclusive. The user enables the 480p setting to indicate that his TV supports 480p (progressive scan). This does not require a game to support 480p. The same holds true for 720p and 1080i.

If the user has enabled 480p and the game is not doing field rendering, the game should select 480p rather than 480i (interlaced) because the picture quality is better. A game selects the 480p rendering mode by specifying the D3DPRESENTFLAG_PROGRESSIVE flag in the Flags field of D3DPRESENT_PARAMETERS structure that is specified when the device is created or reset. The back buffer height must be 480 and the back buffer width must be either 640 or 720.

If the user has enabled 720p, the game can select 720p by setting the back buffer height to 720, the back buffer width to 1280, and specifying both D3DPRESENTFLAG_PROGRESSIVE and D3DPRESENTFLAG_WIDESCREEN in the Flags member of the D3DPRESENT_PARAMETERS structure that is specified when the device is created or reset. The 720p mode automatically implies widescreen.

If the user has enabled 1080i, the game can select 1080i by setting the back buffer height to 1080 and the back buffer width to 1920, and specifying both D3DPRESENTFLAG_INTERLACED and D3DPRESENTFLAG_WIDESCREEN in the Flags member of the D3DPRESENT_PARAMETERS structure that is specified when the device is created or reset. The 1080i mode automatically implies widescreen.

PAL-60

If the Xbox console was built for a game region outside of North America or Japan, the user will also have the ability to enable support for PAL by selecting the following:

  • Enable PAL-60

Support for video standards, such as PAL, are set in the Xbox console at manufacturing time. A title can detect video standard support at runtime by calling XGetVideoStandard. If PAL is supported, the function will return XC_VIDEO_STANDARD_PAL_I. Older PAL televisions only support PAL at 50 Hz. New PAL televisions support both PAL at 50 Hz (PAL) and at 60 Hz (PAL-60). Titles can check for support of PAL-60 by calling XGetVideoFlags and checking to see if XC_VIDEO_FLAGS_PAL_60Hz is one of the flags returned. If this option has been set, the game can select a video mode with a refresh rate of 60 Hz. This is done by setting the FullScreen_RefreshRateInHz field to 60 in the D3DPRESENT_PARAMETERS structure that is specified when the device is created. Normally, this field is 50 Hz for PAL modes. The Xbox video hardware will set up the proper output signal for PAL-60. The only difference that the game will notice is that the vertical blank will occur 60 times per second rather than 50. A game that runs well at 60 Hz will not likely have performance problems running at 50 Hz. However, some content is authored with a specific playback frame rate in mind. For example, video for cut scenes or motion capture sequences may be authored to be decompressed and displayed at 60 Hz.

There are several approaches for dealing with this type of content. The best solution is to author two versions of this content. One should be ideal for playback at 50 Hz, and the other should be ideal for playback at 60 Hz. The game can choose the appropriate version at startup time when it determines the type of display. There will likely be ample room on the game disc to hold both versions of the content. This solution offers the best performance and quality. Another solution is to interpolate the content at run-time based on the time between frames. The system time is read at the start of each frame, and the animation or video is interpolated based on that time. This solution may be more difficult with some types of content than with others, and it does cost some run-time performance. It does have the added advantage that it properly handles cases in which the frame rate is changing due to system overload. The third solution is to drop frames of video or motion capture samples. If one frame in every six is dropped on content authored for 60 Hz, it will play back at the proper rate for 50 Hz. This solution doesn't cost much performance but can result in noticeable quality degradation and is not recommended.

A game can select the PAL-60 mode by setting the back-buffer height to 480 and the back-buffer width to either 640 or 720. It is also necessary to set the FullScreen_RefreshRateInHz field of the D3DPRESENT_PARAMETERS structures to be 60. PAL-60 does not support 640×576 or 720×576.

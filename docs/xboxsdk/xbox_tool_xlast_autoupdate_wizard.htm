The New Autoupdate Wizard

To create a new autoupdate package, select New from the File menu. Choose Autoupdate Package and click OK. This starts the New Autoupdate Package Wizard.

  1. After displaying the welcome dialog box, the New Autoupdate Package Wizard presents a dialog box asking for the following information.
    2. Autoupdate Package Name
    Specifies the friendly name of the autoupdate package. This name is used as the name of the XML project file for the submission. It cannot contain any folder or path information. The maximum length of the autoupdate package name is 38 characters. Valid characters for the autoupdate package name are the characters a–z, A–Z, the numerals 0–9, and the characters !, #, $, &, ', (, ), -, @, [, ], ^, _, `, {, }, and ~. You do not need to provide a file name extension.
    Project Directory
    Specifies the path to the directory containing the project file, all temporary files that XLAST creates, and the completed autoupdate package (.aup file).
    Retail ID of the Game
    Contains the 8-digit hexadecimal title identifier that you received from Microsoft when the game was released.
    Base Disc Version
    Defines a two-digit integer value that typically takes the form 00, 01, 02, and so on. This value differentiates individual releases of the game in the marketing channel. For example, it can be used to differentiate Japanese, German, and other localized releases of the game.
    Path to original game build
    Specifies the path to the released version of the game. This can be on a PC hard disk or on a network share. The directory structure of this folder's subdirectories must be the same as the directory structure on the released version of the game disk. At a minimum, this path, or its subdirectories, must contain the original versions of every XBE file in the game, whether they are being updated in the autoupdate or not. It must also contain the original versions of all the files you want to update or patch.
    Path to updated game binaries
    Specifies the path to the updated version of the game. This must be a full release build. At a minimum, this path, or its subdirectories, must contain all XBE files in the game, whether or not they are being updated in the autoupdate. It must also contain the updated versions of all the files you want to update or patch.

    Note    It is strongly recommended that the path to the updated binaries use the same folder structure as the original game. However, this is not required.

    Autoupdate version number
    Contains the version number of the autoupdate submission. The autoupdate version number can never be zero, and must be incremented by 1 for each autoupdate submission for a specific game.

    Note    XLAST does not modify any files that reside in the paths to the original or updated game files. Instead, it creates the autoupdate package in the project directory.

  2. After you click the Next button, the New Autoupdate Package Wizard scans both the original and updated builds. If there are errors, it notifies you. Otherwise, it displays the Wizard's final dialog box containing the Finish button.

    3. Note    When the Wizard finishes, it automatically addes some files that you should not remove. It does not matter whether these files have changed or not. They are required in every autoupdate package.

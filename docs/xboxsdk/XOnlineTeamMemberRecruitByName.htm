XOnlineTeamMemberRecruitByName

Submits a request to recruit a team member by gamertag.

HRESULT XOnlineTeamMemberRecruitByName(
    DWORD dwUserIndex,
    XUID xuidTeam,
    LPCSTR lpPeerName,
    const XONLINE_TEAM_MEMBER_PROPERTIES *pPeerTeamMemberInfo,
    XONLINE_MSG_HANDLE hMsg,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwUserIndex
[in] Controller index (zero-based) of the player making the request.
xuidTeam
[in] XUID that uniquely identifies the team.
lpPeerName
[in] String that uniquely identifies the gamertag of the player to recruit.
pPeerTeamMemberInfo
[in] Pointer to an XONLINE_TEAM_MEMBER_PROPERTIES structure containing team member information.
hMsg
[in] Handle to a message. If this parameter is NULL, a default message with no voice attachment will be generated.
hWorkEvent
[in, optional] Handle to an event.
phTask
[out] Pointer to an XONLINETASK_HANDLE returned.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

The message passed to XOnlineTeamMemberRecruitByName must be of type XONLINE_MSG_TYPE_TEAM_RECRUIT.

When recruiting to a family title ID team (see XOnlineTeamSetFamilyTitleID), titles must also specify that family title ID to XOnlineMessageEnableReceivingFamilyTitleIDs in order to receive notifications of the incoming XONLINE_MSG_TYPE_TEAM_RECRUIT message via XOnlineGetNotificationEx or to have the message returned by XOnlineMessageEnumerate.

The title should not destroy the message object it specified through the hMsg parameter until the XOnlineTeamMemberRecruitByName operation is complete.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

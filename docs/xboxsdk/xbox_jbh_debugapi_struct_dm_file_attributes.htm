DM_FILE_ATTRIBUTES

The DM_FILE_ATTRIBUTES structure is used by DmGetFileAttributes and DmSetFileAttributes.

typedef struct _DM_FILE_ATTRIBUTES {
    char Name[256];
    FILETIME CreationTime;
    FILETIME ChangeTime;
    DWORD SizeHigh;
    DWORD SizeLow;
    DWORD Attributes;
} DM_FILE_ATTRIBUTES, *PDM_FILE_ATTRIBUTES;

Members

Name
Name of the file or directory. For DmGetFileAttributes and DmSetFileAttributes, this member is unused. It is always empty.
CreationTime
A FILETIME structure that specifies when the file or directory was created. For FATX (the file system used by the Xbox hard disk), only year values between 2000 and 2127 (inclusive) are valid.
ChangeTime
A FILETIME structure. For a file, the structure specifies when the file was last written to. For a directory, the structure specifies when the directory was created. For FATX (the file system used by the Xbox hard disk), only year values between 2000 and 2127 (inclusive) are valid.
SizeHigh
High-order DWORD of the file size. This member has no meaning for directories.
SizeLow
Low-order DWORD of the file size. This member has no meaning for directories.
Attributes
FAT-style attribute information for the file or directory. When setting file attributes, passing a zero for this member signifies that the attributes flag for the file should not be changed in this operation. For example, it may be desirable to change the file times or size without changing the attributes flag. A zero value for this field will not cause the file attributes to be set to FILE_ATTRIBUTE_NORMAL. You must explicitly use the FILE_ATTRIBUTE_NORMAL flag to set the file attributes flag to FILE_ATTRIBUTE_NORMAL.

The following attributes are defined.

Attribute Meaning
FILE_ATTRIBUTE_NORMAL The file or directory has no other attributes set. This attribute is valid only if used alone.
FILE_ATTRIBUTE_DIRECTORY The handle identifies a directory.
FILE_ATTRIBUTE_HIDDEN The file or directory is hidden. It is not included in an ordinary directory listing.
FILE_ATTRIBUTE_READONLY The file or directory is read-only. Applications can read the file but cannot write to it or delete it. In the case of a directory, applications cannot delete it.

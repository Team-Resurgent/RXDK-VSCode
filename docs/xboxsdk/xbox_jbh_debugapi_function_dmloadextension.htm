DmLoadExtension

Loads debugger extensions.

DMHRAPI DmLoadExtension(
  LPCSTR  szName, 
  PHANDLE phModule, 
  PVOID   *pvBase
  );

Parameters

szName
File name of the extension to load.
phModule
Pointer to memory to receive the loaded module's handle. This parameter can be NULL.
pvBase
Pointer to the loaded module. This parameter can be NULL.

Return Values

Returns XBDM_NOERR if successful, an error value otherwise.

Loading Debugger Extensions

When a debugger extension is loaded, all debugger extensions that the loaded extension depends on (those that were linked with the loaded extension) are loaded as well. After a debugger extension is successfully loaded, its entry point is called. The entry point for a debugger extension takes the following form:

typedef ULONG (__stdcall *PDM_ENTRYPROC)(ULONG, ULONG, ULONG);

The three parameters are ignored, they exist solely for Win32 compatibility.

E:\DXT\*.DXT

Debugger extensions can be loaded manually using the DmLoadExtension function, or they can be loaded automatically by the Xbox System Software. When the target system boots, the system software searches the E:\DXT directory for debugger extension files (*.DXT). It attempts to load them as described above (see Loading Debugger Extensions).

Building a Debugger Extension

  • Compile your debugger extension as a Win32 driver (link using the "/DRIVER" switch).
  • Rename your binary to *.dxt.
  • Do not use the multithreaded C run-time library.
  • Do not link to xapi.lib.

Requirements

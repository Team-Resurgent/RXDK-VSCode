DirectSound Warning Messages

DirectSound is designed to display various warning messages in the debugger to make developers aware of performance characteristics that their code may encounter. For example, voices in DirectSound do not stop instantaneously. Calling certain methods on a DirectSound buffer immediately after stopping the buffer could trigger the following message: 

DSOUND: CMcpxVoiceClient::WaitForVoiceOff: Warning: Busy-waiting for the voice to turn off
This message is to let developers know that your call blocks momentarily while it waits for the voice to stop. For more information on calls can generate busy-wait states, see the following sections:

  • IDirectSoundBuffer8::Play
  • IDirectSoundBuffer8::PlayEx
  • IDirectSoundBuffer8::SetFormat
  • IDirectSoundBuffer8::SetBufferData
  • IDirectSoundStream::Flush
  • IDirectSoundStream::FlushEx

Calling IDirectSoundBuffer8::Release and IDirectSoundStream::Release will also cause busy-wait states.

DirectSound Global Debug Settings

DirectSound includes three global variables to assist with warnings and debugging:

  • g_dwDirectSoundDebugLevel - Determines the level for debug messages that will appear in the debugger.
  • g_dwDirectSoundDebugBreakLevel - Determines whether a particular level of debug message will cause a break in the debugger.
  • g_pfnDirectSoundDebugCallback - Provides the ability for a game to intercept all debug output calls from DirectSound.

See the above reference sections for detailed information on each of these global variables.

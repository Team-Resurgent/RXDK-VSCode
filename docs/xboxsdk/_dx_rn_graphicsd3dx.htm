rn

There are 12 temporary registers, r0 through r11. Each register holds a 4D floating-point vector. A given instruction can use up to three temporary registers for input and at most one temporary register for output.

No values in a temporary register that remain from preceding invocations of the vertex shader can be used. Vertex shaders that read a value from a temporary register before writing to it will fail the Microsoft® Direct3D® API call to create the vertex shader.

r12 is a special thirteenth register that can be use to read back the current value of oPos. r12 can only be used as an input register and oPos can only be used as an output register.

ILiveEngine::NotificationSetState

Sets the current online state information for the specified user.

HRESULT NotificationSetState(
    DWORD Port,
    DWORD StateFlags,
    XNKID SessionID,
    DWORD StateDataSize,
    const PVOID *pStateData
);

Parameters

Port
[in] The controller port index (zero-based) of the user whose state information is being set.
StateFlags

[in] Online state flags to set. This parameter can be a combination of the following values.
Value Description
XONLINE_FRIENDSTATE_FLAG_JOINABLE The user is in a session that other users can join.
XONLINE_FRIENDSTATE_FLAG_ONLINE The user is online.
XONLINE_FRIENDSTATE_FLAG_PLAYING The user is playing a game.
XONLINE_FRIENDSTATE_FLAG_VOICE The user has voice capability.

To have a player appear to be offline—that is, "cloaked"—clear the XONLINE_FRIENDSTATE_FLAG_ONLINE flag; even though the player is online, the player will not appear on the friends lists of other players. However, if the cloaked player sends an invitation to a player on his or her friends list, the cloaked player will appear online to that player until the invitation is resolved.

SessionID
[in] XNKID structure that identifies the user's current session. Setting this structure to all zeros expires any previously sent game invitations.
StateDataSize
[in] Size, in bytes, of the buffer pointed to by pStateData. This value can range from zero to MAX_STATEDATA_SIZE. If StateDataSize is zero, pStateData must be NULL.
pStateData
[in] Pointer to a buffer containing title-specific state information. This parameter can be NULL if no such information is provided. The Xbox Drop-in UI friends implementation does not use this data, so titles are free to use it in whatever manner they wish.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

Titles using Xbox Drop-in UI must call NotificationSetState rather than XOnlineNotificationSetState when setting user states and state flags.

texm3x2depth

Sets the depth value for the current fragment.

texm3x2depth   tDest, tSrc0

Registers

tDest
Destination register, holding the result of the operation.
tSrc0
Temporary source register, specifying the input argument.

Remarks

The texm3x2depth texture mode is similar to the texm3x2tex mode except that the result of the transformation is not used to look up a value in a texture map; it is used to set the depth value for the current fragment. The value calculated in the previous stage is used as the z value. This stage calculates the w value as follows:

w = (scalar)((s,t,r) · (r,g,b))

No texture lookup is done with these values, so no texture should be set for this stage. The fragment depth is set to the value z/w. Note that the z/w value must be in the same format and range as the current depth buffer. The scaling of the value can be controlled by the scale applied to the texture coordinates sent to this stage and the previous stage.

This instruction actually performs a dot product between the texture coordinates of the current stage and the output value from the specified input stage. Color values are unsigned integers in the range 0x0 to 0xff, but through the use of input modifiers, these values can be interpreted as signed numbers.

If the _bx2 input modifier is applied to the specified input argument, the color values are mapped to signed numbers using the formula

( ColorValue – 0x80 ) / 127.0f

So,

  • 0x0 is mapped to –1.007874016f (–128 / 127),
  • 0x1 is mapped to –1.0f,
  • 0x80 is mapped to 0.0f, and
  • 0xff is mapped to 1.0f.

If there is no input modifier applied to the specified input argument, the color values are mapped to the range zero to one using the formula

ColorValue / 255.0f

So,

  • 0x0 is mapped to 0.0f, and
  • 0xff is mapped to 1.0f.

This instruction is an Xbox extension.

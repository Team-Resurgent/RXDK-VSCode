BaseFrameNode (XbView)

This BaseFrameNode class stores a node in the skeleton hierarchy. This class gathers the data from the XBR class and its helper classes necessary to draw the geometry. It stores pointers to the world matrix needed to transform the geometry, the actual geometry itself (in the m_pMesh pointer), and a flag that stores whether or not the geometry needs any alpha blending to render itself.

There are two constructors for this class and both are called from the BaseSkeletonNode constructor.

  1. The first constructor is passed three parameters. The first is a DWORD index into the list of matrices stored in the skeleton hierarchy. This index points to the matrix used by this node to transform itself when it draws. The other two parameters for the first constructor are pointers to the node's Skeleton class and a pointer to the XBR class itself so that this class can find and save the pointers it needs to draw itself. During the first constructor, the class runs through the XBR class's list of resources and attempts to find an Effect structure with the same ID as the Skeleton class pointed to by the DWORD index passed in. If it finds an Effect with the same ID, it stores a pointer to the Effect, stores a pointer to the matrix needed to transform the Effect from the Skeleton class, and sets a flag that tells the BaseFrameNode if the Effect uses any alpha blending.

  2. The second constructor is called when there is an Effect structure that is not attached to any skeleton node. In this case, the Effect itself is passed into the constructor, and the constructor uses the identity matrix for the transform. It also checks the Effect to see if any alpha blending is needed to draw it and sets the m_bHasAlpha flag accordingly.

In both cases, the m_pChild and m_pNext pointers are set to NULL. These are fixed up in the BaseSkeletonNode after the lists are created. The class stores a pointer to its Skeleton class if one was passed in and NULL if not.

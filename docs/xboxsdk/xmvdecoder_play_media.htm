XMVDecoder::Play

Plays an entire Xbox Media Video (XMV) file, blocking until the video and audio portion of the stream is complete. This method creates a surface that is the size of the video, and then plays it back on the screen.

HRESULT Play(
    DWORD Flags,
    RECT *pRect
);

Parameters

Flags
[in] Determines what kind of playback will occur. It must be one of the following values:
Value Description
XMVFLAG_NONE No looping occurs.
XMVFLAG_FULL_LOOP The movie will loop until either XMVDecoder::TerminateLoop or XMVDecoder::TerminatePlayback is called.
pRect
[in] Pointer to a RECT structure describing what part of the screen to render the output video. If pRect is NULL, the video will be displayed on the full screen.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

Either this function or XMVDecoder::GetNextFrame should be used to play back XMV files. Do not try intermixing the two methods.

Because this function uses overlays, there must be at least one call to either IDirect3DDevice8::Present or IDirect3DDevice8::Swap before calling Play.

This function cannot be used with an XMV file opened using XMVFLAG_UNSYNCHRONIZED_PLAYBACK. Direct3D must have been initialized before calling Play. Play uses overlays to render the video, and can be run on background threads without interfering with other Direct3D rendering functions.

The XMVDecoder::TerminateImmediately routine must be called on the same thread that the decoder is running on and therefore cannot be used with XMVDecoder::Play.

Requirements

Header: Declared in Xmv.h.

Import Library: Use Xmv.lib.

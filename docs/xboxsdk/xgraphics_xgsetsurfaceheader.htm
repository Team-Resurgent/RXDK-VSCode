XGSetSurfaceHeader

Manually creates an image surface.

DWORD WINAPI XGSetSurfaceHeader(
  UINT Width,
  UINT Height,
  D3DFORMAT Format,
  IDirect3DSurface8 *pSurface,
  UINT Data, 
  UINT Pitch
);

Parameters

Width
[in] Width of the image surface, in pixels.
Height
[in] Height of the image surface, in pixels.
Format
[in] Member of the D3DFORMAT enumerated type, describing the format of the image surface.
pSurface
[in, out] Address of the IDirect3DSurface8 interface, representing the image surface to be filled.

Data
[in] Offset to the data held by this resource.
Pitch
[in] Specifies the pitch of the texture.

Return Values

Returns the number of bytes required by the resource, not counting the header.

Remarks

All resources, including surfaces, include a header that contains all of the information needed by the driver to process the resource and a data allocation that contains information used by the GPU. The resource headers are used only by the driver and so should be allocated from normal cached memory. The header contains all of the information needed by the driver to set up the hardware so that it can use the information stored in the data allocation.

Unless the title is manually creating a resource, it is recommended that the title use IDirect3DDevice8::CreateImageSurface.

After calling XGSetSurfaceHeader, it is necessary to call IDirect3DResource8::Register in order to register the memory address of the resource.

The following code snippet shows how to register a manually created surface resource.

IDirect3DSurface8* pSurface = new IDirect3DSurface8;
DWORD dwSurfaceSize = XGSetSurfaceHeader( dwWidth, dwHeight, 
                                          1, 0, D3DFMT_LIN_D24S8, 0, pSurface, 0, 0 );
void* pBuffer = D3D_AllocContiguousMemory( dwSurfaceSize, D3DSURFACE_ALIGNMENT );
pSurface->Register( pBuffer );

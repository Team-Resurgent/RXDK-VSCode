SetLastError

The SetLastError function sets the last-error code for the calling thread. 

VOID SetLastError(
  DWORD dwErrCode   // per-thread error code
);

Parameters

dwErrCode
[in] Specifies the last-error code for the thread.

Return Values

This function does not return a value.

Remarks

Error codes are 32-bit values (bit 31 is the most significant bit). Bit 29 is reserved for title-defined error codes; no system error code has this bit set. If you are defining an error code for your title, set this bit to indicate that the error code has been defined by your title and to ensure that your error code does not conflict with any system-defined error codes.

This function is intended primarily for library functions. Calling this function after an error occurs lets the library emulate the error condition behavior of the Xbox Title Libraries (XTL).

Most XTL functions call SetLastError when they fail. Function failure is typically indicated by a return value error code such as zero, NULL, or –1. Some functions call SetLastError under conditions of success; those cases are noted in each function's reference topic.

Titles can retrieve the value saved by this function by using the GetLastError function. The use of GetLastError is optional; it can be called by a title to find out the specific reason for a function failure. Note that, unless specifically documented in the Return Values section of a function's reference page, the error codes set by a function are intended to be primarily used as a debugging aid, not as a method for controlling execution flow within a title.

The last-error code is kept in thread local storage so that multiple threads do not overwrite each other's values.

Requirements

Header: Declared in Winbase.h, for include use Xtl.h.
Import Library: Use Xapilib.lib

XNetQosLookup

The XNetQosLookup function is called by a title to probe the quality of service (QoS) between itself and specified remote hosts. The remote hosts can be Xbox consoles, security gateways, or both. 

INT XNetQosLookup(
    UINT cxna,
    XNADDR * apxna[],
    XNKID * apxnkid[],
    XNKEY * apxnkey[],
    UINT cina,
    IN_ADDR aina[],
    DWORD adwServiceId[],
    UINT cProbes,
    DWORD dwBitsPerSec,
    DWORD dwFlags,
    WSAEVENT hEvent,
    XNQOS * * ppxnqos
);

Parameters

cxna
Number of remote Xbox consoles to probe.
apxna[]
Array of pointers to XNADDR structures that contain Xbox network addresses for the remote Xbox consoles.
apxnkid[]
Array of pointers to XNKID structures that contain session IDs for the remote Xbox consoles. The title on the local Xbox console need not register these session IDs with XNetRegisterKey.
apxnkey[]
Array of pointers to XNKEY structures that contain key-exchange keys for the remote Xbox consoles.
cina
Number of security gateways to probe.
aina[]
Pointer to an array of IN_ADDR structures that contain the IP addresses of the security gateways.
adwServiceId[]
Pointer to an array of service IDs for the security gateways. The title on the local Xbox console need not call XNetServerToInAddr with these service IDs.
cProbes
Number of desired probe replies to be received. Each probe involves sending two 100-byte packets to the host and receiving two 100-byte packets in return. If the desired number of probe replies has not been received, the QoS client sends more probes. The larger the number specified by cProbes, the longer the QoS lookup takes. The recommended value for cProbes is eight. To test only for connectivity—that is, to find out whether the remote host is active—the title can set cProbes to zero.
dwBitsPerSec
Maximum upstream bandwidth that the outgoing QoS probe packets can consume. Specifying zero will use the default of 16,384 bps (16 Kbps). Decreasing the value of dwBitsPerSec extends the time it takes to complete all of the probes. Increasing this value speeds up the process. A title should limit this value to a maximum of 65,536 (64 Kbps) to prevent consuming all the upstream bandwidth on a cable modem or DSL line.
dwFlags
The title should set this parameter to zero; no flags are currently supported.
hEvent
Handle to an event that is signaled whenever any of the XNQOSINFO structures in the result set has the XNET_XNQOSINFO_COMPLETE flag set in its bFlags member. This parameter is optional and may be NULL.
ppxnqos
Pointer to a pointer to an XNQOS structure that receives the results from the QoS probes. The value of the cxnqos member of this structure is equal to the sum of the cxna and cina parameters. The axnqosinfo member of the XNQOS structure points to an array of XNQOSINFO structures. Each XNQOSINFO structure contains the results for a single remote host. In the array, the structures for the remote Xbox consoles come before the structures for the security gateways. The structures are in the same order as that specified in the apxna and apina parameters. The title must call XNetQosRelease to free the memory associated with these structures.

Return Values

Returns zero if successful, an error code otherwise.

Remarks

The XNetQosLookup function is asynchronous. The function returns immediately even though the QoS probes are still in progress. For each of the XNQOSINFO structures pointed to by the XNQOS structure, the calling application must check the value of the bFlags member to determine whether the results for that remote host have been received.

To cancel the lookup before it is complete, your program can call XNetQosRelease. This terminates the lookup immediately without waiting for any further network traffic, and there are no side effects. However, your program cannot dereference the XNQOS structure returned through the XNetQosLookup function's ppxnqos parameter after calling XNetQosRelease.

Requirements

Header: Declared in Winsockx.h.

Import Library: Use Xonline.lib or xnets.lib.

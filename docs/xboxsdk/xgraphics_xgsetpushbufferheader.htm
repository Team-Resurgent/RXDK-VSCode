XGSetPushBufferHeader

Creates an empty push-buffer resource.

VOID WINAPI XGSetPushBufferHeader(
  UINT Size,
  BOOL RunUsingCpuCopy,
  IDirect3DPushBuffer8 *pPushBuffer,
  UINT Data
);

Parameters

Size
[in] Size of the push-buffer in bytes. The size must be a multiple of four bytes. Additionally, the size can minimally be 512 bytes, and must be at least as big as any single command recorded into the push-buffer (so if IDirect3DDevice8::DrawPrimitiveUP is called while recording, for example, Size must be at least as big as all the vertex data and the call overhead). If the Size is not as large as any single command recorded into the push-buffer, Direct3D will overrun the end of the buffer while recording (no check is done on retail builds for this case so as not to slow down any of the Direct3D methods while not recording a push-buffer). Typically, the size should be at least 1100 bytes.

Size should also be at least as large as the total size of the push-buffer to be recorded. However, even on retail builds, Direct3D is guaranteed not to overrun the end of the buffer while recording a push-buffer that is too big (subject to the constraint that no single command is larger than the size). This permits GetSize to be called on a too-small push-buffer to get an exact, accurate size required.

RunUsingCpuCopy
[in] If TRUE, the push-buffer resource data is created in non-contiguous, cached memory, and D3DPUSHBUFFER_RUN_USING_CPU_COPY is set in the Common field of IDirect3DPushBuffer8. If FALSE, the push-buffer resource data is created in physically contiguous, write-combined memory.
pPushBuffer
[in, out] Address of the IDirect3DPushBuffer8 interface, representing the push-buffer resource that is to be filled.
Data
[in] Offset to the data held by this resource.

Return Values

None.

Remarks

In general, RunUsingCpuCopy should be FALSE when the push-buffer is large, because the push-buffer can be directly executed in-place by the GPU, which saves in CPU time and system bandwidth. But RunUsingCpuCopy should be TRUE for small push-buffers, because the per-call overhead of IDirect3DDevice8::RunPushBuffer is extremely high for the direct execution case.

As with any Direct3D resource, Release should be called when finished with the push-buffer resource.

All resources, including push-buffers, include a header that contains all of the information needed by the driver to process the resource and a data allocation that contains information used by the GPU. The resource headers are used only by the driver and so should be allocated from normal cached memory. The header contains all of the information needed by the driver to set up the hardware so that it can use the information stored in the data allocation.

Unless the title is manually creating a resource, it is recommended that the title use IDirect3DDevice8::CreatePushBuffer.

After calling XGSetPushBufferHeader, it is necessary to call IDirect3DResource8::Register in order to register the memory address of the resource.

The following code snippet shows how to register a manually created push-buffer resource.

IDirect3DPushBuffer8* pPushBuffer = new IDirect3DPushBuffer8;

// Assume that dwSize is the size of the push-buffer, in bytes.
XGSetPushBufferHeader( dwSize, FALSE, pPushBuffer, 0 );
void *pData = D3D_AllocContiguousMemory( dwSize, D3DPUSHBUFFER_ALIGNMENT );
pPushBuffer->Register( pData );

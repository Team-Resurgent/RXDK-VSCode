XOnlineChangeLogonUsersTaskGetResults

Returns the results of a completed (or failed) XOnlineChangeLogonUsers call.

HRESULT XOnlineChangeLogonUsersTaskGetResults(
    XONLINETASK_HANDLE hLogonTask,
    HRESULT *phr
);

Parameters

hLogonTask
[in] Online task handle returned by the call to XOnlineChangeLogonUsers.
phr
[out] Pointer to a buffer that will receive the result codes for each player specified in the XOnlineChangeLogonUsers call. The specified buffer should be large enough to hold XONLINE_MAX_LOGON_USERS values of type HRESULT. This argument can be NULL if this information is not needed.

Return Values

The return value is the result status of the completed XOnlineChangeLogonUsers call.

Remarks

XOnlineChangeLogonUsersTaskGetResults returns the results of a logon change task initiated with a call to XOnlineChangeLogonUsers, and optionally retrieves the result status for each player that was specified in the logon change request.

Typically, XOnlineChangeLogonUsersTaskGetResults is called in response to a return code of XONLINE_E_LOGON_CHANGE_USER_FAILED from XOnlineTaskContinue for the logon change task. That error return code indicates that there was some problem logging on one or more of the new players (or possibly one of the existing players). If the problem can be associated with one or more of the new players specified in the XOnlineChangeLogonUsers call, then the error result codes returned in the phr array indicate specific result codes for each individual player, in the same order that they were specified in the call to XOnlineChangeLogonUsers. It is possible that no error codes specific to the new players can be determined, in which case all of the result codes returned in phr will be S_OK.

The possible result codes returned in the phr array are:

  • S_OK: The player was successfully logged on, and has no messages.
  • XONLINE_S_LOGON_USER_HAS_MESSAGE: The player was successfully logged on, and has a dynamic message available for viewing. The player should be given the option of rebooting into the Xbox Dashboard in order to view the message.
  • XONLINE_E_LOGON_USER_ACCOUNT_REQUIRES_MANAGEMENT: The player cannot be successfully logged on. The player must reboot into the Xbox Dashboard to determine the reason for this failure and to resolve the issue before they may participate in online game play. Note that the failure may simply be caused by a required message that must be read.

Requirements

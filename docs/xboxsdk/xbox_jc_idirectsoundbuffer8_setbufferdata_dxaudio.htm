IDirectSoundBuffer8::SetBufferData

Sets DirectSoundBuffer memory using pointers.

HRESULT SetBufferData(
    LPVOID pvBufferData,
    DWORD dwBufferBytes
);

Parameters

pvBufferData
Address of the wave data for playback. Can be set to NULL.
dwBufferBytes
Value that specifies the length of audio data, in bytes.

Return Values

Returns DS_OK if successful, an error code otherwise.

Remarks

DirectSound allocates the buffer memory on your behalf when pvBufferData is set to NULL and dwBufferBytes is greater than zero. The allocated memory is tracked by DirectSound and reclaimed either when Release is called or when you call SetBufferData with a non-NULL pointer (or when pvBufferData is set to NULL and dwBufferBytes is greater than zero).

Because of restrictions in the hardware, this call can cause a busy-wait state. Before performing additional operations on the buffer, you must make sure that the buffer is stopped by polling IDirectSoundBuffer8::GetStatus. Check the DSBSTATUS_PLAYING flag after calling IDirectSoundBuffer8::Stop or IDirectSoundBuffer8::StopEx as in the following example:

DWORD dwStatus;
pDSBuffer->StopEx( 0, DSBSTOPEX_IMMEDIATE );
do
{
    pDSBuffer->GetStatus( &dwStatus );
} while( dwStatus & DSBSTATUS_PLAYING );

Additionally, calling some methods on the buffer immediately after stopping the buffer can generate debug warning messages. See DirectSound Warning Messages for more information.

Data alignment restrictions apply when using this function. PCM data must be DWORD aligned (4 bytes), and ADPCM data must be block aligned (36 bytes).

While changing the channel count is possible, calling either SetBufferData or SetFormat with a different channel count is inefficient. For example, when converting from stereo to 6-channel, a better solution is to free the stereo voice and reallocate it as a native 6-channel voice.

DirectSound buffers are managed in a scatter gather entry (SGE) list. There is a maximum of 2,047 SGEs, which each point to a 4-KB page. This means that a maximum of 8 MB are available for allocating or playing DirectSound buffers simultaneously. Attempting to allocate/play more than the maximum will result in the following error message:

DSOUND: CMcpxBuffer::MapInputBuffer: Resource failure: Out of scatter/gather entries.
        Either your buffer is too big, or too many buffers have been mapped at one time.

XOnlineStorageUploadFromMemory

Uploads a file attachment to title storage from memory.

HRESULT XOnlineStorageUploadFromMemory(
    DWORD dwFacility,
    DWORD dwUserIndex,
    LPCWSTR wszStorageFileName,
    FILETIME ftServerExpirationDate,
    PBYTE pbDataToUpload,
    DWORD cbDataToUpload,
    DWORD dwUploadFlags,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwFacility

[in] Storage facility; this value can be one of the following:
Value Description
XONLINESTORAGE_FACILITY_TEAMS Teams storage facility.
XONLINESTORAGE_FACILITY_PER_USER_TITLE Per title, per user storage.

For a complete list of facility values, see the XONLINESTORAGE_FACILITY page.

dwUserIndex
[in] Controller index (zero-based) of the player for whom to download the attachment.
wszStorageFileName
[in] Service's storage path for uploaded file. This path can be created using the XOnlineStorageCreateServerPath function.
ftServerExpirationDate
[in] Date of expiration of this item on the server. A value of zero means that the item never expires. This value must be in UTC format.
pbDataToUpload
[in] Buffer of data to upload.
cbDataToUpload
[in] Size, in bytes, of the data buffer to upload.
dwUploadFlags
[in] Flags that qualify the upload process. This value must be set to zero.
hWorkEvent
[in] Handle to an event that is signaled when there is work to do on the asynchronous task returned by the phTask parameter. This parameter can be NULL if such synchronization is not required.
phTask
[out] Pointer to a variable of type XONLINETASK_HANDLE that receives a task handle. Use this handle in calls to the XOnlineTaskContinue function to process the file upload.

Return Values

If XOnlineStorageUploadFromMemory succeeds, the return value is S_OK.

If XOnlineStorageUploadFromMemory fails, the return value is one of the following error codes:

Value Description
XONLINE_E_STORAGE_INVALID_REQUEST The request is invalid.
XONLINE_E_STORAGE_ACCESS_DENIED The client doesn't have sufficient permissions to upload the file.
XONLINE_E_STORAGE_INVALID_ACCESS_TOKEN The access token signature is invalid.
XONLINE_E_STORAGE_CANNOT_FIND_PATH Name resolution failed.
XONLINE_E_STORAGE_INVALID_STORAGE_PATH The storage path is invalid.
XONLINE_E_STORAGE_INVALID_FACILITY The facility code is invalid.

Remarks

This function is similar to the XOnlineStorageUploadByServerPath function, except that it uploads the file from an in-memory buffer instead of a directory on disk. The specified buffer must remain valid and unmodified until the upload operation completes.

Only one upload-from-memory operation at a time is supported.

Your title must start the signature service before it can call this function.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.

XOnlineOfferingEnumerate

The XOnlineOfferingEnumerate function begins enumeration of the available online content and subscription offerings. The function creates an asynchronous task to perform the enumeration.

HRESULT XOnlineOfferingEnumerate(
    DWORD dwUserIndex,
    const XONLINEOFFERING_ENUM_PARAMS *pEnumParams,
    PBYTE pbBuffer,
    DWORD dwBufferSize,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwUserIndex
[in] Controller index (zero-based) of the user for whom offerings will be enumerated. The user must have been successfully logged on with a previous call to XOnlineLogon.
pEnumParams
[in] Pointer to an XONLINEOFFERING_ENUM_PARAMS structure that specifies additional enumeration parameters.
pbBuffer
[out, optional] Pointer to a buffer that will be used internally by the function to store enumeration results while the enumeration is in progress. The enumeration results are not returned in this buffer; they are obtained with a call to XOnlineOfferingEnumerateGetResults. If this parameter is NULL, the function will automatically allocate a buffer.
dwBufferSize
[in] Size, in bytes, of the buffer pointed to by pbBuffer. If pbBuffer is NULL, this parameter specifies the size of the results buffer that the function will automatically allocate. The XOnlineOfferingEnumerateMaxSize function can be used to determine the buffer size needed to contain a specified number of enumeration records. The buffer must be large enough to hold at least one offering record.
hWorkEvent
[in, optional] Handle to an event that will be signaled when there is work to be done on the enumeration task. This parameter can be NULL if no such event is required.
phTask
[out] Address of an XONLINETASK_HANDLE that receives the offering enumeration task. This task sends the enumeration request to the offering service and receives the response. This handle must be pumped to completion by XOnlineTaskContinue. After the task has completed, call XOnlineOfferingEnumerateGetResults to extract the offering enumeration list. Call XOnlineTaskClose to close the task.

Return Values

If the function succeeds, the return value is S_OK.

Remarks

XOnlineOfferingEnumerate begins the enumeration of content offerings available from the online service. This function creates an asynchronous task and returns a handle to that task in the phTask parameter. In order to perform work on (and eventually complete) this asynchronous task, repeatedly call XOnlineTaskContinue with the enumeration task handle until that function indicates that partial results are available (by returning XONLINETASK_S_RESULTS_AVAIL), or that the task is complete (by returning XONLINETASK_S_SUCCESS).

If partial results are available or the task has completed, call XOnlineOfferingEnumerateGetResults to retrieve the actual results of the enumeration. A limited number of results will be returned for each enumeration request. That number will be the lesser of the available records or the number of records specified in the dwMaxResults member of pEnumParams, and will also be limited by the number of records that can fit into the buffer of size cbBuffer. When XOnlineOfferingEnumerateGetResults is called, the variable pointed to by pfMoreResults will be set to TRUE if more offerings are available.

As long as more enumeration records are available, continue to call XOnlineTaskContinue to completion, then XOnlineOfferingEnumerateGetResults. When no more records are available, call XOnlineTaskClose with the enumeration task handle in order to free the resources associated with the enumeration.

The hWorkEvent parameter can be used to pass an event handle to the enumeration function. That event handle will be signaled when there is work to be performed on the asynchronous task. A secondary thread could be waiting on this event handle to know when asynchronous task work needs to be done, and call XOnlineTaskContinue to do so. Thus, the secondary thread can avoid spinning on calls to XOnlineTaskContinue when no work needs to be done.

Note that some caching is done when determining available content. It may be up to three hours between the time that new content is posted to a server and the time that titles will see the content in enumerations.

To enumerate offline content (from the hard disk or DVD), use XFindFirstContent.

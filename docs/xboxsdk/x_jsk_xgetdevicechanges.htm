XGetDeviceChanges

The XGetDeviceChanges function tracks dynamic insertions and removals of peripheral devices.

BOOL XGetDeviceChanges(
  PXPP_DEVICE_TYPE DeviceType,
  PDWORD pdwInsertions,
  PDWORD pdwRemovals
);

Parameters

DeviceType
[in] The device type to query. DeviceType must be of one the following values:
ValueDescription
XDEVICE_TYPE_GAMEPADAn Xbox Controller.
XDEVICE_TYPE_MEMORY_UNITAn Xbox Memory Unit.
XDEVICE_TYPE_VOICE_MICROPHONEXbox Communicator microphone.
XDEVICE_TYPE_VOICE_HEADPHONEXbox Communicator headphone.
XDEVICE_TYPE_DEBUG_KEYBOARDA debug keyboard. Do not use in shipping titles. See Remarks below.
XDEVICE_TYPE_DEBUG_MOUSEA debug mouse. Do not use in shipping titles. See Remarks below.
pdwInsertions
[out] Pointer to DWORD variable that receives a bitmap of insertions since the last call to XGetDeviceChanges or XGetDevices.
pdwRemovals
[out] Pointer to DWORD variable that receives a bitmap of removals since the last call to XGetDeviceChanges or XGetDevices.

Return Values

The return value is TRUE if there were any dynamic insertions or removals since the last successful call to XGetDeviceChanges or XGetDevices.

If there were no changes in status, the return value is FALSE.

Remarks

XGetDeviceChanges is used to get dynamic insertions and removals since the last time XGetDevices or XGetDeviceChanges was called. This function reports changes only for the specified device type. Like XGetDevices, the bitmap fields returned must be interpreted based upon the device type parameter. For specifics of the bitmap, please see XGetDevices.

XGetDeviceChanges is more efficient than XGetDevices when called in a frequent loop. Additionally, XGetDeviceChanges will indicate when a device has been removed and reinserted between calls (by setting bits in both pdwInsertions and pdwRemovals), whereas XGetDevices will not indicate a change if a device is inserted and removed between calls. Detecting the removal and insertion of a device is critical because handles become invalid when devices are removed, so open handles to those devices must be closed and reopened.

XGetDeviceChanges may be called without first calling XGetDevices. On the first call, pdwInsertions will indicate all of the devices that have been detected up to that point.

To use XDEVICE_TYPE_DEBUG_KEYBOARD, #define DEBUG_KEYBOARD before including Xtl.h. Do not use XDEVICE_TYPE_DEBUG_KEYBOARD in shipping retail game titles.

To use XDEVICE_TYPE_DEBUG_MOUSE, #define DEBUG_MOUSE before including Xtl.h. Do not use XDEVICE_TYPE_DEBUG_MOUSE in shipping retail game titles.

Requirements

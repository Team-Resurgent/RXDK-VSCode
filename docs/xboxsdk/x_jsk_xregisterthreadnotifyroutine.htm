XRegisterThreadNotifyRoutine

The XRegisterThreadNotifyRoutine function is used to register a title-defined notification routine that will receive notifications when a new thread is created or when an existing thread is deleted. Typically, this is used by a library that maintains per-thread state or resources.

VOID XRegisterThreadNotifyRoutine(
    PXTHREAD_NOTIFICATION pThreadNotification,
    BOOL fRegister
);

Parameters

pThreadNotification
[in, out] Pointer to an XTHREAD_NOTIFICATION structure. The pfnNotifyRoutine member of this structure specifies the title-defined function of type XTHREAD_NOTIFY_PROC that will receive thread notifications. This structure must remain valid and unmodified until the thread notification routine is unregistered. A global or allocated variable must be used; do not use a local variable inside a function.
fRegister
[in] Specifies whether to register or unregister the notification routine. If fRegister is TRUE, the new notification routine is registered (added to the list of thread notification routines); if fRegister is FALSE, the existing notification routine is unregistered (removed from the list of thread notification routines).

Return Values

This function does not return a value.

Remarks

XRegisterThreadNotifyRoutine registers a notification function that will be called by the system whenever a thread is created or deleted. The notification function is of type XTHREAD_NOTIFY_PROC and must be defined by the title.

Registering a thread notification routine allows a library to track thread creation and deletion. A library that needs per-thread storage can use these notifications to allocated and free resources as necessary.

The XTHREAD_NOTIFICATION structure pointed to by the pThreadNotification parameter must remain valid until the thread notification routine is unregistered. The system continues to access the structure at the address specified by pThreadNotification. Some form of persistent memory (such as a global variable or memory allocated from the heap) must be used. Do not use a local stack variable inside a function (because the variable will become invalid when the containing function returns).

You cannot register the same notification routine more than once.

When a library is being unloaded, it must unregister the notification routine. To unregister the notification routine, call XRegisterThreadNotifyRoutine using the same structure that was specified when the routine was registered, and use FALSE for the fRegister parameter.


The following example demonstrates registering a thread notification routine:

// the persistent XTHREAD_NOTIFICATION structure
XTHREAD_NOTIFICATION MyThreadNotification;
	
// the title-defined notification routine
VOID WINAPI MyThreadNotifyProc(BOOL fCreate) {
    if (fCreate) {
        // A new thread is being created.
    } else {
        // An existing thread is being deleted.
    }
}

    ...
	
    // in the library's initialization code,	
    // register the notification routine with the system
    MyThreadNotification.pfnNotifyRoutine = MyThreadNotifyProc;
    XRegisterThreadNotifyRoutine( &MyThreadNotification, TRUE );

    ...

Continuing the example above, after the notification routine is registered, the system will call MyThreadNotifyProc whenever a thread creation or deletion event occurs. When a new thread is created, MyThreadNotifyProc is called with fCreate == TRUE. When an existing thread is deleted, MyThreadNotifyProc is called with fCreate == FALSE. The notification routine is called in the context of the thread being created or deleted.

To unregister the notification routine used in the above example, the call would be:

    XRegisterThreadNotifyRoutine( &MyThreadNotification, FALSE );

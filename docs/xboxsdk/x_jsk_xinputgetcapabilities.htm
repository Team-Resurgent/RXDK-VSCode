XInputGetCapabilities

The XInputGetCapabilities function is used to determine the specific capabilities and features of a device connected to the Xbox game system.

DWORD XInputGetCapabilities(
  HANDLE hDevice,                          
  PXINPUT_CAPABILITIES pCapabilities       
);

Parameters

hDevice
[in] Handle to a valid device opened with XInputOpen.
pCapabilities
[out] Pointer to a structure that receives the functional capabilities for the specified device.

Return Values

If the function successfully retrieves the device capabilities, the return code is ERROR_SUCCESS.

If the function fails, it will return a valid Win32 error code.

Remarks

XInputGetCapabilities is used to determine the specific capabilities and features of a device. Because XID provides backward and forward device compatibility, some fields in the device packets may be placeholders only. Additionally, the bit resolution of specific fields may vary from device to device. Finally, titles may wish to expose additional functionality or modify gameplay based upon a specific device subtype, such as a steering wheel or light gun. Titles that need to determine the specific capabilities of a device can use the XInputGetCapabilities function.

Interpreting Device Capabilities

XInputGetCapabilities returns the following union in pCapabilities:

typedef struct _XINPUT_CAPABILITIES
{
    BYTE    SubType;
    WORD    Reserved;
    union  
    {
       XINPUT_GAMEPAD Gamepad;
    } In;
    union
   {
      XINPUT_RUMBLE Rumble;
   } Out;
} XINPUT_CAPABILITIES, *PXINPUT_CAPABILITIES;

Titles must use the device type used to open the device to correctly use the union. If the device opened was an Xbox controller, titles should access the XINPUT_GAMEPAD and XINPUT_RUMBLE portions of the union to determine device capabilities. For details on the specifics of this structure, see XINPUT_CAPABILITIES.

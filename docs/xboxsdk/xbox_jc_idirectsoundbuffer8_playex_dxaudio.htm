IDirectSoundBuffer8::PlayEx

Plays the contents of the sound buffer.

HRESULT PlayEx(
    REFERENCE_TIME rtTimeStamp,
    DWORD dwFlags
);

Parameters

rtTimeStamp
Time for start to occur. Reference time units are hundreds of nanoseconds from an arbitrary point when the Xbox is powered on. If this value is zero, playback will start immediately.

Note    Time stamps cannot be used to guarantee sample-accurate synchronization.

dwFlags
Flags specifying how to play the buffer. The following flags are defined:
Value Description
DSBPLAY_FROMSTART Play the buffer from the beginning, regardless of current position.
DSBPLAY_LOOPING After the end of the audio buffer is reached, play restarts at the beginning of the buffer. Play continues until explicitly stopped.
DSBPLAY_SYNCHPLAYBACK Synchronize playback of multiple buffers and streams. This causes the buffer to reset the play cursor to zero, the equivalent of specifying the DSBPLAY_FROMSTART. See the IDirectSound8::SynchPlayback function for more information on using this flag.

Note    These flags are not supported with buffers created using the DSBCAPS_FXIN2 flag.

Return Values

If the method succeeds, the return value is DS_OK.

Remarks

Because of restrictions in the hardware, this call can cause a busy-wait state. Before performing additional operations on the buffer, you must make sure that the buffer is stopped by polling IDirectSoundBuffer8::GetStatus. Check the DSBSTATUS_PLAYING flag after calling IDirectSoundBuffer8::Stop or IDirectSoundBuffer8::StopEx as in the following example:

DWORD dwStatus;
pDSBuffer->StopEx( 0, DSBSTOPEX_IMMEDIATE );
do
{
    pDSBuffer->GetStatus( &dwStatus );
} while( dwStatus & DSBSTATUS_PLAYING );

Additionally, calling some methods on the buffer immediately after stopping the buffer can generate debug warning messages. See DirectSound Warning Messages for more information.

This command is valid for buffers created using the DSBCAPS_FXIN2, but is not valid for buffers created using the DSBCAPS_FXIN or DSBCAPS_MIXIN flags.

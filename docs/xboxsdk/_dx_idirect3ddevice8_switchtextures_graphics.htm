IDirect3DDevice8::SwitchTexture

Switches a texture to a stage for a device.

HRESULT SwitchTexture(
  DWORD Stage,
  IDirect3DBaseTexture8* pTexture
);

Parameters

Stage
[in] Stage identifier to which the texture is set. Stage identifiers are zero-based. Devices can have up to four set textures, so the maximum value allowed for Stage is 3.
pTexture
[in] Pointer to an IDirect3DBaseTexture8 interface, representing the texture being set. For complex textures, such as mipmaps and cube textures, this parameter must point to the top-level surface.

Return Values

If the method succeeds, the return value is S_OK.

Remarks

This method is a restricted form of IDirect3DDevice8::SetTexture and allows a texture to be changed in ten x86 instructions. The previous IDirect3DDevice8::SetTexture call for the corresponding stage must have been non-NULL. The new texture must have the same cubemap-ness and dimensionality as the corresponding previously set texture. The texture cannot be a linear texture.

IDirect3DDevice8::SwitchTexture does no reference counting or lifetime management, so it is up to the caller to ensure that the texture still exists and is unchanged when the graphics processing unit (GPU) gets around to rendering it. One method of doing this is to use IDirect3DDevice8::InsertFence.

The state of IDirect3DDevice8::SwitchTexture is not queryable and is not recorded by state blocks.

IDirect3DDevice8::SwitchTexture has the same signature as IDirect3DDevice8::SetTexture, so D3DFixUp::SetTexture can be used to do fix-ups of IDirect3DDevice8::SwitchTexture calls recorded in push-buffer resources.

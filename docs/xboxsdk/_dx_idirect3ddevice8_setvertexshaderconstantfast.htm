IDirect3DDevice8::SetVertexShaderConstantFast

Sets values in the vertex constant array. This function is is just like SetVertexShaderConstant except it has less CPU overhead and doesn't keep a copy of the constant data, even when running on a non-D3DCREATE_PUREDEVICE device.

HRESULT SetVertexShaderConstantFast(
    INT Register,
    CONST void *pConstantData,
    DWORD ConstantCount
);

Parameters

Register
[in] Register address at which to start loading data into the vertex constant array.
pConstantData
[in] Pointer to the data block holding the values to load into the vertex constant array. The size of the data block is (ConstantCount * 4 * sizeof(float)).
ConstantCount
[in] Number of constants to load into the vertex constant array. Each constant is comprised of four floating-point values.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

Since SetVertexShaderConstantFast doesn't keep track of the constants it sets, GetVertexShaderConstant will not work for any constants set by SetVertexShaderConstantFast (it will return stale data), and state blocks that use vertex shader constants may also not work.

This is the method used to load the constant registers for the vertex shader program. When loading transformation matrices, the application should transpose them row/column and load them into consecutive constant registers.

There are either 96 or 192 constant registers available for use, depending on the constant mode as set by IDirect3DDevice8::SetShaderConstantMode. The permissible register range is 0 through 95 when in D3DSCM_96CONSTANTS mode, and –96 through 95 otherwise. Registers D3DVS_XBOX_RESERVEDCONSTANT1 and D3DVS_XBOX_RESERVEDCONSTANT2 (–38 and –37) are reserved, and may never be used, unless constant mode D3DSCM_NORESERVEDCONSTANTS is used in conjunction with a screenspace vertex shader. If D3DSCM_NORESERVEDCONSTANTS is set using SetShaderConstantMode, then all constants are available.

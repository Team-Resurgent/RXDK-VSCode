Creating an XACT Sound Bank

After the XACT engine has been created, the next step is to create an XACT sound bank. This requires three things:

  1. Create an XACT sound bank (.XSB) file using the graphical user interface component of XACT. For an overview of how to use the tool to generate sound banks, see the XACT Authoring Guide. This example requires a sound bank file with a single cue. However, XACT does not allow sound banks consisting only of empty cues (that is, cues that do not bind to audio files). Empty cues can reside among other functioning cues, but not alone. So, we'll need to have a single small wave associated with the cue. Throughout the examples that follow, the wave bank containing the wave will never actually be loaded and/or played. It merely serves as a placeholder.

    Once the .XSB file has been created, simply copy it into your resource folder on the Xbox console. The example below uses a folder named "media", which is a subfolder of the application directory.

  2. Load the sound bank data file using our utility function, XACTLoadFile (listed later in the Input and Support Routines section).
  3. Create the XACT sound bank from our loaded data using IXACTEngine::CreateSoundBank.
void CreateSoundBank()
{
    // Load the data image of the sound bank
    DWORD dwFileSize;
    XACTLoadFile( "d:\\media\\sample.xsb", (VOID**)&g_pbSoundBankData, &dwFileSize );

    // Register our (almost) empty sound bank with XACT
    HRESULT hr = g_pXACT->CreateSoundBank( g_pbSoundBankData, dwFileSize, &g_pSoundBank ); 
    assert( SUCCEEDED(hr) );
}

CrashDump Sample

The CrashDump sample demonstrates how the Xbox crash dump system works.

Overview

Like Win32, the Xbox provides crash dump information that you can use to debug applications.

Note    Crash dump information from the Xbox is not compatible with WinDBG.

Path

Source: (XDK Root)\Samples\Xbox\Misc\CrashDump

Remarks

You must copy the contents of the Media directory to the target Xbox before running this sample.

For proper generation of crash dump information, ensure that your Xbox is using the August 2002 or later version of XbDM. If you aren't sure what version you're using, run the August 2002 or later XDKRecovery application to update your Xbox.

Make sure you don't have a kernel debugger (such as KD or WinDBG) connected to your Xbox. (If you do, the bug check will be caught by that debugger, rather than generate a crash dump.)

Note that when you build and run the CrashDump sample, the Vertical Blank Count field on the Xbox screen increments continuously. Press the A button on your Xbox game controller to cause the application to perform a bug check by dereferencing a null pointer during a vertical blank interrupt callback. After you do this, the Vertical Blank Count field no longer increments.

Note    If you have a kernel debugger (such as KD or WinDBG) connected to your Xbox when you press the A button, you get a breakpoint in that debugger that allows you to debug the bug check. If you don't have a kernel debugger connected to your Xbox, XbDM writes out a crash dump that you can use to debug the problem using Visual Studio .NET.

It takes about two minutes for the Xbox to write a crash dump and return to the XDK Launcher. During that time the Xbox's LEDs flash various red and green patterns.

If you have the Visual Studio .NET debugger attached when the crash occurs, the VS.NET debugger eventually notices that the Xbox has crashed and ends the debugging session.

After the Xbox finishes writing the crash dump, it return to XDK Launcher. When this happens, copy the crash dump file from the Xbox to the development PC. The crash dump file is located at

xe:\crashdump.xdmp

To make it easier for Visual Studio .NET to find the symbols for the application, copy the crash dump to the same directory as your application's PDB file. For the CrashDump sample, the path is similar to the following.

C:\Program Files\Microsoft Xbox SDK\Samples\Xbox\Misc\CrashDump\Debug

To debug the crash dump:

  1. Start Visual Studio .NET.
  2. Choose File:Open Project from the Visual Studio .NET menus.
  3. Open the crashdump.xdmp file.
  4. Press F5 to begin debugging the crash dump.

When the sample starts, it displays a dialog box describing what caused the crash dump. You can then use the standard debugger features (such as the stack crawl window) to determine what caused the crash.

Dashboard Metadata

A Simple Walk-Through

For this walk-through, use the DolphinClassic sample that comes with the XDK. Although it's not nearly as fun as plugging your game into the Xbox Dashboard, using an XDK sample will at least help you get an idea of how things work. When you're finished, you'll have a game called "DolphinClassic" listed in the Games Area of the Dashboard, along with the required title image.

  1. Load the DolphinClassic sample into Microsoft® Visual C++®. For this example, the XDK should be installed in the default location, C:\Program Files\Microsoft Xbox SDK. You can find the project file under C:\Program Files\Microsoft Xbox SDK\Samples\Xbox\Graphics\DolphinClassic. In Project / Settings / Xbox Image, select Title Info from the Category drop-down menu. You can assign the title name, the publisher name, and the title ID by filling in the appropriate fields. These values are already completed for you in the DolphinClassic sample.

  2. Create a title image that can be displayed in the Games Area of the Xbox Dashboard. To do this, create an image using your favorite graphics utility. The image can be any 128 x 128 Microsoft® Direct3D® texture. For the purposes of this walk-through, create a bitmap image in Microsoft® Paint. After opening the Paint program (from the Start menu, click Accessories, and then click Paint), use the Image drop-down menu and select Attributes. There you can set both the width and the height of the image to 128. Draw a picture to represent your game. Once you've created your image, save the bitmap as C:\Program Files\Microsoft Xbox SDK\Samples\Xbox\Graphics\DolphinClassic\title.bmp.

  3. Open a command-line shell and change your directory to the location where you saved the bitmap (in the previous step). Now you need to create a resource definition file (.rdf). This file describes the bitmap so that it can be built into a Direct3D resource file. Launch Microsoft® Notepad (Start / Accessories / Notepad) and enter the following information. Save this file as "title.rdf" in the DolphinClassic directory.

    Texture MyTex
{
   Source title.bmp
   Format D3DFMT_DXT1
   Width 128
   Height 128
}

  4. Once you've saved the file, exit Notepad. You can now use the Bundler utility (included with the XDK) to transform your bitmap into a Direct3D resource file. For our walk-through, type "bundler title.rdf -o titleimage.xbx." You should now have a file called "titleimage.xbx." Close the command-line window and return to Visual C++.

  5. Add the parameters necessary to embed the resource file you just created into your Xbox executable. In Project / Settings / Xbox Image, select Title Info from the Category drop-down menu and enter "titleimage.xbx" in the Title Image field.

  6. Build the DolphinClassic sample. Visual C++ automatically calls ImageBld with the proper switches to incorporate your image within your .xbe, and it also calls xbcp to copy the .xbe to your target system. The ImageBld settings are acquired from the data that you enter on the Project / Settings / Xbox Image tab in Visual C++. See the documentation for ImageBld for more information.

Note   The first time you run the sample, the metadata is automatically extracted from your .xbe and copied to the appropriate title directory. Because of this, you need to run the sample once on your target machine from the Title Launcher before it will be displayed in the Dashboard.

Save Game Metadata

Besides having to create the title image, you are also required to include metadata for saved games. The textual metadata is created automatically when you call the XCreateSaveGame function. The name of the saved game passed to this function will be displayed by the Dashboard.

Saved games must have an associated image. You can use the Bundler utility to create an image file for saved games. The process for creating this image is similar to that for creating the title's image file. Saved game images must be stored in the saved game's directory. It must be named "saveimage.xbx" and must be 64 × 64 pixels. The XDK also includes a function that will allow you to create an "saveimage.xbx" file from a screen shot or other drawing surface (see XGWriteSurfaceOrTextureToXPR).

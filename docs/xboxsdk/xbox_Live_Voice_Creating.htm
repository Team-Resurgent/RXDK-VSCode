Creating and Using Voice Media Objects

After a game has determined that a headset is attached to the player's Xbox, it must create the appropriate Xbox Media Objects (XMOs) to enable voice chats. To create voice device XMOs, call XVoiceCreateMediaObject or XVoiceCreateMediaObjectEx. The primary difference between these functions is that XVoiceCreateMediaObjectEx enables you to specify a callback function that is invoked as a kernel deferred procedure call (DPC).

Games obtain input from the microphone and send output to the earphones by calling the XMediaObject::Process function. Your game must supply input and output buffers for this function. The buffers consist of arrays of XMEDIAPACKET structures.

Sending Voice Packets to Xbox Live

Before sending voice packets across the Internet to another player on Xbox Live, titles first must encode the packets. This requires the creation of a voice encoder XMO. Voice encoder XMOs compress voice data for transmission. Create a voice encoder XMO with one of the following functions.

Function Description
XVoiceCreateOneToOneEncoder Creates a one-to-one encoder that takes one input channel, compressess the data, and sends the compressed results to the output channel.
XVoiceCreateTwoToTwoEncoder Creates a two-to-two encoder that takes two input channels, compressess the data from each of these input channels, and sends the compressed results to the corresponding output channels.
XVoiceCreateFourToFourFullEncoder Creates a four-to-four encoder that takes four input channels, compressess the data from each, and sends the compressed results to the corresponding output channels.
XVoiceCreateFourToFourRoundRobinEncoder Creates a four-to-four encoder that takes four input channels, selects two of these input channels to compress, and sends the compressed results to the corresponding output channels. On the following frame, the two input channels that were not compressed previously are given priority over the two that were compressed.

These functions provide a pointer to an IXVoiceEncoder interface. Call the IXVoiceEncoder::ProcessMultiple function to compress the data packets and send them across the network.

Recieving Voice Packets from Xbox Live

To receive voice packets from another player on Xbox Live, titles must create at least one voice decoder XMO to decompress the voice data and store it in an output array. Call one of the following functions to create the voice decoder XMO.

Function Description
XVoiceCreateOneToOneDecoder Creates a one-to-one decoder that decompresses a single input channel and outputs the result to a single channel.
XVoiceCreateTwoToOneMixingDecoder Creates a two-to-one decoder that decompresseses two input channels and outputs the result to a single channel.
XVoiceCreateFourToOneMixingDecoder Creates a four-to-one decoder that decompresseses four input channels and outputs the result to a single channel.
XVoiceCreateFourToFourFullDecoder Creates a four-to-four decoder that decompresses a four input channels and outputs the result to the corresponding channels.

These functions provide a pointer to an IXVoiceDecoder interface. Call the IXVoiceEncoder::ProcessMultiple function to decompress the data packets and send them to the appropriate channels.

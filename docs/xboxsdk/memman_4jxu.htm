IsBadStringPtr

The IsBadStringPtr function verifies that the title has read access to a range of memory pointed to by a string pointer.

BOOL IsBadStringPtr(
  LPCTSTR lpsz,      // string
  UINT_PTR ucchMax   // maximum size of string
);

Parameters

lpsz
[in] Pointer to a null-terminated string, either Unicode or ASCII.
ucchMax
[in] Specifies the maximum size, in TCHARs, of the string. The function checks for read access in all bytes up to the string's terminating null character or up to the number of bytes specified by this parameter, whichever is smaller. If this parameter is zero, the return value is zero.

Return Values

If the title has read access to all characters up to the string's terminating null character or up to the number of characters specified by ucchMax, the return value is zero.

If the title does not have read access to all characters up to the string's terminating null character or up to the number of characters specified by ucchMax, the return value is nonzero.

If the application is compiled as a debugging version and the title does not have read access to the entire memory range specified, the function causes an assertion and breaks into the debugger. Leaving the debugger, the function continues as usual and returns a nonzero value. This behavior is by design, as a debugging aid.

Remarks

This function is typically used when working with pointers returned from third-party libraries, where you cannot guarantee the memory management behavior in that library.

Threads are expected to cooperate in such a way that one will not free memory that the other needs. Use of this function does not negate the need to do this. If this is not done, the application may fail in an unpredictable manner.

If the title has read access to some, but not all, of the specified memory range, the return value is nonzero.

In a preemptive multitasking environment, it is possible for some other thread to change the access to the memory being tested. Even when the function indicates that the title has read access to the specified memory, you should use structured exception handling when attempting to access the memory. Use of structured exception handling enables the system to notify the title if an access violation exception occurs, giving the title an opportunity to handle the exception.

Requirements

Header: Declared in Winbase.h, for include use Xtl.h.
Import Library: Use Xapilib.lib
Unicode: Implemented as both Unicode and ANSI versions.

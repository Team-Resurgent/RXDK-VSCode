Writing the needsCompiling Function (XbRC)

After writing the addData function, the next step to extend XbRC is to write the needsCompiling member function for the class derived from XBRCRawData. The needsCompiling function must first check for the presence of the intermediate file. If it's not there, you need to recompile. If the intermediate file is there, check its time-date stamp against the XDX file you are compiling and linking. If the intermediate file is older than the XDX file, load the intermediate file to see if any of the local data changed between the XDX file and the intermediate file. Remember, the XDX file is made up of lots of resources (in theory!), and only one of them may have been changed.

Checking whether any of the local data changed saves having to recompile the entire file for a change to one resource. The intermediate file names are generated using the resource identifier from the XDX file and the resource type number. This means you can even reorder the resources in the XDX file, and it will not recompile them; just link them in the new order! Duplicate resources get merged at present. You must also check any external files for changes against the intermediate file. You could, for example, have included a new external texture for the cube map, which probably would not have updated the XDX file if the file name for the new texture was identical.

Warning! This is the one place that can become fairly complicated because, if you make a mistake here, you will either compile every time (way slow!) or not recompile when you need to (way bad!). Be careful when writing your dependency checking, and remember that always recompile always works!

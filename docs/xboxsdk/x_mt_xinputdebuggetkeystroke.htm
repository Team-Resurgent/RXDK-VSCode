XInputDebugGetKeystroke

The XInputDebugGetKeystroke function retrieves a keyboard event from the keyboard input queue.

To use this function, you must define the DEBUG_KEYBOARD compiler define before including xtl.h. Do not use this function in actual shipping retail game titles.

DWORD XInputDebugGetKeystroke(
  PXINPUT_DEBUG_KEYSTROKE pKeystroke
);

Parameters

pKeystroke
[out] Pointer to an XINPUT_DEBUG_KEYSTROKE structure that receives information about a single keyboard event.

Return Values

If the function succeeds, it fills the structure pointed to by pKeystroke with information about the next keyboard event and returns ERROR_SUCCESS.

If the keyboard input queue is empty, the return value is ERROR_HANDLE_EOF.

If the function fails, the return value is an error code defined in winerror.h. The function does not use SetLastError to set the calling thread's last-error code.

Remarks

Before XInputDebugGetKeystroke can be called, the title must initialize the debug keyboard by calling XInputDebugInitKeyboardQueue, XGetDevices, and XInputOpen.

XInputDebugGetKeystroke retrieves and removes one keyboard event from the keyboard input queue. The options specified when the debug keyboard was initialized determine the types of keyboard events that will be reported.

The debug keyboard does not toggle the keyboard lights when corresponding keys are pressed (for example, CAPS LOCK).

Requirements

Header: Declared in xkbd.h, for include use xtl.h.
Import Library: Use xkbd.lib

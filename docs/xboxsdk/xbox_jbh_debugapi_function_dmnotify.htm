DmNotify

Registers notification handlers for debugging events.

DMHRAPI DmNotify(
  PDMN_SESSION Session, 
  DWORD dwNotification, 
  PDM_NOTIFY_FUNCTION pfnHandler
  );

Parameters

Session
Pointer to a notification session that was created with a call to DmOpenNotificationSession.
dwNotification
Contains one of the following values:
Value Meaning
DM_NONE Notifications are no longer sent.
DM_BREAK A notification is issued if a breakpoint is reached.
DM_DEBUGSESSION When this flag is used, the debug manager will return TRUE for all calls to DmIsDebuggerPresent as long as this session remains open. This flag is valid only on the development system (xboxdbg.lib).
DM_DEBUGSTR A notification is issued if text is sent to debug out.
DM_EXEC A notification is issued if the debugging subsystem causes all execution on the Xbox to start or stop.
DM_SINGLESTEP A notification is issued if the CPU generates a single-step interrupt.
DM_MODLOAD A notification is issued if an XTL-derived module (an Xbox title) is loaded.
DM_MODUNLOAD A notification is issued if an XTL-derived module (an Xbox title) is unloaded.
DM_CREATETHREAD A notification is issued if a thread is created.
DM_DESTROYTHREAD A notification is issued if a thread is destroyed.
DM_EXCEPTION A notification is issued if an exception occurs.
DM_ASSERT A notification is issued if an assertion occurs.
DM_DATABREAK A notification is issued if a data break occurs.
DM_RIP A notification is issued if a RIP debug event occurs.
DM_CLOCKINT A notification is issued if a clock interrupt occurs.
DM_THREADSWITCH A notification is issued if an execution switches from the current thread.
DM_SECTIONLOAD A notification is issued if a section from an XTL-derived module (an Xbox title) is loaded.
DM_SECTIONUNLOAD A notification is issued if a section from an XTL-derived module (an Xbox title) is unloaded.
pfnHandler
Pointer to a notification handler. Debugging notification handlers must take the following form: 
DWORD __stdcall my_handler(ULONG dwNotification, DWORD dwParam);

Return Values

Returns XBDM_NOERR if successful, an error value otherwise.

Remarks

Once you have registered your handler to receive notifications, the Xbox debugging subsystem will call your handler when the specified debugging event occurs. The first parameter, dwNotification, contains one of the notification constants listed below. The contents of the second parameter, dwParam, vary according to the type of debugging event that occurred.

Value Meaning
DM_BREAK The dwParam parameter contains a pointer to a DMN_BREAK structure.
DM_DEBUGSTR The dwParam parameter contains a pointer to a DMN_DEBUGSTR structure.
DM_EXEC The dwParam parameter contains one of the DMN_EXEC constants listed below.
DM_SINGLESTEP The dwParam parameter contains a pointer to a DMN_BREAK structure.
DM_MODLOAD The dwParam parameter contains a pointer to a DMN_MODLOAD structure.
DM_MODUNLOAD The dwParam parameter contains a pointer to a DMN_MODLOAD structure.
DM_CREATETHREAD The dwParam parameter contains a pointer to a DMN_CREATETHREAD structure.
DM_DESTROYTHREAD The dwParam parameter contains the ID of the thread that was destroyed.
DM_EXCEPTION The dwParam parameter contains a pointer to a DMN_EXCEPTION structure.
DM_ASSERT The dwParam parameter contains a pointer to a DMN_DEBUGSTR structure.
DM_DATABREAK The dwParam parameter contains a pointer to a DMN_DATABREAK structure.
DM_RIP The dwParam parameter contains a pointer to a RIP_INFO structure.
DM_THREADSWITCH The dwParam parameter contains a pointer to a DMN_THREADSWITCH structure.

Note   If the thread has stopped, the DM_STOPTHREAD flag is set in the dwNotification parameter. In addition, all other threads are suspended as if DmStop had been called. Bitwise-and dwNotification with DM_NOTIFICATIONMASK to strip this flag, or bitwise-and with DM_STOPTHREAD to see if the flag was set. If the thread has stopped, you will have to restart it using the DmContinueThread function, and you will have to call DmGo to resume execution of all other threads.

DM_ASSERT

If an assertion occurs, a DM_ASSERT notification is sent. The dwParam parameter contains a pointer to a DMN_DEBUGSTR structure that contains the assert message. If an assert occurs, you will have to tell the debugging subsystem how to proceed by setting the EAX register (in the context of the thread that took the assertion) on the Xbox to one of these three values: "B" (ASCII 66) to suspend and break into the process, "I" (ASCII 73) to ignore the assertion and continue with title execution, or "T" (ASCII 84) to terminate the thread. To accomplish this, you will need to call the DmSetThreadContext function.

DM_DEBUGSTR

If text is sent to debug out, a DM_DEBUGSTR notification is sent.

DM_EXEC

DM_EXEC notifications are sent immediately, when a handler is registered for DM_EXEC notification events (to relay the current state), and whenever the Xbox execution state changes. dwParam will contain one of the following values:

  • DMN_EXEC_STOP – All title threads are stopped.
  • DMN_EXEC_START – All title threads are restarted, except those marked with the DM_STOPTHREAD flag.
  • DMN_EXEC_REBOOT – The Xbox is about to reboot.
  • DMN_EXEC_PENDING – The Xbox has rebooted, but no title has been loaded yet.

DM_CREATETHREAD

The DM_CREATETHREAD notification is sent just after a new thread is created and just before that thread begins execution.

DM_CLOCKINT, DM_THREADSWITCH

There are a few things to keep in mind regarding the DM_CLOCKINT and DM_THREADSWITCH notification event types.

  • No system services are available while processing these notifications.
  • These events are only supported on the Xbox (via the xbdm.h header).
  • Details of these notification events are intentionally left undocumented.

Cease Notifications

To tell the debugging subsystem to quit sending notifications to a specific handler, call DmNotify, sending DM_NONE as the dwNotification value and a pointer to the handler as the pfnHandler value.

To tell the debugging subsystem to quit sending notifications to all handlers in the given session, call DmNotify, sending DM_NONE as the dwNotification value and NULL as the pfnHandler value.

Registering a Handler Multiple Times

If the same handler is registered in several sessions, each handler will receive a notification when a debug event occurs. If you attempt to register the same handler multiple times in the same session, it is only registered once.

Requirements

Header: Declared in XbDm.h and XboxDbg.h
Import Library: XbDm.lib for target system code; XboxDbg.lib for development system code.

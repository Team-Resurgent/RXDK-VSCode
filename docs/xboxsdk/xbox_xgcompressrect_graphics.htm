XGCompressRect

This function will compress a rectangle into one of the Xbox supported compressed texture formats: DXT1, DXT2, DXT3, DXT4 or DXT5.

HRESULT WINAPI XGCompressRect( 
  LPVOID pDestBuf,
  D3DFORMAT DestFormat,
  DWORD dwDestPitch, 
  DWORD dwWidth,
  DWORD dwHeight,
  LPVOID pSrcData,
  D3DFORMAT SrcFormat,
  DWORD dwSrcPitch,
  FLOAT fAlphaRef,
  DWORD dwFlags
);

Parameters

pDestBuf
[out] Either pDestBuf is a pointer to a Direct3D created texture (using IDirect3DDevice8::CreateTexture) or pDestBuf is a pointer to a contiguous memory allocation to be used with the IDirect3DResource8::Register function. In either case, the compressed texture is written out to pDestBuf.
DestFormat
[in] Member of the D3DFORMAT enumeration, describing the texture compression format of the destination. It must be one of the following values:
  • D3DFMT_DXT1
  • D3DFMT_DXT2
  • D3DFMT_DXT3
  • D3DFMT_DXT4
  • D3DFMT_DXT5
In order to get DXT2 or DXT4 compression, XGCOMPRESS_PREMULTIPLY must be one of the flags passed to dwFlags.
dwDestPitch
[in] Pitch of the destination, in terms of a row of 4x4 blocks. For example, a DXT1 texture with a width of 16 pixels has a pitch of 4 blocks * 8 bytes per block = 32 bytes.
dwWidth
[in] Width of the source rectangle measured in pixels.
dwHeight
[in] Height of the source rectangle measured in pixels.
pSrcData
[in] A pointer to linear (not swizzled) texture data.
SrcFormat
[in] Member of the D3DFORMAT enumerated type, describing the format of the source texture. This needs to be one of the Xbox linear formats below:
  • D3DFMT_LIN_A8R8G8B8
  • D3DFMT_LIN_X8R8G8B8
  • D3DFMT_LIN_R5G6B5
  • D3DFMT_LIN_A1R5G5B5
  • D3DFMT_LIN_X1R5G5B5
  • D3DFMT_LIN_A4R4G4B4
  • D3DFMT_LIN_R6G5B5
  • D3DFMT_LIN_A8B8G8R8
  • D3DFMT_LIN_B8G8R8A8
  • D3DFMT_LIN_R4G4B4A4
  • D3DFMT_LIN_R5G5B5A1
  • D3DFMT_LIN_R8G8B8A8
dwSrcPitch
[in] Pitch of the source texture measure in bytes.
fAlphaRef
[in] Specifies the cutoff value between transparent and opaque for DXT1 compression.
dwFlags
[in] Flags that describe compression behavior. This parameter must be the value zero, or one or more of the following flags:
XGCOMPRESS_PREMULTIPLY
Setting this flag ensures that alpha is premultiplied which will allow DXT2 and DXT4 compression.
XGCOMPRESS_NEEDALPHA0
Ensures that zero is available with interpolated alpha.
XGCOMPRESS_NEEDALPHA1
Ensures that one is available with interpolated alpha.
XGCOMPRESS_PROTECTNONZERO
Ensures that non-zero values don't get quantized to zero.

Normally, using interpolated alpha, there is no guarantee as to what the decoded alpha values will end up being. It is unlikely that 0.0 or 1.0 will end up as any of the interpolants. If you want to make sure that these end up as values, it is necessary to use XGCOMPRESS_NEEDALPHA0 and XGCOMPRESS_NEEDALPHA1. However, using these two flags will take away from other possible interpolated values, and some accuracy will be lost from the intermediate values.

When XGCOMPRESS_NEEDALPHA0 is set and alphakill is enabled, small non-zero alpha values may end up at zero. The XGCOMPRESS_PROTECTNONZERO flag adjusts the interpolant endpoints so that this doesn't happen. If both XGCOMPRESS_NEEDALPHA0 and XGCOMPRESS_PROTECTNONZERO are set, 6-value interpolation is used instead of 8-value interpolation.

Return Values

If this method succeeds, it returns S_OK.

Remarks

In Direct3D on the PC, D3DFMT_DXT2 and D3DFMT_DXT3 are two distinct texture formats. On Xbox, the hardware treats these two formats the same, so the value of their enumerants is the same. The same situation applies for DXT5 and DXT4. The only difference between these two texture types is that one type (DXT2 and DXT4) requires that each of the RGB channels be multiplied by the Alpha channel before the compression takes place. With DXT3 and DXT5, no premultiplication of Alpha occurs.

On Xbox, the values of the D3FORMAT enumeration are different than in DirectX 8.0. The Xbox D3DFORMAT values need to be used for the DestFormat and SrcFormat parameters of XGCompressRect, even when it is used on Win32. When compiling on Win32, the Xbox version of these constants can be obtained from the default location of "C:\Program Files\Microsoft Xbox SDK\Include\d3d8-xbox.h."

IDirect3DDevice8::SetSwapCallback

Sets the function that will be called on each swap or present.

void SetSwapCallback(
  D3DSWAPCALLBACK pCallback
);

Parameters

pCallback
[in] Specifies the address of a callback function of type D3DSWAPCALLBACK that will be called during each Swap or Present. Set this to NULL to disable the swap callback.

Remarks

When the GPU encounters a Swap or Present, the callback function will be called. SetSwapCallback is provided in order to notify the title information when the GPU handles its Swap or Present calls. Swap and Present are like all other rendering functions in that they are enqueued in the push-buffer. So the handling of the command is not synchronous with the function call. Even if the title calls the Swap or Present before a vertical blank, the GPU may not handle it until after the vertical blank. This is because the GPU must still process all of the commands in the push-buffer that are before the Swap or Present. SetSwapCallback enables the title to find out when the GPU handles the Swap or Present and at which vertical blank it will be visible.

The callback is defined as:

typedef void (__cdecl * D3DSWAPCALLBACK)(D3DSWAPDATA *pData);

The value passed as the pData parameter of the callback function is a D3DSWAPDATA structure containing information about the Swap or Present that initiated the callback.

The earliest that a title will be notified that it sucessfully made the Swap or Present in time for the frame is through SetSwapCallback.

This method is called at the time of deferred procedure calls (DPC) and should do little processing. Only a few XTL functions are safe to call at DPC level: The Interlocked functions are safe to use, as are the following XTL functions, if given a valid object handle:

  • SetEvent
  • ResetEvent
  • PulseEvent
  • ReleaseSemaphore
  • ReleaseMutex

When these functions are called at DPC with an invalid handle, their behavior is undefined.

Do not perform floating-point math, SSE instructions, and MMX instructions during DPC unless the XSaveFloatingPointStateForDpc and XRestoreFloatingPointStateForDpc functions are used.
